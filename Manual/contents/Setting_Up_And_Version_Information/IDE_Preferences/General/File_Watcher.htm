Préférences de File Watcher

Le File Watcher est le nom donné à la partie de l'EDI qui surveille les fichiers liés à un projet pour tout changement dû à un logiciel externe (c'est-à-dire: modification d'une image dans un éditeur externe) ou en raison de changements internes comme l'édition d'un actif de script. En général, ces paramètres ne devraient pas avoir besoin d'être modifiés, mais sur certains systèmes - comme ceux qui utilisent un SSD comme lecteur de sauvegarde pour les fichiers du projet - l'observateur de fichiers peut vous informer des modifications lorsque vous faites des choses comme simplement enregistrer le fichier. projet ou chaque fois que vous modifiez un élément de projet, auquel cas vous devrez peut-être modifier ces paramètres.

Les paramètres disponibles sont:

  • Activer File Watcher: Ceci peut être utilisé pour activer ou désactiver le système File Watcher. Si cela est désactivé, l'EDI ne détectera plus les fichiers modifiés par des sources externes, il faut donc faire attention lors de la désactivation de cette fonctionnalité. File Watcher est activé par défaut.
  • Définir la tolérance de File Watcher dans MS après l'enregistrement Dans l'EDI: Ceci ajuste le temps entre l'enregistrement d'un fichier et le File Watcher vérifiant les modifications. Vous ajusteriez cette valeur si vous enregistriez votre projet et que File Watcher apparaissait par erreur indiquant que quelque chose avait changé, auquel cas vous augmenteriez le délai jusqu'à ce que le message n'apparaisse plus. La valeur par défaut est de 2000 ms, mais peut aller de 1 ms à 10 000 ms.
