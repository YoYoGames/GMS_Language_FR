Préférences du chemin

Les préférences de chemin d'accès contrôlent les différents chemins d'accès aux fichiers requis par les éléments de l'IDE GameMaker Studio 2 ainsi que pour les éditeurs externes dont vous avez besoin. Les options suivantes existent pour l'IDE :

  • Répertoire temporaire: L'emplacement pour enregistrer tous les dossiers temporaires. Par défaut, sous Windows, il s'agit de

    \Users\<Username>\AppData\Local\Temp\GameMakerStudio2

    Et sur macOS, c'est

    /var/folders/<Path_Hash>/GameMakerStudio2

  • Répertoire du cache de l'IDE: L'emplacement pour sauvegarder le cache de l'IDE. Par défaut, sous Windows, il s'agit de

    \Users\<Username>\AppData\Roaming\GameMakerStudio2\Cache

    Et sur macOS, c'est :

    /Users/<Username>/.config/GameMakerStudio2/Cache

  • Répertoire du cache des actifs: L'emplacement de sauvegarde du cache des actifs pour chaque projet. Par défaut, sous Windows, il s'agit de

    \Users\<Username>\AppData\Roaming\GameMakerStudio2\Cache

    Et sur macOS, c'est :

    /Users/<Username>/.config/GameMakerStudio2/Cache

  • Emplacement "Mes projets" : L'emplacement où GameMaker Studio 2 créera initialement les nouveaux projets. Par défaut, sous Windows, il s'agit de

    \Users\<Username>\Documents\GameMakerStudio2

    Et sous macOS, il s'agit de

    /Users/<Username>/GameMakerStudio2

  • Supprimer automatiquement le répertoire temporaire à la fermeture: Si vous cochez cette option, GameMaker Studio 2 supprimera automatiquement le dossier temporaire qu'il crée par projet pour la compilation etc... Ce paramètre est activé par défaut, et le fait de le décocher le désactivera (ce qui signifie que vous devrez supprimer manuellement tous les fichiers temporaires). Ce paramètre est activé par défaut et le décocher le désactivera (ce qui signifie que vous devrez supprimer manuellement tous les fichiers temporaires par la suite).
  • Supprimer automatiquement le cache des actifs à la fermeture: Si cette option est cochée, le dossier de cache du compilateur d'actifs sera supprimé lorsque vous quitterez GameMaker Studio 2. Cette option est désactivée par défaut, et son activation signifie qu'à chaque fois que vous un projet, le cache devra être reconstruit (ce qui peut prendre du temps en fonction de la taille du jeu).
  • Supprimer automatiquement le cache de l'IDE à la fermeture: Si cette option est cochée, le dossier cache du compilateur de l'IDE sera supprimé lorsque vous quitterez GameMaker Studio 2. Cette option est désactivée par défaut, et l'activer signifiera qu'à chaque fois que vous démarrerez GameMaker Studio 2, le cache devra être reconstruit.
  • Supprimer le dossier temporaire: En cliquant sur ce bouton, vous supprimez le dossier temporaire du projet.
  • Supprimer le cache d'actifs: En cliquant sur ce bouton, vous supprimez le cache des actifs du compilateur pour le projet.
  • Supprimer le cache de l'IDE: En cliquant sur ce bouton, le cache de l'IDE sera supprimé

Les options suivantes existent pour définir les chemins d'accès aux éditeurs externes :

  • Chemin d'accès à l'éditeur/visualisateur externe pour les fichiers SWF: Si vous travaillez avec des sprites au format SWF, vous pouvez définir le chemin d'accès à votre visionneur/éditeur préféré et lorsque vous cliquez sur le bouton Editer l'image dans l'éditeur de sprites, il ouvrira le programme donné. ouvrira le programme donné. La valeur par défaut ici est de ne pas avoir de chemin.
  • Chemin d'accès à l'éditeur/visualisateur externe pour les fichiers Spine: Si vous travaillez avec des sprites au format Spine, vous pouvez indiquer le chemin de votre visualiseur/éditeur préféré et lorsque vous cliquez sur le bouton Editer l'image dans l'éditeur de sprites alors il ouvrira le programme donné. La valeur par défaut ici est de ne pas avoir de chemin.
