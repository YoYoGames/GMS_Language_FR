Préférences de la place de Marketplace

Les préférences de la place de Marketplace sont utilisées pour configurer le certificat Marketplace pour le téléchargement ou l'achat d'actifs. Le formulaire de certificat se compose des sections suivantes lorsque vous avez déjà créé un certificat (voir plus bas cette page pour plus de détails sur la création d'un certificat):

  • Fichier de certificat: dans cette section, vous spécifiez le *.pem fichier que la place de Marketplace utilise pour la certification des actifs. Si vous avez un certificat précédemment créé, vous pouvez cliquer sur le bouton du navigateur de fichiers ici et l'ajouter au forum, mais si vous n'avez pas de certificat et que vous souhaitez en créer un, vous devez omettre toute information ici et cliquer à la place sur le Bouton Créer un nouveau certificat (expliqué ci-dessous).
  • Mot de passe du certificat: Ici, vous devez saisir le mot de passe du certificat pour le fichier sélectionné ci-dessus. Le mot de passe est créé lorsque vous créez pour la première fois le *.pem (voir Créer un nouveau certificat, ci-dessous) et sera lié de manière irréversible au fichier de certificat créé.
  • Vérifier: Une fois que vous avez rempli les champs ci-dessus, vous devez cliquer sur ce bouton pour vous assurer que le mot de passe et la sélection de fichiers sont corrects et fonctionnent ensemble.

Confirmer les boîtes de dialogue

Ici, vous pouvez sélectionner le comportement des boîtes de dialogue de confirmation lorsque vous travaillez avec une ressource à partir du navigateur d'actifs. Les options disponibles sont:

  • Réponse automatique à la suppression de contenu hors ligne: lorsque vous supprimez un élément de la place de marché de votre bibliothèque sans connexion Web, vous recevez normalement un avertissement vous demandant si vous êtes sûr de vouloir continuer. En définissant cette option sur "OK", le message sera supprimé comme si vous aviez cliqué sur le bouton "OK" pour continuer. La valeur par défaut est "Afficher le message".

Créer un nouveau certificat

Si vous n'avez pas encore créé de fichier de certificat pour utiliser le Marketplace, vous devez cliquer sur le bouton intitulé Créer un nouveau certificat au bas de ce formulaire. Lorsque vous faites cela, une nouvelle fenêtre s'ouvre pour vous demander de saisir les informations suivantes:

  • Nom de l'éditeur: il s'agit du nom qui sera utilisé comme nom de l'éditeur pour l'élément. Dans le Marketplace vous pouvez configurer différents éditeurs si vous le souhaitez et chacun doit avoir son propre fichier de certificat *.pem correspondant.
  • Mot de passe du certificat: Ici, vous devez donner un mot de passe unique pour le fichier de certificat à créer. Faites attention à ce que vous choisissez car vous ne pouvez pas le modifier ultérieurement et si vous ne disposez pas du mot de passe correct pour le fichier de certificat dans les sections ci-dessus, vous ne pourrez pas mettre à jour les Marketplace (ni en télécharger de nouveaux).

Avec ces détails, vous pouvez ensuite cliquer sur le bouton Créer un certificat pour générer le *.pem déposer. Vous serez invité à donner un emplacement de sauvegarde et un nom pour le fichier, et vous devriez en prendre note pour référence future. Une fois que vous avez fait cela, le nouveau fichier et le nouveau mot de passe seront automatiquement ajoutés dans les champs du formulaire listés ci-dessus et vous pouvez cliquer sur le bouton Vérifier pour tester que le certificat a été généré correctement.

REMARQUE: nous vous recommandons de prendre une capture d'écran des paramètres réels ainsi que de faire une copie du *.pem  déposer. Les deux doivent être enregistrés ensemble dans un emplacement sûr pour référence future, ou au cas où vous auriez besoin de réinstaller GameMaker Studio 2 à un moment donné. Vous ne pouvez pas mettre à jour les actifs qui ont déjà été téléchargés à moins que le package mis à jour ne soit signé à l'aide du même certificat avec lequel l'original a été créé.
