Préférences des flux d'exécution

Les préférences relatives auruntime permettent d'afficher et de modifier le runtime actuellement installé Runtime ainsi que le runtime en cours d'utilisation. Plusieurs outils d'exécution différents peuvent être installés dans GameMaker Studio 2 à un moment donné et vous pouvez définir celui à utiliser à partir de cette préférence. Idéalement, vous souhaitez utiliser le runtime le plus récent, mais il peut arriver qu'il présente un problème, ou qu'il soit nécessaire d'en utiliser un plus ancien pour des raisons de comparabilité, etc... Vous pouvez donc modifier ici les outils d'exécution que GameMaker Studio 2 utilise pour créer votre jeu.

La section initiale de cette préférence concerne l'origine des outils d'exécution et vous demande de saisir l'URL d'un flux RSS. Ce flux sera ensuite analysé pour détecter les mises à jour des outils d'exécution, qui pourront ensuite être téléchargées et installées. Cette section affiche par défaut l'URL du fil d'exécution principal, mais vous pouvez cliquer sur le bouton Ajouter pour créer un nouveau fil d'exécution dans lequel vous indiquez le nom et l'URL du flux RSS, puis cliquez sur Appliquer pour indiquer à GameMaker Studio 2 d'y accéder. Vous devrez fermer et rouvrir la fenêtre des Préférences pour que le nouveau flux s'affiche.

Une fois qu'un fil d'exécution a été créé, il sera répertorié dans cette section comme une sous-catégorie, et vous devriez toujours avoir au moins un fil répertorié (par défaut, il s'agit du fil principal ). Les préférences du fil d'exécution sont utilisées pour sélectionner, installer et désinstaller les différents outils d'exécution. Sur en haut, vous avez les outils d'exécution actuellement utilisés, la date de leur création et l'emplacement où ils sont stockés sur le disque dur. En dessous de ces détails, vous trouverez une liste de tous les outils d'exécution disponibles, et vous pouvez choisir n'importe lequel de cette liste pour qu'il devienne l'outil d'exécution. liste pour en faire le runtime "en cours" utilisé pour la compilation de vos projets.

Pour sélectionner un autre runtime à utiliser, il suffit de le trouver dans la liste, puis de double-cliquer sur . Si le runtime n'a pas été téléchargé et installé, vous serez invité à le faire. Si le runtime n'a pas été téléchargé et installé, vous serez invité à le faire et, une fois qu'il aura été installé, il vous sera demandé de fermer et de redémarrer l'IDE afin que le nouveau runtime puisse être initialisé.
