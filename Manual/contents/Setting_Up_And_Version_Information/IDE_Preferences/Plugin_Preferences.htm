Préférences du plugin

Les préférences des plugins sont utilisées pour contrôler tous les plugins supplémentaires qui ont été ajoutés à l' GameMaker Studio 2. Par défaut, cela a une section dédiée au plugin Source Control pour intégrer les GameMaker Studio 2 avec Git.

Contrôle de la source ( Git )

Cette section traite des préférences que vous devez définir pour que le plug-in Git SCM GameMaker Studio 2. D' abord, vous avez l'ensemble des options de G GENERAUX:

  • Ouvrir automatiquement la fenêtre de sortie de contrôle de source: lorsque cette case est cochée, la fenêtre de sortie de contrôle de source s'affiche automatiquement lorsque quelque chose change. Ceci est activé par défaut.
  • Journalisation détaillée: activez la journalisation détaillée pour le journal de sortie Git Ceci est désactivé par défaut.

Vient ensuite la section Identité de l'utilisateur avec les champs suivants qui doivent être remplis:

  • Nom d'utilisateur pour valider en tant que: Ici, vous devez entrer le nom d'utilisateur que vous utiliserez pour effectuer des validations sur Git (cela doit être le même que le nom d'utilisateur spécifié lors de la configuration de votre Git ).
  • E-mail de validation en tant que: e-mail associé à cette identité Git

Après avoir configuré l'identité de l'utilisateur, vous pouvez ensuite ajouter une nouvelle combinaison nom d'utilisateur / mot de passe pour l'authentification en cliquant sur le bouton intitulé Ajouter une nouvelle authentification d'utilisateur / passe, qui ouvrira la fenêtre suivante avec différents champs à remplir:

Vous pouvez également ajouter une nouvelle authentification de paire de clés SSH en cliquant sur le bouton avec cette étiquette. Cela ouvrira la fenêtre suivante que vous devrez remplir:

Sous ces boutons, vous pouvez ensuite configurer l' outil de fusion et l' outil de différence pour SCM avec ces options:

  • Chemin d'accès à l'outil de fusion: ici, vous spécifiez le chemin d'accès complet au fichier vers l'outil de fusion Git
  • Options pour l'outil de fusion: dans ce champ, vous pouvez ajouter des commandes supplémentaires à exécuter chaque fois que vous utilisez l'outil de fusion.
  • Path to Diff Tool: Ici, vous spécifiez le chemin d'accès complet au fichier vers l' Git diff.
  • Options pour l'outil Diff: Dans ce champ, vous pouvez ajouter des commandes supplémentaires à exécuter chaque fois que vous utilisez l'outil Diff.

Enfin, après avoir configuré ceux-ci, vous avez une dernière option:

  • Activer automatiquement le contrôle de source: lorsque cette case est cochée, le contrôle de source sera activé automatiquement pour tous les nouveaux projets. Ceci est désactivé par défaut.
