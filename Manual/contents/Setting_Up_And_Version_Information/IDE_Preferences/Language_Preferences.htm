Préférences linguistiques

Les préférences de langue sont utilisées pour définir comment certaines des fonctionnalités de langage pour la programmation seront affichées dans l'EDI de GameMaker Studio 2. Les options pour GML sont les suivantes:

  • Afficher les types d'arguments dans la barre d'état: Cette option concerne le moment où vous utilisez vos propres scripts personnalisés et avez utilisé la notation JSDoc pour identifier les différents arguments, etc... (voir ici pour plus d'informations). Lorsque cette option est cochée et que vous avez spécifié les types d'arguments dans le script, ils seront affichés dans la barre d'état en bas de l'éditeur, entre {}. Ceci est désactivé par défaut.
  • Afficher les descriptions des arguments dans la barre d'état: si cette case est cochée, toutes les descriptions que vous avez données à vos scripts personnalisés en utilisant la notation JSDoc seront affichées dans la barre d'état en bas de la fenêtre de l'éditeur. Ceci est désactivé par défaut.
  • Afficher l'orthographe américaine pour les fonctions système: par défaut, GameMaker Studio 2 utilise l'orthographe anglaise GB pour les fonctions, par exemple draw_text_colour(), cependant ce n'est peut-être pas exactement ce à quoi vous êtes habitué et vous pouvez donc passer aux orthographes américaines en utilisant cette option, de sorte que les fonctions deviennent draw_text_color(), par exemple. Ceci est désactivé par défaut.
  • Nombre maximal d'erreurs de syntaxe à afficher: cette valeur définit le nombre maximal d'erreurs de syntaxe qui seront affichées à tout moment. Si vous trouvez que l'EDI est en retard, vous pouvez essayer de réduire cette valeur. Il est défini sur 50 par défaut.
  • Activer les avertissements de référence de variable: lorsqu'il est activé, cela signalera tous les problèmes potentiels avec les variables, comme quand un a été défini quelque part mais n'est pas réellement utilisé dans le code. Lorsqu'ils sont désactivés, ces avertissements ne seront plus affichés. La valeur par défaut est que cela est activé.
