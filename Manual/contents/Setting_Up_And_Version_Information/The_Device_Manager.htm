Le gestionnaire de périphériques

On accède au gestionnaire de périphériques en cliquant sur sur le bouton crayon dans la liste des cibles:

En cliquant sur ce bouton, vous ouvrez le gestionnaire de périphériques où vous pouvez configurer différents périphériques pour les plates-formes cibles sur lesquelles vous souhaitez compiler et tester. et tester (notez que les plates-formes disponibles dépendent de la licence que vous possédez et qu'elles ne sont pas toutes visibles). Vous trouverez ci-dessous des détails sur la configuration des dispositifs pour chacune des plates-formes cibles :

Android

Ici, vous pouvez cliquer sur le bouton Ajouter un nouveau dispositif pour ajouter manuellement un nouveau dispositif Android à la liste, ou vous pouvez cliquer sur le bouton bouton Détecter le périphérique pour que GameMaker Studio 2 détecte automatiquement tous les périphériques connectés via une connexion filaire. Vous pouvez également configurer un appareil virtuelAndroid en cliquant sur le bouton marqué Exécuter AVD. Le gestionnaire de dispositifs virtuels Android s'ouvre alors, et vous pouvez créer un nouveau dispositif virtuel Android (voir ici pour plus d'informations). Une fois que vous en avez créé un, vous devez le lancer à partir du gestionnaire AVD, puis dans GameMaker Studio 2, cliquez sur le bouton Detect Device. Detect Device et il devrait être ajouté automatiquement à la liste des appareils :

Une fois que vous avez ajouté un périphérique (physique ou virtuel), vous pouvez alors vérifier la connexion à l'aide du bouton Tester la connexion ou le retirer de la liste en cliquant sur le bouton bouton Supprimer le périphérique.


HTML5

Le gestionnaire de périphériques HTML5 permet d'ajouter différents navigateurs à la liste des périphériques pour les cibler lors de l'exécution. Pour commencer, vous devez cliquer sur le bouton Détecter les navigateurs, ce qui devrait créer des périphériques pour chacun des navigateurs installés sur votre machine. Si votre navigateur ne figure pas dans la liste, vous pouvez également cliquer sur le bouton Ajouter un nouveau navigateur, puis donner un nom à l'appareil et indiquer à GameMaker Studio 2 le chemin d'accès à l'exécutable du navigateur.

Les champs supplémentaires disponibles peuvent être configurés pour aider à contrôler la façon dont le navigateur exécute vos projets, avec les options suivantes disponibles :

  • Arguments de ligne de commande du navigateur: Vous pouvez ajouter ici tout argument de ligne de commande à ajouter au chemin du navigateur. Vous pouvez trouver une liste de ces arguments dans les liens suivants vers les principaux navigateurs populaires :
    • Chromium (Chrome)
    • Firefox
    • Internet Explorer
  • Adresse du serveur Web: L'adresse du serveur Web à cibler à cibler (par défaut, 127.0.0.1). Il se peut que vous deviez la modifier en fonction des spécifications des différentes API. Par exemple, Facebook exigerait que cette adresse soit définie sur http://localhost (et l'avoir sur la liste blanche de la console de l'application).
  • Port du serveur Web: Définissez le numéro de port pour le serveur Web (51264 par défaut).
  • IP des serveurs Web autorisés / Plages d'IP CIDR: Définissez les serveurs Web autorisés, les noms d'hôtes et les plages d'adresses IP CIDR selon vos besoins.

macOS

Pour ajouter un périphérique Mac au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Ajouter un nouveau périphérique qui ajoutera le périphérique et créera quelques champs vides que vous devrez remplir pour activer la connexion.

Vous devez d'abord nommer l'appareil pour pouvoir l'identifier dans la liste des cibles, puis donner le nom d'hôte (ou l'IP) du Mac. nom d'hôte (ou IP) du Mac. Vous devez ensuite donner le nom d'utilisateur de l'utilisateur auquel vous souhaitez vous lier, et le mot de passe qu'il a utilisé. Si vous le souhaitez, vous pouvez également modifier le Install Folder sur le Mac à l'endroit où vous préférez que GameMaker Studio 2 stocke les fichiers de construction de vos projets.

Lorsque tout cela est terminé, vous pouvez cliquer sur le bouton Test Connection button pour vous assurer que GameMaker Studio 2 peut communiquer avec le Mac.

REMARQUE: si vous développez pour iOS, vous devez d'abord effectuer cette opération et établir une connexion fonctionnelle avec le Mac que vous souhaitez utiliser pour le développement.

NOTE: Vous devez avoir activé la connexion à distance sur le mac cible. Cela peut être fait en allant dans Préférences -> Partage et Activez la connexion à distance pour l'utilisateur auquel vous voulez permettre de se connecter.

Ubuntu(Linux)

Pour ajouter un périphérique Ubuntu(Linux) au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Add New Device (Ajouter un nouveau périphérique ). ajoutera le périphérique et créera quelques champs vides que vous devrez remplir pour activer la connexion.

Vous devez d'abord nommer le périphérique afin de pouvoir l'identifier dans la liste des cibles, puis donner le nom d'hôte (ou l'IP) de la machine Ubuntu (ou VM). Vous devez ensuite donner le nom d'utilisateur de l'utilisateur que vous souhaitez lier, et le mot de passe qu'il a utilisé. Vous devez également indiquer un dossier d'installation sur la machine Ubuntu que vous souhaitez utiliser pour installer vos projets. Lorsque tout cela est terminé, vous pouvez cliquer sur le bouton Tester la connexion pour vous assurer que GameMaker Studio 2 peut communiquer avec la machine Ubuntu.

Notez que l'exécutable final créé par GameMaker dépendra de l'architecture de l'appareil ciblé, qui peut être x64 ou ARM. Par exemple, si vous connectez un Raspberry Pi 3 ou 4, l'exécutable créé sera un exécutable 32bit ARM, et si vous connectez un PC Linux, il sera un exécutable 64bit x64.

Windows UWP

Avant de pouvoir ajouter un appareil pour la cible Windows 10 UWP, vous devez avoir activé le mode développeur et configuré le portail de périphériques sur l'appareil. Si vous ne le faites pas, GameMaker Studio 2 ne pourra pas communiquer avec l'appareil ni créer correctement vos applications UWP. avec l'appareil et créer correctement vos applications UWP.

Le moyen le plus simple d'activer le mode Développeur est de taper " Mode Développeur " dans la recherche Windows, puis de cliquer sur l'entrée " Paramètres " qui s'affiche. La fenêtre suivante s'ouvre alors :

Vous devez sélectionner l'option Developer Mode (qui peut nécessiter le téléchargement et l'installation par Windows de certains paquets supplémentaires), puis sélectionner l'option située en dessous, Enable Device Portal. télécharger et d'installer des paquets supplémentaires), puis sélectionnez l'option située en dessous, Enable Device Portal. Le portail de l'appareil est ce qui permettra à GameMaker Studio 2 de communiquer avec le PC Windows 10 et, une fois activé, vous devrez d'indiquer un nom d'utilisateur et un mot de passe:

Notez que le nom d'utilisateur et le mot de passe doivent être uniques pour le portail et ne doivent pas être ceux utilisés par votre compte Microsoft ou d'autres informations d'identification Windows. Une fois que vous avez fourni ces informations, cliquez sur le bouton intitulé Appliquer , puis fermez le gestionnaire de paramètres et retournez dans GameMaker Studio 2.

Pour plus d'informations sur le Device Portal, veuillez consulter cet article de Microsoft : Device Portal for Desktop.

Pour ajouter un périphérique compatible Windows 10 UWP au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Add New Device (Ajouter un nouveau périphérique ). ajoutera le périphérique et créera quelques champs vides que vous devrez remplir pour activer la connexion.

Tout d'abord, vous devez nommer l'appareil afin de pouvoir l'identifier dans la liste des cibles, puis donner l'hôte - il s'agit de l'adresse IP de l'appareil Windows 10 sur le réseau qui doit héberger le jeu et doit être suffixé par " :50443" pour définir le numéro de port. Après cela, ajoutez le nom d'utilisateur et le mot de passe que vous avez configurés dans le portail des appareils (comme mentionné précédemment). Vous pouvez laisser le code PIN vide car il n'est pas nécessaire pour le bureau UWP Windows 10.

Pour des détails de configuration plus spécifiques qui couvrent les différentes sorties UWP (Desktop, XboxOne), veuillez consulter le service d'assistance YYG : Configuration de l'UWP.
