Social

Cet événement ne peut être déclenchée par le rappel s de différentes fonctions sociales spécifiques et de UWP Xbox Fonctions en direct et retourne une DS carte stockée dans la variable async_load, contenant différentes paires clé / valeur en fonction du rappel de la fonction qui a déclenché l'événement.

La carte DS contiendra un certain nombre de clés, dont la plus importante est "id ". Cela renverra une constante qui peut ensuite être vérifiée dans le code pour déterminer lequel des nombreux rappels que l'événement a reçus. Une fois que vous avez analysé cette clé et comparé la valeur renvoyée avec les constantes disponibles, vous pouvez continuer à extraire le reste des informations de la carte.

Les pages de fonctions sur les réalisations et les classements dans la section Référence détaillent les fonctions et les rappels ainsi que les constantes utilisées pour déclencher cet événement, mais il existe également certaines circonstances dans lesquelles il sera déclenché sans appel de fonction du jeu en cours d'exécution:

  • lorsque votre jeu est lancé depuis le tableau de bord du système d'exploitation
  • lorsque vous terminez un défi
  • quand un autre joueur termine un défi
  • lorsque le jeu est lancé à partir du tableau de bord du système d'exploitation en sélectionnant un défi

N'importe lequel des rappels ci-dessus déclenchera l'événement social et un async_load la carte sera générée avec les détails suivants (notez qu'il existe un autre "id "valeur clé pour définir chacune des différentes raisons pour lesquelles l'événement a été appelé, mais tous les autres contenus de la carte sont identiques):

  • "id "- La valeur de cette clé dépendra du type de rappel déclenchant l'événement. Il peut s'agir de l'une de ces constantes:
Constant Description
achievement_challenge_received Un défi a été reçu
achievement_challenge_completed Un défi a été relevé.
achievement_challenge_completed_by_remote Un défi a été complété par l'autre joueur.
achievement_challenge_launched Le jeu a été lancé à partir du tableau de bord du système d'exploitation avec le défi donné.

  • "playerid "- L'identifiant du joueur pour le défi.
  • "issuerid "- L'identité de la personne qui a lancé le défi.
  • "state "- L'état du défi, qui aura une valeur de 0 à 3 (sous forme de chaîne) pour invalide, en attente, terminé ou refusé.
  • "message "- Le message texte pour le défi.
  • "issueddate "- La date d'émission de la contestation
  • "completeddate "- La date de fin du défi.
  • "type "- Le type de défi donné. Il peut s'agir de l'une des deux constantes suivantes:
Constant Description
achievement_type_score_challenge Un défi basé sur la valeur du score.
achievement_type_achievement_challenge Un défi basé sur une réalisation.

  • "identifier "- La chaîne d'identification du défi.
  • "score "- Le score est lié au défi.

Vous trouverez ci-dessous un petit extrait de code comme exemple d'utilisation:

var ident = ds_map_find_value(async_load, "id" );
if ident == achievement_challenge_completed;
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
 state = ds_map_find_value(async_load, "state");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }
