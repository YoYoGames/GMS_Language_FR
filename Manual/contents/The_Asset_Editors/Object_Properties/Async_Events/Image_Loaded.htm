Image chargée

Cet événement est déclenché lorsque vous chargez une image dans GameMaker Studio 2, tant que vous avez utilisé une URL valide ou un chemin de fichier inclus avec la fonction de chargement de fichier applicable. Par exemple, disons que vous souhaitez charger une image de sprite et que vous ne modifiez le sprite actuel de l'instance par le nouveau que lorsqu'il est chargé. Eh bien, vous auriez quelque chose comme ça dans un événement de création ou un événement d'alarme (par exemple):

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

Cela va maintenant commencer à charger l'image dans l'appareil ou le navigateur, mais cela ne bloquera pas GameMaker Studio 2 pendant qu'il attend que le fichier soit chargé. Au lieu de cela, GameMaker Studio 2 continuera à fonctionner normalement jusqu'à ce que l'image soit chargée et que le rappel déclenche l' événement Image Loaded, où une carte DS est créée et stockée dans la variable spéciale async_load. La carte contient les informations suivantes:

  • "filename ": Le chemin complet du fichier que vous avez demandé.
  • "id ": ID de la ressource que vous avez chargée. Ce sera le même que la variable à laquelle vous avez affecté la ressource.
  • "status ": Renvoie une valeur inférieure à 0 pour une erreur.

Vous affecteriez alors l'image nouvellement chargée à un sprite dans cet événement. L'exemple de code suivant montre comment les informations renvoyées seraient utilisées:

if ds_map_find_value(async_load, "id") == spr
   {
   if ds_map_find_value(async_load, "status") >= 0
      {
      sprite_index = spr
      }
   }

Le code ci-dessus vérifie d'abord le id de la carte DS créée, puis vérifiez le status du rappel. Si la valeur est supérieure ou égale à 0 (signalant la réussite), le résultat du rappel sera alors utilisé pour définir l'index du sprite sur l'image nouvellement chargée.
