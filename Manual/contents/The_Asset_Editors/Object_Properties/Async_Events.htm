Les événements Async

Un événement asynchrone est un événement qui est déclenché lorsque GameMaker Studio 2 reçoit un rappel d'une source externe, qui peut provenir du Web ou de l'appareil exécutant votre jeu. Fondamentalement, vous dites à GameMaker Studio 2 de faire quelque chose - comme charger une image - et il commencera à le faire mais continuera à faire tout ce qu'il a à faire pendant ce temps. Ensuite, une fois la demande terminée, un rappel sera envoyé à GameMaker Studio 2 et tous les événements asynchrones définis pour ce type de rappel seront déclenchés.

Veuillez noter que les événements asynchrones sont déclenchés pour toutes les instances qui les ont, tout comme les événements de clavier, de sorte que vous pouvez - par exemple - faire un http_get() appeler dans une instance, mais avoir l' événement HTTP asynchrone dans une autre pour gérer le rappel.

REMARQUE: vous devez savoir qu'en raison de la protection XSS dans les navigateurs, les demandes et les tentatives de chargement de ressources à partir de plusieurs domaines sont bloquées et peuvent sembler renvoyer des résultats vides lors de l'utilisation de l'un des événements suivants.

Il se peut que si vous créez des extensions pour différentes plates-formes, vous souhaitiez cibler un ou plusieurs de ces événements asynchrones pour les rappels de vos fonctions d'extension. À cette fin, vous pouvez trouver un tableau ci-dessous qui décrit chacune des constantes de nom d'événement interne et leur valeur d'ID correspondante. Notez que ces constantes d'événement ne sont à utiliser qu'avec des extensions!
