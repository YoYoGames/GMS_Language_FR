L'éditeur de script

Cette section traite de l' éditeur de script (également appelé éditeur de texte ) qui est l'endroit où vous tapez tout le code dont votre jeu aura besoin pour s'exécuter. Cet éditeur est utilisé pour créer des ressources de script personnalisées, ainsi que pour coder des événements d'objet, pour ajouter du code de création de salle, et pour beaucoup d'autres choses, et en tant que telle, cette section détaille simplement le fonctionnement de l'éditeur de script, et non comment créer un script ressources ni comment utiliser Drag and Drop™ pour créer des scripts. Pour cela, veuillez consulter les sections suivantes:

  • Scripts GML
  • Scripts DnD

L'éditeur de script est l'endroit où vous écrivez votre GameMaker Language, qui est le langage de programmation GameMaker Studio 2 (voir la section Référence GML pour plus de détails). Une fois que vous vous êtes familiarisé avec GameMaker Studio 2 et que vous souhaitez l'utiliser au maximum, il est conseillé de commencer à apprendre à utiliser ce langage, car il élargit considérablement vos possibilités lors de la création de jeux.

Un élément de script peut être nommé en cliquant avec le bouton droit de la souris sur celui-ci dans le navigateur de ressources et en sélectionnant Renommer, mais notez que le nom du script doit être conforme aux règles de script des fonctions, de sorte qu'elles ne peuvent être que alphanumériques, doivent commencer par une lettre et que le seul symbole qu'elles peuvent contenir est le " _ "symbole sous la barre. L'éditeur de script des autres endroits ne peut pas être renommé et aura un nom spécifique à ce qui est en cours d'édition, comme un événement d'objet par exemple.

Lorsque vous ouvrez la fenêtre de l'éditeur de script, elle aura les options et la disposition suivantes:

Onglet Script / Événement

L'éditeur de script s'ouvre dans une fenêtre avec des onglets en haut pour vous permettre d'avoir plusieurs scripts ou événements dans une fenêtre (bien que ce comportement puisse être modifié à partir des Préférences pour donner une nouvelle fenêtre à chaque script ou événement). Vous pouvez cliquer sur un onglet, puis faites-le glisser pour modifier l'ordre de tabulation, ou - si vous préférez - vous pouvez le retirer de la fenêtre actuelle et le placer sur l'espace de travail pour créer une nouvelle fenêtre pour ce script (ou l'ajouter à une fenêtre différente ), et vous pouvez également agrandir l'éditeur de script pour créer également un nouvel espace de travail. Si vous faites glisser un onglet de script hors de la fenêtre de l'EDI, un nouvel IDE sera généré pour contenir cette ressource de script, et il peut être utilisé comme vous le feriez pour la fenêtre principale.

Notez que si vous modifiez du code à partir d'un événement d'objet dans l'éditeur de script et que vous avez agrandi l'éditeur de script ou l'avez placé dans une fenêtre ou un espace de travail séparé, cliquez avec le bouton droit de la souris Le menu de l'éditeur aura quelques options supplémentaires:

  • Restaurer dans l'espace de travail: cela sortira l'éditeur de script d'un état maximisé / espace de travail et le réenchaînera avec l'objet dans l'espace de travail.
  • Aller à l'objet: Cela vous mènera à l'espace de travail sur lequel se trouve l'objet avec le code et vous concentrera sur l'objet.
  • Ajouter / ouvrir un événement: Cela vous permet d'ajouter un nouvel événement à l'objet auquel appartient le script actuel et ouvrira un nouvel onglet de code dans l'éditeur de script pour l'événement ajouté. Si l'événement sélectionné contient déjà du code, celui-ci sera ouvert dans un nouvel onglet.

Pour plus d'informations sur le reste du clic droit options du menu, voir la section sur l' édition, ci-dessous.

Gouttière

La gouttière est utilisée pour afficher les numéros de ligne de votre code et également pour transmettre des informations spécifiques sur l'état du code. Plus important encore, si vous faites une faute de GameMaker Studio 2 frappe ou construire un code incorrect, le GameMaker Studio 2 IDE vous informera de la question en signalant la ligne de code avec un point d'exclamation rouge montrant une erreur de syntaxe.   • Rapport d'erreurs

La gouttière marquera également toute ligne du script ayant un point d'arrêt ajouté. Un point d'arrêt est simplement un endroit dans le script où vous voulez que le débogueur suspende l'exécution de votre jeu lorsqu'il est atteint. Vous pouvez basculer un point d'arrêt à partir de n'importe quelle ligne de n'importe quel script ou événement d'objet en appuyant sur "f9 "ou en utilisant le clic droit menu. Rechercher / remplacer

Tout en travaillant dans l'éditeur de script, vous pouvez appuyer sur / + "F "pour ouvrir le champ de recherche ou / + "H "pour ouvrir Rechercher et remplacer:

Ici, vous pouvez effectuer une recherche locale pour le mot-clé que vous entrez et une fois que vous avez entré votre terme de recherche, vous pouvez ensuite utiliser les flèches en haut à droite de la fenêtre de recherche pour passer d'un terme trouvé au suivant dans le script. Lorsque vous activez cette option, la fonction de recherche ne mettra en évidence que les chaînes qui correspondent à la chaîne d'entrée entière. Par exemple, si elle est désactivée, une recherche sur " aléatoire " affichera tous les mots contenant cette chaîne - comme irandom(), ou randomise() - tout en l'activant ne montrerait que la fonction random().
Lorsque cette option est activée, vous GameMaker Studio 2 de vérifier non seulement le contenu de la chaîne de recherche, mais également la casse. Par exemple, si vous avez un sprite appelé "spr_Dog "et faites une recherche pour" chien "avec cette option désactivée, alors la chaîne de sprite sera mise en surbrillance, cependant si la bascule est activée , ce ne sera plus le cas car" Dog "n'est plus considéré comme" chien ".

Si vous avez ouvert la fenêtre de recherche en utilisant / + "F ", la fonctionnalité de remplacement ne sera pas visible, vous devez donc cliquer sur le bouton Remplacer pour l'ouvrir. Notez que si vous souhaitez effectuer une recherche globale (c'est-à-dire: rechercher l'ensemble du projet plutôt que le script ou l'événement en cours), vous pouvez appuyer sur / + + "F".

Éditeur de code principal

L'éditeur de code principal est l'endroit où vous allez écrire tout votre code pour créer l'actif de script ou remplir un événement d'objet. L'édition réelle de votre code est couverte dans la section sur la modification plus loin sur cette page.

Informations

La barre d'informations en bas de l'EDI vous montre le numéro de ligne actuel sur lequel vous vous trouvez et la position le long de la ligne. C'est également là que vous pouvez voir l'assistant de code, qui est une ligne de texte qui montre la fonction que vous êtes en train de modifier avec les arguments dont elle a besoin. Au fur et à mesure que vous remplissez la fonction dans l'éditeur, les arguments seront mis en surbrillance pour vous montrer lequel vous êtes en train de modifier. Si vous avez utilisé les commentaires de script JSDoc dans un actif de script, les informations que vous avez fournies s'affichent également ici.
