Modifier le code

Lorsque vous commencez à utiliser l'éditeur de script, il vous suffit de taper le code que vous souhaitez avoir pour l'actif de script ou pour l'événement ou autre, en suivant la syntaxe GML correcte. Pendant que vous faites cela, vous obtiendrez de l'aide dans la fenêtre de saisie semi-automatique:

Cette fenêtre apparaîtra pendant que vous écrivez votre code (normalement après une légère pause, mais cela peut être défini dans les Préférences ) et vous donnera une sélection de réponses «les mieux adaptées» dans l'ordre alphabétique.

Lors de l'utilisation de l'éditeur de code, vous pouvez cliquer sur et faites glisser la souris pour sélectionner une ligne de code ou maintenez puis clique et faites glisser pour sélectionner des colonnes de code. Tout ce qui est sélectionné de cette manière peut ensuite être copié, coupé et collé à l'aide des raccourcis clavier standard de / + " C ", / + "X " et / + "V ".

Il est à noter que vous pouvez commenter votre code de différentes manières. Il est très important de commenter votre code pour maintenir la productivité, car cela permet de garder les choses claires et de séparer les sections de code de manière à ce que chaque partie soit évidente et pourquoi elle est là. Cela signifie également que si vous travaillez en équipe, vous pouvez laisser des notes aux autres membres et qu'ils sauront exactement ce que vous avez fait ou à quoi sert une section spécifique de code. Même si vous êtes un développeur solo, laisser des commentaires dans votre code est important, surtout si vous devez quitter le projet et y revenir plus tard.

Commenter votre code est simple, car il vous suffit d'utiliser "// "avant toute ligne de texte ou vous pouvez commenter des paragraphes entiers en utilisant"/* text here */ ". Il existe même deux raccourcis clavier pour commenter et annuler le commentaire des lignes de code sélectionnées: / + "K "pour commenter, et / + + "K "pour annuler le commentaire. Certains commentaires de style JSDoc peuvent également être ajoutés aux scripts pour définir les propriétés des fonctions qu'ils contiennent, de sorte que lorsque vous utilisez les fonctions de votre code, elles s'affichent en saisie semi-automatique avec leurs arguments et autres détails. Vous pouvez en savoir plus à ce sujet dans la section Commentaires de script JSDoc pour plus d'informations.

Par défaut, tout le code écrit aura des icônes de pliage de code à côté de chaque ensemble d'accolades correspondantes {}. Ceux-ci prennent la forme d'un petit plus + ou moins - icônes, et en cliquant dessus "pliera" le code entre les accolades correspondantes afin qu'il occupe moins d'espace visuel.

Vous pouvez également définir manuellement les régions de pliage de code à l'aide de la fonction spéciale #region balises pour marquer une section de code. Comment cela fonctionne, c'est que vous ajoutez un #region balise (avec tout texte de commentaire) en haut d'une section de code, puis ajoutez un #endregion balise en bas de la section et ensuite vous pouvez "plier" le code, c'est-à-dire: le cacher pour pouvoir réduire le nombre de lignes de code visibles dans l'éditeur à tout moment:

Le code réel ressemble à ceci:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Comme vous l'avez peut-être remarqué, certaines parties du texte du script sont colorées. L'éditeur de script "connaît" les objets existants, les variables et fonctions intégrées, etc... et les code en couleur en conséquence. Le codage couleur aide beaucoup à éviter les erreurs, car vous pouvez voir immédiatement si vous avez mal orthographié un nom ou utilisé un mot-clé comme variable, ou si vous avez une erreur de syntaxe. Cependant, si vous n'aimez pas le code couleur, dans les Préférences, vous pouvez l'activer et le désactiver (vous pouvez également utiliser F10 pour l'activer ou le désactiver dans l'éditeur) et personnaliser les couleurs utilisées.

Vous pouvez faire un clic droit n'importe où dans l'éditeur aussi pour ouvrir le menu suivant:

Outre les fonctions habituelles que vous attendez de couper, copier et coller des lignes de code, vous avez également la possibilité de commenter ou de dé-commenter tout texte sélectionné, d'ajouter ou de supprimer des retraits, ainsi que de diviser la fenêtre de code en 1, 2 ou 3 volets séparés (notez que les options relatives à #regions ne sera pas disponible si vous n'en avez défini aucun dans le code en cours d'édition).

La dernière option pour diviser en volets divisera la fenêtre réelle en différentes parties où vous pouvez ensuite faire glisser un script ou un événement et l'afficher à côté des autres pour la comparaison et l'édition.

Ceci est exceptionnellement utile pour comparer deux scripts ou pour afficher des scripts connectés en même temps, sans avoir besoin d'une autre fenêtre ou d'avoir à basculer constamment entre les onglets.

Les autres options de ce menu sont de basculer un point d'arrêt à utiliser avec le débogueur et de convertir le code GML DnD™ ou vice versa. Notez que si vous convertissez du code en DnD™, il ne créera pas d' actions individuelles pour chaque ligne de code, mais utilisera plutôt une action de code unique pour contenir tout ce qui était auparavant dans l'événement ou le script.
