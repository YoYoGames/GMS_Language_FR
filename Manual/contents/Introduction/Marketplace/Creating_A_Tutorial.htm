Créer un didacticiel sur la place de Marketplace

La création d'un didacticiel est similaire à la création d'un package d'actifs Marketplace classique, mais vous devez maintenant ajouter des fichiers inclus spécifiques ainsi qu'une page de didacticiel initiale (et une page vidéo facultative). L'espace de travail du didacticiel ressemblera à ceci: Comme expliqué ci-dessus pour les Marketplace générales du marché, vous devez sélectionner un éditeur, puis attribuer à votre didacticiel un nom et un identifiant unique (format d'URL inversée), puis dans la liste déroulante «type», sélectionnez Didacticiel. Cela vous donnera ensuite des options supplémentaires pour ajouter un fichier HTML de base à partir duquel démarrer le didacticiel, ainsi qu'un fichier vidéo (facultatif) et une URL. Le fichier vidéo doit utiliser le modèle fourni par Marketplace - comme le reste du didacticiel - que vous pouvez obtenir ici, et lorsque l'utilisateur charge le didacticiel, la vidéo sera chargée dans un cadre séparé dans l'EDI. Notez que les pages que vous donnez pour la page de démarrage et la page vidéo sont des chemins relatifs dans les fichiers inclus, et vous pouvez ouvrir un explorateur pour choisir autre chose que ceux par défaut avec lesquels GameMaker Studio 2 remplit les champs.

En fait, la création de votre didacticiel vous obligera à ajouter des fichiers inclus dans votre projet de didacticiel, et ceux-ci doivent être ajoutés d'une manière très spécifique pour que l'EDI détecte correctement les fichiers à utiliser pour le didacticiel. Nous avons donc deux dossiers principaux:

  • TutorialContent: Tous vos tutoriels HTML fichiers, le CSS pour les styliser, et les images nécessaires pour eux (pas les images requises pour les ressources, mais celles à afficher dans le didacticiel), ainsi que le modèle vidéo si nécessaire, allez ici. Vous pouvez également avoir des sous-dossiers si vous souhaitez diviser le didacticiel en chapitres, par exemple, mais vous devez avoir au moins une base HTML fichier à utiliser comme page de démarrage du didacticiel.
  • TutorialResources: Ce dossier contient deux sous-dossiers supplémentaires pour les images et les ressources audio. Si votre didacticiel nécessite que l'utilisateur ajoute un sprite (par exemple) et que vous souhaitez l'inclure dans le didacticiel, vous devez placer la ressource image dans le dossier Images. Ces dossiers sont importés dans le projet du didacticiel et lorsque l'utilisateur ouvre l'éditeur de sprite et ajoute une image, le chemin par défaut de la fenêtre de l'explorateur se trouve à cet emplacement, ce qui permet à l'utilisateur de trouver plus facilement les ressources nécessaires.

Pour que votre didacticiel fonctionne dans l'EDI, vous devez avoir cette structure de dossiers ainsi qu'une base HTML page de démarrage avec un CSS fichier pour le style. Il existe un certain nombre de liens spéciaux qui peuvent être utilisés dans vos projets de didacticiel pour accéder à des parties d'une vidéo ou ouvrir le manuel sur une section ou une fonction spécifique, et ils sont tous fournis dans les modèles de didacticiels officiels de la place de Marketplace.

IMPORTANT! Nous vous suggérons fortement de télécharger les modèles de didacticiels à partir de la place de Marketplace avant de tenter de créer vos propres didacticiels. Ils contiennent des exemples de toutes les parties importantes du HTML ainsi que du CSS et des images pour recréer des boutons et des menus dans le didacticiel, vous laissant libre de vous soucier du contenu plutôt que de la façon de l'afficher.

Une fois que vous avez configuré les fichiers inclus et fourni les informations requises, vous pouvez cliquer sur OK et le didacticiel sera téléchargé sur le Marketplace, ouvrant un espace de travail sur la liste des actifs lorsque vous avez terminé afin que vous puissiez la modifier et ajouter des icônes, du texte, des captures d'écran, etc.... Une fois la ressource du didacticiel publiée, les utilisateurs pourront la trouver dans la section Didacticiels de la page de démarrage, où elle peut être téléchargée et installée.
