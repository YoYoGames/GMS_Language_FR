Création d'un actif de marché

Lorsque vous sélectionnez l'option Créer un paquet dans le menuMarché de l'IDE, vous accédez à l'espace de travail suivant :. Voici vous devez d'abord sélectionner l'identité de l'éditeur sous laquelle vous souhaitez publier le package d'actifs (que vous avez dû configurer depuis votre tableau de bord de l'éditeur), puis donner un nom d'affichage et un numéro de version. Un ID de package sera alors créé pour vous sur la base du nom d'affichage et du compte d'éditeur utilisés, mais vous pouvez le modifier pour qu'il soit différent si vous le souhaitez, tant qu'il conserve le format URL inversé. Vous devez ensuite choisir le type de package de ressources à créer :

  • Actif: Il s'agit d'une collection d'actifs (sprites, scripts, objets, etc...) que vous souhaitez vendre sur le Marché.
  • Démo: Il s'agit d'une démo pour démontrer un concept ou une idée de jeu.
  • Tutoriel: La ressource téléchargée est un tutoriel, conçu pour être exécuté à partir de l'IDE GameMaker Studio 2 (voir cette page pour plus de détails).

Une fois que vous avez fait cela, vous pouvez choisir les actifs du projet que vous voulez ajouter dans le paquet d'actifs en utilisant la section sur la droite. Vous pouvez ajouter toutes vos ressources, ou seulement certaines d'entre elles, et supprimer celles que vous auriez ajoutées par erreur. Nous supposons que vous avez créé les ressources à télécharger précédemment dans GameMaker Studio 2 et que le projet actuel contient toutes les ressources que vous souhaitez télécharger.

RAPPEL ! Pour pouvoir créer ou utiliser des packages d'actifs, vous devez avoir préalablement créé un certificat de profil utilisateur. Vous pouvez le faire à partir des préférences de la place de marché, ce qui est expliqué ici.

Lorsque vous êtes satisfait de la sélection des ressources et que vous souhaitez télécharger votre package sur le Marché, il vous suffit de cliquer sur le bouton OK en bas, et le package de ressources sera créé et téléchargé sur le Marché. Lorsque le téléchargement est terminé, un nouvel espace de travail s'ouvrira avec une vue du navigateur de la page de ressources que vous pourrez modifier. Pour plus d'informations sur la façon de configurer cette fonction, veuillez consulter la page suivante :

  • Inscription d'un actif

Si vous devez mettre à jour un actif Marketplace, vous faites plus ou moins la même chose que lorsque vous avez créé le package original, sauf que maintenant vous devez spécifier le package à mettre à jour à partir d'une liste déroulante de packages disponibles :

Lors de la mise à jour, vous ne pourrez pas modifier l'ID du paquet, et vous devez savoir que le numéro de version affiché sera le numéro de version actuel et que vous devez l'incrémenter comme il convient, sinon la mise à jour échouera (puisque vous ne pouvez pas avoir deux paquets d'actifs avec le même numéro de version). Le site est également un champ dans lequel vous pouvez ajouter du texte pour le journal des modifications de la ressource, qui sera téléchargé avec la ressource et affiché sur la page Marketplace.

IMPORTANT ! Les ressources créées avec GameMaker Studio version 2.3+ ne seront disponibles que pour les utilisateurs utilisant la version 2.3 et supérieure. Si vous mettez à jour une ressource d'une version antérieure, cette ressource sera toujours disponible pour les utilisateurs qui n'ont pas fait la mise à jour et coexistera avec la version 2.3+ mise à jour. qui n'ont pas fait la mise à jour et coexistera avec la version 2.3+ mise à jour.

Téléchargement à travers la page Web

Vous pouvez télécharger des ressources via la page Web Marketplace plutôt que via l'EDI, ce qui peut s'avérer nécessaire si le téléchargement via l'EDI a échoué pour une raison quelconque. Pour commencer, vous devez créer le paquet et essayer de le télécharger via l'IDE comme indiqué ci-dessus. l'IDE comme indiqué ci-dessus, et si cela a échoué, vous pouvez alors aller dans la fenêtre de sortie et vérifier le journal pour une ligne qui montre où le paquet que vous créez a été enregistré avant de tenter le téléchargement. Par exemple :

Sur Windows:

Fichier package créé :
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMakerStudio2\GMS2USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

Sur macOS:

Fichier de paquetage créé :
/Users/MACUSERNAME/.config/GameMakerStudio2/GMS2USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Vous devez vous rendre à l'emplacement de ce fichier et copier le fichier *.YYMP dans un endroit sûr, car il s'agit du fichier que vous allez télécharger sur le marché.

À ce stade, vous devez ouvrir le Marketplace dans votre navigateur Web et naviguer jusqu'à la liste des actifs pour l'actif, puis aller à la section Versions, et ensuite cliquer sur le bouton Télécharger la version, comme indiqué dans l'image ci-dessous :

Vous accédez alors à la page de téléchargement de la nouvelle version où vous pouvez ajouter des informations sur la version en cours de téléchargement, si nécessaire. et également fournir le fichier en cliquant sur le bouton Choose FIle et en sélectionnant le fichier *.YYMP créé précédemment :

L'ajout du fichier permet de télécharger les versions et d'afficher une barre de progression. Lorsque le téléchargement est terminé et que vous êtes satisfait des détails, vous pouvez cliquer sur le bouton Save pour enregistrer la ressource dans la liste. Notez que la ressource sera en phase de traitement pendant quelques minutes avant de pouvoir être publiée.
