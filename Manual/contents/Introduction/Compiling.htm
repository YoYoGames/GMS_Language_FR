Compiler

Compiler votre jeu peut signifier l'une des deux choses suivantes: le compiler pour le tester ou le compiler pour créer un package exécutable ou une plate-forme cible spécifique. La compilation de votre jeu pour le test peut être effectuée en appuyant simplement sur le bouton Lecture en haut de l'EDI, qui lancera le jeu pour le test en utilisant la cible spécifiée. Vous pouvez également exécuter le jeu en mode débogage en testant à l'aide du bouton Déboguer. Cela lancera le jeu, mais ouvrira également la fenêtre de débogage, où vous pourrez surveiller les performances de votre jeu et voir les problèmes éventuels (voir la section sur le débogage pour plus d'informations).

Par défaut, GameMaker Studio 2 s'exécutera et déboguera à l'aide de la machine virtuelle (VM) intégrée, qui est plus ou moins la même que celle exécutée sur le système d'exploitation de bureau utilisé. Cependant, GameMaker Studio 2 est un moteur multiplateforme et vous pouvez tester, déboguer et compiler des packages exécutables de vos projets sur un certain nombre de plates-formes cibles différentes (les plates-formes exactes disponibles dépendront des détails de votre licence). Pour changer la plate-forme cible actuelle, vous pouvez cliquer sur sur le bouton Cibles pour ouvrir la fenêtre des cibles, qui ressemblera à ceci (les détails exacts varieront en fonction de votre type de licence):

En haut, à côté du bouton Cibles, vous avez les paramètres actuels qui vous indiquent la plate-forme et les paramètres spécifiques réellement utilisés, puis le reste de la fenêtre est occupé avec les détails et les options de toutes les cibles disponibles que vous pouvez sélectionnez pour utiliser à la place. Plateformes

Cette section répertorie toutes les plates-formes cibles disponibles. Le contenu de cette liste variera en fonction de la licence dont vous disposez, mais aura toujours au moins la cible «Test». Pour sélectionner une cible, cliquez simplement sur dessus. Cela mettra ensuite à jour le reste des fenêtres d'options pour afficher différents détails en fonction de la plate-forme sélectionnée.

Production

Chaque plateforme cible peut avoir un ou plusieurs formats de sortie, les principaux étant:

  • VM: la cible VM (Virtual Machine) utilise un runner générique pour chaque plateforme, puis interprète le code de votre jeu. En général, cette option est utilisée pour les tests en raison de ses temps de construction plus rapides, mais elle n'offre pas la même amélioration des performances que celle offerte par l'option YYC (si disponible). Vous pouvez l'utiliser pour compiler des packages exécutables pour des jeux plus petits ou des jeux où les performances ne seront jamais un problème.
  • YYC: Le YYC ( YoYo Compiler) prend la GameMaker Studio 2 et la compile en code natif pour la plate-forme cible, "supprimant" les fonctions inutiles et exécutant une foule d'autres techniques d'optimisation pour créer un package exécutable plus petit et plus performant. Cela peut augmenter les performances de vos jeux d'au moins deux ou trois fois, en particulier sur les jeux à forte logique, et est idéal pour les projets plus importants ou gourmands en ressources processeur. Les temps de compilation peuvent prendre plus de temps et vous devez toujours vider le cache du compilateur avant de créer un package d'actifs complet final pour une plate-forme cible. Notez que la cible YYC peut nécessiter l'installation d'outils supplémentaires pour la plate-forme sélectionnée, sinon cela ne fonctionnera pas - vous pouvez trouver plus d'informations à ce sujet dans le centre d'aide de YoYo Games ainsi que sur les pages de préférences des cibles individuelles détaillées dans ce manuel.
  • JavaScript: la cible JavaScript ne sera disponible que pour quelques cibles, comme la cible HTML5, et définit le jeu pour qu'il soit compilé en JavaScript pur.

    Création d'un package exécutable final

    La compilation de votre projet de jeu sur n'importe quelle plate-forme cible nécessitera que vous ayez configuré les bons outils de construction (voir ici ) et que vous ayez également rempli les préférences de plate-forme appropriées. Une fois que vous avez fait cela, vous pouvez simplement cliquer sur le bouton Créer un exécutable dans l'EDI pour démarrer la génération du compilateur ou sélectionnez Créer un exécutable dans le menu Générer. L'une ou l'autre option ouvrira une fenêtre d'explorateur de fichiers dans laquelle vous pourrez donner le nom final que vous souhaitez utiliser pour votre exécutable de jeu avant de cliquer sur Enregistrer pour démarrer le processus de compilation et de construction. Une fois que vous avez fait cela, les fichiers nécessaires seront générés afin que vous puissiez les distribuer comme vous le souhaitez.

    IMPORTANT! Si vous disposez de la licence d'essai gratuite de 30 jours, la compilation d'un exécutable final ne sera pas disponible et vous devrez passer à l'une des licences payantes pour compiler votre jeu en vue de sa distribution.

    REMARQUE: avant de faire une version finale de votre projet pour la publication, vous devez toujours effacer le cache du compilateur d'actifs (à l'aide de l'icône «balai» en haut de l'IDE). GameMaker Studio 2 mettra en cache plusieurs de vos fichiers de jeu pour réduire au minimum la vitesse de compilation.

    Chaque option cible est enregistrée dans un format spécifique à la plate-forme, répertorié ci-dessous:

    • Windows - La compilation pour le Windows général vous demandera d'abord de choisir entre la création d'un programme d' installation ou d'un package Zip, où le programme d'installation sera un seul exécutable qui installera votre jeu, et le fichier Zip sera un seul *.zip formater un fichier compressé avec tous vos fichiers de jeu stockés (les fichiers devront être extraits pour que le jeu fonctionne). Si vous cochez la case intitulée Remember Packaging Option, GameMaker Studio 2 se souviendra du choix pour toutes les compilations futures (cela peut être réinitialisé ou modifié à partir des Préférences Windows ). Pour en savoir plus, consultez le centre d'aide de YoYo Games. Veuillez noter que la Windows peut compiler des exécutables 32 bits ou 64 bits en fonction de ce que vous avez sélectionné dans les options générales du jeu. Nous vous recommandons vivement que tous les nouveaux projets utilisent le x64 Runtime pour créer 64bit executables que le 32bit runtime va obtenir désapprouvée et retirée à l'avenir.
    • Ubuntu ( Linux ) - Les applications Ubuntu ( Linux ) peuvent être compilées comme *.zip fichiers - contenant un exécutable x64 ou ARM - selon le périphérique ciblé. Pour en savoir plus, consultez le centre d'aide de YoYo Games.
    • HTML5 - Si vous avez choisi de créer HTML5, alors un index.html fichier (c'est le nom par défaut, mais vous pouvez également donner votre propre nom dans les préférences de la plate-forme) ainsi qu'un dossier contenant vos fichiers de jeu seront créés et enregistrés à l'emplacement spécifié. Pour que votre jeu fonctionne, vous aurez besoin de ces deux éléments à télécharger sur un serveur. Le index.html peut également être personnalisé pour afficher votre jeu avec une couleur d'arrière-plan différente, ou à une position différente etc... mais une connaissance du HTML est nécessaire pour cela, et vous pouvez également spécifier votre propre fichier d'index personnalisé lorsque vous construisez le package (voir les options de jeu HTML5 ). Pour en savoir plus, consultez le centre d'aide de YoYo Games.
    • Android - Pour Android appareils Android, vous pouvez choisir de créer un *.apk ou un *.aab ( Android App Bundle) dans la fenêtre qui s'affiche pour enregistrer le jeu:

      Le type de fichier que vous choisissez dépendra du magasin que vous souhaitez cibler, avec le *.aab le fichier est requis pour Google Play, tandis que le *.apk Le fichier peut être utilisé sur d'autres magasins. Pour en savoir plus, consultez le centre d'aide de YoYo Games.
    • iOS - La compilation vers iOS créera un xarchive fichier qui est ensuite utilisé dans Xtools pour créer le package iOS Notez que pour compiler pour iOS vous aurez besoin d'un ordinateur Apple Mac exécutant OSX ou supérieur ainsi que des certificats et autorisations appropriés. Pour en savoir plus, consultez le centre d'aide de YoYo Games.
    • macOS - La compilation pour la macOS vous demandera d'abord de choisir entre la création d'un fichier DMG ou Zip:
      "Package as DMG" construira un package d'installation.dmg qui peut être utilisé pour installer le jeu sur un ordinateur Mac. Notez que pendant ce processus de construction, vous verrez le programme d'installation apparaître sur le Mac, qui disparaîtra une fois le processus de construction terminé.

      "Package as Zip" construira soit un *.app fichier ou un *.pkg, selon que vous souhaitez le télécharger ultérieurement sur le Mac App Store ou non. Comme avec iOS vous aurez besoin d'un ordinateur Apple Mac exécutant OSX ou supérieur, ainsi que des certificats et autorisations appropriés. Pour en savoir plus, consultez le centre d'aide de YoYo Games.
    • Windows UWP - Pour Windows UWP, GameMaker Studio 2 créera un *.appxbundle package qui peut ensuite être téléchargé sur le Windows Store ou distribué ailleurs. Lorsque vous cliquez sur le bouton Créer Executable vous serez invité à dire GameMaker Studio 2 quel type de package que vous souhaitez créer (peut être pour ARM, x64 et / ou x86 architectures ), et vous devez choisir les cibles que le jeu sera pris en charge sur (vous devez en sélectionner au moins un, mais vous pouvez sélectionner les trois ou n'importe quelle combinaison):

      Si vous cochez la case Package For Store Upload, le package final créé sera un *.appxupload, qui est ce que Microsoft spécifie à utiliser pour soumettre des applications à leur magasin, comme expliqué dans cet article ici (toutes les architectures choisies seront incluses, tout comme avec le*.appxbundle ). Vous pouvez en savoir plus sur la configuration et la compilation sur la plate-forme UWP dans le centre d'aide de YoYo Games.

    Une fois que vous avez créé votre package d'actifs exécutables, vous pouvez ensuite donner le fichier à d'autres personnes ou le placer sur votre site Web pour le télécharger, ou télécharger ces fichiers vers les différents services d'hébergement pour une distribution individuelle ou même vers des magasins en ligne (comme Google Play, iTunes ou le MS Store) pour la distribution générale et la vente au détail.

    Notez que vous êtes libre de distribuer les jeux que vous créez avec GameMaker Studio 2 comme vous le souhaitez, y compris en les vendant. Bien sûr, cela suppose que les sprites, images et sons que vous avez utilisés pour le créer peuvent également être distribués ou vendus et que vous avez les droits légaux sur tous les actifs, et cela suppose également que le jeu est conforme au CLUF de YoYo Games GameMaker Studio 2.
