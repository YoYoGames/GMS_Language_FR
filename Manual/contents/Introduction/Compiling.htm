Compiler votre jeu peut signifier deux choses : le compiler pour le tester, ou le compiler pour créer un fichier paquet exécutable ou une plate-forme cible spécifique. Pour compiler votre jeu à des fins de test, il suffit d'appuyer sur le bouton Play en haut de l'IDE, ce qui lancera le jeu à des fins de test en utilisant la cible spécifiée. Vous pouvez également exécuter le jeu en mode débogage en le testant à l'aide du bouton Debug . Cela lancera le jeu, mais ouvrira également la fenêtre de débogage (Debug Window), dans laquelle vous pourrez surveiller le fonctionnement du jeu. Sortie

Chaque plateforme cible peut avoir un ou plusieurs formats de sortie, les principaux étant :

  • VM: La cible VM (Virtual Machine) utilise un runner générique pour chaque plateforme et interprète ensuite le code de votre jeu. En général, cette option est utilisée pour les tests en raison de ses temps de construction plus rapides, mais elle n'offre pas le même gain de performance que l'option YYC (si disponible). performances que l'option YYC (si elle est disponible). Vous pouvez utiliser cette option pour compiler des paquets exécutables pour des jeux plus petits ou des jeux pour lesquels les performances ne seront jamais un problème.
  • YYC: Le YYC(YoYo Compiler) prend la sortie normale de GameMaker Studio 2 et la compile en code natif pour la plate-forme cible, en supprimant les fonctions inutiles et en effectuant une multitude d'autres techniques d'optimisation. afin de créer un paquet exécutable plus petit et plus performant. Les performances de vos jeux peuvent être multipliées par deux ou trois, notamment pour les jeux à forte composante logique, et sont idéales pour les projets de grande envergure ou à forte intensité de traitement. Les temps de compilation peuvent Les temps de compilation peuvent être plus longs et vous devez toujours vider le cache du compilateur avant de créer un paquet d'actifs complet pour une plateforme cible. Notez que la cible YYC peut nécessiter l'installation d'outils supplémentaires pour la plate-forme sélectionnée, sinon elle ne fonctionnera pas. vous trouverez de plus amples informations à ce sujet dans le centre d'aide deYoYo Games ainsi que sur les pages de préférences des cibles individuelles. détaillées dans ce manuel.
  • JavaScript: La cible JavaScript ne sera disponible que pour quelques cibles, comme la cible HTML5, et permet de compiler le jeu en pur JavaScript.

Création d'un paquet final exécutable

La compilation de votre projet de jeu sur n'importe quelle plateforme cible nécessitera que vous ayez mis en place les outils de compilation appropriés. outils de compilation (voir ici) et que vous ayez également rempli les préférences de plate-forme appropriées. Une fois que vous avez fait cela, vous pouvez simplement cliquer sur le bouton Create Executable dans l'IDE pour lancer la compilation ou sélectionner Create Executable dans le menu Build. L'une ou l'autre de ces options ouvrira une fenêtre d'exploration de fichiers où vous pourrez donner le nom final que vous souhaitez utiliser pour l'exécutable du jeu. que vous souhaitez utiliser pour l'exécutable de votre jeu avant de cliquer sur Save pour lancer le processus de compilation et de construction. Une fois que vous avez fait cela, les fichiers nécessaires seront générés afin que vous puissiez le distribuer comme vous le souhaitez.

IMPORTANT ! Si vous disposez de la licence d'essai gratuite de 30 jours, la compilation d'un exécutable final ne sera pas disponible et vous devrez passer à l'une des licences payantes pour compiler votre jeu en vue de sa distribution.

REMARQUE: avant d'effectuer une construction finale de votre projet en vue de sa diffusion, vous devez toujours vider le cache du compilateur d'actifs (en utilisant l'icône "balai" en haut de l'IDE). GameMaker Studio 2 mettra en cache un grand nombre de vos fichiers de jeu afin de réduire au maximum la vitesse de compilation.
