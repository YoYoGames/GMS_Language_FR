La fenêtre de sortie

Lorsque vous ouvrez un projet dans GameMaker Studio 2 pour la première fois, la fenêtre de sortie s'affichera ancrée au bas de l'écran. Cette fenêtre ancrée contient divers onglets qui affichent les différentes informations de sortie de votre projet, selon certaines circonstances. Le dock peut être fermé en cliquant sur le bouton en bas de l' EDI, et vous pouvez cliquer sur et faites glisser n'importe quel onglet de la fenêtre ancrée vers un autre dock pour changer sa position, ou vous pouvez cliquer dans la barre de contenu du dock et faites-le glisser dans l'espace de travail pour créer une fenêtre autonome:

Vous pouvez également cliquer sur et faites glisser une sortie ancrée dans une autre pour créer une fenêtre de sortie de vue fractionnée, comme indiqué dans l'exemple ci-dessous où les deux sorties d'erreur ont été placées dans le même onglet (vous pouvez cliquer lentement sur un onglet pour changer son nom aussi):

Vous pouvez récupérer l'état par défaut de l'EDI à tout moment en sélectionnant Réinitialiser la mise en page dans la fenêtre des mises en page et vous pouvez rouvrir n'importe quel onglet fermé à partir du menu Fenêtre. Vous pouvez également modifier la façon dont le contenu de chacune des fenêtres de sortie est affiché en modifiant les préférences de la fenêtre de sortie.

Production

La fenêtre de sortie générale est l'endroit où toutes les sorties du compilateur sont affichées, ainsi que tous les messages que vous avez peut-être ajoutés à votre jeu en utilisant leshow_debug_message() fonction. La plupart des informations initiales affichées sont simplement des informations de débogage sur la façon dont le jeu est en cours de construction et, en tant que telles, peuvent généralement être ignorées. Cependant, si vous rencontrez un problème lors de la construction de votre projet pour une plate-forme cible, cela peut s'avérer utile pour trouver la cause et fournir des informations pour le support si vous les contactez. Notez que vous pouvez modifier la quantité d'informations affichées ici à partir de la section Compilation des Préférences.

Résultats de recherche

Vous pouvez ouvrir la fenêtre Rechercher et remplacer à l'aide du raccourci clavier / + + " F ", ou allez dans le menu Edition. Une fois que vous avez entré vos termes de recherche, les résultats seront affichés dans cette fenêtre avec le format:

[object] - [event] - [Line Number]: [search string]

Si le terme de recherche se trouve dans un script, ce sera simplement:

[script] - [Line Number]: [search string]

Vous pouvez ensuite double-cliquer sur l'une des entrées renvoyées pour ouvrir l'actif donné à la bonne position pour l'édition.

Contrôle de la source

Cette fenêtre affichera toute la sortie de votre plugin SCM. Vous pouvez en savoir plus sur la configuration du contrôle de la source ici.

Points d'arrêt

Les points d'arrêt sont des emplacements dans votre code de jeu ou DnD™ où vous avez demandé à GameMaker Studio 2 de suspendre l'exécution du projet en mode débogage. Vous pouvez ajouter un point d'arrêt n'importe où dans la boucle de jeu en utilisant la touche " F9 ", et quand vous le faites, il apparaîtra dans cet onglet de sortie. Vous pouvez les activer et les désactiver (sans les supprimer) en cliquant sur la case à cocher à gauche et si vous utilisez le bouton droit de la souris sur l'un, vous obtenez un petit menu qui vous permet de supprimer le point d'arrêt ou d'ouvrir la DnD™ où se trouve le point d'arrêt.

Erreurs de syntaxe

Les erreurs de syntaxe sont les erreurs dans votre code que le vérificateur de syntaxe de GameMaker Studio 2 Il existe de nombreuses raisons pour lesquelles vous pourriez obtenir une erreur de syntaxe - comme avoir un ";" dans votre code lorsqu'un ":" est attendu - et ceux-ci doivent être corrigés avant d'essayer de tester ou de compiler votre jeu. Chaque entrée d'erreur de syntaxe dans la fenêtre de sortie peut être double-cliquée à l'aide du bouton gauche de la souris pour ouvrir une fenêtre à la position marquée afin qu'elle puisse être résolue.

Les erreurs de syntaxe seront mises à jour au fur et à mesure que vous écrivez votre code (il y aura une courte pause entre la saisie de quelque chose et les erreurs apparaissant dans cette fenêtre pour éviter que des erreurs ne soient signalées pour du code non terminé), et suivront le format:

[object] - [event] - [Line Number] - [Position In Line]: [error string]

Ou si l'erreur est dans un script, ce sera:

[script] - [Line Number] - [Position in line]: [error string]

Notez que les erreurs de syntaxe empêcheront généralement le jeu d'être compilé, mais il y a des erreurs qui seront signalées qui n'empêcheront pas la compilation, mais qui devraient néanmoins être résolues. Les erreurs en question sont:

  • pour les variables déclarées qui ne sont utilisées nulle part
  • pour les variables utilisées quelque part qui n'ont pas encore été déclarées

En général, l'un ou l'autre suggérerait une faute de frappe dans un nom de variable, ou un autre type d'erreur facile à faire - et ils sont donc signalés pour que vous puissiez les réviser - mais cela peut aussi être intentionnel, car vous avez peut-être déclaré la variable pour une utilisation future, mais je n'ai pas encore réussi à l'utiliser. C'est pourquoi ces erreurs permettront toujours à votre jeu de se compiler. Pour une liste complète des messages d'erreur de syntaxe possibles, veuillez cliquer ici.

Erreurs de compilation

Une erreur de compilation se produit lorsque votre jeu rencontre un type d'erreur que le vérificateur de syntaxe n'a peut-être pas été en mesure de détecter, ou lorsqu'une erreur est liée à la façon dont vous avez configuré les options de compilation. Lorsque cela se produit, votre jeu vous donnera une erreur de compilation et cessera de fonctionner. Toutes les erreurs du compilateur seront également affichées dans la fenêtre Sortie générale, mais elles seront également répertoriées ici séparément (puisqu'elles peuvent être «perdues» dans le reste de la sortie générale).

Les messages d'erreur du compilateur suivront tous le même format:

[object] - [event] - [Line Number]: [error string]

Si l'erreur est trouvée dans un script, ce sera simplement:

[script] - [Line Number]: [error string]

Vous pouvez ensuite double-cliquer sur l'une des entrées d'erreur du compilateur pour ouvrir l'actif donné à la position marquée comme donnant l'erreur. Pour une liste complète des messages d'erreur possibles, veuillez consulter ici.
