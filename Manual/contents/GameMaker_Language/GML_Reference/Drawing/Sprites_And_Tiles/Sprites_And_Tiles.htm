Sprites et tuiles

Bien que vous puissiez configurer des arrière-plans dans l'éditeur de salle et que vous puissiez assigner des sprites aux instances, souvent cela n'est pas assez flexible pour le jeu que vous créez. Par exemple, si vous avez une instance qui doit dessiner un menu, vous pouvez vouloir afficher plusieurs sprites de boutons à la fois. ou si vous avez un jeu avec des arrière-plans défilants en parallaxe, vous devrez être en mesure de les définir par le biais du code d'une instance de contrôleur quelconque. Dans ces situations, vous devriez utiliser les fonctions suivantes qui peuvent dessiner des sprites et des arrière-plans n'importe où dans la pièce. Certaines de ces fonctions vous permettent également de manipuler la forme, la taille, la couleur et l'alpha du sprite pour obtenir des effets spéciaux.

Vous pouvez affecter un sprite à une instance et, par défaut, GameMaker Studio 2 le dessinera avec les propriétés données par les différentes variables du sprite, Cependant, si vous ajoutez quoi que ce soit dans l'événement draw, vous "prenez le contrôle" et devez indiquer à GameMaker Studio 2 ce qu'il faut dessiner exactement. Cela signifie que si vous devez utiliser les fonctions suivantes pour définir exactement ce que l'instance doit et à quel endroit de la pièce elle doit le dessiner :

  • dessiner_soi
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • dessiner_sprite_part
  • dessiner_sprite_part_ext.
  • dessiner_sprite_étiré
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • dessiner_sprite_inclinaison
  • draw_sprite_tiled_ext

Si vous utilisez des sprites au format SWF, il existe des options supplémentaires pour contrôler l'anticrénelage utilisé lors du dessin (voir Importation de sprites non bitmap pour plus d'informations sur ces types de sprites). plus d'informations sur ces types de sprites). Notez que ces fonctions fonctionnent à un niveau global et définiront l'AA sur tous les sprites SWF vectoriels dessinés après leur appel.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Outre les fonctions énumérées ci-dessus, il existe également deux fonctions spéciales conçues pour être utilisées avec les animations squelettiques (voir Importation de sprites non bitmap pour de plus amples informations sur ces types de sprites et la section Animations squelettiques pour les fonctions spécifiques liées à ces types de sprites). plus d'informations sur ces types de sprites, et la section Animations squelettiques pour les fonctions spécifiques liées à ces types de sprites) :

  • dessiner_squelette
  • draw_skeleton_instance
  • dessiner_squelette_collision
  • draw_skeleton_time

Les tuiles sont dessinées automatiquement en fonction des données qu'elles contiennent, du jeu de tuiles utilisé et des propriétés de la carte des tuiles. Vous pouvez toutefois dessiner des tuiles individuelles ou des cartes de tuiles à l'aide des fonctions suivantes :

  • dessiner_carreau
  • dessiner_carreau
