dessiner_chemin

Avec cette fonction, vous pouvez demander à GameMaker Studio 2 de dessiner un chemin à l'écran. Le chemin sera dessiné comme une simple ligne, et peut être soit relatif à l'instance appelante, soit à la position absolue à laquelle il a été créé dans l'éditeur de chemin ou par le biais du code. Cette fonction est extrêmement utile pour déboguer les chemins dynamiques (par exemple, ceux créés pour les instances avec la fonction mp_grid_path() ).

NOTE: Si vous souhaitez dessiner une forme à l'aide d'un shader, sachez que la plupart des shaders attendent les entrées suivantes : vertex, texture, couleur. Cependant, en utilisant cette fonction, seules les données de vertex et de couleur sont transmises, et donc le shader peut ne rien dessiner (ou dessiner quelque chose mais pas correctement). shader peut ne rien dessiner (ou dessiner quelque chose mais pas correctement). Si vous avez besoin de dessiner des formes de cette manière, le shader doit être personnalisé en conséquence.

Syntaxe :

draw_path(path, x, y, absolute) ;

Arguments Description
chemin Le chemin à dessiner
x La coordonnée x de l'endroit où le chemin est dessiné
y La coordonnée y de l'endroit où le chemin est dessiné
absolu Si le chemin est dessiné à la position absolue(true) ou à la position relative(false).

Les retours :

N/A

Exemple :

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
draw_path(path, x, y, false) ;
}

Le code ci-dessus utilisera la fonction mp_grid_path pour générer un chemin et le stocker dans la variable "path". Si le chemin est créé avec succès, il est ensuite dessiné à l'écran à une position relative à l'instance qui exécute le code.
