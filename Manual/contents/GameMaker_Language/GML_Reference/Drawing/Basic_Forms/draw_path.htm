draw_path

Avec cette fonction, vous pouvez demander à GameMaker Studio 2 de tracer un chemin vers l'écran. Le chemin sera dessiné comme une simple ligne et peut être relatif à l'instance appelante ou à la position absolue à laquelle il a été créé dans l'éditeur de chemin ou via le code. Cette fonction est extrêmement utile lors du débogage de chemins dynamiques (par exemple, ceux créés pour des instances avec le mp_grid_path() fonction).

REMARQUE: Si vous souhaitez dessiner une forme à l'aide d'un shader, sachez que la plupart des shaders attendent les entrées suivantes: sommet, texture, couleur. Cependant, lors de l'utilisation de cette fonction, seules les données de sommet et de couleur sont transmises, et le shader peut ne rien dessiner (ou dessiner quelque chose mais pas correctement). Si vous devez dessiner des formes de cette manière, le shader doit être personnalisé dans cet esprit.

Syntaxe:

draw_path(path, x, y, absolute);

Argument Description
path Le chemin à dessiner
x La coordonnée x de l'endroit où le chemin est dessiné
y La coordonnée y de l'endroit où le chemin est dessiné
absolute Si le tracé est tracé à la position absolue (true ) ou la position relative (false )

Retour:

N/A

Exemple:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
    {
    draw_path(path, x, y, false);
    }

Le code ci-dessus utilisera le mp_grid_path fonction pour générer un chemin et le stocker dans la variable "chemin". Si le chemin est créé avec succès, il est ensuite dessiné à l'écran à une position relative à l'instance exécutant le code.
