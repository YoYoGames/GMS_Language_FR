surface_exists

Cette fonction est essentielle lorsque vous travaillez avec des surfaces en raison de leur nature volatile. Les surfaces sont toujours conservées dans la mémoire de texture, ce qui signifie qu'elles peuvent être détruites d'un instant à l'autre (par exemple, lorsqu'un économiseur d'écran démarre sur Windows, ou lorsqu'il est minimisé sur un Android ), vous devez donc toujours vérifier qu'une surface existe avant faire quoi que ce soit avec lui (cela inclut le dessiner à l'écran). L'exemple de code ci-dessous montre une utilisation typique de cette commande dans l'événement draw d'une instance pour rechercher une surface et la recréer si elle a été supprimée (notez que la surface aura été créée à l'origine dans l'événement create de l'objet ).

Syntaxe:

surface_exists(surface_id);

Argument Description
surface_id L'ID de la surface à vérifier.

Retour:

Boolean

Exemple:

if !surface_exists(surf)
    {
    surf = surface_create(1024, 1024);
    }
else
    {
    if view_current = 1
        {
        draw_surface(surf,0,0);
        }
    }

Le code ci-dessus vérifiera si une surface indexée dans la variable "surf" existe, et si ce n'est pas le cas, il la recréera. S'il existe, il vérifie alors quelle vue est actuellement dessinée et s'il s'agit d'une vue [1], il dessine la surface.
