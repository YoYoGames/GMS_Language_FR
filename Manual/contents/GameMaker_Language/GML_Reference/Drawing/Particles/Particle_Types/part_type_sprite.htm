part_type_sprite

Cette fonction peut être utilisée pour définir un type de particule pour utiliser un sprite personnalisé à partir des ressources du jeu. Vous pouvez sélectionner le sprite, puis dire à GameMaker Studio 2 d'utiliser le sprite et de l'animer ou non, et si vous choisissez animé, vous pouvez également choisir de modifier les sous-images à un intervalle tel que l'animation commence et se termine. au début et à la fin de la durée de vie de chaque particule créée (de sorte que si vous avez une durée de vie définie sur un minimum de 15 pas et un maximum de 30 pas, quelle que soit la valeur de la durée de vie, l'animation sera adaptée, avec une exécution plus rapide pour les durées de vie plus courtes et plus lentes pour les plus longues). Vous pouvez également choisir de choisir une sous-image aléatoire de sorte que si le sprite n'est pas animé, une image différente soit choisie au hasard pour créer le sprite, ou si la particule est animée, elle démarrera l'animation à un point aléatoire.

Syntaxe:

part_type_sprite(ind, sprite, animate, stretch, random);

Argument Description
ind L'indice du type de particule à modifier.
sprite L'indice de l'image-objet sur lequel définir le type de particule.
animate S'il faut suivre l'animation du sprite normalement (vrai) ou non (faux).
stretch Indique s'il faut étirer l'animation de l'image-objet pour qu'elle corresponde à la durée de vie des particules (vrai) ou non (faux).
random Choisir une sous-image aléatoire (vrai) ou non (faux).

Retour:

N/A

Exemple:

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

Le code ci-dessus définit le type de particule indexé dans la variable globale "p1" sur la forme du sprite indexé dans "spr_Coins", puis définit diverses autres propriétés de particule.
