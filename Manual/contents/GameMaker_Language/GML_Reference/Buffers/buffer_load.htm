Chargement de la mémoire tampon

Cette fonction est utilisée pour charger un tampon qui a été précédemment sauvegardé à l'aide des fonctions buffer_save() fonctions, ainsi que tous les fichiers inclus ou les fichiers chargés en externe. Elle renvoie l'ID du nouveau tampon qui est créé par la fonction comme un tampon "grow" (voir ici) avec un alignement d'octets de 1. Cet ID doit être stocké dans une variable et utilisé dans tous les appels de fonction ultérieurs à ce tampon. appels de fonction ultérieurs à ce tampon. Si le chargement échoue pour une raison quelconque, la fonction renverra -1 et un message sera affiché dans la fenêtre de sortie du compilateur indiquant que le chargement a échoué.

Notez que sur HTML5, tous les tampons chargés depuis le stockage local seront décodés en utilisant base64 (puisqu'ils sont enregistrés en tant que chaînes base64). Cependant, l'utilisation de cette fonction sur les fichiers inclus ou sur tout fichier externe n'utilisera pas le décodage base64, ce qui devrait convenir dans la plupart des cas.

REMARQUE: il est important de supprimer de la mémoire toute ressource créée dynamiquement comme celle-ci lorsque vous n'en avez plus besoin afin d'éviter les fuites de mémoire. Ainsi, lorsque vous avez fini d'utiliser le tampon que vous avez créé, vous devez le libérer à nouveau. en utilisant buffer_delete().

Syntaxe :

buffer_load(nom du fichier) ;

Arguments Description
nom de fichier Le nom du fichier à charger.

Les retours :

réel

Exemple :

player_buffer = buffer_load("Player_Save.sav") ;

Le code ci-dessus va charger en mémoire un tampon précédemment enregistré, créant ainsi un nouveau tampon. L'index de ce nouveau tampon est stocké dans la variable "player_buffer".
