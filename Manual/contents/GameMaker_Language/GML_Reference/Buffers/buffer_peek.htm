buffer_peek

Avec la fonction buffer_read() vous pouvez lire les données d'un tampon donné à la position actuelle de "recherche", chaque donnée étant lue en avançant cette position des octets lus ou écrits. Cependant, il peut être nécessaire de lire un élément de données donné sans vouloir modifier la position de recherche actuelle, et c'est dans ce cas que vous utiliserez cette fonction. Il vous suffit de fournir à la fonction un identifiant de tampon, puis une position de décalage (à partir du début du tampon) dans ce tampon, afin d'obtenir une position de recherche. du début de la mémoire tampon) dans cette mémoire tampon à lire, ainsi que le type de données que vous souhaitez lire.

REMARQUE: L'utilisation d'un type de données incorrect pour les données lues entraînera des valeurs erronées !

Si la fonction réussit, une valeur (Real, string, int, etc...) sera renvoyée en fonction du type de données, tandis qu'un échec renverra simplement 0.

Syntaxe :

buffer_peek(buffer, offset, type) ;

Arguments Description
tampon L'index du tampon à utiliser.
décalage La position de décalage (en octets) dans le tampon pour lire les données données.
type Le type de données qui doit être lu à partir du tampon (voir la liste des constantes ici).

Les retours :

Real/String ou 0 s'il échoue.

Exemple :

var rouge = buffer_peek(buff, 1, buffer_u8) ;
var vert = buffer_peek(buff, 2, buffer_u8) ;
var bleu = buffer_peek(buff, 3, buffer_u8) ;
image_blend = make_colour_rgb(red, green, blue) ;

Le code ci-dessus récupère trois valeurs à partir de trois positions différentes dans le tampon indexé dans la variable "buff" et utilise ensuite ces valeurs pour définir le mélange d'images de l'instance.
