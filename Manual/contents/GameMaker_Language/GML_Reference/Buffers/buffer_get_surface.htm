buffer_get_surface

Avec cette fonction, vous pouvez écrire les informations d'une surface dans un tampon donné. Le tampon doit avoir été créé précédemment et doit être un tampon de croissance aligné sur 1 octet d'une taille approximative (en octets) de la surface que vous allez écrire. Les informations pour la surface seront toujours écrites au début du tampon (quelle que soit la position de recherche) mais vous pouvez choisir une valeur de décalage (en octets) pour déplacer la position d'écriture là où vous le souhaitez à partir du début du tampon. Notez que la fonction écrit chaque pixel de la surface dans la mémoire tampon en utilisant un formatage BGRA sur la Windows, mais sur d'autres cibles, cela peut être différent selon le système d'exploitation ou même le périphérique.

REMARQUE: cette fonction a changé depuis la mise à jour 2.3.1 de GameMaker Studio Les versions précédentes utilisaient la constante buffer_surface_copy, qui est désormais obsolète.

Syntaxe:

buffer_get_surface(buffer, surface, offset);

Argument Description
buffer L'index du tampon à utiliser.
surface L'indice de la surface à utiliser.
offset La valeur de décalage des données (en octets).

Retour:

N/A

Exemple:

buffer_get_surface(buff, application_surface, 0);

Le code ci-dessus copiera les données stockées dans la surface de l'application vers le tampon indexé dans la variable "buff" sans décalage.
