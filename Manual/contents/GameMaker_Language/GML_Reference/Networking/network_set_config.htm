network_set_config

Avec cette fonction, vous pouvez définir différentes configurations de réseau.

REMARQUE: cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet exécuté sur la cible HTML5.

Ces configurations sont données sous forme de constantes et le réglage dépendra de la constante que vous avez choisie. Le tableau ci-dessous répertorie les constantes disponibles et leurs paramètres:

Constant Description Réglage
network_config_connect_timeout Définir une valeur de délai d'expiration de la connexion A value in milliseconds
network_config_use_non_blocking_socket Dites à GameMaker Studio 2 ne pas bloquer lors de la connexion. 0 = block, 1 = don't block
network_config_enable_reliable_udp Active le protocole "UDP fiable" pour un socket UDP existant The socket ID to target
network_config_disable_reliable_udp Désactive le protocole «UDP fiable» pour un UDP existant enfermé. The socket ID to target
network_config_avoid_time_wait Définit leSO_LINGER valeur du délai d'expiration à 0 pour un socket TCP existant The socket ID to target



La première option (network_config_connect_timeout ) définit simplement le délai de connexion à un serveur mais ne change pas le comportement de connexion en dehors du temps d'attente. Il est recommandé de ne pas définir ce paramètre trop bas et de le maintenir à environ 1000 pour un jeu LAN uniquement ou environ 4000 pour Internet, sinon le jeu peut ne pas se connecter de manière aléatoire. Si vous souhaitez définir une valeur de délai d'attente pour l'envoi / la réception de paquets, utilisez la fonction network_set_timeout().

La deuxième option (network_config_use_non_blocking_socket ) signifie que les fonctions de connexion au réseau renverront toutes un socket_id instantanément, mais vous ne pouvez pas envoyer ni recevoir sur ce socket tant que vous n'avez pas reçu un événement réseau asynchrone. L'événement déclenché aura la clé «type» définie sur network_type_non_blocking_connect (vous pouvez trouver plus de détails sur la page sur l' événement asynchrone de mise en réseau ). Il s'agit d'un paramètre global GameMaker Studio 2 ne prend pas en charge un mélange de blocage et de non-blocage dans une seule application.

Les troisième et quatrième options (network_config_enable_reliable_udp, network_config_disable_reliable_udp ) permettent d'activer ou de désactiver le GameMaker Studio 2 et, en tant que tel, ne doivent être utilisés qu'avec des connexions UDP. Ce qui signifie «fiable» dans ce cas, c'est que, lorsqu'il est activé aux deux extrémités d'une connexion, GameMaker Studio 2 tentera de vérifier que vos paquets sont arrivés correctement et de renvoyer ceux qui n'arrivent pas (notez que nous ne garantissons pas dans vos paquets arrivera dans l'ordre de transmission). Lors de l'utilisation d'un type de socket fiable, il y aura un en-tête de 12 octets ajouté à tous les paquets udp qui contient les informations nécessaires à GameMaker Studio 2 pour vérifier le paquet pour les erreurs et renvoyer les paquets manquants. Les points suivants méritent d'être notés lors de l'utilisation de cette option:

  • Le protocole doit être activé / désactivé des deux côtés avant d'envoyer vos données, et toutes les données envoyées ou reçues avant que l'option ait été activée / désactivée ne seront pas affectées.
  • Vous pouvez activer ou désactiver le protocole pour un ID de socket donné plusieurs fois, à condition que cela soit correctement géré à la fois côté serveur et côté client
  • Il est possible d'avoir deux sockets UDP séparés, l'un avec UDP fiable activé pour l'un avec UDP fiable désactivé pour l'autre.

La dernière option,network_config_avoid_time_wait est pour les connexions TCP uniquement et peut être utilisé pour définir le SO_LINGER valeur à 0 o éviter un grand nombre de connexions dans le TIME_WAIT state, bloquant toutes les ressources disponibles sur un serveur. Ce changement est permanent tant que le socket est actif et ne peut pas être annulé.

Syntaxe:

network_set_config(config_value, setting);

Argument Description
config_value La constante de valeur de configuration à définir (répertoriée ci-dessous).
setting Le réglage de la valeur de configuration.

Retour:

String

Exemple:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

Le code ci-dessus définira le délai d'expiration de la connexion réseau à GameMaker Studio 2 ne pas bloquer lors de la connexion.
