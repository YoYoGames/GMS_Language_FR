network_send_udp

Avec cette fonction, vous pouvez envoyer des données sur le réseau en utilisant UDP à un serveur. La fonction prend l' identifiant de socket pour se connecter, l'URL à laquelle se connecter et le port à utiliser. Vous devez ensuite fournir le buffer id qui contient les données à envoyer (pour plus d'informations sur les buffers voir Référence - Buffers ) et enfin la taille (en octets) des données. UDP est "sans connexion" en ce sens que vous ne faites pas réellement de connexion, vous envoyez simplement un paquet directement à une adresse IP, et le serveur reçoit les données entrantes d'une adresse IP et doit les traiter "telles quelles". La fonction renverra le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué. Il est à noter que la taille finale des données envoyées qui sont renvoyées par cette fonction comprendra également les GameMaker, qui représentent 12 octets supplémentaires.

REMARQUE: Cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet exécuté sur la cible HTML5, et les projets HTML5 ne pourront pas non plus recevoir UDP.

Syntaxe:

network_send_udp(socket, url, port, buffer, size);

Argument Description
socket L'identifiant du socket à utiliser.
url L'url ou l'adresse IP à laquelle se connecter (une chaîne).
port Le port auquel se connecter.
buffer L'identifiant du tampon à partir duquel obtenir les données.
size La taille (en octets) des données.

Retour:

Real

Exemple:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Le code ci-dessus enverra un paquet UDP au serveur défini par l'URL sur le port 6510. Les données sont extraites du tampon indexé dans la variable "buff".
