réseau_send_broadcast

Cette fonction permet de diffuser localement les données d'un tampon vers une série d'adresses IP (pour plus d'informations sur les tampons, voir Référence - Tampons). La plage est limitée à celle de l'appareil qui exécute le serveur, Ainsi, si l'adresse IP du périphérique est 92.168.11.130, les données seront diffusées sur la plage 92.168.11.*. La fonction renvoie le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué.

REMARQUE: Cette fonction ne fonctionne que lorsqu'elle est utilisée avec UDP - votre serveur doit être TCP et votre client doit avoir une socket client UDP créée avec la fonction network_create_socket_ext() afin de recevoir les diffusions envoyées par le serveur.

NOTE: Cette fonction ne fonctionnera pas si elle est utilisée dans un projet fonctionnant sur la cible HTML5.

Syntaxe :

network_send_broadcast(socket, port, buffer, size) ;

Arguments Description
prise de courant L'identifiant du socket à utiliser.
port Le port que le serveur utilisera.
tampon L'identifiant du tampon à partir duquel les données sont récupérées.
taille La taille (en octets) des données.

Les retours :

Real

Exemple :

buffer_seek(broadcast_buffer, buffer_seek_start, 0) ;
buffer_write(broadcast_buffer, buffer_string, global.ServerName) ;
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer)) ;

Le code ci-dessus écrit la chaîne du nom du serveur actuel (stocké dans "global.ServerName"), puis l'écrit dans un tampon binaire avec l'id "broadcast_buffer". Ces données sont ensuite diffusées localement vers une plage d'adresses IP (l'IP du périphérique est actuellement implicite comme plage de base de diffusion) au port 6511.
