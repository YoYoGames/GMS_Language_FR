network_send_broadcast

Avec cette fonction, vous pouvez diffuser les données d'un tampon localement vers une plage d'adresses IP (pour plus d'informations sur les tampons, voir Référence - Tampons ). La plage est limitée à celle du périphérique exécutant le serveur, de sorte que si le périphérique a une adresse IP de 92.168.11.130, les données seront diffusées sur la plage 92.168.11. *. La fonction renverra le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué.

REMARQUE: Cette fonction ne fonctionnera que lorsqu'elle est utilisée avec UDP - votre serveur doit être TCP et votre client doit avoir un socket client UDP créé avec network_create_socket_ext () afin de recevoir toutes les diffusions envoyées par le serveur.

REMARQUE: cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet exécuté sur la cible HTML5.

Syntaxe:

network_send_broadcast(socket, port, buffer, size);

Argument Description
socket L'identifiant du socket à utiliser.
port Le port que le serveur utilisera.
buffer L'identifiant du tampon à partir duquel obtenir les données.
size La taille (en octets) des données.

Retour:

Real

Exemple:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Le code ci-dessus écrit la chaîne de nom du serveur actuel (stocké dans "global.ServerName"), puis l'écrit dans un tampon binaire avec l'id "broadcast_buffer". Ces données sont ensuite diffusées localement vers une plage d'adresses IP (l'IP de l'appareil est actuellement implicite comme plage de base de diffusion) vers le port 6511.
