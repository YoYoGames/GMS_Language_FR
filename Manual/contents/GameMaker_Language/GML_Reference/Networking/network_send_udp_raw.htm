réseau_send_udp_raw

Avec cette fonction, vous pouvez envoyer des données sur le réseau en utilisant UDP à un serveur. La fonction prend l'identifiant de la prise par laquelle se connecter, l'URL à laquelle se connecter et le port à utiliser. Vous devez ensuite fournir l'identifiant du tampon qui contient les données à être envoyées (pour plus d'informations sur les tampons, voir Référence - Tampons) et enfin la taille (en octets) des données. UDP est "sans connexion" dans le sens où vous n'effectuez pas de connexion, vous envoyez juste un paquet directement à une IP. directement à une adresse IP, et le serveur reçoit les données entrantes d'une adresse IP et doit les traiter "telles quelles". Les données envoyées ne sont en aucun cas formatées par GameMaker Studio 2 et les appareils récepteurs recevront les données sous forme de flux. ce qui signifie que vous devrez les gérer vous-même. La fonction renvoie le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué.

NOTE: Cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet fonctionnant sur la cible HTML5, et les projets HTML5 ne pourront pas non plus recevoir d'UDP.

Syntaxe :

network_send_udp_raw(socket, url, port, buffer, size) ;

Arguments Description
prise de courant L'identifiant du socket à utiliser.
url L'url ou l'IP à laquelle se connecter (une chaîne).
port Le port auquel se connecter.
tampon L'identifiant du tampon à partir duquel les données sont récupérées.
taille La taille (en octets) des données.

Les retours :

Real

Exemple :

network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff)) ;

Le code ci-dessus va envoyer un paquet UDP brut au serveur défini par l'URL sur le port 6510. Les données sont extraites du tampon indexé dans la variable "buff".
