ligne de collision

Collision_line vérifie le long d'une ligne allant du point x1,y1 au point x2,y2 s'il y a collision avec une instance de l'objet spécifié par l'argument "obj". cette vérification peut être précise ou non, mais pour que les collisions précises soient activées, l'objet ou l'instance que vous vérifiez doit également avoir des collisions précises activées pour son sprite. collisions précises soit activée, l'objet ou l'instance que vous vérifiez doit également avoir des collisions précises activées pour son sprite. Sinon, la vérification par défaut est basée sur les boîtes englobantes. L'image suivante illustre ce fonctionnement :

N'oubliez pas que pour que les collisions précises soient prises en compte, le sprite de l'objet et la fonction de collision doivent tous deux être activés. et la fonction de collision doivent être marqués comme précis. Il faut également noter que la valeur de retour de la fonction peut être l'identifiant de n'importe laquelle des instances considérées comme étant en collision le long de la ligne. ligne définie, n'importe lequel de leurs ids pourrait être la valeur de retour de la fonction.

Notez qu'au lieu d'un index d'objet, vous pouvez fournir une instance id pour rechercher une instance spécifique, ou les mots-clés d'instance all, ou other (selon l'événement et l'étendue du code actuel).

Syntaxe :

collision_line(x1, y1, x2, y2, obj, prec, notme) ;

Arguments Description
x1 La coordonnée x du début de la ligne.
y1 La coordonnée y du début de la ligne.
x2 La coordonnée x de la fin de la ligne.
y2 La coordonnée y de la fin de la ligne.
obj. L'objet à vérifier pour les collisions d'instance.
préc Si la vérification est basée sur des collisions précises(vrai, ce qui est plus lent) ou sur sa boîte de délimitation en général(faux, plus rapide).
notme Si l'instance appelante, si elle est pertinente, doit être exclue(true) ou non(false).

Les retours :

Identité de l'instance ou personne

Exemple :

with (obj_Enemy)
{
if collision_line(100, 400, 100, 600, id, false, false) instance_destroy() ;
}

Ce code demande à toutes les instances de "obj_Enemy" de vérifier le long d'une ligne de 100x, 400y à 100x, 600y s'il y a une collision avec elles-mêmes, et si c'est le cas, elles sont détruites.
