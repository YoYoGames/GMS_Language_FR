point de collision

Collision_point vérifie que le point spécifié par les arguments x1,y1 n'entre pas en collision avec une instance de l'objet spécifié par l'argument "obj". cette vérification peut être précise ou non, mais pour que les collisions précises soient activées, l'objet ou l'instance que vous vérifiez doit également avoir des collisions précises activées pour son sprite. soit activée, l'objet ou l'instance que vous vérifiez doit également avoir des collisions précises activées pour son sprite. Sinon, la vérification par défaut est basée sur les boîtes englobantes. L'image suivante illustre ce fonctionnement :

N'oubliez pas que pour que les collisions précises soient prises en compte, le sprite de l'objet et la fonction de collision doivent tous deux être activés. et la fonction de collision doivent être marqués comme précis. Il faut également noter que la valeur de retour de la fonction peut être l'identifiant de n'importe laquelle des instances considérées comme étant en collision, donc si trois instances se chevauchent à ce point, n'importe lequel de leurs identifiants pourrait être la valeur de retour de la fonction. l'un de leurs ids pourrait être la valeur de retour de la fonction.

Notez qu'au lieu d'un index d'objet, vous pouvez fournir une instance id pour rechercher une instance spécifique, ou les mots-clés d'instance all, ou other (en fonction de l'événement et de la portée actuelle du code).

Syntaxe :

point de collision(x, y, obj, prec, notme) ;

Arguments Description
x La coordonnée x du point à vérifier.
y La coordonnée y du point à vérifier.
obj. L'objet à vérifier pour les collisions d'instance.
préc Si la vérification est basée sur des collisions précises(vrai, ce qui est plus lent) ou sur sa boîte de délimitation en général(faux, plus rapide).
notme Si l'instance appelante, si elle est pertinente, doit être exclue(true) ou non(false).

Les retours :

Identité de l'instance ou personne

Exemple :

if collision_point(x, y, obj_Cursor, false, true)
{
Score += 10S ;
}

Ici, nous vérifions le point à la position de l'objet qui a le code de l'objet "obj_Cursor". S'il y en a un, alors nous ajoutons 10 à la variable de score.
