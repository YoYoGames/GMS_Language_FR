lieu_réunion

Cette fonction vous permet de vérifier la présence d'une collision entre une position et une autre instance ou toutes les instances d'un objet en utilisant le masque de collision de l'instance qui exécute le code de vérification. Lorsque vous utilisez cette fonction, vous demandez en fait à GameMaker Studio 2 de déplacer l'instance vers la nouvelle position, de vérifier s'il y a une collision, de revenir en arrière et de vous dire si une collision a été trouvée ou non. Cela fonctionnera pour les collisions précises, mais seulement si l'instance et l'objet à vérifier ont tous deux des masques de collisions précises sélectionnés, sinon seules les collisions de la boîte englobante sont appliquées. Cette fonction accepte également les mots-clés spéciaux all et other. Veuillez noter que si vous avez besoin d'obtenir l'identifiant unique de l'instance de l'objet avec lequel la collision a lieu, vous devez devez utiliser instance_place().

Notez que les coordonnées x/y données seront ramenées à l'entier le plus proche avant que la vérification ne soit effectuée. contrôle.

Syntaxe :

place_meeting(x, y, obj) ;

Arguments Description
x La position x à vérifier.
y La position y à vérifier.
obj. L'instance ou l'objet à vérifier.

Les retours :

Booléen

Exemple :

if keyboard_check(vk_left)
{
if !place_meeting(x - 5, y, obj_wall) x -=5 ;
}

Le code ci-dessus vérifie qu'il n'y a pas de collision à gauche de l'instance et déplace l'instance s'il n'y en a pas.
