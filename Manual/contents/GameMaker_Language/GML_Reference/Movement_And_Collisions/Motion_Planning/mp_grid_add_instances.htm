mp_grid_add_instances

Cette fonction utilise le masque de collision d'instance (décidé par les propriétés du sprite ou l'index de masque de l'instance appelante) pour marquer les cellules comme interdites ou non dans une grille MP, où les cellules interdites ne peuvent être traversées par aucune des fonctions de recherche de chemin. Vous pouvez spécifier dans la fonction s'il faut prendre en compte des collisions précises ou non et les cellules interdites marquées changeront en fonction de ce paramètre. Cette image illustre ce comportement:

Les deux instances ci-dessus ont été ajoutées à la grille MP en utilisant le mp_grid_add_instances() fonction avec precise étant défini comme vrai. Comme vous pouvez le voir, l'instance verte n'a marqué que les carrés de la grille qu'elle "touche" comme interdits, du fait que son masque de sprite est également défini sur précis. Cependant, la deuxième instance a également marqué d'autres cellules qui ne semblent pas se toucher. Cela est dû au fait que le masque de sprite n'est pas défini sur precise, ce qui signifie que même si vous utilisez le paramètre précis dans la fonction, seule la boîte englobante de l'image-objet sera prise en compte. C'est également ce qui se passerait si vous définissiez l'argument précis de la fonction sur false - même les instances avec un masque de sprite précis seront ajoutées à la grille en fonction de leurs cadres de délimitation.

Syntaxe:

mp_grid_add_instances(id, obj, prec);

Argument Description
id Index du mp_grid à utiliser
obj Index d'objet, ou id d'instance, des instances à ajouter dans mp_grid
prec Si la vérification est basée sur des collisions de pixels parfaits (vrai = lent) ou sur sa boîte englobante en général (faux = rapide).

Retour:

N/A

Exemple:

mp_grid_add_instances(grid, obj_Wall, true);

Le code ci-dessus ajoutera toutes les instances de "obj_Wall" dans le mp_grid indexé dans la variable "grid" en utilisant le masque de collision précis plutôt que la boîte englobante.
