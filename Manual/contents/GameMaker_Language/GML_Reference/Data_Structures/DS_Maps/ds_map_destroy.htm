ds_map_destroy

Les cartes DS prennent de l'espace en mémoire, qui leur est alloué lors de leur création. Cela signifie que vous devez également libérer cette mémoire lorsque la carte DS n'est pas nécessaire pour éviter les erreurs, les fuites de mémoire et les pertes de performances lors de l'exécution de votre jeu. Cette fonction fait exactement cela. Notez que la destruction d'une carte dé-référencera toutes les structures de données stockées dans la carte donnant une fuite de mémoire, vous devrez donc parcourir la carte et détruire tous les éléments de la structure de données manuellement avant de la détruire pour éviter cela. Le seul moment où cela n'est pas nécessaire est lorsque vous avez marqué des éléments de la carte comme une liste DS ou comme une autre carte DS, auquel cas ces éléments seront détruits et leur mémoire nettoyée automatiquement également.

IMPORTANT! Lorsque vous créez une structure de données, la valeur d'index pour l'identifier est une valeur entière commençant à 0. Cela signifie que les structures de données de différents types peuvent avoir la même valeur d'index, donc en cas de doute, vous devriez utiliser la fonction ds_exists () avant y accéder. Notez également que les index sont réutilisés, donc une valeur d'index de structure de données détruite peut être utilisée par une valeur nouvellement créée par la suite, nous vous recommandons donc de toujours définir la variable qui contenait l'index DS à -1 après la destruction.

Syntaxe:

ds_map_destroy(id);

Argument Description
id L'identifiant de la carte à détruire.

Retour:

N/A

Exemple:

ds_map_destroy(inventory);
inventory = -1;

Le code ci-dessus détruira la carte DS dont l'ID est indexé dans la variable "inventaire".
