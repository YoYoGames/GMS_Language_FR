ds_stack_read

Avec cette fonction, vous pouvez recréer une pile DS enregistrée (une qui a été précédemment écrite sous forme de chaîne en utilisant ds_stack_write() ). Vous devez d'abord créer une nouvelle pile DS dans laquelle lire la chaîne, et si la pile DS existe déjà et contient des informations stockées, cela sera effacé avant la lecture. Cette fonction est d'une importance vitale lors de la création de mécanismes de sauvegarde / chargement pour votre jeu. Notez que si la structure de données a été créée avec les versions précédentes de GameMaker vous devez ajouter l'argument facultatif «hérité», en le définissant sur true car le format de la chaîne a changé avec cette version.

Syntaxe:

ds_stack_read(id, str [, legacy] );

Argument Description
id L'identifiant de la structure de données à lire.
str La chaîne à partir de laquelle lire.
legacy ( optional ) Peut être soit true ou false ou omis complètement.

Retour:

N/A

Exemple:

stack = ds_stack_create();
ini_open("save.ini");
var str = ini_read_string("Stacks", "0", "");
if (str != "")
    {
    ds_stack_read(stack, str);
    }
ini_close();

Le code ci-dessus crée une pile et stocke l'index dans la variable "pile". Il ouvre alors un ini file et lit une chaîne à partir de celui-ci, en vérifiant que la chaîne n'est pas renvoyée comme vide en premier. Cette chaîne est ensuite lue dans la pile DS nouvellement créée.
