camera_apply

Cette fonction appliquera les paramètres de caméra donnés à la vue de caméra actuelle faisant l'objet d'un rendu immédiat. Bien que vous puissiez définir n'importe quelle vue de caméra en utilisant les fonctions spécifiques camera_set_*, les nouveaux paramètres ne seront pas utilisés pour le rendu avant le prochaine image du jeu, mais avec cette fonction, vous pouvez effectuer ces changements immédiatement. Si vous utilisez plusieurs vues, la fonction doit être utilisée dans le Draw Event et vous pouvez utiliser la variable view_current pour appliquer sélectivement la caméra donnée uniquement à la vue de la caméra en cours de rendu. Notez que cette fonction ne modifie PAS la valeur de l'ID de la caméra, mais uniquement les paramètres de cette caméra. GameMaker Studio 2 utilise lorsqu'aucune caméra de vue n'est active), les modifications seront écrasées dès la prochaine image de jeu par les paramètres internes de dessin de la pièce.

Syntaxe :

camera_apply(camera_id)

Arguments Description
camera_id La valeur d'identification unique de la caméra renvoyée lorsque vous avez créé la caméra.

Les retours :

N/A

Exemple :

if view_current == 0
{
camera_apply(cutscene_cam) ;
}

Le code ci-dessus (dans un événement de dessin) vérifie quelle vue de caméra est actuellement rendue et s'il s'agit de la vue de caméra [0], les paramètres de la caméra référencée par la variable "cutscene_cam" lui sont appliqués.
