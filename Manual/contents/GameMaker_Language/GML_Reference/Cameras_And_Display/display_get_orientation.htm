affichage_get_orientation

Cette fonction renvoie l'une des deux constantes dont dispose GameMaker Studio 2 pour vous indiquer si le périphérique exécutant le jeu est tenu en mode paysage ou portrait (voir le tableau ci-dessous). Veuillez noter que cette fonction peut ne pas détecter correctement l'orientation de l'appareil lorsqu'elle est utilisée dans le module cible HTML5. Cependant, il est facile d'imiter cette fonction en utilisant le script suivant :

retourner (largeur du navigateur < hauteur du navigateur) ;

Il retournera true pour le portrait et false pour le paysage.

Syntaxe :

display_get_orientation()

Les retours :

Constant

Constant Description
afficher_paysage L'appareil est tenu horizontalement, c'est-à-dire que le bord le plus long est de gauche à droite, et le bouton de menu est à droite.
afficher_paysage_flippé Comme ci-dessus, sauf que maintenant le bouton de menu est à gauche.
affichage_portrait L'appareil est tenu verticalement, c'est-à-dire que le bord le plus long est de haut en bas et que le bouton de menu est en bas.
afficher_portrait_flipped Comme ci-dessus, sauf que maintenant le bouton du menu est en haut.

Exemple :

si display_get_orientation() == display_landscape
{
global.Config = 0 ;
}
else
{
global.Config = 1 ;
}

Le code ci-dessus vérifie l'orientation de l'appareil et définit une variable globale en fonction de la valeur renvoyée.
