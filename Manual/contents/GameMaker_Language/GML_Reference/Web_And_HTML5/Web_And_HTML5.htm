Web et HTML5

GameMaker Studio 2 fournit diverses fonctions permettant d'interagir avec Internet ou avec un serveur, et elles peuvent être utilisées sur toutes les plateformes cibles disponibles. Ces fonctions sont les suivantes :

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Outre les fonctions générales énumérées ci-dessus, la cible HTML5 possède quelques fonctions web supplémentaires qui sont spécifiquement conçues pour être utilisées par les jeux qui la ciblent :

  • hauteur du navigateur
  • largeur du navigateur
  • capture d'entrée du navigateur
  • webgl_enabled
  • clickable_exists
  • ajouts cliquables
  • Ajout d'un texte cliquable
  • changement cliquable
  • Texte de changement cliquable
  • Style de jeu cliquable
  • clickable_delete
  • événement_analytique
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Outre ces fonctions pour le module cible HTML5, il existe également une variable JavaScript spéciale qui peut être utilisée dans les rares occasions où le contenu du dossier html5game est stocké à un endroit complètement différent (URL). Normalement, ce chemin est codé en dur dans le fichier JavaScript et est relatif à l'emplacement à partir duquel le JavaScript est appelé, mais vous pouvez définir la variable g_GameMakerHTML5Dir avant l'appel au script de jeu et le jeu sera exécuté à partir de l'emplacement nouvellement défini. Par exemple

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Il convient également de noter que vous pouvez signaler les fonctions de script en utilisant le préfixe gmcallback_, ce qui signifie qu'elles ne seront pas obscurcies lorsque vous compilerez le projet pour HTML5. Ceci est très utile lorsque vous utilisez les fonctions clickable_* listées ci-dessus (pour plus d'informations, voir ici).
