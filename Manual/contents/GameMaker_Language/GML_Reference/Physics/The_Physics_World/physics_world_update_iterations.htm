physics_world_update_iterations

Tout dans GameMaker Studio 2 est basé sur des étapes et, normalement, chaque événement ne se produira qu'une fois par étape. Cependant, pour obtenir la précision nécessaire avec les fonctions physiques, on leur fait faire plusieurs calculs à chaque étape qui sont appelés "itérations" dont le nombre est contrôlé par cette fonction. Le nombre par défaut du système physique est 10, mais il peut être changé en plus ou moins en fonction du nombre de fois que vous voulez que la physique calcule chaque étape, mais il est recommandé de ne pas le définir inférieur à 5 ni supérieur à 30. Sachez également que cette fonction dépend de la physics_world_update_speed() fonction.

Syntaxe:

physics_world_update_iterations(iterations)

Argument Description
y1 les itérations (par étape) que le système de physique effectuera

Retour:

N/A

Exemple:

physics_world_update_iterations(20);

Le code ci-dessus définit le monde de la physique dans la pièce actuelle pour effectuer des calculs 20 fois par pas.

Retour: Le monde de la physique
