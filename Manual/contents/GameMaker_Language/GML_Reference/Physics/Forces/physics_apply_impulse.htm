physics_apply_impulse

Non seulement vous pouvez appliquer la force et la gravité à un objet avec la physique de GameMaker Studio 2 mais vous pouvez également appliquer une impulsion. Ceci est légèrement différent d'une force en ce que lorsqu'elle est appliquée, elle affectera immédiatement la vitesse, et éventuellement le couple (ou «spin») de l'objet, en particulier si le point choisi pour appliquer l'impulsion n'est pas aligné avec le centre de masse (attention: le centre de masse n'est pas forcément le même que l'origine!). Voici une illustration:

Comme vous pouvez le voir, le joueur a cliqué sur la balle et cela lui a donné une impulsion pour se déplacer vers le coin supérieur droit, en tournant au fur et à mesure. L'impulsion exacte est définie par le vecteur que nous obtenons des composantes ximpulse / yimpulse par rapport aux coordonnées xpos / ypos - ce qui signifie simplement que l'impulsion est calculée comme la distance de xpos / ypos à ximpulse / yimpulse en Newtons, et la direction est l'angle que nous obtenons de xpos / ypos à ximpulse / yimpulse.

REMARQUE: Le monde de la physique utilise des compteurs pour tous ses calculs, et le rapport de pixels par mètre est défini dans l'éditeur de salle ou à l'aide de physics_world_create ().

Syntaxe:

physics_apply_impulse(xpos, ypos, ximpulse, yimpulse)

Argument Description
xpos La coordonnée x dans la pièce où l'impulsion sera appliquée
ypos La coordonnée y dans la pièce où l'impulsion sera appliquée
ximpulse la composante x du vecteur d'impulsion
yimpulse la composante y du vecteur d'impulsion

Retour:

N/A

Exemple:

if mouse_check_button_pressed(mb_left)
    {
    with (place_meeting(mouse_x, mouse_y, all))
        {
        physics_apply_impulse(mouse_x, mouse_y, -10 + irandom(20), -10 + irandom(20));
        }
    }

Le code ci-dessus appliquera une impulsion avec un vecteur aléatoire à une instance qui se trouve à la position de la souris lorsque le bouton gauche est enfoncé.
