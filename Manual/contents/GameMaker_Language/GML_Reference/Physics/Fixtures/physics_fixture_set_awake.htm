physics_fixture_set_awake

Lorsque vous démarrez une salle avec la physique, ou liez des appareils à des instances, les appareils sont toujours considérés comme "éveillés"... c'est-à-dire qu'ils traitent des événements et interagissent avec les instances environnantes. Cependant, cela peut parfois entraîner des problèmes, en particulier si vous avez un certain nombre d'instances avec des appareils qui sont côte à côte lorsqu'une pièce démarre (pensez à une tour composée de plusieurs instances avec des appareils) car s'ils sont «éveillés», ils se déplaceront et éventuellement changer de position en raison du début soudain de la gravité et des collisions les affectant. Dans ces cas, vous pouvez utiliser cette fonction pour marquer le projecteur comme étant "endormi", auquel cas il ne traitera rien lors de sa création dans la pièce.

Syntaxe:

physics_fixture_set_awake(fixture, state)

Argument Description
fixture l'index du luminaire
flag si un appareil est éveillé (true ) ou non (false )

Retour:

N/A

Exemple:

physics_fixture_set_awake(fix_Cloud, true);

Le code ci-dessus indique que l'appareil est "éveillé" lorsqu'il est lié à une instance.
