Dialogue

La plupart des ordinateurs et appareils mobiles modernes n'aiment pas quand un jeu est arrêté alors qu'une fenêtre de dialogue est affichée. Ils ont tendance à interpréter cela comme un coureur de jeu arrêté, et peuvent le considérer comme une erreur et fermer l'application prématurément. Pour éviter cela, GameMaker Studio 2 dispose de fonctions de dialogue asynchrones qui vous permettent de demander certaines formes de commentaires ou d'entrée de l'utilisateur, mais sans bloquer le lanceur de jeu à tout moment, ce qui signifie que - lorsqu'il est appelé - le jeu fonctionnera normalement en arrière-plan pendant que l'utilisateur répond à la fenêtre de dialogue affichée. Une fois que l'utilisateur a répondu, un événement asynchrone de dialogue sera appelé afin que vous puissiez gérer les résultats.

REMARQUE: cela signifie que s'il est possible pour le joueur de perdre la partie, de se faire tuer, etc... pendant que cette boîte de dialogue est affichée, alors vous devez prendre des mesures pour créer un système de pause, sinon ce sera un utilisateur désagréable vivre.

Vous trouverez ci-dessous une liste de toutes les fonctions de dialogue asynchrones disponibles:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
