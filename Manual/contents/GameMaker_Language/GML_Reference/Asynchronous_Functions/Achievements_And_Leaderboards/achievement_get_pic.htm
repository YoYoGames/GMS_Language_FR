achievement_get_pic

Cette fonction enverra une requête au serveur pour l'image d'un joueur ou d'un ami et déclenchera un événement social asynchrone de rappel qui contient le async_load carte remplie avec les paires clé / valeur pertinentes. La clé d' identification de ce DS Map est utilisé pour identifier le rappel correct (il peut y avoir plus d'une fonction de déclenchement pour un événement asynchrone donné), et sera associé à la constante achievement_pic_loaded ainsi qu'un certain nombre d'autres paires clé / valeur. Le contenu exact de la carte est indiqué ci-dessous:

REMARQUE: cette fonction est pour iOS uniquement.

  • " id " - Pour cette fonction, il devrait être achievement_pic_loaded
  • " playerid " - L'identifiant du joueur ou de l'ami dont nous recevons l'image.
  • " spriteid " - L'identifiant du sprite pour l'image qui peut ensuite être utilisé pour la dessiner à l'écran en utilisant l'une des fonctions de sprite de dessin

ATTENTION! Cette fonction crée des sprites et en tant que tels, leurs identifiants doivent être stockés dans des variables puis supprimés de la mémoire à l'aide de la fonction sprite_delete (), sinon une utilisation répétée vous donnera une fuite de mémoire qui finira par ralentir ou planter votre jeu.

Syntaxe:

achievement_get_pic(char)

Argument Description
char L'identifiant de la personne dont vous souhaitez obtenir l'image (à partir d'une liste d'amis ou d'un classement)

Retour:

N/A

Exemple étendu:

Le code suivant sera appelé après avoir obtenu une liste d'amis ou une liste de classements en utilisant achievement_load_friends() ou achievement_load_leaderboard(). Ces fonctions renverront des valeurs "id" uniques pour chaque personne, qui sont ensuite utilisées dans la fonction pour récupérer la photo de profil pour elles:

achievement_get_pic(global.player_id[0]);

Cela enverra une demande pour l'image associée à l'identifiant du joueur stocké dans le tableau global et générera un rappel asynchrone avec le spécial async_load Carte DS contenant les données suivantes:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
    {
    var picid = ds_map_find_value(async_load, "playerid");
    var spriteid = ds_map_find_value(async_load, "spriteid");
    if global.playerid[0] == picid)
        {
        global.playerpic[0] = spriteid;
        }
    }

Le code ci-dessus vérifie le retour DS Map dans l' événement social asynchrone et si son "id" correspond à la constante en cours de vérification, il vérifie alors que l'id associé à l'image est le même que celui demandé et si c'est le cas, il définit un autre tableau global pour contenir l'id unique pour l'image (qui peut être utilisée pour dessiner l'image sous forme de sprite).
