steam _ugc_create_query_user_ex

Cette fonction peut être utilisée pour interroger la base de données UGC. La fonction nécessite la valeur d'ID de l'utilisateur et l'ID du jeu qui va consommer l'objet et / ou l'ID du jeu qui a créé l'objet. Vous devez également utiliser les constantes suivantes pour le type de données à interroger, le type d'élément à interroger et l'ordre dans lequel les éléments retournés seront triés, ainsi qu'un numéro de page - notez qu'une requête renverra un nombre maximum de 50 articles.

La fonction renvoie une valeur de descripteur de requête unique qui doit être stockée dans une variable pour être utilisée dans les autres fonctions de requête. Notez que cette fonction prépare uniquement la requête mais ne l'envoie pas réellement - pour cela, vous devez appeler la fonction steam _ugc_send_query.htm ">steam_ugc_send_query() - et vous pouvez utiliser davantage steam_ugc_query_*() fonctions pour affiner la demande de recherche avant qu'elle ne soit réellement envoyée.

Syntaxe:

steam_ugc_create_query_user_ex(list_type, match_type, sort_order, page, account_id, creator_app_id, consumer_app_id);

Argument Description
list_type Le type de liste de données à créer (voir les constantes ci-dessous).
match_type Le type d'éléments UGC à interroger (voir les constantes ci-dessous).
sort_order La façon dont les données doivent être ordonnées (voir les constantes ci-dessous).
page Numéro de page à interroger.
account_id L'identifiant du compte Steam
creator_app_id ID de l'application du créateur de l'élément.
consumer_app_id L'ID de l'application grand public.

Retour:

Query Handle

Exemple:

query_handle = steam_ugc_create_query_user_ex(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, page, global.AccountID, 0, global.GameID);

Le code ci-dessus crée une requête de requête et stocke son handle dans une variable pour une utilisation future.

Retour: UGC
