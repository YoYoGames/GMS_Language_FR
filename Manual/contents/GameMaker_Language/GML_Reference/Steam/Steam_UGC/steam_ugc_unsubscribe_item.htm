steam _ugc_unsubscribe_item

Cette fonction peut être utilisée pour se désinscrire d'un article UGC. Vous fournissez l'ID de fichier publié, puis la fonction renvoie une valeur d'ID asynchrone unique qui peut être utilisée pour vérifier que l'élément a été correctement désabonné dans l' événement Steam Async. Cet événement aura le async_load Carte DS qui sera remplie avec les paires clé / valeur suivantes:

  • " id " - L'ID asynchrone renvoyé par la fonction qui a déclenché l'événement
  • " result " - Le résultat de l'opération (une valeur réelle). Ce sera soit la constante GML ugc_result_success ou un autre nombre réel. Vous devez donc vérifier cette constante pour vous assurer que l'appel a réussi et, dans le cas contraire, quelque chose n'a pas fonctionné correctement. Le reste des valeurs possibles renvoyées est affiché comme le résultat de la valeur Steam "EResult" et vous devriez voir steamclientpublic.h dans les en-têtes du SDK pour les 89 valeurs possibles.
  • " event_type " - La chaîne " ugc_unsubscribe_item ".
  • " published_file_id " - L'ID unique de l'élément publié.

Syntaxe:

steam_ugc_unsubscribe_item(published_file_id);

Argument Description
published_file_id ID de fichier unique pour lequel l'UGC doit se désabonner.

Retour:

Async ID

Exemple:

steam_sub = steam_ugc_subscribe_item(global.pubFileID);

Le code ci-dessus abonnera (et téléchargera) l'élément avec l'ID de fichier stocké dans la variable globale "pubFileID".

Retour: UGC
