clavier_virtuel_show

Cette fonction peut être utilisée pour afficher le clavier virtuel sur le périphérique exécutant le jeu. Lorsque vous appelez cette fonction, vous devez fournir l'une des constantes suivantes pour chacun des trois premiers arguments :

  • Type de clavier: Cette constante est utilisée pour définir le type de clavier qui sera disponible sur le clavier virtuel. Les types de clavier disponibles sont les suivants :
Type de clavier Description
kbv_type_default Le type de clavier par défaut pour le système actuel.
kbv_type_ascii Un clavier uniquement ASCII.
kbv_type_url Un clavier normal optimisé pour la saisie d'URL. Il comporte généralement un ".com" ou d'autres touches de domaine, ainsi que des touches "/" et ".".
kbv_type_email Un clavier normal optimisé pour la saisie d'e-mails. Il comporte généralement les caractères "@" et ".".
kbv_type_numbers Un clavier uniquement composé de chiffres, généralement représenté par un pavé numérique.
kbv_type_phone Un clavier de téléphone. Généralement, il ne contient que des chiffres avec les touches "*" et "#".
kbv_type_phone_name Un clavier optimisé pour la saisie d'un numéro de téléphone et d'un nom. Généralement similaire à un clavier ASCII, mais avec une sélection limitée de caractères spéciaux.

  • Type de retour: Cette constante est utilisée pour définir ce qui est affiché sur la touche retour/action du clavier virtuel. Les types de retour disponibles sont les suivants :
Type de retour Description
kbv_returnkey_default Le titre de la touche de retour par défaut pour le système actuel.
kbv_returnkey_go Définit le titre de la touche de retour sur "Go".
kbv_returnkey_google Définit le titre de la clé de retour à "Google", ou à une icône de recherche générique dans certains cas.
kbv_returnkey_join Définit le titre de la touche de retour sur "Go".
kbv_returnkey_next Définit le titre de la touche de retour à "Suivant".
kbv_returnkey_route Définit le titre de la touche de retour à "Route".
kbv_returnkey_search Définit le titre de la touche de retour à "Recherche", ou à une icône de recherche générique dans certains cas.
kbv_returnkey_send Définit le titre de la touche de retour sur "Envoyer".
kbv_returnkey_yahoo Définit le titre de la clé de retour à "Yahoo", ou à une icône de recherche générique dans certains cas.
kbv_returnkey_done Définit le titre de la touche de retour à "Done".
kbv_returnkey_continue Définit le titre de la touche de retour à "Continuer".
kbv_returnkey_emergency Définit le titre de la touche de retour sur "Appel d'urgence".

  • Type de capitalisation automatique: Cette constante est utilisée pour déterminer comment les mots tapés via le clavier virtuel doivent être capitalisés automatiquement. Les types de capitalisation automatique disponibles sont :
Type de capitalisation automatique Description
kbv_autocapitalize_none La capitalisation automatique est désactivée.
kbv_autocapitalize_words Les mots seront mis en majuscules.
kbv_autocapitalize_sentences Les phrases seront mises en majuscules automatiquement.
kbv_autocapitalize_caracteres Tous les caractères seront en majuscules.

Le dernier argument permet d'activer/désactiver le texte prédictif. Il doit être défini sur true (vrai) pour l'autoriser et sur false (faux) dans le cas contraire, mais notez que ce n'est pas parce qu'il est autorisé qu'il sera utilisé, car cela dépendra des préférences de l'utilisateur du périphérique. de l'utilisateur de l'appareil. En mode texte prédictif, le clavier virtuel ne génère pas d'événements de frappe normaux dans GameMaker Studio 2. Au lieu de cela, il mettra uniquement à jour le dernier caractère appuyé et les variables de chaîne de caractères du clavier. Cela est dû à l'incapacité de détecter si un changement dans le champ de texte interne utilisé pour détecter les pressions sur les touches provient d'une touche réelle du clavier virtuel ou d'une suggestion de texte. Dans ce cas, il est préférable de lire l'entrée keyboard_string plutôt que de plutôt que de lire n'importe quel type d'entrée de touche brute.

Il est également important de noter que l'utilisateur obtiendra des claviers différents avec des capacités différentes selon le système d'exploitation de la plate-forme, avec les réserves suivantes pour une utilisation par cible :

  • UWP: L'implémentation UWP ne prend pas en charge les paramètres return_key_type, autocapitalization_type et predictive_text_enabled. Le passage de ces paramètres dans une fonction ne causera aucune erreur, ils n'auront simplement aucun effet sur le clavier. effet sur le clavier.

  • Android: Sur Android, le type de clavier kbv_type_phone_name n'est pas pris en charge et affichera un clavier ASCII à la place, et le type de touche de retour ne peut être que kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send ou kbv_returnkey_done. Si d'autres types de clés de retour sont utilisés sur cette plate-forme, le système utilisera par défaut les clés de retour de remplacement énumérées ci-dessous :
Touche de retour non disponible Clé de remplacement
kbv_returnkey_google
kbv_returnkey_yahoo 		kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency 		kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV/ FireTV: Les types de touches de retour personnalisés ne sont pas pris en charge sur les claviers ASCII - la touche de retour par défaut sera toujours affichée.

  • tvOS: Le texte prédictif et la capitalisation automatique ne sont pas pris en charge sur tvOS, et la hauteur du clavier renvoyée par les événements système et la fonction keyboard_virtual_height() correspondra par défaut à la hauteur de l'écran en raison du fait qu'il s'étend sur la totalité de l'écran et que les fonctions de dimensionnement des rectangles de clavier sont désactivées sur tvOS. l'écran et que les fonctions de dimensionnement du clavier sont désactivées sur tvOS. Notez également que les événements du clavier physique (bluetooth) ne seront pas diffusés à moins que le clavier virtuel n'ait été ouvert.

L'appel de cette fonction génère un événement asynchrone du système, dans lequel la carte async_load DS est remplie des paires clé/valeur suivantes :

  • "event_type" - ce sera "l'état du clavier virtuel" lorsqu'il sera déclenché par les fonctions du clavier virtuel.

  • "screen_height " - la hauteur du clavier virtuel (en pixels). Elle sera de 0 si le clavier est invisible.

  • "keyboard_status" - l'état actuel du clavier, renvoyé sous la forme d'une des chaînes suivantes :
    • "caché"
    • "caché"
    • "visible"
    • "visible"

Syntaxe

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled) ;

Arguments Description
Type de clavier Détermine le jeu de touches disponible sur le clavier virtuel.
Type de clé de retour Détermine ce qui est affiché sur la touche retour/action du clavier virtuel.
Type d'autocapitalisation Détermine comment/si les mots tapés via le clavier virtuel seront autocapitalisés.
texte_prédictif_activé Définissez cette option sur true/false pour activer/désactiver la saisie de texte prédictive.

Les retours :

N/A

Exemple :

if input == false
{
input = true ;
keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false) ;
}

Le code ci-dessus fera apparaître le clavier virtuel de l'OS si la variable donnée n'est pas définie comme vraie.
