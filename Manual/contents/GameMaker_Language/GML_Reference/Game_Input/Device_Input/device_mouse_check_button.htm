device_mouse_check_button

Cette fonction renvoie true (1) ou false (0) selon que le périphérique que vous spécifiez est utilisé ou non. Désormais, l' appareil fait référence au numéro de la souris, qui peut être compris entre 0 et n et cet événement est déclenché lorsque l'écran tactile d'un appareil mobile est touché - où n est le nombre maximal de «doigts» pouvant toucher l'écran à une fois (le nombre maximum de touches pouvant être détectées dépendra beaucoup de l'appareil utilisé et du système d'exploitation qu'il exécute, mais la plupart des appareils devraient en détecter au moins jusqu'à 4). Cette fonction met constamment à jour chaque étape et les constantes standard de mb_left et mb_right peut être utilisé pour vérifier l'un ou l'autre des boutons. Notez que mb_right ne sera détecté que si une double pression est détectée (ce comportement peut être désactivé à l'aide de la fonction device_mouse_dbclick_enable().

Syntaxe:

device_mouse_check_button(device, button);

Argument Description
device L'appareil (de 0 à n ) qui est vérifié
button Le bouton de l'appareil en cours de vérification.

Retour:

Boolean

Exemple:

if device_mouse_check_button(0, mb_left) && device_mouse_check_button(1, mb_left)
{
room_goto(rm_Menu);
}

Le code ci-dessus vérifie si les deux touches sont maintenues enfoncées en même temps et si elles vont ensuite dans une autre pièce.

Retour: Entrée de périphérique
