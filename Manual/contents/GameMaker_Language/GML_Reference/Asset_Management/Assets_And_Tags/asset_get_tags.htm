Balises de l'actif

Cette fonction vous permet de récupérer toutes les étiquettes affectées à un poste dans le navigateur de postes. Vous fournissez soit le nom du bien (sous forme de chaîne), soit son index, et la fonction renvoie un tableau de tags pour ce bien. Si aucune balise n'est trouvée ou s'il y a une erreur (par exemple, la chaîne de nom donnée n'existe pas), le tableau retourné sera vide. Si vous fournissez une valeur d'index de poste, vous devrez fournir l'argument optionnel de type de poste (une constante). (une constante), car des biens de types différents peuvent avoir le même index, même s'ils ne peuvent pas avoir le même nom. Les types de biens disponibles sont répertoriés dans le tableau ci-dessous :

Constant Description
objet_actif Le nom donné fait référence à un objet.
actif_sprite Le nom donné fait référence à un sprite.
actif_son Le prénom fait référence à un son.
salle des actifs Le prénom fait référence à une pièce.
piliers_de_l'actif Le nom donné fait référence à un ensemble de carreaux.
chemin_de_l'actif Le nom donné fait référence à un chemin.
script_de_l'actif Le nom donné fait référence à un script.
asset_font Le nom donné fait référence à une police de caractères.
chronologie des actifs Le prénom fait référence à une ligne de temps.
asset_shader Le nom donné fait référence à un shader.
courbe d'animation de l'actif Le nom donné fait référence à une courbe d'animation.
séquence_d'actifs Le nom donné fait référence à une Séquence.

Syntaxe :

asset_get_tags(name_or_index, [asset_type]) ;

Arguments Description
nom_ou_index Le nom du poste (une chaîne de caractères) ou sa valeur d'index (un nombre entier).
[type d'actif] FACULTATIF ! Le type de bien pour lequel il faut obtenir les balises, utilisé uniquement lorsqu'un index est fourni pour le premier argument.

Les retours :

Array

Exemple :

my_tags = asset_get_tags(object_get_name(object_index)) ;

Le code ci-dessus récupère toutes les balises attribuées à l'objet à partir duquel l'instance qui exécute le code a été créée.
