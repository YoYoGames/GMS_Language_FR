event_perform

Cette fonction exécutera l'événement spécifié, avec l'argument désigné, pour l'instance exécutant le code. Il existe de nombreuses options ici qui permettent une simulation complète de tous les événements possibles, mais notez que cela exécute littéralement tout le code de l'événement et le jeu ne modifiera rien pour le faire se déclencher manuellement, par exemple si vous choisissez d'effectuer un clavier appuyez sur l'événement, l'événement sera déclenché mais la touche correspondante ne sera pas reconnue comme ayant été enfoncée. Ou si vous effectuez un événement d'alarme, le compteur d'alarme ne sera pas réglé sur -1 mais continuera plutôt à décompter.

Le tableau ci-dessous présente toutes les constantes incluses dans GameMaker Studio 2 pour référencer les types d'événements, ainsi que leur valeur numérique. Les événements eux-mêmes sont indiqués en gras et sont les mêmes que les valeurs de retour du event_type variable, tandis que leurs valeurs numériques sont marquées en italique et ce sont les mêmes que les valeurs de retour pour le event_number variable:

Nom Description
ev_create Créer un évènement.
ev_destroy Détruire l'événement
ev_cleanup Événement de nettoyage
ev_step Événement d'étape
---- ev_step_normal Étape
---- ev_step_begin Étape de début
---- ev_step_end Étape de fin
ev_alarm Événement d'alarme
---- 0... 11 Le numéro correspondant à laquelle des 12 alarmes vous souhaitez déclencher l'événement de
ev_keyboard, ev_keypress, ev_keyrelease Clavier / clavier enfoncé / clavier libéré
---- n'importe quel code clé vk_space ou ord ("W") par exemple.
ev_mouse Événement souris
---- ev_left_button Bouton gauche enfoncé sur l'objet
---- ev_right_button Bouton droit maintenu enfoncé sur l'objet
---- ev_middle_button Bouton du milieu (ou molette cliquable) maintenu enfoncé sur l'objet
---- ev_no_button Aucun bouton enfoncé
---- ev_left_press Le bouton gauche vient d'appuyer sur l'objet
---- ev_right_press Le bouton droit vient d'appuyer sur l'objet
---- ev_middle_press Bouton du milieu (ou molette cliquable) juste pressé sur l'objet
---- ev_left_release Le bouton gauche vient d'être relâché sur l'objet
---- ev_right_release Le bouton droit vient d'être relâché sur l'objet
---- ev_middle_release Le bouton du milieu vient d'être relâché sur l'objet
---- ev_mouse_enter La souris vient d'entrer dans la zone de délimitation de l'objet

---- ev_mouse_wheel_up La molette de la souris fait défiler vers le haut
---- ev_mouse_wheel_down La molette de la souris défile vers le bas
---- ev_global_left_button Bouton gauche maintenu n'importe où
---- ev_global_right_button Bouton droit maintenu n'importe où
---- ev_global_middle_button Bouton central (ou molette cliquable) maintenu n'importe où

---- ev_global_right_press Bouton droit juste pressé n'importe où

ev_gesture Un événement de geste (appuyez, faites glisser, feuilletez, pincez ou faites pivoter)

---- ev_gesture_pinch_in Le mouvement entre deux touches pour une instance a été détecté comme vers l'intérieur

---- ev_gesture_rotating Le mouvement entre deux touches pour une instance a été détecté comme tournant
---- ev_gesture_rotate_end Le relâchement d'une (ou des deux) touches pour une rotation a été détecté pour une instance
---- ev_global_gesture_tap Un simple clic / toucher et relâchement a été détecté n'importe où dans la pièce
---- ev_global_gesture_double_tap Deux touches / clics rapides et relâchements ont été détectés n'importe où dans la pièce
---- ev_global_gesture_drag_start Le début d'un geste de glissement a été détecté n'importe où dans la pièce

---- ev_global_gesture_pinch_in Le mouvement entre deux touches n'importe où dans la pièce a été détecté comme vers l'intérieur
---- ev_global_gesture_pinch_out Le mouvement entre deux touches n'importe où dans la pièce a été détecté comme vers l'extérieur
---- ev_global_gesture_pinch_end Le relâchement d'une (ou des deux) touches pour une pincée a été détecté n'importe où dans la pièce
---- ev_global_gesture_rotate_start Le mouvement entre deux touches n'importe où dans la pièce a été détecté comme une rotation
---- ev_global_gesture_rotating Le mouvement entre deux touches n'importe où dans la pièce a été détecté comme tournant
---- ev_global_gesture_rotate_end Le relâchement d'une (ou des deux) touches pour une rotation a été détecté n'importe où dans la pièce
ev_collision Collision avec un objet
---- L'index de l'objet à vérifier. object1 par exemple.
ev_other L'une des actions répertoriées sous «Autre»
---- ev_outside Si l'instance se trouve à l'extérieur de la pièce
---- ev_boundary Si l'instance croise la frontière
---- ev_game_start Déclenchée uniquement au début du jeu
---- ev_game_end Uniquement déclenché à la fin du jeu
---- ev_room_start Déclenchée uniquement au début d'une pièce
---- ev_room_end Déclenchée uniquement à la fin d'une pièce

ev_draw Événement de tirage au sort. REMARQUE: Cet événement ne peut pas être forcé en dehors d'un événement de dessin et les constantes et les constantes servent uniquement à identifier l'événement lorsqu'elles sont exécutées dans ces cas.
---- ev_draw_begin L'événement de début de tirage au sort.
---- ev_draw_end L'événement de fin de tirage.
---- ev_draw_pre L'événement de pré-tirage.
---- ev_draw_post L'événement post-tirage.
---- ev_gui L'événement de l'interface graphique de tirage au sort.
---- ev_gui_begin L'événement de début de l'interface graphique de tirage au sort.
---- ev_gui_end L'événement de fin de l'interface graphique de tirage.

Syntaxe:

event_perform(type, numb);

Argument Description
type Le type d'événement à réaliser. Voir ci-dessous.
numb La constante ou la valeur de l'événement spécifique, si nécessaire (sinon, utilisez simplement 0).

Retour:

N/A

Exemple étendu


Pour vous aider à comprendre comment cette fonction peut être utilisée, veuillez consulter les quelques exemples suivants:

event_perform(ev_keypress, ord("W"));

Cela effectuerait l'événement associé à la touche Keyboard Check Pressed "W" (sans générer réellement une pression sur le clavier).

event_perform(ev_step, ev_step_begin);

Cela exécuterait l' événement Begin Step (s'il était appelé à partir de l'un des événements d'étape, le code d'événement de début d'étape serait exécuté deux fois).

event_perform(ev_create, 0);

Cela exécuterait l' événement Create de l'instance.
