Génération d'événements d'objets

GameMaker Studio 2 est entièrement piloté par les événements, c'est-à-dire que toutes les actions se produisent à la suite de certains événements qui sont déclenchés dans un ordre spécifique ou par une occurrence spécifique dans le jeu. Il existe un certain nombre d'événements différents, chacun avec une tâche ou un timing spécifique et certains d'entre eux sont même divisés en "sous" événements séparés. Normalement, ces événements sont lancés par GameMaker Studio 2 lorsqu'il détecte un déclencheur dans le jeu, ou à chaque tic-tac du jeu (comme c'est le cas pour le pas de jeu). (comme c'est le cas pour les événements step et draw), mais vous pouvez également appliquer un événement à une instance de la pièce actuelle à partir d'un morceau de code.

Les fonctions suivantes permettent de générer des événements :

  • événement_hérité
  • événement_performer
  • événement_perform_objet
  • utilisateur de l'événement

Il existe également un certain nombre de variables en lecture seule qui peuvent être utilisées avec les fonctions énumérées ci-dessus et qui sont particulièrement utiles pour le débogage :

  • événement_action
  • numéro de l'événement
  • Objet de l'événement
  • type d'événement

Pour plus d'informations sur les événements, veuillez consulter la section Événements relatifs aux objets.
