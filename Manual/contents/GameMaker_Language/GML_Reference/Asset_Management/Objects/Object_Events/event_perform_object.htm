event_perform_object

Cette fonction fonctionne de la même manière que event_perform() sauf que cette fois, vous pouvez spécifier des événements à partir d'un autre objet. Il existe de nombreuses options ici qui permettent une simulation complète de tous les événements possibles, mais notez que cela exécute littéralement tout le code de l'événement et le jeu ne modifiera rien pour le faire se déclencher manuellement, par exemple si vous choisissez d'effectuer un clavier appuyez sur l'événement, l'événement sera déclenché mais la touche correspondante ne sera pas reconnue comme ayant été enfoncée. Ou si vous effectuez un événement d'alarme, le compteur d'alarme ne sera pas réglé sur -1 mais continuera plutôt à décompter. Vous pouvez trouver une liste complète des constantes disponibles que cette fonction requiert sur la page de la fonction event_perform().

REMARQUE: les actions de l'événement appelé avec cette fonction sont appliquées à l' instance actuelle et non aux instances de l'objet donné.

Syntaxe:

event_perform_object(obj, type, numb);

Argument Description
obj L'objet qui doit avoir son événement déclenché.
type Le type d'événement à réaliser. Voir ci-dessous.
numb L'événement spécifique, si nécessaire (sinon, utilisez simplement 0).

Retour:

N/A

Exemple:

event_perform_object(obj_Player, ev_keypress, ord("W"));

Cela effectuerait l'événement associé à la touche Keyboard Check Pressed> W de l'objet "obj_Player" dans l'instance actuelle.
