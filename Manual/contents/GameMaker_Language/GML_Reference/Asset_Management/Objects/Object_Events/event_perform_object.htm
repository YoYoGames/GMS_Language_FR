événement_perform_objet

Cette fonction fonctionne de la même manière que event_perform() sauf que cette fois, vous pouvez spécifier des événements provenant d'un autre objet. Il existe de nombreuses options qui permettent de simuler tous les événements possibles. événements possibles, mais notez que cette fonction ne fait qu'exécuter tout le code de l'événement et que le jeu ne modifiera rien pour que l'événement se déclenche manuellement. mais la touche correspondante ne sera pas reconnue comme ayant été pressée. Ou encore, si vous déclenchez un événement d'alarme, le compteur d'alarme ne sera pas mis à -1 mais continuera à décompter. Vous trouverez une liste complète des constantes disponibles que cette fonction requiert sur la page de la fonction. requiert à partir de la page de la fonction event_perform().

REMARQUE: Les actions de l'événement appelé avec cette fonction sont appliquées à l'instance actuelle, et non aux instances de l'objet donné.

Syntaxe :

event_perform_object(obj, type, numb) ;

Arguments Description
obj. L'objet dont l'événement doit être déclenché.
type Le type d'événement à réaliser. Voir ci-dessous.
engourdi L'événement spécifique, s'il est nécessaire (sinon, utilisez simplement 0).

Les retours :

N/A

Exemple :

event_perform_object(obj_Player, ev_keypress, ord("W")) ;

Cela permettrait d'exécuter l'événement associé à la touche Keyboard Check Pressed > W de l'objet "obj_Player" dans l'instance actuelle.
