La structure de la piste

La struct track peut être utilisée soit pour lespistes d'actifs deniveau supérieur , soit pour lespistes de paramètres de sous-pistes , et le comportement de la piste est défini par deux choses, son nom et son type . Les structures de piste sont créées à l'aide de la fonction sequence_track_new() et peuvent être récupérées à partir des actifs de la séquence en utilisant la propriété activeTracks de la Sequence Instance Struct ou la propriété tracks de la Sequence Object Struct.

Les propriétés disponibles pour la structure de la piste sont les suivantes :

  • type: Les constantes suivantes peuvent toutes être utilisées pour définir le type de piste qui est créé dans la séquence (ces constantes sont également utilisées lors de la génération des images clés et des données d'images clés, expliquées dans les sections ci-dessous) :
Constant Description Valeur
seqtracktype_graphic Il s'agit d'une piste d'actif graphique (sprite). 1
seqtracktype_audio Il s'agit d'une piste d'actif audio. 2
seqtracktype_instance Il s'agit d'une instance de suivi des actifs. 14
seqtracktype_sequence Il s'agit d'une piste d'actifs séquentiels. 7
seqtracktype_clipmask Il s'agit d'une piste d'actifs de groupe de masques de clips. 8
seqtracktype_clipmask_mask Il s'agit d'une piste d'actif de sprite de masque de clip utilisée pour générer le masque de clip. 9
seqtracktype_clipmask_subject Il s'agit d'une piste d'actif de sprite de masque de clip qui est masquée. 10
seqtracktype_group Ce groupe de dossiers de suivi des actifs. 11
seqtracktype_colour Il s'agit d'une piste de paramètres de données de couleur. 4
seqtracktype_real Il s'agit d'une piste de paramètres à valeur numérique réelle. 3
seqtracktype_message Il s'agit d'une piste de message de diffusion. 15
seqtracktype_moment Il s'agit d'une piste d'événement/moment. 16
seqtracktype_bool Non utilisé actuellement. 5
seqtracktype_string Non utilisé actuellement. 6
seqtracktype_spriteframes Non utilisé actuellement. 13
seqtracktype_empty Non utilisé actuellement. 12

  • name: lors de la création d'une piste d'actif, le nom que vous donnez ici peut être n'importe quelle chaîne de caractères nécessaire pour identifier la piste . que vous créez (ces chaînes ne sont applicables qu'aux types de pistes de paramètres seqtracktype_real et seqtracktype_colour ) :
    • "position" - lorsqu'elle est définie sur une piste de type seqtracktype_real, elle indique que le contenu de cette piste doit être utilisé comme données de position.
    • "scale" - lorsqu'il est défini sur une piste de type seqtracktype_real, cela indique que le contenu de cette piste doit être utilisé comme données de mise à l'échelle.
    • "rotation" - lorsqu'il est défini sur une piste de type seqtracktype_real, cela indique que le contenu de cette piste doit être utilisé comme données de rotation.
    • "blend_multiply" - lorsqu'il est défini sur une piste de type seqtracktype_colour, ce paramètre indique que le contenu de cette piste doit être utilisé comme données de multiplication des couleurs. Les données d'image clé pour une telle piste doivent être être un tableau de quatre valeurs au format [A, R, G, B]. Notez que les valeurs de chaque composant sont exprimées entre 0 et 1, où 0 correspond à la valeur HEX #00 et 1 correspond à la valeur HEX #FF (0 - 255 comme indiqué dans le sélecteur de couleurs pour les pistes de mélange d'images dans la fenêtre d'aperçu). pour les pistes de fusion d'images dans l'éditeur de séquences).
    • "image_speed" - lorsqu'il est défini sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme valeur de vitesse de l'image.
    • "image_index" - lorsqu'il est défini sur une piste de type seqtracktype_real, cela indique que le contenu de cette piste doit être utilisé comme valeur d'index d'image.
    • "Image_angle" - Lorsqu'il est défini sur une piste de type seqtracktype_real, cela indique que le contenu de cette piste doit être utilisé comme valeur d'angle de l'image.
    • "image_blend" - lorsqu'il est défini sur une piste de type seqtracktype_colour, ce paramètre indique que le contenu de cette piste doit être utilisé comme valeur de fusion de l'image. Les données d'image clé d'une telle piste doivent être un tableau de quatre valeurs au format [image_blend]. tableau de quatre valeurs au format [A, R, G, B]. Notez que les valeurs de chaque composant sont exprimées entre 0 et 1, où 0 correspond à la valeur HEX #00 et 1 correspond à la valeur HEX #FF (0 - 255 comme indiqué dans le sélecteur de couleur pour les pistes à couleurs multiples dans l'éditeur de séquences).
  • pistes: Cette propriété permet d'accéder à la liste des pistes qui sont les enfants de cette piste. Lorsque vous obtenez cette propriété, un tableau de structures de pistes est renvoyé, et lorsque vous définissez cette propriété un tableau de structures de pistes doit être spécifié.
  • visible: Ceci indique si cette piste est visible (la valeur est true) ou non (la valeur est false). Vous pouvez obtenir ou définir cette valeur et si une piste n'est pas visible, aucune de ses pistes enfants ne sera dessinée non plus.
  • images-clés: Cette propriété permet d'accéder à la liste des structures d'images clés pour la piste. Lorsque vous obtenez cette propriété, un tableau de structures d'images clés est retourné, et lorsque vous définissez cette propriété, un tableau de structures d'images clésdoit être spécifié. doit être spécifié.
