audio_queue_sound

Cette fonction ajoutera les données d'un tampon dans la file d'attente audio que vous avez précédemment créée à l'aide de la fonction audio_create_play_queue(). Vous spécifiez l'index de la file d'attente à ajouter, et l'ID du tampon à partir duquel les données doivent être récupérées. l'ID du tampon d'où proviennent les données, ainsi que la position (décalage) dans le tampon et le nombre d'octets à ajouter. Une fois que vous avez ajouté des données audio d'un tampon à une file, le tampon ne peut pas être supprimé tant que vous n'avez pas libéré la file à l'aide de la fonction audio_free_play_queue()Les propriétés du tampon doivent correspondre à celles de la file à laquelle vous ajoutez le son.

Syntaxe :

audio_queue_sound(queueIndex, bufferId, bufferOffset, bufferLength) ;

Arguments Description
queueIndex L'index de la file d'attente à ajouter.
bufferId Le tampon à ajouter à la file d'attente.
bufferOffset Le décalage dans le tampon source à partir duquel il faut commencer.
longueur du tampon La longueur du tampon (le nombre d'octets dans le tampon).

Les retours :

N/A

Exemple :

len = async_load[ ? "data_len"] ;
audio_buff = buffer_create(len, buffer_fast, 1) ;
buffer_copy(async_load[ ? "buffer_id"], 0, len, buff, 0) ;
audio_queue_sound(audio_queue, audio_buff, 0, len) ;
audio_play_sound(audio_queue, 10, 0) ;

Le code ci-dessus est appelé dans l'événement asynchrone d'enregistrement audio et attribue certaines données enregistrées à un tampon, qui est ensuite ajouté à une file d'attente audio. ensuite ajouté à une file d'attente audio. Celle-ci est ensuite lue.
