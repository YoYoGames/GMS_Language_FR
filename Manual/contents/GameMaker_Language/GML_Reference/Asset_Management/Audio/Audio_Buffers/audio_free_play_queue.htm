audio_free_play_queue

Cette fonction permet de libérer la mémoire associée à la file d'attente audio donnée. L'index de la file d'attente est la valeur renvoyée lorsque vous avez créé la file d'attente à l'aide de la fonction audio_create_play_queue(), et cette fonction doit être appelée lorsque la file d'attente n'est plus nécessaire pour éviter les fuites de mémoire. La libération de la file d'attente arrêtera tout son en cours de lecture et vous ne pourrez pas supprimer la mémoire tampon à partir de laquelle un son est diffusé tant que la file d'attente à laquelle il est affecté n'a pas été libérée. Cette fonction déclenchera un événement asynchrone de lecture audio, et dans ce cas, une carte DS spéciale sera créée dans la variable async_load avec les paires clé / valeur suivantes:

  • " queue_id " - la file d'attente qui a fini de lire les données
  • " buffer_id " - le tampon qui n'est plus lu à partir de
  • " queue_shutdown " - il est mis à 0 pendant la lecture normale et à 1 lorsque l'événement est reçu car vous avez libéré la file d'attente avec cette fonction. Lorsqu'il est défini sur 1, vous ne souhaitez pas mettre d'autres données en file d'attente.

Syntaxe:

audio_free_play_queue(queueIndex);

Argument Description
queueIndex L'index de la file d'attente à libérer.

Retour:

N/A

Exemple:

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
    {
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
    }

Le code ci-dessus ira dans l' événement de lecture audio asynchrone et vérifie si l'audio qui a fini de jouer provient de "audio_queue" et si c'est le cas, il libère la file d'attente et le tampon qui lui est associé.
