file d'attente_audio_créer_lecture

Cette fonction prépare une file d'attente tampon pour l'audio. Vous passez le format des données (seuls buffer_u8 ou buffer_s16 sont actuellement pris en charge), la fréquence d'échantillonnage (qui peut être comprise entre 1000hz et 48000hz), et les canaux que l'audio requiert à partir de l'une des constantes listées ci-dessous :

Constant Description
audio_mono Audio mono (un seul canal).
audio_stereo Audio stéréo (double canal).
audio_3D Audio 3D (5.1).


Notez que vous devez libérer toutes les files d'attente audio créées avec cette fonction lorsqu'elles ne sont plus nécessaires afin d'éviter les fuites de mémoire en utilisant la fonction audio_free_play_queue().

Cette fonction déclenchera un événement asynchrone de lecture audio lorsque la file d'attente aura atteint sa fin . sera créée dans la variable async_load avec les paires clé/valeur suivantes :

  • "queue_id" - la file d'attente qui a terminé la lecture des données.
  • "buffer_id" - le tampon qui n'est plus lu.
  • "queue_shutdown" - ceci est mis à 0 pendant la lecture normale et à 1 lorsque l'événement est reçu car audio_free_play_queue() a été appelé. Lorsque ce paramètre est défini à 1, vous ne voulez pas ne voulez pas mettre en file d'attente d'autres données.

Syntaxe :

audio_create_play_queue(queueFormat, queueRate, queueChannels) ;

Arguments Description
queueFormat Le format des données de la mémoire tampon à utiliser(buffer_u8 ou buffer_s16).
taux de file d'attente Le taux d'échantillonnage des données dans la file d'attente de la mémoire tampon.
canaux de file d'attente Les canaux à utiliser à partir d'une des constantes listées ci-dessous.

Les retours :

Real (ID de la mémoire tampon)

Exemple :

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono) ;

Le code ci-dessus crée une nouvelle file d'attente audio et attribue son ID à une variable pour référence ultérieure.
