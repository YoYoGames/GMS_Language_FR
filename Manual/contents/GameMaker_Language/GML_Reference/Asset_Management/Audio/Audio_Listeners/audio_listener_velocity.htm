vitesse de l'auditeur audio

Cette fonction peut être utilisée pour donner à l'auditeur des effets Doppler et simuler un mouvement audio basé sur le vecteur qui est résolu à partir des positions relatives x, y et z données. Si l'auditeur lui-même ne se déplace jamais, ou si le mouvement n'est pas constant, il n'est normalement pas nécessaire de définir ces paramètres. Si l'écouteur lui-même ne se déplace jamais, ou si le mouvement n'est pas constant, vous n'avez normalement pas besoin de définir ces valeurs, mais, par exemple, si vous créez un jeu de défilement où le joueur a un mouvement constant de bas en haut et les ennemis un mouvement constant de haut en bas, vous définirez la vitesse de l'écouteur et de l'émetteur en fonction de la position de l'écouteur. vitesse de l'auditeur et de l' émetteur (pour les émetteurs, vous utiliserez audio_emitter_velocity()) aux vecteurs appropriés pour simuler l'effet Doppler correct lorsqu'ils passent devant l'instance du joueur. correct lorsqu'ils passent devant l'instance du joueur.

NOTE : si vous avez plusieurs écouteurs, vous devez utiliser la fonction audio_listener_set_velocity().

L'image ci-dessous montre comment cela pourrait être configuré pour l'exemple de jeu donné ci-dessus :

Syntaxe :

audio_listener_velocity(vx, vy, vz) ;

Arguments Description
vx La composante de vitesse x de l'auditeur (par défaut 0).
vy La composante y de la vélocité de l'auditeur (par défaut 0).
vz La composante z de la vélocité de l'auditeur (par défaut 0).

Les retours :

N/A

Exemple :

si vitesse > 0
{
audio_listener_velocity(abs(hspeed), abs(vspeed), 0) ;
}

Le code ci-dessus vérifie si la vitesse de l'instance du lecteur est supérieure à 0 et si c'est le cas, il utilise les composants absolus appropriés hspeed et vspeed pour définir la vitesse de l'auditeur.
