audio_debug

Cette fonction peut être utilisée pour afficher des informations de débogage sur le système audio, avec true pour l'activer et false pour le désactiver. Lorsque vous déboguez l'audio de cette manière, la superposition affichera les informations suivantes :

  • Handle - Le handle de la source OpenAL (unique pour chaque son joué)
  • Data Handle - Le handle du buffer OpenAL (montre la même valeur pour plusieurs sons avec le même handle source)
  • Processed Buffers - combien de buffers ont été lus ?
  • Buffers en file d'attente - combien de buffers ont été mis en file d'attente (et sont toujours en attente de lecture)
  • Gain - La valeur du gain pour l'audio (0 - 65536)
  • Play Offset - la position de l'échantillon (position de lecture actuelle) dans le tampon actuel.
  • Barres des tampons en file d'attente - un petit diagramme en forme de boîte des tampons traités/en file d'attente avec le décalage de lecture (position actuelle) dessiné comme une ligne bleue.

Pour les groupes audio synchronisés, vous disposez également de la fonction audio_sync_group_debug().

REMARQUE : cette fonction ne fonctionne pas sur la plateforme cible HTML5.

Syntaxe :

audio_debug(enable) ;

Arguments Description
activer Active (à true) ou désactive(à false) le débogage audio.

Les retours :

N/A

Exemple :

if (debug_mode)
{
audio_debug(true) ;
}
else
{
audio_debug(false) ;
}

Le code ci-dessus permet d'activer ou de désactiver la superposition de débogage audio selon que le jeu fonctionne en mode de débogage ou non.
