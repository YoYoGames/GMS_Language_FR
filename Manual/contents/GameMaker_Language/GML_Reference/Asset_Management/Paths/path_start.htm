path_start

Un tracé est créé à partir d'une série de points de définition qui sont liés entre eux, puis utilisés pour planifier les mouvements d'une instance. Ils peuvent être créés avec du code ou dans l'éditeur de chemin et ils sont affectés à une instance à utiliser dans le jeu. Vous utiliseriez ensuite cette fonction pour indiquer à votre instance quel chemin suivre, quelle vitesse suivre le chemin (mesurée en pixels par étape), comment se comporter lorsqu'il atteint la fin du chemin et s'il faut suivre le chemin absolu ou relatif. positionner. Cette dernière partie signifie qu'il commence et suit le chemin exactement comme vous l'avez conçu et placé dans l' éditeur de chemin (absolu), ou qu'il commence et suit le chemin à partir de la position à laquelle l'instance a été créée (relative).

Comme vous pouvez le voir dans l'image ci-dessus, les deux instances suivent le même chemin commencé par path_start(), mais l'une des instances suit le chemin absolu tandis que l'autre suit le chemin relatif. Il convient de noter que lorsque relative est défini sur false (absolu), l'instance «sautera» au début du chemin, peu importe où dans la pièce elle est placée.

Les constantes utilisées pour définir le comportement de l'instance lorsqu'elle atteint la fin du chemin sont les suivantes:

Constant Description
path_action_stop Terminer le chemin
path_action_restart Continuez le chemin depuis le début, en sautant à nouveau à la position de départ si le chemin n'est pas fermé
path_action_continue Continuer à partir de la position actuelle
path_action_reverse Reculer le long du chemin (obtenu en inversant la vitesse de déplacement du chemin)

Syntaxe:

path_start(path, speed, endaction, absolute);

Argument Description
path L'index de chemin pour commencer.
speed La vitesse à laquelle suivre le chemin en pixels par pas, signifiant négatif en arrière.
endaction Que faire lorsque la fin du chemin est atteinte (voir les constantes répertoriées ci-dessous).
absolute Si l'instance appelante doit suivre le chemin absolu tel qu'il est dans l'éditeur (true) ou un chemin relatif vers sa position actuelle (false).

Retour:

N/A

Exemple:

path_start(path, 4, path_action_reverse, 0);

Cela obligera l'instance appelante à suivre le chemin indexé dans la variable "chemin" à une vitesse de 4 pixels par étape. Il suivra le chemin par rapport à sa propre position, et quand il atteindra la fin, il inversera la direction et reviendra dans le sens où il est venu.
