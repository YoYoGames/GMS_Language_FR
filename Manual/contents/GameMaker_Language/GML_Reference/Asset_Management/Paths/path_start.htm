Début du chemin

Un chemin est créé à partir d'une série de points de définition qui sont reliés entre eux et ensuite utilisés pour planifier les mouvements d'une instance. Ils peuvent être créés avec du code ou dans l'éditeur de chemin et sont assignés à une instance pour être utilisés dans le jeu. à utiliser dans le jeu. Vous utiliserez alors cette fonction pour indiquer à votre instance quel chemin suivre, à quelle vitesse suivre le chemin (mesurée en pixels par pas), comment se comporter lorsqu'elle atteint la fin du chemin et si elle doit suivre la position absolue ou relative du chemin. position. Cette dernière partie signifie qu'il démarre et suit le chemin exactement comme vous l'avez conçu et placé dans l'éditeur de chemin (absolu), ou qu'il démarre et suit le chemin à partir de la position à laquelle l'instance a été créée (relatif). à laquelle l'instance a été créée (relative).

Comme vous pouvez le voir dans l'image ci-dessus, les deux instances suivent le même chemin commencé avec path_start(), mais l'une des instances suit le chemin absolu tandis que l'autre suit le chemin relatif. Il est à noter que lorsque relative est définie sur false (absolu), l'instance "sautera" au début du chemin, peu importe où elle se trouve dans la pièce. la pièce où elle est placée.

Les constantes utilisées pour définir le comportement de l'instance lorsqu'elle atteint la fin du chemin sont les suivantes :

Constant Description
path_action_stop Terminer le chemin
path_action_restart Poursuivre le chemin depuis le début, en revenant à la position de départ si le chemin n'est pas fermé.
path_action_continue Continuer à partir de la position actuelle
path_action_reverse Revenir en arrière sur le chemin (en inversant la vitesse de déplacement du chemin).

Syntaxe :

path_start(path, speed, endaction, absolute) ;

Arguments Description
chemin L'index du chemin pour commencer.
vitesse La vitesse à laquelle il faut suivre le chemin en pixels par pas, négatif signifiant aller en arrière.
action Ce qu'il faut faire lorsque la fin du chemin est atteinte (voir les constantes listées ci-dessous).
absolu Si l'instance appelante doit suivre le chemin absolu tel qu'il est dans l'éditeur (true) ou un chemin relatif à sa position actuelle (false).

Les retours :

N/A

Exemple :

path_start(path, 4, path_action_reverse, 0) ;

L'instance appelante suivra le chemin indexé dans la variable "path" à une vitesse de 4 pixels par pas. Elle suivra le chemin par rapport à sa propre position, et lorsqu'elle atteindra la fin, elle inversera sa direction et retournera sur ses pas. chemin qu'elle a suivi.
