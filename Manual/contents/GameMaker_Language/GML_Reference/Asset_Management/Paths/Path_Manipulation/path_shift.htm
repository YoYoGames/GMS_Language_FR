path_shift

Avec cette fonction, vous pouvez déplacer un chemin le long de l'axe horizontal et vertical. Veuillez noter que cela n'aura aucun effet visible dans le jeu si l'instance suivant le chemin en cours de déplacement le suit relativement (plutôt qu'absolument) car la position du chemin est alors ignorée. Cette fonction modifie l'actif du chemin réel, et affectera donc de manière permanente la façon dont le chemin est utilisé par toutes les instances du jeu à partir du moment où la fonction est utilisée jusqu'à la fin du jeu. Si ce n'est pas ce dont vous avez besoin, vous devez utiliser une fonction telle que path_duplicate() pour créer d'abord une copie du chemin, puis appelez cette fonction sur l'actif dupliqué (n'oubliez pas d'appeler path_delete () sur l'actif lorsqu'il n'est plus nécessaire).

Syntaxe:

path_shift(index, xshift, yshift);

Argument Description
index L'index du chemin à changer.
xshift Le nombre de pixels horizontaux pour décaler le chemin. Négatif = gauche, positif = droite.
yshift Le nombre de pixels verticaux pour décaler le chemin. Négatif = haut, positif = bas.

Retour:

N/A

Exemple:

my_path = path_duplicate(pth_AI);
path_shift(my_path, 50, 50);
path_start(my_path, 4, path_action_reverse, true);

Le code ci-dessus duplique un élément de chemin et stocke sa valeur d'identification dans une variable. Cette variable est ensuite utilisée pour décaler la position du chemin dupliqué, puis l'instance démarre le chemin.

Retour: Manipulation de chemin
