Constantes de Shader

Outre les shader définies dans les pages de référence OpenGL ES Shading Language (GLSL ES), il existe également un certain nombre de shader qui sont uniques à GameMaker Studio 2.

Les constantes de matrice d'affichage suivantes peuvent être utilisées comme indices de tableau lors de l'utilisation de la constante de tableau shader gm_Matrices:

Constant Description
MATRIX_VIEW Cette constante d'index de tableau contient l'index de la matrice de vue actuelle. L'index retourné serait utilisé comme valeur de tableau lors de l'appel du gm_Matrices constante dans le code shader
MATRIX_PROJECTION Cette constante d'index de tableau contient l'index de la matrice de projection actuelle. L'index retourné serait utilisé comme valeur de tableau lors de l'appel du gm_Matrices constante dans le code shader
MATRIX_WORLD Cette constante d'index de tableau contient l'index de la matrice mondiale actuelle. Cela peut être utilisé pour des choses comme l'éclairage si vous avez des informations sur la lumière dans l'espace-monde. L'index retourné serait utilisé comme valeur de tableau lors de l'appel du gm_Matrices constante dans le code shader
MATRIX_WORLD_VIEW Cette constante d'index de tableau contient l'index du résultat du monde et des matrices de vue multipliées ensemble. Ceci est souvent utilisé pour des choses comme le brouillard. L'index retourné serait utilisé comme valeur de tableau lors de l'appel du gm_Matrices constante dans le code shader
MATRIX_WORLD_VIEW_PROJECTION Cette constante d'index de tableau contient l'index du résultat des matrices monde, vue et projection multipliées ensemble. Il s'agit de la matrice de transformation normale utilisée pour les positions des sommets. L'index retourné serait utilisé comme valeur de tableau lors de l'appel du gm_Matrices constante dans le code shader
MATRIX_MAX Ce n'est pas un index de tableau, mais renvoie plutôt la taille du tableau matriciel dans le vertex shader.
MAX_VS_LIGHTS Ce n'est pas un index de tableau, mais renvoie plutôt le nombre de lumières dans le vertex shader.

La constante suivante est également disponible pour l'éclairage:

Constant Description
MAX_VS_LIGHTS Le nombre maximum de lumières ponctuelles et directionnelles disponibles dans le shader

Les uniformes et constantes de matrice prédéfinis suivants peuvent être utilisés dans votre shader pour accéder aux valeurs spécifiques de GameMaker Studio 2

Constant Description
gm_Matrices[matrix] Cette constante de tableau renvoie une matrice de transformation et est l'un des uniformes prédéfinis disponibles que GameMaker Studio 2 crée pour que vous puissiez l'utiliser dans l'éditeur de code de shader L'index du tableau est choisi parmi l'une des constantes répertoriées ci-dessus.
gm_BaseTexture Il s'agit d'une constante d'échantillonnage 2D qui renvoie la texture de l'objet actuel, telle que définie par GameMaker Studio 2. Ce serait donc le sprite, la surface ou la texture actuel qui serait normalement utilisé lors du dessin sans que le shader soit appelé.
gm_LightingEnabled Cela peut être utilisé pour obtenir ou définir l' GameMaker Studio 2 lors de l'utilisation de la 3D.
gm_FogStart Cela peut être utilisé pour obtenir la distance à laquelle les polygones commencent à être fusionnés avec la couleur du brouillard.
gm_RcpFogRange Cela peut être utilisé pour obtenir la distance à laquelle le brouillard est maximal et rien ne peut plus être vu.
gm_PS_FogEnabled Cela reviendra true ou false si le GPU a activé ou non le brouillard de pixels.
gm_FogColour Cela peut être utilisé pour obtenir la couleur de brouillard utilisée par GameMaker Studio 2.
gm_VS_FogEnabled Cela reviendra true ou false si le GPU a activé ou non le brouillard de sommet.
gm_AlphaTestEnabled Cela peut être utilisé pour obtenir des tests alpha dans le shader. Voir gpu_set_alphatestenable() pour plus d'informations sur les tests alpha.
gm_AlphaRefValue Cela peut être utilisé pour obtenir la valeur de référence actuelle du test alpha. Voir gpu_set_alphatestref() pour plus d'informations sur la référence du test alpha.
