instance_create_depth

Avec cette fonction, vous pouvez créer une nouvelle instance de l'objet spécifié à n'importe quel point donné dans la pièce et à la profondeur spécifiée. La profondeur peut être n'importe quelle valeur, où plus la profondeur est basse, plus les objets sont "proches" de la caméra et plus la profondeur est élevée, plus ils sont éloignés, de sorte qu'une instance à une profondeur de -200 sera dessinée sur une instance à une profondeur de +300 ( par exemple).

Notez que cette fonction créera en fait une couche de pièce pour l'instance, puisque toutes les instances doivent être sur une couche dans la pièce, mais puisqu'il s'agit d'une couche gérée (c'est-à-dire: pas celle que vous avez créée via le code ou dans la pièce, mais celui que GameMaker Studio 2 a créé automatiquement) vous ne pouvez pas y accéder, et le layer la variable d'instance renverra -1.

IMPORTANT! Il existe une profondeur de couche minimale et maximale de -16 000 à 16 000. Tout ce qui est placé à une profondeur en dehors de cette plage ne sera pas dessiné bien que tous les événements se déroulent toujours normalement.

Cette fonction renvoie le id de la nouvelle instance qui peut ensuite être stockée dans une variable ou utilisée pour accéder à cette instance. Notez que cette fonction appellera également l' événement Create de l'instance en cours de création avant de continuer avec le code ou les actions pour l'événement qui a appelé la fonction.

Syntaxe:

instance_create_depth(x, y, depth, obj);

Argument Description
x La position x à laquelle l'objet sera créé
y La position y à laquelle l'objet sera créé
depth La profondeur à laquelle affecter l'instance créée
obj L'index d'objet de l'objet pour créer une instance de

Retour:

Real (instance ID value)

Exemple:

var inst = instance_create_depth(x, y, -10000, obj_Bullet);
with (inst)
    {
    speed = other.shoot_speed;
    direction = other.image_angle;
    }

Le code ci-dessus crée une nouvelle instance de l'objet "obj_Bullet" et stocke l'ID d'instance dans une variable. Cette variable est ensuite utilisée pour attribuer speed et direction à la nouvelle instance.
