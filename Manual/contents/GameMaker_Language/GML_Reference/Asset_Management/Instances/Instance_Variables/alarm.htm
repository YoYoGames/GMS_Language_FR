alarm

Ce tableau à 1 dimension est utilisé pour obtenir la valeur actuelle de toutes les alarmes que l'instance peut avoir, ou il peut être utilisé pour définir ces alarmes. Il y a douze alarmes intégrées dans chaque instance d'un objet, et chacune a sa propreevent qui s'exécutera lorsque cette variable atteindra 0.

Les alarmes sont décomptées au début de chaque étape, après l'événement Begin Step mais avant l'événement Step principal et avant que quoi que ce soit ne soit rendu pour l'étape. Il convient de noter que l'alarme n'est pas terminée lorsqu'elle atteint 0 (bien que l'événement ait été déclenché) à l'étape suivante, elle descendra à -1, donc si vous devez arrêter une alarme pour une raison quelconque, vous devez définir son tableau. valeur -1 et non 0. Les temps d'alarme sont calculés par pas de jeu, avec une valeur de 30 correspondant à 30 pas, 60 à 60 pas, etc...

REMARQUE: une alarme sans actions ni code ne sera pas exécutée. Cependant, même avec juste un commentaire et aucun code, l'alarme compte à rebours.

Vous pouvez définir le tableau d'alarme directement (et récupérer la valeur d'alarme actuelle directement en utilisant le tableau également), mais dans certaines circonstances, ce n'est pas la méthode la plus appropriée, vous pouvez donc également utiliser la fonction alarm_set() pour régler une alarme et la fonction alarm_get() pour obtenir la valeur d'une alarme.

Syntaxe:

alarm[val];

Retour:

Integer (-1 if the alarm isn't running)

Exemple:

if canshoot
    {
    if keyboard_check_pressed(vk_space)
        {
        canshoot = false;
        alarm[0] = room_speed;
        instance_create_layer(x, y, "Bullets", obj_Bullet);
        }
    }

Le code ci-dessus vérifie une variable, puis si elle est vraie, vérifie si une touche est enfoncée et si cela est vrai, elle définit finalement la variable sur false, définit une alarme (où elle sera à nouveau définie sur true) et crée une instance.
