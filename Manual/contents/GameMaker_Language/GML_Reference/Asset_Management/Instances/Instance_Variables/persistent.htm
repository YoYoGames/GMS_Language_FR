persistent

Cette variable peut être lue pour savoir si l'instance est marquée comme persistent ou pas, ou il peut être utilisé pour définir la persistance sur true (persistant) ou false (non persistant) pour l'instance. Une instance persistante est une instance qui sera "reportée" d'une pièce à l'autre, ce qui signifie (par exemple) qu'elle ne doit être créée qu'une seule fois au début du jeu et qu'elle sera présente dans toutes les autres pièces. Des précautions doivent être prises avec la persistance car il est facile de perdre la trace des instances persistantes qui peuvent conduire à des problèmes plus tard dans le développement du jeu.

Lorsque vous créez une instance avec la persistance activée dans une salle, cette variable sera définie sur true et il se verra attribuer un calque ou une profondeur (selon la fonction que vous avez utilisée pour créer l'instance de l'objet). Lorsque la pièce est modifiée et si la pièce suivante n'a pas de calque portant le même nom ou la même profondeur que celui attribué, un nouveau calque est créé pour l'instance qui persiste dans les pièces. Si vous avez donné un calque nommé lors de la création de l'instance, le nouveau calque sera également nommé de la même manière que le calque d'origine, tandis que si vous attribuez une profondeur à l'instance, le nouveau calque sera nommé "_layer_XXX ", où" XXX "est une valeur hexadécimale utilisée pour donner au calque un nom unique. Ces calques seront supprimés de la pièce à l'extrémité de la pièce. Une autre chose à noter concernant les calques et les instances persistantes est que si vous avez attribué à l'instance un nom lors de la création, et il existe un autre calque dans les pièces suivantes portant le même nom, l'instance persistante sera affectée au calque portant le même nom, quelle que soit la profondeur du calque. Enfin, si une occurrence persistante se déplace vers une pièce avec un calque à la même profondeur que celle sur laquelle l'instance a été créée, il ne sera pas affecté à ce calque, mais un nouveau calque sera créé à la même profondeur (selon la convention de dénomination expliquée précédemment).

Un objet persistant aura toujours ses événements Game Start, Game End, Room Start et Room End déclenchés, cependant si vous redémarrez le jeu (avec, par exemple, le game_restart() fonction) tous les objets persistants seront supprimés et n'existeront que lorsqu'ils seront créés à nouveau par le jeu. Notez également que si vous désactivez un objet persistant, il ne passera plus d'une pièce à une autre à moins d'être réactivé avant le déclenchement de l'événement de fin de pièce.

Syntaxe:

persistent;

Retour:

Boolean

Exemple:

if !persistent
    {
    persistent = true;
    }

Le code ci-dessus vérifiera l'instance pour voir si la persistance est marquée comme false et si l'instance n'est PAS persistante, l'indicateur de persistance est défini sur true.
