depth

Lorsque vous créez un objet, vous pouvez lui attribuer une profondeur initiale qui définit comment les instances de cet objet seront dessinées dans la pièce lorsque le jeu est joué et cette variable peut être utilisée pour obtenir et modifier cette valeur de profondeur pendant le jeu. fonctionnement. Vous n'aurez normalement pas besoin d'utiliser cette variable car vous devriez définir des instances à ajouter à des couches discrètes, qui à leur tour sont définies sur une profondeur spécifique, mais il se peut que vous souhaitiez modifier la profondeur d'une instance à l'aide de cette valeur, auquel cas un "calque temporaire" sera créé spécifiquement pour l'instance à la profondeur donnée. Notez que lorsqu'aucune instance n'est sur la même profondeur, cette couche temporaire sera supprimée de la mémoire (contrairement aux couches régulières qui resteront même si elles n'ont rien sur elles).

IMPORTANT! Lorsque vous modifiez le depth et GameMaker Studio 2 gère les couches, la variable de couche intégrée renverra -1 et non l'ID de couche, car les couches gérées ne peuvent pas être manipulées via le code.

Dans GameMaker Studio 2 plus la valeur de profondeur d'une instance est basse, plus cette instance sera "plus proche de la caméra", tandis qu'une valeur de profondeur plus élevée signifie que l'instance sera dessinée "plus loin de la caméra", c'est-à-dire: -1000 est dessiné au-dessus de -100, qui est dessiné au-dessus de 0, qui est dessiné au-dessus de 100, qui est dessiné au-dessus de 1000.

Notez que les instances d'un objet qui ont la même profondeur peuvent être dessinées au -dessus ou en dessous les unes des autres même lorsqu'elles semblent être placées les unes sur les autres dans l'éditeur de pièce. Si vous voulez garantir que quelque chose est dessiné sur ou sous tout le reste, vous devez toujours définir la profondeur (ou le calque) explicitement. Notez également qu'il existe une valeur de profondeur minimale (-16000) et maximale (+16000) en dehors de laquelle les instances ne seront pas dessinées, bien qu'elles existent toujours et traitent les événements.

REMARQUE: vous ne pouvez pas définir la profondeur d'une instance dans son événement de dessin (tous les autres événements conviennent).

Syntaxe:

depth;

Retour:

Real (single precision floating point value)

Exemple:

if y != yprevious
    {
    depth = -y;
    }

Le code ci-dessus vérifiera si la position y a changé et si c'est le cas, la profondeur sera également définie pour y correspondre.
