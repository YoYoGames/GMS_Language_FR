Variables d'instance

Lorsque vous créez un nouvel objet, il viendra avec certaines variables déjà initialisées avec des valeurs par défaut. Nous appelons ces variables des variables d' instance, car elles seront créées pour chaque instance que vous placez de l'objet dans vos salles de jeux, et une fois qu'une instance est créée, les valeurs de ces variables sont uniques à cette instance et uniquement à cette instance. Certaines de ces variables que vous utiliserez beaucoup, comme x et y, tandis que d'autres sont moins nécessaires en fonction de ce que vous voulez faire, mais en général, ils sont très utiles et, dans la mesure du possible, vous devriez chercher à les utiliser plutôt que de créer vos propres variables personnalisées.

La plupart des variables d'instance peuvent être définies aussi bien que lues, ce qui vous permet de modifier les propriétés et le comportement d'une instance simplement en modifiant la valeur d'une certaine variable - vous pouvez empêcher une instance de dessiner, par exemple, en définissant simplement le visible variable intégrée à false. Vous trouverez ci-dessous les différentes variables initialisées pour toutes les instances de tous les objets de votre jeu.

IMPORTANT! Si une instance est dans unsequence puis certaines de ces variables - commeimage_xscale /image_yscale / image_angle /x /y - sera écrasé lorsque la séquence sera mise à jour à chaque étape après le début de la lecture.

Ces variables traitent des propriétés générales des instances:

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

Ces variables traitent de la position et du mouvement de l'instance:

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

Propriétés des objets

La variable suivante contient l'ID d'index unique de l'objet à partir duquel l'instance a été créée:

  • object_index

Ces variables sont toutes liées au sprite affecté à l'instance et peuvent être utilisées pour changer ce qui est dessiné et comment:

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

Ces variables concernent le masque de collision:

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Ces variables sont liées aux chemins et à la manière dont l'instance interagit avec une, si elle est affectée:

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Ces variables permettent de définir une instance pour utiliser une chronologie:

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

Séquences

La variable suivante est utilisée lorsqu'une instance est utilisée dans une séquence:

  • in_sequence

Il existe un grand nombre de variables intégrées à utiliser avec les fonctions physiques de GameMaker Studio 2, et pour que les choses soient plus claires, elles peuvent être trouvées dans la section du manuel qui couvre tout ce qui concerne la simulation physique:

  • La physique
