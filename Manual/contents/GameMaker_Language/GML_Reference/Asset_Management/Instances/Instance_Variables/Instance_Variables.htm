Variables d'instance

Lorsque vous créez un nouvel objet, il est livré avec certaines variables déjà initialisées avec des valeurs par défaut. Ces variables sont appelées variables d'instance, car elles seront créées pour chaque instance que vous placerez de l'objet dans vos salles de jeu, et une fois qu'une instance est créée, les valeurs de ces variables sont uniques à cette instance et uniquement à cette instance. Certaines de ces variables seront très utilisées, comme x et y, tandis que d'autres seront moins nécessaires en fonction de ce que vous voulez faire. Mais en général, elles sont très utiles et, dans la mesure du possible, vous devriez les utiliser plutôt que de créer vos propres variables personnalisées.

La plupart des variables d'instance peuvent être définies et lues, ce qui vous permet de modifier les propriétés et le comportement d'une instance en modifiant simplement la valeur d'une certaine variable. visible à false. Vous trouverez ci-dessous les différentes variables qui sont initialisées pour toutes les instances de tous les objets de votre jeu.

IMPORTANT ! Si une instance se trouve dans une séquence, certaines de ces variables - comme image_xscale / image_yscale / image_angle / x / y - seront écrasées lors de la mise à jour de la séquence met à jour chaque étape après avoir commencé à être jouée.

Ces variables traitent des propriétés générales de l'instance :

  • id
  • visible
  • solide
  • persistant
  • profondeur
  • couche
  • alarme

Ces variables concernent la position et le mouvement de l'instance :

  • direction
  • friction
  • gravité
  • direction_gravité
  • hspeed
  • vspeed
  • vitesse
  • xstart
  • ystart
  • x
  • y
  • xprécédent
  • yprécédent

Propriétés des objets

La variable suivante contient l'ID d'index unique de l'objet à partir duquel l'instance a été créée :

  • objet_index

Ces variables sont toutes liées au sprite attribué à l'instance et peuvent être utilisées pour modifier ce qui est dessiné et comment :

  • index_spirit
  • largeur de sprite
  • hauteur du sprite
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • angle_image
  • mélange d'images
  • image_index
  • numéro de l'image
  • vitesse de l'image
  • image_xscale
  • image_yscale

Ces variables concernent le masque de collision :

  • masque_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Ces variables sont liées aux chemins et à la façon dont l'instance interagit avec l'un d'eux, s'il est assigné :

  • chemin_index
  • path_position
  • path_positionprevious
  • vitesse du chemin
  • path_scale
  • orientation_du_chemin
  • path_endaction

Ces variables permettent de configurer une instance pour utiliser une ligne de temps :

  • index de la chronologie
  • chronologie_fonctionnement
  • vitesse_chronologique
  • position_temporaire
  • boucle de temps

Séquences

Les variables suivantes stockent des informations concernant la séquence qui peut contrôler l'instance à un moment donné :

  • dans_séquence
  • séquence_instance

Il existe un grand nombre de variables intégrées à utiliser avec les fonctions physiques de GameMaker Studio 2, et pour que les choses soient plus claires, elles se trouvent dans la section du manuel qui couvre tout ce qui concerne la simulation physique :

  • Physique
