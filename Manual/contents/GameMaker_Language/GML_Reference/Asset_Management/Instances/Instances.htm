Les instances

Les instances sont créées à partir d' objets et, une fois créées, elles peuvent être modifiées et manipulées via le GML et les DnD™. Les principales fonctions d'instance liées aux instances disponibles et les plus couramment utilisées sont:

  • instance_create_layer
  • instance_create_depth
  • instance_destroy
  • instance_exists
  • instance_change
  • instance_copy
  • instance_find
  • instance_furthest
  • instance_nearest
  • instance_number
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Notez qu'il existe deux variables intégrées supplémentaires (ce sont des variables de portée globale ) et une fonction liée à l'utilisation d'instances qui peuvent être utiles pour récupérer certains détails:

  • instance_id_get
  • instance_id
  • instance_count

Il existe un certain nombre de variables intégrées disponibles dans chaque instance d'un objet qui peuvent être utilisées pour contrôler différents aspects de la façon dont l'instance est affichée, comment elle utilise d'autres actifs et comment elle réagit aux collisions, etc. énumérés ici:

  • Variables d'instance

Les instances ont une variable de tableau qui peut être utilisée pour obtenir ou définir des valeurs d'événement d'alarme, mais il existe également deux fonctions qui peuvent être utilisées pour faire la même chose:

  • alarm_get
  • alarm_set

Enfin, vous avez également la possibilité de désactiver les instances, ce qui signifie qu'elles ne traiteront aucun de leurs événements, même si elles existeront toujours dans le jeu. Les fonctions qui s'y rapportent sont disponibles à partir de la page suivante:

  • Désactiver des instances
