Instances

Lesinstances sont créées à partir des objets et, une fois créées, elles peuvent être modifiées et manipulées par le biais du code GML et des actions DnD™. Les principales fonctions liées aux instances qui sont disponibles et les plus couramment utilisées sont :

  • instance_créer_couche
  • instance_création_profondeur
  • instance_destroy
  • instance_existe
  • instance_change
  • instance_copy
  • instance_find
  • instance_furthest
  • instance_nearest
  • numéro d'instance
  • instance_place
  • liste_de_lieux_instances
  • instance_position
  • liste_position_instance

Notez qu'il y a deux variables supplémentaires intégrées (ce sont des variables de portée globale ) et une fonction liée à l'utilisation des instances qui peut être utile pour récupérer certains détails :

  • instance_id_get
  • identifiant de l'instance
  • nombre d'instances

Il y a un certain nombre de variables intégrées disponibles dans chaque instance d'un objet qui peuvent être utilisées pour contrôler différents aspects de la façon dont l'instance est affichée, comment elle utilise les autres ressources, et comment elle réagit aux collisions, etc... qui sont toutes listées ici :

  • Variables d'instance

Les instances possèdent une variable de type tableau qui peut être utilisée pour obtenir ou définir les valeurs des événements d'alarme, mais il existe également deux fonctions qui peuvent être utilisées pour faire la même chose :

  • alarme_get
  • alarme_set

Enfin, vous avez également la possibilité de désactiver des instances, ce qui signifie qu'elles ne traiteront plus aucun de leurs événements, bien qu'elles existeront toujours dans le jeu. Les fonctions relatives à cela sont disponibles à partir de la page suivante page :

  • Désactivation des instances
