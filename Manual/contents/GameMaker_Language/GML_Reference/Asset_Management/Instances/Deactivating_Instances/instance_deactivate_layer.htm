instance_deactivate_layer

Avec cette fonction, vous pouvez désactiver toutes les instances affectées à une couche spécifique. Vous devez fournir l' ID de couche, qui peut être soit le nom de la couche tel qu'écrit dans l'éditeur de code (sous forme de chaîne), soit la valeur réelle de l'ID de couche renvoyée par le layer_create(), et notez que vous ne pouvez désactiver les couches d'instances qu'avec cette fonction. Notez que si vous avez désactivé un calque qui a des instances d'objets marqués comme persistants, vous devrez réactiver le calque avec la fonction instance_activate_layer() avant de changer de salle, sinon les instances persistantes sur le calque ne seront pas reportées et seront supprimées. Notez également que la désactivation n'est pas instantanée et qu'une instance ainsi désactivée ne sera considérée comme inactive qu'à la fin de l'événement au cours duquel la fonction a été appelée.

REMARQUE: Si vous désactivez une instance au démarrage de la salle (c'est-à-dire à partir du code de création de salle, ou à partir d'un événement de création d'instance d'une instance dans la salle), toutes les instances placées dans la salle à partir de l'éditeur de salle continueront d'exécuter leur événement de création. avant d'être désactivé.

AVERTISSEMENT: la désactivation des instances sur lesquelles la physique est activée n'empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur variable phy_active sur true ou false lorsque vous activez / désactivez les instances.

Syntaxe:

instance_deactivate_layer(obj);

Argument Description
layer La chaîne d'ID de couche (ou la valeur) à utiliser

Retour:

N/A

Exemple:

instance_deactivate_layer("Enemy Layer");
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Le code ci-dessus désactive toutes les instances affectées à la couche "Enemy_Layer" puis active une région dans la salle.

Retour: Désactivation d'instances
