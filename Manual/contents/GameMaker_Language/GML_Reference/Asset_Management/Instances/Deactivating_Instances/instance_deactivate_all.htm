instance_désactivation_tous

Cette fonction désactivera toutes les instances qui se trouvent dans la salle au moment où le code est exécuté. Cela peut inclure l'instance qui exécute le code si l'indicateur "notme" est mis à false, mais normalement vous voudriez que cette instance soit active, auquel cas l'indicateur "notme" devrait être mis à true. soit active, auquel cas l'indicateur "notme" doit être défini sur true. Notez que la désactivation n'est pas instantanée, et une instance qui a été désactivée de cette manière ne sera pas considérée comme inactive jusqu'à la fin de l'événement dans lequel la fonction a été appelée.

NOTE: Si vous désactivez une instance au démarrage de la room (c'est-à-dire à partir du code de création de la room, ou à partir d'un événement de création d'une instance dans la room), toutes les instances qui sont placées dans la room à partir de l'éditeur de room exécuteront toujours leur événement de création avant d'être désactivées.

AVERTISSEMENT: La désactivation des instances dont la physique est activée n' empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur phy_active variable à true ou false lorsque vous activez/désactivez les instances.

Syntaxe :

instance_deactivate_all(notme) ;

Arguments Description
notme Si l'instance appelante doit rester activée (true) ou non (false).

Les retours :

N/A

Exemple :

instance_deactivate_all(true) ;
var _vx = camera_get_view_x(view_camera[0]) ;
var _vy = camera_get_view_y(view_camera[0]) ;
var _vw = camera_get_view_width(view_camera[0]) ;
var _vh = camera_get_view_height(view_camera[0]) ;
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false) ;

Le code ci-dessus désactive toutes les instances, sauf celle qui exécute le code, puis active une région dans la pièce.
