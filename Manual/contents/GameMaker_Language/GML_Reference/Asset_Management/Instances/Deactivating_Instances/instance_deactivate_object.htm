instance_désactivation_objet

Cette fonction vous permet de désactiver une seule instance ou toutes les instances d'un objet spécifique parmi toutes celles qui ont été activées précédemment. Notez que si vous avez désactivé une instance ou un objet qui a été marqué comme Persistant, alors vous devrez le réactiver à nouveau avec la fonction. vous devrez le réactiver à nouveau avec la fonction instance_activate_object() avant de changer de pièce, sinon elle ne sera pas reportée et sera rejetée à la place. Notez également que la désactivation n'est pas instantanée, et une instance qui a été désactivée de cette manière ne sera pas considérée comme inactive avant la fin de l'événement dans lequel la fonction a été appelée.

NOTE: Si vous désactivez une instance au démarrage de la room (c'est-à-dire à partir du code de création de la room, ou à partir d'un événement de création d'une instance dans la room), toutes les instances qui sont placées dans la room à partir de l'éditeur de room exécuteront toujours leur événement de création avant d'être désactivées.

AVERTISSEMENT: La désactivation des instances dont la physique est activée n' empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur phy_active variable à true ou false lorsque vous activez/désactivez les instances.

Syntaxe :

instance_deactivate_object(obj) ;

Arguments Description
obj. L'objet ou l'instance à désactiver (le mot clé all peut également être utilisé).

Les retours :

N/A

Exemple :

instance_deactivate_object(obj_Enemy) ;
var _vx = camera_get_view_x(view_camera[0]) ;
var _vy = camera_get_view_y(view_camera[0]) ;
var _vw = camera_get_view_width(view_camera[0]) ;
var _vh = camera_get_view_height(view_camera[0]) ;
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false) ;

Le code ci-dessus désactive toutes les instances de l'objet "obj_Enemy", puis active une région dans la pièce.

Retour : Désactiver les instances
