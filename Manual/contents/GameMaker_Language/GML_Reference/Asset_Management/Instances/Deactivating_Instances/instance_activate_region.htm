instance_activate_region

Avec cette fonction, vous pouvez définir une région dans la salle pour activer les instances qui ont été précédemment désactivées. Cette région peut être signalée comme "à l'intérieur" ou "à l'extérieur", comme illustré dans l'image suivante:

Vous pouvez voir dans l'image ci-dessus que l'instance "apple" est toujours active car, même si le sprite lui-même ne chevauche pas la région, le cadre de délimitation le fait. Ainsi, les instances sont considérées comme étant dans la région spécifiée lorsque leur boîte englobante la recouvre, et l'état du masque de collision (précis ou non) n'est pas pris en compte. Notez que l'activation n'est pas instantanée et qu'une instance ainsi activée ne sera considérée comme active qu'à la fin de l'événement au cours duquel la fonction a été appelée.

Syntaxe:

instance_activate_region(left, top, width, height, inside);

Argument Description
left Coordonnée x de la gauche de la région rectangulaire à activer.
top Coordonnée y du haut de la région rectangulaire à activer.
width La largeur de la région à activer.
height La hauteur de la région à activer.
inside S'il faut activer les instances à l'intérieur de la région (vrai) ou à l'extérieur (faux).

Retour:

N/A

Exemple:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Le code ci-dessus désactive toutes les instances sauf celle qui exécute le code, puis active une région dans la salle.

Retour: Désactivation d'instances
