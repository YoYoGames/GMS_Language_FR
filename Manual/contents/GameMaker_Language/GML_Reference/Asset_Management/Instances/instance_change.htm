instance_change

Vous pouvez utiliser cette fonction pour transformer une instance d'un objet en une autre instance d'un objet différent, et ce faisant, décider d'exécuter les événements Destroy et Clean Up des instances initiales et l'événement Create des nouvelles instances. De cette façon, vous pouvez (par exemple) faire en sorte qu'une bombe se transforme en une explosion - dans ce cas, l'argument "perf" serait probablement vrai, car vous voudriez que la bombe exécute ses événements Destroy et Clean Up, et que l'explosion exécute son événement Create. ou vous pouvez faire en sorte que votre personnage joueur se transforme en un autre personnage - dans ce cas, l'argument "perf" sera probablement faux car vous ne voulez pas que les instances exécutent leurs événements Create et Destroy/Clean Up.

Il convient de noter que la modification de l'instance signifie que vous ne devez effectuer aucune autre action avec cette instance jusqu'à l'étape suivante, en particulier essayer d'accéder aux variables etc... car cela provoquera une erreur. En fait, vous modifiez l'instance mais elle n'est pas réellement disponible avant la fin de l'étape en cours, donc accéder directement à l'une des variables qu'elle contient (par exemple, en appelant obj_Changed.x) ne fonctionnera pas.

ATTENTION ! Lors du changement d'une instance activée par la physique, les propriétés physiques ne seront pas transférées à la nouvelle instance. Par conséquent, vous devez avoir un code en place pour "transférer" l'état physique de l'instance actuelle vers la nouvelle instance, ou avoir défini les propriétés physiques de la nouvelle instance. instance actuelle vers la nouvelle instance, ou avoir défini les propriétés physiques de la nouvelle instance dans son événement de création, ou dans l'éditeur d'objets. Pour cette raison, il est recommandé de ne pas utiliser cette fonction avec des instances ayant des propriétés physiques, mais plutôt d'utiliser une combinaison de instance_destroy() et instance_create_layer().

Syntaxe :

instance_change(obj, perf) ;

Arguments Description
obj. Le nouvel objet dans lequel l'objet appelant va se transformer.
perf Si les événements de création et de destruction de ce nouvel objet doivent être exécutés (true) ou non (false).

Les retours :

N/A

Exemple :

if keyboard_check(vk_enter)
{
instance_change(obj_Player_Swimming, false) ;
exit ;
}

Le code ci-dessus transformera l'instance appelante en une instance de "obj_Player_Swimming" sans exécuter l'événement de destruction de l'instance d'origine ni l'événement de création de la nouvelle instance lorsque l'utilisateur appuiera sur la touche "Entrée". bloc de code une fois terminé.
