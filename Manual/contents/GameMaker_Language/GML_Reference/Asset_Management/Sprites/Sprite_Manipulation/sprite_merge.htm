sprite_merge

Cette fonction fusionnera le sprite indexé en argument 1 ("ind2") avec celui qui est indexé en argument 0 ("ind1"). Les images elles-mêmes ne sont PAS fusionnées ensemble, mais plutôt les indices d'image sont fusionnés, avec les sous-images du sprite "ind2" ajoutées à celles du sprite "ind1", c'est-à-dire: elles sont ajoutées à la fin. Notez que si les sprites sont de tailles différentes, alors les sprites ajoutés sont étirés pour s'adapter à la taille de l'image pour "ind1".

Ce changement est permanent, et à partir du moment où vous utilisez cette fonction jusqu'à ce que le jeu soit fermé ou que le sprite soit supprimé, le sprite qui est fusionné sera changé, mais le sprite avec lequel il est fusionné restera le même.

REMARQUE: ni l'élément d'image qui est fusionné dans, ni l'élément d'image qui doit fusionner, ne peuvent être l'une des ressources de jeu d'origine. Vous devez d'abord utiliser sprite_duplicate () pour créer deux nouvelles ressources (si vous fusionnez des éléments de jeu inclus), puis les fusionner.

REMARQUE: Cette fonction n'est utile que pour les sprites bitmap et ne fonctionnera pas avec les sprites SWF ou JSON (Spine).

Syntaxe:

sprite_merge(ind1, ind2);

Argument Description
ind1 L'index du sprite à fusionner.
ind2 L'index du sprite avec lequel ind1 doit être fusionné.

Retour

N/A

Exemple:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

Le code ci-dessus charge un sprite dans une variable locale puis le fusionne avec un autre sprite qui a été dupliqué. Enfin, le sprite fusionné est affecté à l'instance sprite_index tandis que le sprite chargé est supprimé pour libérer la mémoire qu'il utilise. Notez qu'à un moment donné après cette opération, comme dans les événements Destroy ou Room End, vous devrez également utiliser sprite_delete() pour supprimer également le sprite fusionné de la mémoire.
