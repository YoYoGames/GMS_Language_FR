sprite_set_cache_size

Lorsqu'il effectue le mélange d'images, HTML5 ne peut pas le faire de manière dynamique de la même manière qu'un exécutable. GameMaker Studio 2 doit donc enregistrer temporairement une copie mélangée de l'image et la charger. Cette fonction définit le nombre de copies mélangées du sprite donné peuvent être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. Ceci est appliqué à toutes les sous-images du sprite.

Syntaxe :

sprite_set_cache_size(ind, max) ;

Arguments Description
ind L'index du sprite dont la taille du cache doit être modifiée.
max Le nombre maximum de copies en cache du sprite qui peuvent être stockées.

Renvoie à

N/A

Exemple :

sprite_set_cache_size(sprite0, 2) ;

Le cache des sprites de toutes les sous-images de sprite0 sera ainsi réglé sur 2 copies.
