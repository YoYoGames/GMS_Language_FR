sprite_prefetch_multi

Cette fonction peut être utilisée pour pré-extraire (placer dans la mémoire de texture) un certain nombre de pages de texture contenant les sprites donnés. Vous fournissez un tableau rempli avec les indices de sprite (tels que définis lors de la création de la ressource de sprite) et les pages de texture sur lesquelles ils se trouvent seront chargées en mémoire. Notez que la fonction retournera -1 si la prélecture n'est pas prise en charge pour la ressource choisie ou si la plate-forme cible est HTML5, ou elle retournera 0 si tout fonctionnait correctement.

REMARQUE: Il y a un problème de performance car la texture est téléchargée dans la mémoire de texture sur la plupart des appareils, il est donc recommandé d'appeler sprite_prefetch_multi () sur tous les graphiques requis au début d'un niveau pour éviter tout blocage.

Syntaxe:

sprite_prefetch_multi

Argument Description
array Tableau avec les indices de sprite à récupérer

Retour:

Int (-1 or 0)

Exemple:

spr_a[0] = spr_Player_Aura1;
spr_a[1] = spr_Player_Aura2;
spr_a[2] = spr_Player_Aura3;
spr_a[3] = spr_Player_Aura4;
sprite_prefetch_multi(spr_a);

Le code ci-dessus crée un tableau dans lequel chaque élément contient un index de sprite. Ce tableau est ensuite utilisé pour placer ces textures de sprite en mémoire.
