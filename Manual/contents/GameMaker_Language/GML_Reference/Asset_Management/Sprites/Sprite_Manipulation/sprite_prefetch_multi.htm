sprite_prefetch_multi

Cette fonction peut être utilisée pour prélever (placer dans la mémoire de texture) un certain nombre de pages de texture qui contiennent les sprites donnés. Vous fournissez un tableau rempli des indices des sprites (tels que définis lors de la création du sprite) et les pages de texture sur lesquelles ils se trouvent seront chargées en mémoire. sur lesquelles ils se trouvent seront chargées en mémoire. Notez que la fonction renverra -1 si la récupération anticipée n'est pas prise en charge pour la ressource choisie ou si la plate-forme cible est HTML5, ou elle renverra 0 si tout a fonctionné correctement.

NOTE: Il y a un impact sur les performances lorsque la texture est téléchargée dans la mémoire de texture sur la plupart des appareils, il est donc recommandé d'appeler sprite_prefetch_multi() sur tout graphique requis au début d'un niveau pour éviter tout blocage.

Syntaxe :

sprite_prefetch_multi

Arguments Description
tableau Tableau avec les indices des sprites à récupérer

Les retours :

Int (-1 ou 0)

Exemple :

spr_a[0] = spr_Player_Aura1 ;
spr_a[1] = spr_Player_Aura2 ;
spr_a[2] = spr_Player_Aura3 ;
spr_a[3] = spr_Player_Aura4 ;
sprite_prefetch_multi(spr_a) ;

Le code ci-dessus crée un tableau dont chaque élément contient un index de sprite. Ce tableau est ensuite utilisé pour placer ces textures de sprites en mémoire.
