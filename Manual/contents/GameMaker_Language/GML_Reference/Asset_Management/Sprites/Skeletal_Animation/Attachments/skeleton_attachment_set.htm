jeu_attachement_squelette

Un sprite d'animation squelettique peut avoir d'autres images ajoutées en tant qu'attachements, ces images étant ajoutées à un slot nommé (le nom est donné lorsque vous créez le slot d'attachement dans votre programme d'animation) et elles seront dessinées en même temps que l'animation de le sprite actuel. Avec cette fonction, vous pouvez définir un accessoire à un slot donné, où vous devez donner les noms (sous forme de chaînes) du slot et de l'accessoire. Ces noms sont définis par le programme d'animation utilisé, ou (dans le cas de l'accessoire) lorsque vous appelez skeleton_attachment_create().

Notez que vous pouvez également passer un sprite_index en tant que pièce jointe, et ce sprite sera utilisé, ou vous pouvez utiliser -1 pour supprimer la pièce jointe du slot. Lorsque vous passez un index de sprite en tant qu'argument, un slot d'attachement sera créé en utilisant le nom du sprite comme argument. un slot d'attachement en utilisant le nom du sprite comme chaîne de caractères pour nommer le slot (ainsi, l'utilisation de spr_sword, par exemple, créera un slot d'attachement "spr_sword"), et le slot utilisera le premier index d'image (0) du sprite ainsi que son décalage d'origine x et y, avec une échelle de (1,1) et une rotation de 0.

Syntaxe :

skeleton_attachment_set(slot, attachment) ;

Arguments Description
fente Le nom du slot (une chaîne de caractères) pour obtenir l'attachement.
pièce jointe Le nom (sous forme de chaîne ou de sprite_index) de l'image jointe.

Les retours :

Chaîne de caractères

Exemple :

if skeleton_attachment_get("slot_leftHand") == ""
{
skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife")) ;
}

Le code ci-dessus vérifie le nom d'attachement actuellement attribué au slot nommé "slot_leftHand" et si une chaîne vide est retournée, un nouveau sprite est attaché.
