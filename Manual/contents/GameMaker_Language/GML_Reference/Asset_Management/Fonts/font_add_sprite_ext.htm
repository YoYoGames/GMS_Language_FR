font_add_sprite_ext

Avec cette fonction, vous pouvez utiliser une "bande de sprite" (le sprite lui-même doit être un actif de sprite du navigateur d'actifs, ou un sprite que vous avez ajouté aux ressources du jeu à l'aide de sprite_add ) pour créer un nouvel actif de police, où chaque sous-image serait un symbole ou une lettre individuelle. Contrairement à la méthode normale font_add_sprite() normale qui a un ordre spécifique pour les sous-images du sprite, cette fonction mettra en correspondance les sous-images du sprite sur la base de l'argument "string_map" de la fonction. Cet argument est une chaîne de caractères que vous pouvez utiliser pour indiquer à GameMaker Studio 2 l'ordre dans lequel les sous-images de la police du sprite sont placées. Ainsi, par exemple, si vous avez un mappage de chaîne de caractères de "AaBbCcDdEeFfGgHh", les sous-images de votre police sprite doivent être ordonnées de cette manière. Vous pouvez définir "espace" comme étant n'importe quel caractère, par exemple une ligne unique de la taille que vous souhaitez pour l'espace. dans le texte seront rendus à cette largeur (mais l'image choisie ne sera jamais rendue), ou si vous ne fournissez pas de sprite pour l'espace, la largeur du caractère le plus large dans la police sprite sera utilisée à la place.

Vous pouvez également définir l'espacement de la police pour qu'elle soit proportionnelle(true) ou non(false), où une police proportionnelle est espacée sur la base de la largeur réelle des lettres (ainsi, la lettre "i" prend moins de place que la lettre "w", par exemple), tandis qu'une police non proportionnelle sera espacée en fonction de la largeur de la sous-image, comme une police monospace. Enfin, vous pouvez définir l'espace à laisser entre chaque lettre lors de l'écriture, et il peut s'agir de n'importe quelle valeur entière, 0 correspondant à l'absence d'espace (les lettres se "toucheront" si elles sont proportionnelles). lettres se "toucheront" si elles sont proportionnelles). La fonction renvoie une valeur d'index qui doit être stockée dans une variable car elle sera nécessaire dans tous les codes ultérieurs qui se réfèrent à cette police, ou elle renvoie -1 si la fonction échoue pour une raison quelconque. raison.

Il convient de noter que pour que les fonctions d'alignement des polices fonctionnent (comme la fonction draw_set_halign()), l'origine du sprite de la police doit être fixée au coin supérieur gauche. Si vous utilisez d'autres valeurs vous devrez prendre en compte le décalage de l'origine lorsque vous dessinerez du texte en utilisant la police.

REMARQUE : Lorsque vous chargez une police dans GameMaker Studio 2, vous devez vous souvenir de la supprimer (avec la touche font_delete()) lorsqu'elle n'est plus nécessaire, sinon il y a risque d'une fuite de mémoire qui ralentira et finira par faire planter votre jeu.

Syntaxe :

font_add_sprite_ext(spr, string_map, prop, sep) ;

Arguments Description
spr Le sprite sur lequel ajouter une police.
carte_chaîne Chaîne à partir de laquelle l'ordre de la sous-image du sprite est pris.
accessoire Définir comme police proportionnelle ou non.
sep L'espace à laisser entre chaque lettre.

Les retours :

Real

Exemple :

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2) ;

Le code ci-dessus créera une nouvelle police à partir du sprite indexé dans la variable "spr_CalcFont" et stockera l'index de la nouvelle police dans la variable "global.Font". Les sous-images de la police du sprite seront mappées à la chaîne de caractères spécifiée.
