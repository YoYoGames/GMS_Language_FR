script_exécution

Cette fonction a été conçue à l'origine pour appeler des actifs de script de manière dynamique, mais avec l'introduction des fonctions définies par l'utilisateur, elle peut désormais être utilisée de deux manières :

  • La première méthode consiste à prendre en charge les anciens scripts, où chaque script était une fonction unique et où le titre du script était également le nom de la fonction. Utilisé de cette façon, il fonctionnera comme avant et exécutera un actif de script créé par l'utilisateur et lui passera les arguments donnés. les arguments donnés. Notez qu'il n'appelle pas réellement le script dans ce cas, mais la fonction de script du même nom dans le script.
  • La deuxième méthode consiste à appeler une fonction à partir d'un actif de script qui contient plusieurs fonctions ou définitions de variables. Dans ce cas, vous devez fournir le nom de la fonction et les arguments qu'elle requiert plutôt que le nom de la ressource de script. Notez que cela ne fonctionne que pour les fonctions de script et non pour les méthodes, car seules les fonctions de script ont un ID de script qui leur est attribué.

Bien que cette fonction soit principalement conçue pour la prise en charge de l'héritage, elle n'est pas dépréciée car elle a toujours sa place dans les situations de mise en réseau et d'appel de procédure à distance. Il convient également de noter que cette fonction est lente, et son utilisation fréquente peut nuire aux performances.

REMARQUE: Cette fonction ne peut pas être utilisée avec les fonctions intégrées.

Syntaxe :

script_execute(scr, arg0, arg1, arg2..., etc...) ;

Arguments Description
scr Le nom de la fonction/script que vous voulez appeler
arg0, arg1, arg2..., etc... Les différents arguments que vous souhaitez transmettre à la fonction/script.

Les retours :

Cela dépendra de la valeur de retour du script/fonction appelé.

Exemple :

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5)) ;

Le code de l'exemple ci-dessus utilisera script_execute pour choisir l'une des quatre fonctions définies par l'utilisateur (dans ce cas, liée au mouvement) et lui passera un entier aléatoire comme premier argument.
