Couches de mosaïque

L' éditeur de pièce GameMaker Studio 2 vous permet d'ajouter des tuiles dans n'importe quelle pièce à l'aide de calques. Comme son nom l'indique, une couche de carte de tuiles est simplement une collection de tuiles de jeu de tuiles qui sont toutes considérées comme étant à la même profondeur dans la pièce, et en empilant des couches, vous pouvez faire en sorte que certaines choses se dessinent sur ou sous d'autres. Vous pouvez également contrôler certains aspects des couches à partir du code, ajouter ou supprimer des éléments ou modifier certaines propriétés de la couche ou de ce qu'elle contient au moment de l'exécution. Notez que dans l'éditeur de salle, vous êtes limité à une carte de tuiles par couche (la couche de tuiles ), mais lorsque vous travaillez avec elles dans le code, vous pouvez avoir plusieurs cartes de tuiles affectées à une seule couche, chaque carte ayant son propre ID unique et propriétés, donc:

  • Le jeu de tuiles est la ressource que vous utilisez pour créer des couches de tuiles et des cartes de tuiles
  • La couche de tuiles est ce que vous créez dans l'éditeur de pièce pour ajouter des ensembles de tuiles à
  • La carte de tuiles est ce que vous appelez la collection de tuiles qui sont ajoutées à un calque, soit dans l'éditeur de pièce, soit via le code, en tant qu'élément unique sur cette couche.

Outre les fonctions répertoriées ci-dessus pour définir des cartes de tuiles, vous disposez également d'un certain nombre de fonctions qui peuvent être utilisées pour manipuler les données de tuiles individuelles. Un "blob" de données de tuile est composé de l'index de tuile plus quelques bits supplémentaires pour montrer si la tuile a été tournée, retournée ou mise en miroir. Vous avez également la possibilité d'ajouter votre propre masque de bits aux données de tuile, bien que ce ne soit pour les utilisateurs avancés que car vous masquerez les bits qui sont réservés pour l'index de tuile - il y a essentiellement 19 bits réservés pour vos index de tuile, mais si vous n'en utilisez que 8, il vous reste onze bits qui peuvent être utilisés pour créer un masque personnalisé. Ce masque peut ensuite être utilisé et archivé dans le code pour créer des collisions de tuiles personnalisées ou autre. Cependant, vous n'aurez normalement pas besoin de créer vos propres objets blob de données de tuiles, et utiliseriez à la place la fonction tilemap_get() pour obtenir les données de la vignette, vous les manipuleriez à l'aide des fonctions ci-dessous, puis vous la définiriez à nouveau en utilisant tilemap_set().
