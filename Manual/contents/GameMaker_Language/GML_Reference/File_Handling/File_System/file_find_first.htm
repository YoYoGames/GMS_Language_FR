file_find_first

Cette fonction retournera le nom du premier fichier qui satisfait le masque et les attributs. Si aucun fichier de ce type n'existe, une chaîne vide est renvoyée. Le masque peut contenir un chemin et peut contenir des caractères génériques, par exemple 'C: \ temp \ *. Doc'. Les attributs donnent les fichiers supplémentaires que vous souhaitez voir, de sorte que les fichiers normaux sont toujours renvoyés lorsqu'ils satisfont au masque. Vous pouvez ajouter les constantes suivantes pour voir le type de fichiers souhaité (si vous ne souhaitez ajouter aucun attribut, utilisez 0):

Constant Description
fa_readonly Fichiers en lecture seule
fa_hidden Fichiers cachés
fa_sysfile Fichiers système
fa_volumeid Fichiers d'ID de volume
fa_directory Répertoires
fa_archive Fichiers archivés

ATTENTION! Cette fonction peut ne pas fonctionner comme prévu en raison du GameMaker Studio 2! Veuillez consulter la section sur le système de fichiers pour plus d'informations.

REMARQUE: cette fonction ne fonctionnera pas du tout sur la cible HTML5 et les indicateurs de filtre ne fonctionneront que sous Windows (utilisez 0 pour toutes les autres plates-formes).

Syntaxe:

file_find_first(mask, attr);

Argument Description
mask Le masque à utiliser pour la recherche.
attr L'attribut de fichier spécifique à rechercher.

Retour:

String

Exemple:

if directory_exists("\User Content")
    {
    file = file_find_first("/User Content/*.doc", fa_readonly);
    }

Cela vérifiera si le répertoire spécifié existe, puis, si c'est le cas, allez-y et retournez le premier "lecture seule "doc fichier trouvé.
