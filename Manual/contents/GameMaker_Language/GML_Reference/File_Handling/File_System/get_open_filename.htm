get_open_filename

Cette fonction ouvre un dialogue et demande au joueur un nom de fichier à ouvrir avec le filtre donné. Le filtre a la forme " nom1|mask1|nom2|mask2|...", où le masque peut contenir les différentes options avec un point-virgule entre elles et vous pouvez également utiliser un "*" pour signifier n'importe quelle chaîne. Par exemple : " bitmaps|*.bmp;*.wmf", demanderait à l'utilisateur de sélectionner le nom d'un fichier bitmap du système, mais seulement ceux qui ont les deux extensions spécifiées (bmp et png). Cette fonction n' ouvre pas le fichier lui-même ! Elle renvoie seulement une chaîne de caractères avec le nom complet et le chemin d'accès du fichier. Si l'utilisateur appuie sur "Annuler", une chaîne vide "" est retournée.

Une chose importante à noter lorsque vous utilisez cette fonction est qu'elle vous accorde certaines permissions pour ce fichier, pour la durée de votre jeu. Ainsi, une fois que vous avez obtenu le chemin du fichier de la part de l'utilisateur, vous pouvez y accéder encore et encore sans avoir à le demander. Cependant, comme cette fonction sert à lire un fichier, les autorisations accordées ne concernent que la lecture et l'enregistrement ne sera pas autorisé par le système d'exploitation. Si vous avez besoin de lire et d'écrire dans le fichier, vous pouvez obtenir la permission en utilisant la fonction get_save_filename().

REMARQUE: Cette fonction n'est valable que sur les cibles Windows et macOS.

Syntaxe :

get_open_filename(filter, fname) ;

Arguments Description
filtre Le filtre et/ou le nom du type de fichier.
fname Le nom de fichier suggéré à utiliser.

Les retours :

Chaîne de caractères

Exemple :

var file ;
file = get_open_filename("text file|*.txt", "") ;
if file != ""
{
file_text_open_read(file) ;
}

Le code ci-dessus demandera à l'utilisateur de sélectionner un fichier texte à ouvrir, puis vérifiera le nom et le chemin du fichier renvoyé et, s'il existe, ouvrira le fichier pour le lire.
