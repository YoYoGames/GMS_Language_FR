file_text_close

Une fois que vous avez fini de travailler avec un fichier donné (que ce soit en lecture ou en écriture), vous devez refermer le fichier, sinon vous risquez de perdre les informations qu'il contient. Cela empêche également les fuites de mémoire et garantit que vous ne dépassez jamais la limite de fichiers en ayant plus de 32 fichiers ouverts.

Syntaxe:

file_text_close(fileid);

Argument Description
fileid L'identifiant du fichier à fermer.

Retour:

N/A

Exemple:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
    {
    file_text_readln(file);
    }
file_text_write_string(file, level_data);
file_text_close(file);

Le code ci-dessus ouvre un fichier à écrire puis parcourt les lignes de texte déjà écrites dans le fichier jusqu'à ce qu'il atteigne la fin. À ce stade, il écrit une chaîne, puis ferme à nouveau le fichier.
