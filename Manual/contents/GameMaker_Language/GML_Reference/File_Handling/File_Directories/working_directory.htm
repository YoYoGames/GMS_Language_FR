répertoire de travail

Working_directory peut en fait renvoyer deux valeurs différentes selon l'usage que vous en faites. Si vous écrivez un fichier sur le disque, working_directory pointe vers la zone de stockage local qui est réservée à votre jeu sur le périphérique cible (sous Windows, il s'agit de " ", où le " gamedir " est le répertoire portant le nom de votre jeu). cible (sous Windows, il s'agit de "%LOCALAPPDATA%/gamedir/", où le " gamedir " est le répertoire portant le nom de votre jeu). Cependant, lors de la lecture depuis le disque, working_directory peut être soit le stockage local , soit la zone où sont stockés les fichiers inclus.

NOTE: La variable working_directory retournera le chemin d'accès incluant la dernière barre oblique inversée.

Par exemple, si vous avez un fichier *.txt par défaut inclus dans votre jeu et que vous le lisez en mémoire à l'aide de working_directory, ce fichier sera pris dans la zone où sont stockés les fichiers inclus. Si vous choisissez ensuite d'écrire ces Si vous choisissez ensuite d'écrire ces informations dans un fichier texte à l'aide de working_directory, le fichier sera écrit dans le stockage local de la plate-forme cible, où il sera lu la prochaine fois, à moins que vous ne demandiez à GameMaker Studio 2 de supprimer le fichier, auquel cas working_directory indiquera à nouveau la zone où sont stockés les fichiers inclus.

Il convient de noter qu'en général, vous n'avez pas besoin d'utiliser cette fonction et que le simple fait d'appeler le nom du fichier (plus toute information supplémentaire sur le chemin) est suffisant, par exemple ceci :

file_text_open_read(working_directory + "my_file.txt") ;

serait mieux fait comme ça :

file_text_open_read("mon_fichier.txt") ;

AVERTISSEMENT ! Cette fonction peut ne pas fonctionner comme vous l'attendez, car GameMaker Studio 2 est en sandbox ! Veuillez consulter la section sur le système de fichiers pour plus d'informations.

Chaîne de caractères

Exemple :

ini_open(working_directory + "temp_ini.ini") ;

Cela ouvrira un fichier ini dans le répertoire de travail du jeu (en le créant s'il n'existe pas déjà). Il peut s'agir du stockage local, ou de la zone où se trouvent les fichiers inclus, selon que le fichier recherché existe ou non à l'un ou l'autre endroit.
