Fichiers Ini

Les fichiers INI sont de petits fichiers légers qui sont compatibles avec la plupart des plateformes. Ils sont idéaux pour stocker de petits éléments d'information, comme les préférences d'interface, les meilleurs scores locaux, les données de niveau etc... et en tant que tels sont et en tant que tels, ils constituent un moyen simple et efficace de commencer à sauvegarder les informations de vos jeux. Cependant, il convient de noter qu'ils ont une limite sur les données qu'ils peuvent stocker, qui est un maximum de 64KB, donc si vous sauvegardez des choses comme des structures de données, vous pouvez souhaiter d'examiner d'autres fonctions de fichiers comme les fichiers texte.

IMPORTANT ! Tous les projets GameMaker Studio 2 ont un fichier appelé "options.ini", vous ne pouvez donc pas nommer aucun fichier avec ce nom (vous obtiendrez un échec d'uid lorsque vous essaierez de compiler votre projet de jeu).

Veuillez noter que pour les jeux avec localisations, il est très important que les lettres accentuées puissent être lues à partir de fichiers externes. Cela signifie que vous devez d'abord créer le fichier ini au format UTF8, puis l'ajouter dans GameMaker Studio 2 en tant que fichier inclus. puis l'ajouter dans GameMaker Studio 2 en tant que fichier inclus afin qu'il soit exporté lors de l'exécution du jeu et utilisé à la place du fichier ini par défaut au format ANSI qui est créé par les fonctions ini lorsqu'aucun fichier n'a été fourni auparavant. fourni. De cette façon, vous pouvez lire et écrire correctement dans ce fichier en conservant tous les accents et les lettres non romaines.

Les fonctions suivantes existent pour les fichiers ini :

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_existe
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
