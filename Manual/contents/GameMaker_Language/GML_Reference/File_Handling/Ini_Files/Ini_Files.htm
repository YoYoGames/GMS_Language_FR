Fichiers Ini

Les fichiers INI sont des fichiers petits et légers compatibles avec la plupart des plates-formes. Ils sont idéaux pour stocker de petites informations, telles que les préférences d'interface, les meilleurs scores locaux, les données de niveau, etc. Cependant, il convient de noter qu'ils ont une limite sur les données qu'ils peuvent stocker, qui est un maximum de 64 Ko, donc si vous enregistrez des éléments tels que des structures de données, vous souhaiterez peut-être consulter certaines des autres fonctions de fichiers telles que Texte. Fichiers.

IMPORTANT! Tous les GameMaker Studio 2 ont un fichier appelé " options.ini ", vous ne pouvez donc nommer aucun fichier avec ce nom (vous obtiendrez un échec uid lorsque vous essayez de compiler votre projet de jeu).

Veuillez noter que pour les jeux avec des localisations, il est très important que les lettres accentuées puissent être lues à partir de fichiers externes. Cela signifie que vous devez créer le fichier ini au format UTF8 d' abord, puis l' ajouter dans GameMaker Studio 2 en tant que fichier inclus de sorte qu'il est exporté sur l' exécution du jeu et utilisé à la place du format ANSI par défaut fichier ini qui est créé par l'ini fonctionne alors qu’aucun n’a été précédemment fourni. De cette façon, vous pouvez lire et écrire correctement avec tous les accents et les lettres non romaines conservés.

Les fonctions suivantes existent pour les fichiers ini:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
