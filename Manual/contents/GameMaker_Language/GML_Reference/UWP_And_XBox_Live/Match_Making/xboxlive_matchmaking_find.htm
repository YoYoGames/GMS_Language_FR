xboxlive_matchmaking_find

Cette fonction recherchera une session multi-joueurs à l'aide du modèle de session, de la trémie de matchmaking et du modèle Secure Device Association, que vous auriez dû créer au préalable dans le tableau de bord XDP. L'ID utilisateur que vous pouvez récupérer pour une manette de jeu particulière en appelant le xboxlive_user_for_pad() fonction. La fonction renverra une valeur d'ID de demande unique, qui peut ensuite être utilisée pour identifier l' événement social asynchrone correct pour cette fonction. Cet événement sera déclenché lorsque la session aura été créée et contiendra une DS Map dans la variable async_load avec les paires clé / valeur suivantes:

  • "requestid" - contient l'ID de demande tel que renvoyé par la fonction appelante
  • "status" - contient la chaîne "session_created" pour vous informer du type d'événement qui a été déclenché.
  • "num_results" - contient 1 ou plus si une session a été trouvée, sinon contient 0.
  • "sessionid <index> " - contient l'ID de session, ou -1 en cas d'échec ( <index / gt; est une valeur de 0 à "num_results" - 1).
  • "sessionOwner <index> " - contient l'ID de l'hôte de session si une session a été trouvée ( <index> est une valeur de 0 à "num_results" - 1).
  • "erreur" - contient 0 en cas de succès, ou -1 en cas d'échec

Syntaxe:

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

Argument Description
user_id Le pointeur d'ID utilisateur à utiliser
template Le nom du modèle de session
hopper Le nom de la trémie de matchmaking
sdatemplate Le nom du modèle d'association d'appareils sécurisés
matchattributes [OPTIONNEL]

Retour:

Request ID (integer)

Exemple:

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

Ce qui précède récupérera l'ID utilisateur de l'utilisateur sur la manette de jeu donnée, puis essaiera de trouver une session de matchmaking pour eux.
