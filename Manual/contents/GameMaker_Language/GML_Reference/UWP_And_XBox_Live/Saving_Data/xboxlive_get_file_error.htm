xboxlive_get_savedata_user

Cette fonction renverra une constante (illustrée ci-dessous) pour aider à déboguer pourquoi une fonction de fichier a échoué afin que vous puissiez prendre les mesures appropriées dans votre code de jeu. Lorsque l'une des fonctions de fichier échoue, vous pouvez appeler la fonction pour savoir exactement quelle était l'erreur.

REMARQUE: lors de l'utilisation des fonctions de sauvegarde / chargement asynchrones, le async_load map aura une clé supplémentaire, " error ", qui contiendra également l'une des constantes (illustrées ci-dessous).

Syntaxe:

xboxlive_get_file_error();

Retour:

Constant

Constant Description
xboxlive_fileerror_outoflocalstorage Si cela se produit pendant l'enregistrement, cela signifie que tout l'espace de sauvegarde disponible a été utilisé et que l'utilisateur a refusé l'offre (présentée par le système d'exploitation) de libérer de l'espace. Le jeu doit confirmer à l'utilisateur qu'il a l'intention de continuer à jouer sans enregistrer. S'ils veulent toujours pouvoir sauvegarder, le jeu doit réessayer l'opération de sauvegarde.
xboxlive_fileerror_quotaexceeded Si cela se produit, le jeu doit informer l'utilisateur qu'il ne peut plus enregistrer de données jusqu'à ce qu'il libère de l'espace, puis lui présenter un moyen de le faire.
xboxlive_fileerror_noerror La fonction de sauvegarde ou de chargement de fichier a été exécutée sans problème.

Exemple:

buffer_save(buff, "SaveGame.bin");
if (xboxlive_get_file_error() != xboxlive_fileerror_noerror)
    {
    show_debug_message("Save failed. Error code = " + string(xboxlive_get_file_error()));
    }

Le code ci-dessus vérifie s'il y a eu une erreur lors de l'enregistrement et si tel est le cas, le code d'erreur est imprimé sur la console.
