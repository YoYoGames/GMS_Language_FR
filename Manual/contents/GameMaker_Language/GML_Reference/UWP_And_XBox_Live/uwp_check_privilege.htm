uwp_check_privilege

Avec cette fonction, vous pouvez vérifier qu'un utilisateur donné possède un privilège choisi. La fonction renvoie true (vrai) ou false (faux) dans l'événement asynchrone du système, selon que le privilège est activé ou non. true et que le privilège n'est pas activé, elle ouvrira également une boîte de dialogue système (suspendant le jeu) pour inviter l'utilisateur à mettre à niveau le compte ou tout autre élément requis pour obtenir le privilège. Si l'utilisateur active ensuite l'option requise, la fonction retournera vrai.

La fonction exige que vous utilisiez l'une des constantes suivantes pour la demande de privilège :

Constant Description
uwp_privilege_internet_browsing Vérifiez si la navigation sur Internet est autorisée
uwp_privilege_partage_réseau_social Vérifiez si le partage sur les réseaux sociaux est autorisé.
uwp_privilege_share_kinect_content Vérifiez si le partage de la manette Kinect est autorisé.
uwp_privilege_video_communications Vérifiez si la communication vidéo est autorisée
uwp_privilege_communications Vérifiez si la communication par Internet est autorisée
uwp_privilege_utilisateur_créé_contenu Vérifier si l'accès au contenu créé par l'utilisateur est autorisé
uwp_privilege_multiplayer_sessions Vérifiez si les sessions multijoueurs en ligne sont autorisées.
uwp_privilege_sessions Vérifiez si les sessions en ligne sont autorisées
uwp_privilege_fitness_upload Vérifiez si le téléchargement des données de fitness est autorisé.

Une fois que la fonction a été appelée, sur XBox One, elle déclenchera un événement système asynchrone lorsque le résultat du rappel aura été reçu, qui aura la DS spéciale map async_load. Cette carte doit ensuite être analysée pour la clé suivante :

  • "event_type" - doit contenir la valeur "check_privilege_result" si l'événement a été déclenché par cette fonction.

Si le type d'événement concerne cette fonction, il y aura également les touches supplémentaires suivantes lorsque la fonction est appelée sur XBox :

  • "result " - Sera une ou plusieurs des constantes suivantes (valeurs combinées) :
    Constant Description Valeur
    uwp_privilege_result_no_issue Il n'y a pas de problème de privilège avec l'utilisateur 0
    uwp_privilege_result_purchase_required (résultat d'achat requis) L'utilisateur doit acheter quelque chose de plus, généralement un abonnement, pour avoir accès à l'information. 1
    uwp_privilege_resultat_abandonné La vérification a été interrompue 2
    uwp_privilege_result_banned (interdit) L'utilisateur est banni 4
    uwp_privilege_result_restricted L'accès de l'utilisateur est restreint, généralement par le biais du contrôle parental. 8

  • "privilège " - sera l'une des constantes énumérées ci-dessus à utiliser dans la fonction, afin que vous puissiez vérifier quel privilège vous avez demandé dans le cas de vérifications multiples.

Si le projet est exécuté sur une plateforme UWP qui n'est pas XBox One, l'événement asynchrone ne sera pas déclenché et la fonction renverra immédiatement -1, 0 ou 1 :

  • -1 signifie que l'utilisateur n'a pas ce privilège.
  • 0 signifie que la fonction n'a pas été capable d'interroger l'utilisateur.
  • 1 signifie que l'utilisateur a le privilège.

Syntaxe :

uwp_check_privilege(user_id, privilege, attempt_resolution) ;

Arguments Description
nom d'utilisateur Le pointeur de l'ID utilisateur à vérifier.
privilège La constante de privilège à vérifier (voir la description, ci-dessus).
tentative_résolution Permet de tenter une résolution si elle est définie à true, ou de l'ignorer si elle est définie à false.

Les retours :

Entier

Exemple :

var user_one = xboxlive_get_user(0) ;
if (xboxlive_user_is_signed_in(user_one))
{
uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true) ;
}

La procédure ci-dessus vérifie si l'utilisateur est connecté et s'il l'est, elle vérifie l'autorisation des sessions multi-joueurs.
