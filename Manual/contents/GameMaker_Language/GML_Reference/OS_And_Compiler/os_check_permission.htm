os_check_permission

Avec cette fonction, vous pouvez vérifier si une permission spécifique a été accordée au jeu par l'utilisateur. Vous fournissez la permission à vérifier sous forme de chaîne de caractères en utilisant le format " android.permission.<permission> ", ainsi pour vérifier la permission RECORD_AUDIO ( par exemple) vous appellerez

os_check_permission("android.permission.RECORD_AUDIO") ;

La fonction renverra l'une des trois constantes - indiquées ci-dessous - pour indiquer au jeu l'état de la permission. Pour plus d'informations sur les permissions des applications, veuillez consulter la documentationAndroid.

IMPORTANT ! Cette fonction est destinée à la Android cible uniquement.

Syntaxe :

os_check_permission(permission)

Arguments Description
autorisation L'autorisation à vérifier (une chaîne de caractères)

Les retours :

Constant

Constant Description
os_permission_granted Cela indique que l'autorisation a été accordée
os_permission_denied Cela indique que l'autorisation n'a pas été accordée
os_permission_denied_dont_request Cela indique que l'autorisation a été bloquée par les paramètres du téléphone, ou que l'utilisateur a précédemment refusé la demande et sélectionné "Ne plus demander".

Exemple :

if (os_type == os_android)
{
if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
{
os_request_permission("android.permission.INTERNET") ;
}
}

Le code ci-dessus vérifie le type de système d'exploitation et s'il s'agit d'Android, il effectue une vérification des permissions et si la permission "INTERNET" n'a pas encore été accordée, il la demande.
