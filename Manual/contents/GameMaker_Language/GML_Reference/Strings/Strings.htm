Cordes

À un moment ou à un autre de la création de votre jeu, vous aurez besoin d'utiliser du texte. Dans les jeux, le texte est traité à l'aide des fonctions de chaîne de caractères (une chaîne de caractères est une autre façon de désigner une ligne de texte) . GameMaker Studio 2 dispose d'un ensemble complet de fonctions qui vous permettent de manipuler les chaînes de plusieurs façons, notamment en insérant une chaîne dans une autre, en copiant des chaînes et en analysant les chaînes pour trouver les chiffres ou les lettres qu'elles contiennent. En général, une chaîne de caractères ne peut être créée que en ajoutant du texte entre guillemets doubles " " et les chaînes entre guillemets simples ne sont pas acceptées, et vous ne pouvez pas non plus diviser la chaîne sur plusieurs lignes et espérer que GameMaker Studio 2 la rende comme si les sauts de ligne étaient des nouvelles lignes (à moins qu'un identifiant de type chaîne littérale @ soit utilisé, comme expliqué ci-dessous).

Il convient de noter qu'il existe certaines conventions que vous pouvez utiliser lors de la création de chaînes de caractères, principalement en ce qui concerne l'utilisation des caractères d'échappement. Il s'agit de caractères précédés du symbole "\". Ainsi, par exemple Donc, par exemple, si vous voulez mettre des guillemets dans une chaîne de caractères, vous obtiendrez quelque chose comme ceci :

str = "Hello\"World\"" ;

GameMaker Studio 2 prend également en charge les caractères Unicode sur quatre octets, ce qui vous permet de décoder et d'encoder les caractères Unicode dans les limites supérieures de la norme (y compris, mais sans s'y limiter, les emoji). Pour traiter les caractères Unicode, vous pouvez utiliser le "\" pour précéder n'importe quel littéral Unicode - des chiffres hexadécimaux précédés d'un "u", par exemple "\u00e2" pour " á "- où les chiffres sont le numéro de l'alphabet du caractère Unicode. les chiffres sont le numéro du caractère Unicode. Lorsque vous travaillez avec Unicode de cette manière, vous devez être conscient du fait que GameMaker Studio 2 interprétera tous les chiffres suivant le "u", donc si vous voulez écrire "áa" par exemple, vous devez utiliser :

"\u00e2\a"

ou

"\u00e2\u61"

ou

"\u00e2" + "a"

car la simple utilisation de " \u00e2a" donnerait en fait le caractère Unicode " ส " (devenantessentiellement" \ue2a ").

GameMaker Studio 2 peut également traiter n'importe quel littéral hexadécimal - normalement écrit sous forme de chiffres hexadécimaux suivant "0x", par exemple "0xff", où les chiffres correspondent au numéro du caractère à utiliser. le caractère à utiliser. Dans GameMaker Studio 2, ces caractères sont écrits en utilisant " \x", puis la valeur hexadécimale. Ces caractères d'échappement et d'autres caractères prédéfinis sont répertoriés dans le tableau ci-dessous :

Constant Description
\n Nouvelle ligne
\r Retour chariot
\b Retour arrière (0x08)
\f Saut de page (0x0c)
\t Onglet horizontal (0x09)
\v Onglet vertical (0x0b)
\\ Backslash lui-même (0x5c)

NOTE: Les chaînes de caractères supportent le saut de page, la tabulation verticale, etc... mais cela ne veut pas dire que le rendu le fait, et lors du dessin des chaînes de caractères, ces caractères peuvent être ignorés.

Vous pouvez également créer des chaînes litt érales verbatim en faisant précéder la chaîne entière du caractère @:

var test = @"
Coupure de ligne
sur plusieurs lignes

" ;

Le code ci-dessus rendra la chaîne sur plusieurs lignes comme s'il y avait un caractère d'échappement de saut de ligne inclus. Une chaîne littérale verbatim est similaire aux chaînes littérales de la version précédente de GameMaker, mais elle utilise également des guillemets doubles ou simples et doit être préfixée par un symbole . doivent être préfixées par le symbole @, elles peuvent être réparties sur plusieurs lignes dans le fichier de code et ne prennent PAS en charge les caractères échappés. Par exemple, @"Hello\World" n' essaiera pas d'échapper au W sur World et sera stocké textuellement. Notez cependant que lorsque vous utilisez des chaînes littérales comme celle-ci, vous devrez rompre la chaîne si vous souhaitez inclure des guillemets dans la chaîne, par exemple :

var test = @"Hello " + "\"" + @"World" + "\""

Une autre chose à noter est que le caractère Unicode 9647 (▯) est utilisé pour remplacer tous les glyphes manquants que vous pouvez avoir dans votre police désignée lors du rendu dans l'événement draw. Ainsi, si votre police ne possède pas, par exemple, le symbole °, alors écrire 90° produira en fait 90▯.
