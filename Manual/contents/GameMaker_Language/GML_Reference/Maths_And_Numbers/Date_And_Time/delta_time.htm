delta_temps

Le but de la synchronisation delta est d'éliminer les effets du décalage ou de la lenteur des ordinateurs qui essaient de gérer des graphiques complexes ou beaucoup de code. Il s'agit d'une valeur qui peut être ajoutée à la vitesse des objets afin qu'ils se déplacent à la même vitesse, indépendamment du décalage. vitesse, quel que soit le décalage. Dans GameMaker Studio 2, cela est possible en utilisant la variable en lecture seule delta_time à chaque étape, car elle mesure le temps écoulé entre une étape et la suivante en microsecondes (1 microseconde correspond à 1 000 000e de seconde). Par conséquent, la variable delta_time peut être utilisée pour calculer à quelle vitesse (par exemple) un personnage du jeu doit se déplacer pour compenser un pic de retard dans le jeu.

Syntaxe :

delta_temps

Les retours :

Réel (nombre entier)

Exemple :

var _dt = delta_time / 1000000 ;
vitesse = spd * dt ;

Le code ci-dessus définira la vitesse de l'instance en utilisant le temps delta pour corriger le décalage. La variable "_dt" sera utilisée pour contenir la valeur du temps delta calculée précédemment, de sorte que le ratio puisse être utilisé pour multiplier la valeur de la vitesse de base (contenue dans la variable "spd") et ainsi obtenir une vitesse cohérente pour l'instance. dans la variable "spd") et ainsi obtenir une vitesse cohérente pour l'instance.
