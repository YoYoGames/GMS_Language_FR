Fonctions des nombres

Dans GameMaker Studio 2, les nombres réels sont considérés comme des nombres à virgule flottante en double précision - c'est-à-dire qu'ils comportent une virgule décimale sans qu'il y ait un nombre fixe de chiffres avant ou après le point - ou des nombres entiers - c'est-à-dire qu'ils sont des nombres entiers sans valeur de point décimal. 2, par exemple, est un nombre entier mais 2,01 est un réel à virgule flottante.

NOTE: Sur la cible HTML5, tous les nombres réels sont des doubles.

Cette distinction entre les nombres entiers et les nombres flottants est très importante pour le développement multiplateforme, car la précision des calculs effectués sur un PC Windows n' est pas la même que la précision de ces mêmes calculs lorsqu'ils sont effectués sur un appareil mobile. Cela signifie que vous devez faire particulièrement attention lorsque vous effectuez des comparaisons, par exemple :

if (image_index == 1)
{
faire quelque chose...
}

Dans l'exemple ci-dessus, si nous avons réglé l'image_speed sur 0.25 - par exemple - alors après quatre étapes, vous pouvez supposer que la valeur de l'image_index sera de 1, mais il se peut qu' il s'agisse d'une valeur comme 1.0000002 en raison du fonctionnement des mathématiques à virgule flottante. une valeur comme 1.0000002 en raison de la façon dont les mathématiques à virgule flottante fonctionnent et donc l'évaluation ne sera pas vraie. Ces types d'erreurs peuvent être assez difficiles à déboguer, c'est pourquoi il est toujours bon d'en être conscient et d'utiliser d'autres moyens de vérifier les valeurs ou d'utiliser des outils de contrôle. d'autres moyens de vérifier les valeurs ou d'utiliser les fonctions de plancher ou d'arrondi appropriées (énumérées ci-dessous) pour convertir le nombre à vérifier en un nombre entier (pour plus d'informations sur les mathématiques en virgule flottante et sur les raisons de ce problème, voir ici). Par exemple, le code ci-dessus pourrait être écrit comme suit :

if (floor(image_index) == 1)
{
faire quelque chose...
}

Il convient également de noter que lors de l'utilisation des cibles YoYo Compiler, toutes les expressions et fonctions sont évaluées de gauche à droite, alors que sur toutes les autres plateformes cibles, elles sont évaluées de droite à gauche, ce qui signifie que ceci - par exemple - donnera des résultats différents selon la plateforme :

val = max(num, ++num, num++) ;

NOTE: Pour plus d'informations, voir la section sur l'ordre d'évaluation.

Vous pouvez également utiliser une fonction spéciale disponible dans GameMaker Studio 2 pour définir la valeur epsilon pour les mathématiques en virgule flottante. Lorsqu'un nombre réel est arrondi au nombre à virgule flottante le plus proche, l'epsilon (également appelé "epsilon machine") constitue une limite supérieure de l'erreur relative. Vous pouvez obtenir et définir la valeur de l'epsilon à l'aide des fonctions suivantes :

  • math_set_epsilon
  • math_get_epsilon

REMARQUE : lorsque vous utilisez les fonctions aléatoires, GameMaker Studio 2 conserve la même graine aléatoire à chaque fois que vous démarrez le jeu.
