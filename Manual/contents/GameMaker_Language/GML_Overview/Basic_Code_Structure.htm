Structure de code de base

Le code est écrit en blocs et un bloc de code typique consiste en un ensemble d'instructions, appelées instructions s, qui sont ensuite interprétées par GameMaker Studio 2 et utilisées pour que quelque chose se produise dans votre jeu. Ce "quelque chose" peut être aussi simple que d'ajouter 2 et 2 pour obtenir 4, ou aussi complexe que de faire fuir un ennemi lorsque sa santé descend en dessous d'une certaine valeur. La structure réelle du programme peut varier considérablement, en fonction des fonctions qu'il utilise, mais décomposée en bases, elle ressemble à ceci:

<statement>;
<statement>;
...

Les instructions doivent être séparées par un ";" symbole pour éviter les erreurs avec variables déclarations et de garder votre code propre et bien rangé, et ils peuvent consister en des déclarations variables, l' expression s et les appels à des fonctions spécifiques. Vous pouvez également "regrouper" les instructions sous forme de bloc en utilisant les accolades {} afin qu'ils fonctionnent ensemble, comme dans l'exemple conditionnel suivant:

if (<conditional>)
{
<statement>;
<statement>;
...
}

REMARQUE: le GameMaker Language accepte également begin  et end  au lieu des accolades {}, bien que ce ne soit généralement pas la manière la plus courante de le faire:

if (<conditional>)
begin
<statement>;
<statement>;
...
end
