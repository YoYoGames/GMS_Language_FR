throw

Le throw runtime instruction d'exécution vous permet de générer vos propres exceptions d' runtime, de terminer le jeu et d'afficher un message d'erreur, en utilisant la syntaxe suivante:

throw (<expression>);

L'expression utilisée peut être une valeur ou une chaîne ou tout autre type de données, et cela générera alors une erreur d'exception qui est - par défaut - affichée à l'écran et à la fermeture du message d'erreur, le jeu se terminera. Par exemple en appelant ceci:

throw ("Hello World!");

entraînera l'affichage de l'erreur d'exception non gérée suivante:

Vous pouvez cependant reprendre ce message d'erreur et utiliser votre propre code de gestionnaire en appelant la fonction exception_unhandled_handler(). Cette fonction d' runtime vous permet de fournir unmethod pour l'utiliser, il sera appelé chaque fois que des exceptions non gérées se produiront dans votre jeu.
