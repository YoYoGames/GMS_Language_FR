Rapport d'erreurs

La première forme de rapport d'erreur est l' erreur de syntaxe. Il s'agit d'une erreur dans votre code que Game Maker Studio 2 a détectée avant que vous n'ayez essayé de le compiler et celles-ci seront affichées dans l'EDI, soit dans l'éditeur de code, soit dans la fenêtre de sortie des erreurs de syntaxe:

Le niveau suivant de rapport d'erreurs est l' E rror du compilateur. Ces types d'erreurs sont plus généraux que les erreurs du coureur et peuvent être dus à une erreur subtile dans le code du jeu, qui provoque un crash pendant la compilation, avant même qu'il n'ait une chance de s'exécuter. Celles-ci seront également signalées dans la fenêtre de sortie des erreurs du compilateur:

Si le jeu se compile et qu'une erreur se produit, vous obtiendrez une erreur Runner. Ce type d'erreur affichera un message à l'écran, un peu comme celui illustré ci-dessous:

La plupart des messages d'erreur des coureurs suivent le même schéma, avec un message indiquant le type d'erreur, l'instance et l'événement dans lequel l'erreur s'est produite, puis une série de détails sur les coureurs expliquant l'erreur de manière beaucoup plus approfondie et, enfin, une liste de toutes les variables globales, d'instance et locales déclarées dans votre jeu à ce stade. Au bas est le bouton Abandonner qui provoquera l' arrêt du jeu, ainsi que le bouton Copier (qui copie l'erreur dans le presse - papier) et le bouton Effacer qui efface les messages d'erreur de la fenêtre. Les informations fournies par le rapport d'erreurs ici sont très importantes car elles détectent et signalent les gros bogues liés à la syntaxe qui vont faire planter votre jeu fini, mais notez qu'il ne rapportera pas de bogues plus subtils, ni ne donnera de commentaires. sur les performances... pour cela, vous devez utiliser le débogueur.
