Rapport d'erreur

La première forme de rapport d'erreur est l'erreur de syntaxe. Il s'agit d'une erreur dans votre code que Game Maker Studio 2 a détectée avant que vous n'essayiez de le compiler. Syntax Errors Output Window:

Le niveau suivant de signalement des erreurs est celui deserreurs de compilation. Ces types d'erreurs sont plus générales que les erreurs d'exécution et peuvent être dues à une erreur subtile dans le code du jeu, qui provoque un crash pendant la compilation , avant même qu'il n'ait la possibilité de s'exécuter. Celles-ci seront également seront signalées dans la fenêtre de sortie des erreurs du compilateur :

Si le jeu se compile et qu'une erreur se produit, vous obtiendrez une Runner Error. Ce type d'erreur affichera un message à l'écran, comme celui illustré ci-dessous :

La plupart des messages d'erreur du runner suivent le même schéma, avec un message indiquant le type d'erreur, l'instance et l'événement dans lequel l'erreur a été rencontrée, puis une série de détails du runner qui expliquent l'erreur de manière beaucoup plus approfondie et, enfin, une liste de toutes les variables globales, d'instance et locales déclarées dans votre jeu à ce moment-là. Tout en bas se trouve le bouton Abandon qui interrompt le jeu, ainsi que le bouton Copy ( qui copie l'erreur dans le presse-papiers) et le bouton Clear ( qui efface les messages d'erreur de la fenêtre). Les informations fournies par le rapport d'erreur sont très importantes, car elles détectent et signalent les gros bogues liés à la syntaxe qui vont faire planter votre jeu, mais notez qu'elles ne signalent pas les bogues plus subtils, et ne donnent pas d'informations sur les performances... pour cela, vous devez utiliser le débogueur. utiliser le débogueur.
