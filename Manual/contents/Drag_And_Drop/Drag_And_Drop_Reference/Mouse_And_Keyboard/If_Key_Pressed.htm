Si la touche est enfoncée

Cette action vérifie la pression initiale sur une touche du clavier. Il reviendra true à l'étape de jeu unique (cadre) où la touche est initialement enfoncée et false à tous les autres moments. Si vous activez le not modificateur, cette action sera inversée, et vérifier si la touche n'a pas été enfoncée, renvoyant false sur le pas de jeu (cadre) sur lequel il a été enfoncé et true à tous les autres moments. Si vous devez vérifier qu'une touche est maintenue enfoncée et pas seulement l'événement de pression initial, utilisez l'action Si touche enfoncée.

Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme indiqué dans l'image ci-dessous:

Toutes les actions déposées ailleurs seront exécutées après le bloc «si».

Syntaxe de l'action:

Arguments:

Argument Description
Key Le nom de la clé à vérifier.
Not Annuler le chèque (vrai devient faux et vice versa)

Si le chèque est true lors de la pression initiale vers le bas, un son est joué, puis, tandis que la touche est maintenue enfoncée, le mélange de sprite passe au rouge. Si la touche n'est pas maintenue enfoncée, une vérification est effectuée sur le relâchement de la touche pour réinitialiser la couleur de fusion au blanc.
