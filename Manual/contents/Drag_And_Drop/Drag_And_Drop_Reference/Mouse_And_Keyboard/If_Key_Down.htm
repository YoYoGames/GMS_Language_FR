Si touche enfoncée

Avec cette action, vous pouvez interroger le clavier pour voir si une touche est actuellement maintenue enfoncée. L'action reviendra true chaque pas pendant lequel la touche du clavier est maintenue enfoncée, ou false sinon, bien que si vous cliquez sur le not modificateur vous pouvez alors vérifier si la touche n'est pas d'aide, c'est-à-dire: l'action retournera true alors qu'aucune touche n'est enfoncée et false s'il y a. Si vous n'avez besoin de vérifier qu'une seule touche, utilisez l'action Si la touche est enfoncée.

Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme indiqué dans l'image ci-dessous:

Toutes les actions déposées ailleurs seront exécutées après le bloc «si».

Syntaxe de l'action:

Arguments:

Argument Description
Key Le nom de la touche du clavier à vérifier.
Not Annuler le chèque (vrai devient faux et vice versa)

Exemple:

Le code de bloc d'action ci-dessus interroge l'état du clavier à chaque étape et si une touche est maintenue enfoncée, il vérifie la touche initiale enfoncée. Si le chèque est true lors de la pression initiale vers le bas, un son est joué, puis, tandis que la touche est maintenue enfoncée, le mélange de sprite passe au rouge. Si la touche n'est pas maintenue enfoncée, une vérification est effectuée sur le relâchement de la touche pour réinitialiser la couleur de fusion au blanc.
