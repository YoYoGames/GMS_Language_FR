Si non défini

Cette action est utilisée pour vérifier si une variable ou une expression renvoie une valeur "indéfinie", et si c'est le cas, cette action retournera true et si ce n'est pas le cas, il reviendra false. "Undefined" peut être retourné pour un certain nombre de raisons, et une valeur non définie peut provoquer un plantage ou une erreur grave dans votre projet, vous pouvez donc l'utiliser pour vérifier cela et l'attraper avant que cela ne se produise. Si vous marquez l'argument "Non", vous vérifiez si une valeur n'est pas "indéfinie".

Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme indiqué dans l'image ci-dessous:

Ces actions seront désormais exécutées si le "si" est évalué à true, tandis que toutes les actions déposées ailleurs seront exécutées après le bloc "if". Notez que vous pouvez "empiler" des chèques non définis dans la même action en cliquant sur l'icône plus à côté de l'action, et en donnant une autre variable à vérifier. Cela vérifiera si toutes les variables données ne sont pas définies en une seule vérification et n'exécutera le code d'action ajouté que si elles le sont.
