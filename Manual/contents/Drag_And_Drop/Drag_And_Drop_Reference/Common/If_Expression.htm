Si Expression

Cette action est utilisée pour vérifier qu'une expression soit true ou false, et vous pouvez vérifier n'importe quelle expression logique ici tant qu'elle correspond à l'une ou l'autre valeur. Par exemple, l'une des expressions suivantes peut être utilisée:

x < (room_width / 2)
a == b
++val >= 100

Si vous marquez l'argument "Non", les expressions ci-dessus seront annulées, par exemple "== " devient "not= ". Notez également que l' expression n'est pas limitée aux valeurs numériques et que vous pouvez vérifier les chaînes, les pointeurs, les ID de ressources ou tout autre élément qu'une fonction peut renvoyer ou utiliser (voir la section sur les types de données pour plus d'informations), mais vous ne pouvez pas comparer deux différents types de données sans obtenir d'erreur - par exemple, si vous essayez d'utiliser une chaîne et un entier dans l'expression, vous obtiendrez une erreur. Pour plus d'informations sur les expressions, cliquez ici.

REMARQUE IMPORTANTE: en raison de problèmes de précision en virgule flottante, vérifier si deux valeurs sont exactement égales peut renvoyer false, puisque l'un peut être exactement 1, tandis que l'autre peut être 1.00000000000001. Cela peut être évité en utilisant l' action Decimal to Integer avant de vérifier ou d'utiliser les expressions «supérieur à (>)» ou «inférieur à (<)».

Cela vérifiera si toutes les expressions données retournent true en une seule vérification et n'exécutez le code d'action ajouté que s'ils le font (voir l'exemple ci-dessous).
