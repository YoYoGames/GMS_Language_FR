Définir l'état de la chronologie

Cette action peut être utilisée pour démarrer, arrêter, mettre en pause ou reprendre une chronologie. La chronologie doit d'abord avoir été définie pour l'instance à l'aide de l'action Définir la chronologie de l'instance, puis vous pouvez utiliser cette action à tout moment pour modifier son état d'exécution à partir des options suivantes

Option Description
Début Démarrez la chronologie affectée à l'instance (si la chronologie a été mise en pause, cela redéfinira la position de départ sur l'instant 0).
CV Reprenez une chronologie en pause (si la chronologie a été arrêtée et non suspendue, cela ne fera rien).
En pause Mettez en pause la chronologie attribuée à l'instance (pendant la pause, les moments d'une chronologie ne progresseront pas avec les images du jeu).
Arrêter Arrêtez la chronologie attribuée à l'instance.

Syntaxe:

Arguments:

Argument Description
State L'état dans lequel placer la chronologie (voir le tableau ci-dessus)

Le code de bloc d'action ci-dessus vérifiera la valeur contenue dans une variable globale et si elle est true alors la chronologie est mise en pause, et si elle est false puis la chronologie sera reprise.
