Définir la vitesse de la chronologie

Cette action peut être utilisée pour définir la vitesse à laquelle une chronologie se déplacera à travers les moments qu'elle contient. Normalement, dans chaque image de jeu, vous souhaitez que la position dans la chronologie soit augmentée de 1, mais vous pouvez modifier ce montant en définissant cette action sur une valeur différente. La valeur que vous choisissez sera traitée comme un multiplicateur donc une valeur de 2, par exemple, fera tourner la chronologie à deux fois la vitesse, tandis qu'une valeur de 0,5 la ferait tourner à la moitié de la vitesse. Ce que cela signifie pour les moments, c'est que si la valeur est supérieure à un, plusieurs moments peuvent se produire dans le même cadre de jeu (aucun moment n'est jamais sauté), et si la vitesse est inférieure à un, alors après un moment, il y aura des images où rien ne se passe jusqu'à ce que le moment suivant s'exécute - si vous définissez la vitesse de la chronologie sur 0,1, par exemple, vous obtiendrez un moment sur une image de jeu, puis 9 images de jeu sans moment, et sur la dixième image de jeu, le l'instant suivant sera exécuté.

Si vous définissez une chronologie pour commencer à utiliser les actions Définir la chronologie de l'instance ou Définir l'état de la chronologie mais ne définissez pas la vitesse, la chronologie sera par défaut de 1 moment par image de jeu.

Syntaxe de l'action:

Arguments:

Argument Description
Speed La vitesse - en tant que multiplicateur - à laquelle la chronologie doit s'exécuter (la valeur par défaut est 1)

Exemple:

Le code de bloc d'action ci-dessus attribuera une chronologie à l'instance appelant les actions et définira le moment initial sur 10, puis la vitesse de la chronologie sur 2.
