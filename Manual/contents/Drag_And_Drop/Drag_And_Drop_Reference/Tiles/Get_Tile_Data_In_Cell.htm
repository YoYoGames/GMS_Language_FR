Obtenir les données de mosaïque dans la cellule

Avec cette action, vous pouvez obtenir les données de tuile pour une cellule de tuile sur une couche de carte de tuile. Les données de mosaïque sont simplement une valeur qui reflète l'index de la mosaïque avec les valeurs de retournement, de miroir et de rotation, ainsi que tous les masques de mosaïque personnalisés que vous avez utilisés. Avec cette action, vous fournissez d'abord la couche à cibler en utilisant le nom de la couche (une chaîne) tel que défini dans l'éditeur de salle, puis la colonne et la ligne de la couche de carte de tuiles pour lesquelles obtenir les données de tuile. La "cellule" est la zone sur la grille de la carte de tuiles qui contient la tuile dont tu veux obtenir les données, donc si vos tuiles sont 16x16, par exemple, et la pièce est de 1024x768, la tuile-map aura 64 colonnes et 48 lignes.

REMARQUE: le calque sélectionné doit avoir été défini comme un calque de mosaïque dans l'éditeur de salle, sinon vous risquez d'obtenir des erreurs.

La valeur retournée sera alors stockée dans la variable cible qui peut avoir été créée précédemment ou peut être une nouvelle temporaire (si vous cochez la case "Temp"). Pour plus d'informations sur les données de mosaïque, veuillez consulter la GML sur les fonctions de mappage de mosaïques.

Syntaxe de l'action:

Arguments:

Argument Description
Layer La couche à partir de laquelle obtenir la tuile
Column La colonne (de gauche à droite) de la cellule

Exemple:

Le code de bloc d'action ci-dessus parcourt chaque cellule de tuile de la pièce, récupère les données de tuile pour la cellule, les met en miroir, puis définit à nouveau la cellule.
