Définir les données de mosaïque dans la cellule

Avec cette action, vous pouvez définir les données de mosaïque pour une cellule de mosaïque sur une couche de carte de mosaïque. Les données de mosaïque sont simplement une valeur qui reflète l'index de la mosaïque avec les valeurs de retournement, de miroir et de rotation, ainsi que tous les masques de mosaïque personnalisés que vous avez utilisés. Avec cette action, vous fournissez d'abord la couche à cibler en utilisant le nom de la couche (une chaîne) tel que défini dans l'éditeur de salle, puis la colonne et la ligne sur la couche de carte de tuiles pour lesquelles définir les données de tuile. La "cellule" est la zone de la grille de mosaïques qui contient la mosaïque pour laquelle vous souhaitez définir les données. Par conséquent, si vos mosaïques mesurent 16 x 16, par exemple, et que la pièce fait 1024 x 768, la carte des mosaïques aura 64 colonnes et 48 lignes.

REMARQUE: le calque sélectionné doit avoir été défini comme un calque de mosaïque dans l'éditeur de salle, sinon vous risquez d'obtenir des erreurs.

Le dernier argument de l'action est les données de tuile elles-mêmes. Vous devez normalement récupérer les données de mosaïque pour la cellule de mappage de mosaïques à l'aide de l'action Obtenir les données de mosaïque dans la cellule, puis les manipuler à l'aide de l'action Définir la transformation des données de mosaïque avant de la redéfinir avec cette action. Pour plus d'informations sur les données de mosaïque, veuillez consulter la GML sur les fonctions de mappage de mosaïques.

Syntaxe de l'action:

Arguments:

Argument Description
Layer La couche avec la carte de tuiles à cibler
Columns La position de la colonne de cellule le long de l'axe horizontal pour définir les données de mosaïque pour
Row La position de la ligne de cellule le long de l'axe vertical pour définir les données de tuile
Data Les données de mosaïque à utiliser pour définir la cellule

Exemple:

Le code de bloc d'action ci-dessus parcourt chaque cellule de tuile de la pièce, récupère les données de tuile pour la cellule, les met en miroir, puis définit à nouveau la cellule.
