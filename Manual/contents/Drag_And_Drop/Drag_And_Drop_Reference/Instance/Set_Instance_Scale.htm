Définir l'échelle d'instance

Ce bloc d'action définit le image_xscale et image_yscale valeurs de l'instance. Les valeurs d'entrée ici sont des modificateurs qui seront appliqués au sprite affecté à l'instance, où une échelle de 1 (la valeur par défaut) indique aucune mise à l'échelle (1: 1), les valeurs plus petites seront réduites (0,5, par exemple, sera la moitié la largeur du sprite), et des valeurs plus élevées augmenteront. Vous pouvez utiliser des valeurs négatives pour retourner le sprite et le mettre à l'échelle, sauf si la valeur utilisée est exactement -1 (auquel cas le sprite est simplement retourné ou mis en miroir sur son origine sans mise à l'échelle). Vous pouvez choisir de définir l'échelle horizontale ou verticale par rapport aux valeurs actuelles, auquel cas vous allez ajouter ou soustraire la nouvelle valeur des valeurs d'échelle actuelles.

Notez que la modification de ces valeurs modifiera également la façon dont l'instance détecte les collisions, sauf si vous avez fourni un mask_index (masque de collision) dans l' éditeur d'objets. Veuillez noter que pour que les modifications de cette variable soient visibles, l'instance doit avoir soit aucun événement de dessin (et donc GameMaker Studio 2 dessine par défaut le sprite), soit être dessinée à l'aide de l'action Draw Self.

Syntaxe de l'action:

Arguments:

Argument Description
Horizontal Le facteur de mise à l'échelle verticale à utiliser (la valeur par défaut est 1).
Vertical Le facteur de mise à l'échelle horizontale à utiliser (la valeur par défaut est 1).

Exemple:

Le code de bloc d'action ci-dessus définit un nouveau sprite ainsi qu'un certain nombre d'autres propriétés pour la façon dont ce sprite doit être affiché, y compris la mise à l'échelle aux trois quarts de sa taille de base le long des deux axes.
