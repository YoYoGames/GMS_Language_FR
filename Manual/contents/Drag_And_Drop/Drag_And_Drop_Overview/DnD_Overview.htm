Aperçu deDrag And Drop™

Drag and Drop™(DnD™) est un outil de script visuel qui peut être utilisé pour créer vos jeux sans réellement taper de code. Cela ne veut pas dire que vous ne programmez pas de code lorsque vous utilisez DnD™, carvous le faites, mais que votre code est créé et présenté de manière visuelle en utilisant des "blocs" d'actions plutôt que du texte. d'une manière visuelle en utilisant des "blocs" d'actions plutôt que du texte. Les actions sont de simplesexpressions oudéclarations qui peuvent être " enchaînées " pour qu'une instance d'un objet fasse quelque chose lorsqu'elle est placée dans une pièce. Par exemple, vous pouvez avoir une action dans un événement de création pour déplacer une instance dans une direction aléatoire. une instance dans une direction aléatoire, et une autre action dans un événement de collision pour la faire rebondir sur les objets avec lesquels elle entre en collision (pour plus d'informations sur les différents événements disponibles et leur fonctionnement conjoint, consultez la section sur les événements d'objets). Onglets d'événements

Lorsque vous ajoutez un événement à votre objet, un nouvel onglet est ouvert dans la fenêtre de code pour afficher le contenu de cet événement. Vous pouvez ajouter plusieurs événements et avoir plusieurs onglets ouverts en même temps, passer de l'un à l'autre et les modifier comme vous le souhaitez. comme vous le souhaitez. Vous pouvez également fermer les onglets à tout moment pour réduire l'encombrement (en sélectionnant à nouveau l'événement dans l'objet, vous les rouvrirez), et vous pouvez cliquer sur et faire glisser l'onglet hors de la fenêtre dans sa propre fenêtre de code, auquel cas la fermeture de l'objet ou de la fenêtre de code ne fermera pas la nouvelle fenêtre. Les onglets peuvent également être déplacés vers la gauche ou la droite pour être réorganisés.

Aperçu de l'action

Au fur et à mesure que vous ajoutez des blocs d'action à votre événement, ils apparaissent sous une forme "abrégée" sous la forme d'un aperçu en anglais simple de ce qui se passe. L'ordre de l'aperçu est l'ordre dans lequel chacun des blocs d'action sera exécuté pour cet événement. sur n'importe quel élément de la liste pour que la vue du bloc d'action s'agrandisse jusqu'à cette position et puisse être modifiée.

Espace de travail du bloc d'action

Cette zone d'espace de travail est l'endroit où vous faites glisser les actions de la boîte à outils pour créer votre code DnD™. Cliquez sur sur une action et faites-la glisser dans cette zone pour l'ajouter à la liste des actions et la modifier. La plupart des actions ont diversarguments (paramètres) que vous pouvez définir, et la plupart prendront également des mots-clés, variables d'instance, ou même des fonctionsGML - par exemple, vous pouvez appeler l'action Set Direction et utiliser la fonction GML random(val) pour définir une direction aléatoire.

Lorsque vous ajoutez des blocs d'action à votre code, ils sont visuellement "enchaînés" les uns aux autres pour indiquer l'ordre dans lequel les différentes actions vont se produire, bien que la façon dont cette chaîne est créée dépende du bloc initial qui a été utilisé. Par exemple un bloc If Variable aura deux chaînes - une vers les blocs qui seront appelés si la condition vérifiée est true et ensuite le bloc suivant. après. Pour plus d'informations sur la construction de blocs d'action dans le code, consultez les sections liées ci-dessous.

Boîte à outils

La boîte à outils est l'endroit où sont stockées toutes les actions DnD™, les différentes sections contenant des collections (bibliothèques) d'actions similaires ou apparentées. Chaque collection d'actions est liée à un thème spécifique ou à un objectif commun et il vous suffit de cliquer sur et de faire glisser l'action requise de la boîte à outils vers l'espace de travail du bloc d'action pour l'ajouter à l'événement en cours. Pour simplifier les choses Pour simplifier les choses - car il y a beaucoup d'actions - vous pouvez utiliser la barre "Recherche" en haut pour rechercher une action spécifique ou pour filtrer les actions visibles (tapez "dessiner" par exemple pour obtenir toutes les actions de dessin), et vous pouvez également cliquer sur et faire glisser les icônes de n'importe quelle bibliothèque vers votre dossier de favoris pour regrouper les actions les plus utilisées. Cette fonction est particulièrement utile si vous avez l'habitude de faire certaines choses à plusieurs reprises. Le dossier des favoris peut être vidé en cliquant sur la petite icône "poubelle".

Pour une liste complète des actions disponibles, ainsi que des arguments qu'elles prennent et de la façon dont elles peuvent être enchaînées, veuillez consulter la section Référence duglisser-déposer.
