Application d'actions à d'autres instances

La plupart des actions dans les DnD™ ont une option pour appliquer l'action de différentes manières. Cela s'appelle la définition de la portée de l'action et peut être l'une des choses suivantes:

Ce que vous GameMaker Studio 2 est essentiellement l'instance qui doit exécuter l'action. La portée de l'action par défaut est self, ce qui signifie qu'une fois que l'objet est créé en tant qu'instance dans la salle, cette instance exécutera le code d'action. Cependant, ce n'est pas toujours ce que vous voulez et vous voudrez peut-être que certaines actions affectent d'autres, voire toutes les instances de la salle. C'est là qu'intervient la modification de la portée de l'action.

Les différentes portées pour effectuer des actions sont répertoriées ci-dessous, mais il convient de noter que la modification de la portée d'une action de cette manière n'appliquera la nouvelle portée qu'à cette action et non aux actions suivantes de la chaîne. Si vous souhaitez appliquer un changement de portée d'action à plusieurs actions chaînées, utilisez d'abord l'action Appliquer à....

Soi

Il s'agit de la portée par défaut d'une action et indique simplement que l'action ne doit être appelée que par l'instance qui exécute le code.

Autre

Le other scope a deux fonctions principales, et la valeur qu'il renvoie dépendra de l'endroit et de la manière dont vous l'utilisez. Dans l' événement de collision, other renverra la valeur d'ID d'instance unique (une valeur unique qui est utilisée pour distinguer les instances individuelles de chaque objet), de sorte que vous pouvez, par exemple, créer une instance "balle" et avoir un événement de collision avec une instance "joueur" et dans ce Utilisez le other portée pour supprimer les points de vie de l'objet "joueur", puis revenir à self portée pour détruire l'instance "balle".

En dehors de l'événement de collision, le other le réglage se comportera comme s'il était réglé sur noone sauf s'il est appelé à partir d'un bloc d'actions limité. Cela signifie que si vous modifiez la portée d'un groupe d'actions en un objet spécifique, alors que ces actions sont appelées, le other scope renverra l'ID d'instance de l'instance qui a initialement appelé le groupe d'actions. Par exemple, vous pouvez exécuter une action Appliquer à... puis, dans le bloc de code suivant, définir la portée sur other pour effectuer une action sur l'instance exécutant tout le bloc d'événements, et non sur l'instance qui est en cours de délimitation dans les blocs apply to code. Tout

Lorsque vous définissez une action pour tous, vous GameMaker Studio 2 d'exécuter ce bloc pour chaque instance active dans la salle actuelle. Par exemple, créer une action Destroy Object Instance et définir sa portée sur all fera disparaître toutes les instances de la pièce, quel que soit l'objet à partir duquel elles ont été créées.

Objet

Une action peut également recevoir un objet comme sa portée. Cela signifie que toutes les instances de l'objet donné exécuteront cette action en même temps qu'elle sera appelée. Donc, si vous avez 100 instances ennemies dans la pièce, par exemple, et que vous voulez toutes les configurer pour qu'elles pointent vers un point spécifique. Vous appelez la direction du point de consigne et définissez la portée de l'action sur l'objet "obj_Enemy" et lorsqu'il est appelé, toutes les instances de cet objet changeront de direction.

Expression

Le champ de saisie Expression vous permet de saisir l'ID d'une instance spécifique sur laquelle vous souhaitez que l'action fonctionne. Il peut s'agir de la valeur d'ID unique attribuée à une instance à partir de l'éditeur de salle ou de l'ID d'une instance que vous avez stockée dans une variable (où la variable serait la valeur d'entrée) ou il peut même s'agir d'une expression utilisant code, comme indiqué dans l'image ci-dessous:
