Options de jeu deWindows

Cette section présente les différentes options disponibles qui contrôlent la façon dont vos projets de jeux Windows seront compilés. Les différentes sections sont les suivantes :

Général

La page Général pour les jeux Windows est l'endroit où vous devez fournir les informations nécessaires pour que Windows puisse afficher lorsque votre jeu a été installé et est en cours d'exécution. Ces informations se trouvent dans la section Informations sur le produit et doivent être remplies complètement.

La section Options de cette fenêtre vous permet de choisir où le paquetage du jeu doit être installé, soit dans les dossiers "%localappdata%" ou "%appdata%", ainsi que de choisir l'option Copy exe to output folder. ainsi que l'option Copy exe to output folder. En général, vous ne devriez jamais avoir besoin de cette option, mais elle est fournie pour les utilisateurs qui ont plusieurs DLL qui communiquent entre elles et avec votre jeu. Lorsqu'elle est activée, elle maintient l'EXE du jeu dans le dossier de sortie lors des tests, plutôt que de le placer dans le dossier Temp .

Ensuite, vous pouvez définir la marge de veille de Windows. Cette option est liée à la réduction du bégaiement lors de l'exécution de votre jeu sur des systèmes spécifiques. En fait, si votre jeu tourne plus vite que la vitesse de votre pièce, GameMaker Studio 2 " dormira " pendant le temps restant. pour le temps restant, mais cette mise en veille peut être assez imprécise et vous pouvez souvent dormir plus longtemps que nécessaire, ce qui fait que votre image prend plus de temps et provoque des bégaiements. Pour contourner ce problème, nous pouvons dormir pendant moins de temps, puis rester dans une pour le reste du temps afin de le rendre plus précis - bien que le problème avec le fait de rester dans une boucle est qu'il entraîne une augmentation de l'utilisation du CPU, ce qui peut entraîner une augmentation de la température du CPU et un fonctionnement plus rapide du ventilateur. Par défaut, cette valeur est fixée à 10, et dans 99,99% des cas, cela vous conviendra, mais pour les machines bas de gamme ou pour les machines avec beaucoup de processus d'arrière-plan en cours d'exécution, cela peut ne pas être une solution idéale (ou pour les machines très haut de gamme aussi) et une valeur de 1 ou 15 ou autre peut être nécessaire. Notez cependant qu'il s'agit d'une configuration très spécifique au système et que ce qui fonctionne sur votre machine de construction peut ne pas être approprié pour un autre utilisateur, et en tant que tel, en cas de doute, laissez le paramètre à 10.

REMARQUE: en plus de la marge de veille, vous pouvez contrôler davantage vos performances et votre consommation d'énergie sous Windows en ajustant la résolution du planificateur de threads au moment de l'exécution .

L'option suivante consiste à activer Steam pour votre jeu. Si vous activez cette option, assurez-vous d'avoir installé le SDK Steamworks(voir ici), et que vous avez ajouté le Steam App ID dans les options générales du jeu. Notez que si vous exécutez GameMaker Studio 2 sous Windows et à partir de la version Steam de l'IDE et que vous développez des jeux pour Steam, vous devrez peut-être aussi cocher l'option Utiliser un autre lanceur pour l' application Steam. Cette option permet de lancer votre jeu lors des tests à l'aide d'un programme de lancement alternatif afin d'éviter qu'il ne soit signalé par Steam. avec l'ID d'application de GameMaker Studio 2 au lieu de l'ID d'application de votre jeu. Vous ne devez activer cette option que lorsque vous développez pour Steam et si vous constatez que vous avez des problèmes pour lancer votre jeu.

L'avant-dernière option dont vous disposez est l'option Disable file system sandbox. Si vous cochez cette option, GameMaker Studio 2 désactivera le sandboxing pour l'enregistrement et le chargement des fichiers, ce qui permettra à vos jeux d'accéder aux fichiers à partir de n'importe quel endroit sur le système qui exécute le jeu. Cette option n'est pas cochée par défaut, car nous vous recommandons de maintenir le bac à sable de GameMaker Studio 2 pour les fichiers sur tous les systèmes et, lorsque cette option est cochée, certaines zones d'enregistrement/chargement peuvent ne pas être disponibles en fonction des autorisations du système d'exploitation. Pour plus d'informations sur le bac à sable, consultez les pages consacrées au système de fichiers de GameMaker Studio 2.

Enfin, vous avez la possibilité de demander à GameMaker Studio 2 de créer un exécutable x64 pour vos projets Windows. Lorsqu'elle est activée, cette option crée un exécutable 64 bits pour votre jeu lorsqu'il est compilé, et nous recommandons fortement que tous les nouveaux projets utilisent cette option. Nous recommandons fortement à tous les nouveaux projets d'utiliser cette option. Ce format deviendra le format par défaut (et le seul) pour les exécutables Windows à long terme, car l'exécuteur 32 bits sera progressivement abandonné. Notez que lorsque cette option est activée, toute dll d'extension utilisée par le projet devra également être 64 bits (voir ici pour plus d'informations).

Graphiques

Les options graphiques sont celles que vous devez configurer pour déterminer comment votre jeu va utiliser la carte graphique de votre PC Windows cible. Les options suivantes sont incluses pour que vous puissiez les modifier :

  • Afficher le curseur: Lorsque cette option est cochée, le curseur normal de la fenêtre sera affiché. Si vous la décochez, aucun curseur ne sera affiché, à moins que vous n'en ayez créé un dans le code de votre jeu. Cette option est activée par défaut.
  • Démarrer en plein écran: Si cette option est cochée, le jeu démarrera en mode plein écran, sinon il démarrera en mode fenêtré. Cette option est désactivée par défaut.
  • Autoriser le passage au plein écran: si cette option est cochée, l'utilisateur peut passer du plein écran au fenêtré et inversement en utilisant les raccourcis standard de Windows. Cette option est désactivée par défaut.
  • Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques de pixels nets, il est préférable de le désactiver, mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il vaut mieux le laisser activé. mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de laisser cette option activée par défaut.
  • Utiliser la synchronisation pour éviter le déchirement: Ceci permet d'activer ou de désactiver v-sync. Notez que si vous avez un jeu avec une vitesse de 120 et que le joueur a un moniteur avec un taux de rafraîchissement de 60, activer cette option verrouillera votre vitesse de jeu à 60 également. de 60, l'activation de cette option bloquera la vitesse du jeu à 60 également. Cette option est désactivée par défaut.
  • Autoriser le redimensionnement de la fenêtre: En cochant cette case, l'utilisateur peut modifier la taille de la fenêtre du jeu (l'option Fenêtre sans bordure doit être désactivée pour que cela fonctionne). Cette option est désactivée par défaut.
  • Fenêtre sans bordure: Si vous cochez cette case, votre jeu fonctionnera avec une fenêtre sans bordure, ce qui supprimera les boutons normaux de réduction, d'agrandissement et de fermeture, ainsi que le titre du jeu. Cette option est désactivée par défaut.
  • Mise à l'échelle: Vous pouvez choisir de conserver le rapport hauteur/largeur (une pièce 4:3 sera donc "letter boxée" sur un écran 16:9) ou de mettre l'image à l'échelle (en l'étirant pour l'adapter au plein écran).

    ATTENTION ! La désactivationdela surface d'application désactive toutes les options de mise à l'échelle définies dans les options de jeu de Windows jusqu'à ce qu'elle soit réactivée. Voir La surface d'application pour plus de détails.

Enfin, il est possible de définir la taille de la page de texture. La taille par défaut (et la plus compatible) est de 2048x2048, mais vous pouvez choisir entre 256x256 et 8192x8192. Le site est également un bouton marqué Prévisualisation qui génère les pages de texture pour cette plate-forme et ouvre une fenêtre pour que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d'avoir des pages de texture plus grandes. structurées et éviter d'avoir des pages de texture plus grandes (ou plus petites) que nécessaire. Pour plus d'informations sur les pages de texture, voir ici.

REMARQUE: plus la taille de la page de texture est grande, moins votre jeu sera compatible avec les PC ayant des spécifications inférieures.

Images

La section des images est l'endroit où vous fournissez les images dont votre jeu a besoin. Pour Windows, vous devez fournir un fichier d'icône (qui doit être au format .ico ) et un écran d'accueil (qui peut être au format .png, .bmp, .jpg ou .gif ). L'écran d'accueil s'affichera pendant le chargement du jeu et peut être activé ou désactivé en cochant l'option Utiliser l'écran d'accueil (qui est désactivée par défaut).

Il convient de noter que GameMaker Studio 2 dispose d'un outil Project Image Generator qui peut être utilisé pour créer automatiquement toutes les images requises pour les différentes plates-formes cibles sur lesquelles votre jeu est compilé. est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu'elles correspondent à vos besoins.

Installateur

Les jeuxWindows peuvent être créés à l'aide d'un programme d'installation et vous pouvez y définir les différents paramètres et graphiques que GameMaker Studio 2 utilisera lors de sa création. que GameMaker Studio 2 utilisera lors de sa création. Tout d'abord, nous avons les options graphiques qui définissent l'apparence de l'installateur lui-même :

  • Terminé: Le graphique qui apparaît sur la page "Fini" de l'installateur. Il doit être au format *.bmp et mesurer 164x314px.
  • En-tête: Le graphique qui apparaît pendant l'exécution du programme d'installation. Il doit être au format *.bmp et mesurer 150x57px.

Vous pouvez également indiquer un fichier d'accord de licence spécifique (un exemple est fourni avec GameMaker Studio 2) et un fichier de script NSI personnalisé (détails ici) si vous en avez besoin. ou cliquer sur les boutons appropriés pour revenir aux fichiers par défaut fournis avec GameMaker Studio 2.
