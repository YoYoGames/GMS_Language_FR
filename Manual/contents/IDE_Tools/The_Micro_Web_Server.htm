Le serveur Micro Web

La fonction principale de ce petit outil est de se connecter et de servir des fichiers à certaines des plateformes cibles disponibles pour GameMaker Studio 2 (comme la cible HTML5, par exemple). Lorsque vous exécutez un jeu sur une cible qui nécessite le Micro Web Server, vous obtiendrez une nouvelle fenêtre s'ouvre pendant que le jeu est servi sur l'appareil choisi :

REMARQUE: La première fois que vous exécutez un jeu sur une cible qui nécessite le Micro Web Server, il est possible que vous receviez une alerte de pare-feu. Si cela se produit, vous devez autoriser le serveur Web à communiquer à travers le pare-feu.

Le serveur Web affiche un certain nombre d'informations de débogage relatives aux fichiers qui ont été servis, et dispose d'un bouton pour les effacer en bas. À côté du bouton "Clear" se trouve le chemin d'accès au fichier JS racine du projet. vous avez l'option de vitesse du réseau. Celle-ci est réglée sur "No Limit" par défaut, mais si vous cliquez dessus, une liste de différentes options de réseau s'affiche :

La modification de ce paramètre entraîne également la modification de l'option Taille du paquet vers la droite. Vous pouvez également modifier ce paramètre manuellement, dans ce cas, la vitesse du réseau sera réglée sur "Personnalisé". Si vous choisissez une valeur autre que la valeur maximale, la vitesse de transfert du réseau sera réduite et vous pourrez tester des jeux dans un environnement à faible bande passante.

Vous pouvez également voir la liste des IP autorisées ainsi que le numéro de port utilisé, mais ceux-ci ne peuvent pas être modifiés dans le Micro Web Server et doivent être définis dans les paramètres du Device Manager pour le périphérique cible donné.
