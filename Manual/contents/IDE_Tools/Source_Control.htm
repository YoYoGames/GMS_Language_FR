Contrôle des sources

Lagestion du contrôle des sources (SCM) est le nom donné à la méthode permettant de travailler avec des sous-versions ou des sauvegardes de vos projets par le biais d'un dépôt et d'une source locale. Fondamentalement, une solution de GCL est un progiciel indépendant qui contrôle tous les aspects de la maintenance, de la modification et de la comparaison des versions de votre projet pendant que vous travaillez dessus. Cette solution est particulièrement utile pour les personnes qui travaillent en équipe et qui doivent pouvoir contrôler qui fait quoi, sans craindre de perdre des données ou de faire des changements qui devront être annulés. Différents outils SCM sont à votre disposition, et GameMaker Studio 2 est livré avec un plugin Git qui peut être utilisé immédiatement. Vous n'avez pas besoin d'installer de paquets supplémentaires car tout est fourni avec le plugin lui-même. Vous trouverez ci-dessous des liens vers un petit tutoriel expliquant comment configurer ce plugin et utiliser les outils SCM avec un projet.

Lorsque vous avez activé le contrôle de la source pour un projet quelconque (voir la section Configuration du plugin Git de contrôle de la source ci-dessus pour plus de détails), GameMaker Studio 2 affiche le menu contextuel suivant en haut de l'IDE en haut de l'IDE :

Nous présentons ici chacune des options disponibles (la plupart d'entre elles sont expliquées plus en détail dans les sections du tutoriel ci-dessus) :

  • Créer un dépôt de projet: Cette option vous permet de créer un dépôt local dans le même répertoire que vos fichiers de projet.
  • Importer le projet dans le référentiel: Cette option vous permet d'importer un projet dans un référentiel externe.
  • Cloner le référentiel: Cette option vous permettra de cloner un dépôt d'une source vers une nouvelle destination.
  • Livrer les changements: Avec cette option, vous pouvez mettre en scène les fichiers modifiés dans votre projet et ensuite les valider dans le référentiel.
  • Pousser les changements: Après avoir fait un commit ou une fusion, cette option est utilisée pour pousser les changements vers le dépôt maître.
  • Tirer les changements: Avec cette option, vous pouvez mettre à jour le référentiel local en tirant les fichiers modifiés du référentiel maître.
  • Afficher l'historique: Cela ouvrira la fenêtre de l'historique où vous pouvez voir tout l'historique des versions du projet et choisir de revenir en arrière sur des chemins de fichiers spécifiques ou des révisions entières.
  • Afficher les conflits: Cette option ouvre la fenêtre Conflits et répertorie tous les fichiers en conflit qui peuvent exister dans le projet, ce qui vous permet de les traiter soit via l'IDE de GameMaker Studio 2, soit à l'aide d'un outil de fusion spécifique.
