Cloner un référentiel

Lorsque votre projet est stocké dans un référentiel externe, vous devez vous rappeler de pousser vos modifications et d'en extraire de nouvelles. La raison en est que lorsque vous vous engagez, vous stockez vos modifications localement, ce qui vous permet de travailler loin d'une connexion Internet et de vous synchroniser lorsque vous êtes prêt, et pour synchroniser, vous poussez vos modifications vers le référentiel distant. Extraire les modifications signifie que vous souhaitez extraire toutes les modifications que d'autres ont apportées du référentiel distant vers votre référentiel local. Cependant, cela signifie qu'il peut arriver que des utilisateurs modifient les mêmes fichiers, ce qui crée à son tour un conflit de fusion. Vous devez savoir comment gérer ces conflits, nous allons donc en créer un délibérément de temps en temps, puis vous montrer comment y faire face.

REMARQUE: le partage d'un référentiel de projet local (même via un service de synchronisation de fichiers comme Dropbox) n'est pas recommandé, car les validations sont généralement écrites directement dans celui-ci, et vous avez donc très peu de protection contre les conflits.

Pour générer notre conflit, nous devons d'abord avoir deux référentiels locaux, un pour le référentiel maître et un pour notre copie. Pour créer cela, nous devons cloner le référentiel, ce qui se fait avec les étapes suivantes:

  • Accédez à Contrôle de la source > Référentiel de clonage.
  • Dans le champ URL, saisissez le chemin d'accès au projet que nous avons configuré dans le guide Reverting Files ci-dessus (par défaut, ce sera <DRIVE>:\Users\<Username>\Documents\GameMaker Studio2 \ <NomProjet>).
  • Nous devons ensuite définir un chemin vers lequel cloner:
    Lorsque nous cliquons sur le bouton "OK", nous recevrons l'avertissement suivant concernant la tentative de clonage d'un dépôt "non nu":
    Nous pouvons à nouveau cliquer sur "OK".
  • Nous allons maintenant être présenté avec un explorateur de répertoires pour créer un référentiel proxy "nu" (il s'agit d'un référentiel que vous pouvez stocker en toute sécurité sur Dropbox, par exemple). Naviguez jusqu'à l'endroit où nous voulons créer ce référentiel proxy et sélectionnez (ou créez) le dossier à utiliser.

Une fois que nous avons fait cela, GameMaker Studio 2 poussera le projet d'origine vers le proxy, et en tirera un nouveau vers le chemin que nous avons défini dans la fenêtre Clone Repository, puis ouvrira automatiquement un navigateur de fichiers afin que nous puissions ouvrir le projet stocké.. Si nous allons maintenant dans Contrôle de la source > Afficher l'historique, nous verrons nos commits d'origine plus un nouveau "Push For Clone"

Vous avez maintenant cloné votre référentiel et pouvez continuer à travailler sur la version locale du projet.
