Travailleurs à distance

Lorsque vous compilez votre jeu, vous utilisez généralement le travailleur "Local", c'est-à-dire l'ordinateur sur lequel GameMaker Studio 2 a été installé. Cependant, dans certaines circonstances, vous devrez peut-être utiliser un autre ordinateur pour construire vos projets. Par exemple, si vous travaillez sur un Mac et que vous souhaitez créer un projet Windows, vous pouvez configurer un Remote Worker et demander à un PC Windows de créer le projet. En fait, lorsque vous utilisez un travailleur distant, GameMaker Studio 2 envoie tous les fichiers nécessaires depuis le serveur (local). fichiers requis depuis l'ordinateur (local) sur lequel il est exécuté vers l'ordinateur du travailleur à distance, puis utilise cet ordinateur pour créer l'exécutable final du projet (en l'enregistrant sur l'ordinateur distant ou en le renvoyant ensuite sur l'ordinateur local). Cependant, avant de pouvoir utiliser cette fonctionnalité, vous devrez avoir installé certains outils sur la machine de construction et effectué certains réglages, que nous allons expliquer ici.

IMPORTANT ! Vous pouvez configurer un travailleur à distance en utilisant un PC Windows local et un PC Windows distant, ou entre un PC Mac local et un PC Windows distant, mais actuellement vous ne pouvez pas configurer un PC Windows (ou Mac) local et un PC Mac distant, c'est-à-dire que vous ne pouvez pas c'est-à-dire que vous ne pouvez pas avoir un télétravailleur Mac, seulement un Windows.

Configuration initiale

Pour configurer un travailleur à distance, vous devez d'abord installer l'application de l'outil de construction Remote Worker sur le PC à utiliser. Vous pouvez trouver cet outil en allant dans le menu Build de GameMaker Studio 2 sur la machine locale et en sélectionnant l'option " ShowRemote Worker Installer " :

Vous pouvez également cliquer sur l'icône " œil " dans le Target Manager et obtenir l'installateur de Remote Worker de cette façon :

Le programme d'installation s'appellera par exemple "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe ". et vous devez le copier sur le PC que vous voulez utiliser comme travailleur à distance (il n'est pas nécessaire que GameMaker Studio 2 soit installé sur le travailleur à distance, seulement ce programme).

Une fois que vous l'avez copié sur le PC de base, installez-le en suivant les instructions à l'écran. Lorsque le programme d'installation est terminé, le système d'exploitation peut vous demander d'accéder à votre réseau via le pare-feu. Une fois que vous aurez accordé l'accès, le programme d'installation ouvrira alors une page dans votre navigateur Web. Vous devez noter l'adresse IP et le numéro de port indiqués sur cette page car vous en aurez besoin pour l'étape suivante.

Vous devez maintenant retourner sur l'ordinateur sur lequel GameMaker Studio 2 installé et cliquer sur le bouton Target Manager pour ouvrir la fenêtre Target Manager, puis allez dans la section Worker et cliquez sur le bouton Add Worker . La fenêtre suivante s'ouvrira alors fenêtre suivante :

Ici, vous devez donner un nom à votre télétravailleur (c'est ce qui sera affiché dans la liste des télétravailleurs dans Target Manager). dans la liste des travailleurs dans le Target Manager) et dans la section Host Name, ajoutez l'adresse et le port qui vous ont été montrés et notés précédemment sur la machine du travailleur.

Si tout s'est déroulé correctement, vous devriez maintenant recevoir un message indiquant que vous pouvez vous connecter avec succès à la machine du travailleur à distance. Une fois que votre travailleur à distance est installé et connecté à votre ordinateur IDE principal, vous devrez configurer les paramètres de construction locaux pour chaque plateforme (voir la section ci-dessous) avant de pouvoir compiler votre projet. Paramètres à distance

La page des paramètres montre les différents chemins d'accès et les paramètres de construction pour l'application Remote Worker. En haut des paramètres, vous avez les différents chemins qui seront utilisés par l'application Remote Worker. Avant de faire quoi que ce soit d'autre, il convient de les vérifier et de définir d'autres chemins si nécessaire (en général, les chemins par défaut devraient convenir). Compiler

Une fois que votre Remote Worker est configuré et que vous avez rempli les paramètres de compilation locale (voir la section ci-dessus), vous pouvez commencer à l'utiliser pour compiler vos jeux. Pour cela, il suffit de sélectionner le Remote Worker dans la liste Target Manager de sélectionner la plate-forme pour laquelle vous souhaitez tester/compiler, ainsi que toute autre option (comme le type de compilation ou la configuration), puis de cliquer sur Play ou Create Executable .

REMARQUE: les plates-formes disponibles dans le Target Manager changent en fonction des capacités de l'ordinateur du travailleur à distance et toutes les cibles ne sont pas disponibles. Notez également qu'il n'existe pas de cible "Test" pour les travailleurs à distance.

Si vous jouez simplement le projet, la machine Remote Worker le compilera et l'exécutera, mais si vous créez un exécutable, Remote Worker compilera le projet et renverra un fichier *.zip à la machine locale pour qu'elle le sauvegarde. machine locale pour qu'elle l'enregistre.

Il convient de noter que ni le bouton Debug ni le bouton Clean ne seront disponibles lorsque vous utilisez un Remote Worker. Notez également que vous ne pouvez actuellement effectuer qu'une seule construction à distance à la fois.


Une fois que le Remote Worker est installé et fonctionne sur votre machine distante, il apparaît dans la barre des tâches sous forme d'icône et vous pouvez cliquer dessus avec le bouton droit de la souris pour faire apparaître les options suivantes :

  • Ouvrez: Ouvrez l'application Remote Worker dans votre navigateur par défaut.
  • Afficher le journal: Ouvrez le fichier journal du travailleur à distance dans votre navigateur.
  • Paramètres réseau: Ici, vous pouvez filtrer les machines que vous souhaitez autoriser à envoyer des constructions à la machine du télétravailleur. Chaque filtre peut être un filtre de type CIDR ou un nom d'hôte d'une machine à autoriser. Il faut savoir que si vous voulez que plusieurs machines se connectent à un travailleur distant, ou si vous mélangez les types de connexion (par exemple : filaire et wifi), vous devrez peut-être modifier la plage d'adresses IP autorisées ici.
  • Modifier les paramètres: Ouvrez le menu des paramètres du télétravailleur (expliqué dans la section Paramètres, ci-dessus).
  • Quitter: Quittez l'application Remote Worker. Cela signifie que vous ne pourrez plus vous connecter (ni compiler) avec l'application depuis une machine locale jusqu'à ce que vous redémarriez l'application Remote Worker.
