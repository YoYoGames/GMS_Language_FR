Définir les données de mosaïque sur le pixel Avec cette action, vous pouvez définir les données de tuile pour une cellule de tuile trouvée à une position spécifique sur une couche de carte de tuile. Les données de mosaïque sont simplement une valeur qui reflète l'index de la mosaïque avec les valeurs de retournement, de miroir et de rotation, ainsi que tous les masques de mosaïque personnalisés que vous avez utilisés. Avec cette action, vous fournissez d'abord le calque à cibler en utilisant le nom du calque (une chaîne) tel que défini dans l'éditeur de pièce, puis les axes x et y dans la pièce pour lesquels définir les données de mosaïque. Notez que les données de tuile seront définies pour la "cellule" de tuile map dans laquelle la position donnée atterrit. Par exemple, si les tuiles de votre jeu de tuiles mesurent 16x16, le calque de la carte de tuiles contiendra les tuiles dans des cellules de 16x16 pixels, donc si vous utilisez cette action pour cibler la position de la pièce (8, 8), vous changerez la tuile cellule (0, 0) de la carte de tuiles. REMARQUE: le calque sélectionné doit avoir été défini comme un calque de mosaïque dans l'éditeur de salle, sinon vous risquez d'obtenir des erreurs. Le dernier argument de l'action est la donnée de la tuile elle-même. Vous récupérez normalement les données de mosaïque pour la cellule de carte de mosaïque à l'aide de l'action Obtenir les données de mosaïque au pixel, puis vous les manipulez à l'aide de l'action Définir la transformation des données de mosaïque avant de la redéfinir avec cette action. Pour plus d'informations sur les données de mosaïque, veuillez consulter la GML sur les fonctions de mappage de mosaïques. Syntaxe de l'action: Arguments: Argument Description Layer La couche avec la carte de tuiles à cibler X La position le long de l'axe horizontal (X) pour définir les données de mosaïque pour Y La position le long de l'axe vertical (Y) pour définir les données de mosaïque Data Les données de mosaïque à utiliser pour définir la cellule à la position Exemple: Le code de bloc d'action ci-dessus teste un clic de souris et quand on est reçu, il récupère les données de tuile pour la cellule qui coïncide avec la position de la souris. Ces données sont ensuite modifiées et renvoyées à la couche de carte de tuiles, faisant pivoter la tuile dans cette cellule.
