Dessiner des vies d'instance Cette action dessinera un sprite pour représenter chacune des vies de l'instance actuellement. Le nombre à dessiner est tiré de la valeur actuelle de la variable d'instance locale lives, qui est une variable spéciale créée automatiquement dans l'instance si vous utilisez l'une des actions qui le nécessitent. Vous donnez le sprite à dessiner pour les vies et l'ordre de la pile, qui peut être soit une ligne (horizontalement, de gauche à droite), soit une colonne (verticalement, de haut en bas), ainsi que la position. La position peut être une position absolue dans la pièce, ou une position relative à la position de l'occurrence faisant le dessin, et l'espacement entre les images sera basé sur la largeur ou la hauteur de l'image-objet. Notez que cela dessine simplement une image statique - l'image unique initiale (image 0) du sprite donné - et toutes les autres images seront ignorées, de même que toutes les transformations qui ont été ajoutées à l'instance en modifiant les variables d'instance (comme image_xscale ou image_blend ). Le texte de la légende sera dessiné avant la valeur des lives et s'il s'agit de texte, il doit être entouré de "", et les deux seront formatés en utilisant la police, la couleur et l'alignement actuels du dessin. REMARQUE: cette action est uniquement destinée à être utilisée dans les divers événements de tirage au sort et ne tirera rien si elle est utilisée ailleurs. Syntaxe de l'action: Arguments: Argument Description Caption La chaîne à dessiner avec la valeur du score (précédente et peut être laissée vide) Stack Order L'ordre dans lequel dessiner (ligne ou colonne) x La position x pour dessiner les vies à y La position y pour dessiner les vies à Exemple: Le code de bloc d'action ci-dessus dessine des sprites par rapport à l'instance pour représenter le courant lives évaluer.
