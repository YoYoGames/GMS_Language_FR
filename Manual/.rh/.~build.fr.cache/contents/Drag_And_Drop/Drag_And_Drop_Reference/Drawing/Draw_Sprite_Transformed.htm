Dessiner un sprite transformé Cette fonction dessinera le sprite donné similaire à l'action Dessiner un sprite mais avec des options supplémentaires pour changer l'échelle, le mélange, la rotation et le cadre du sprite dessiné. La modification de ces valeurs ne modifie pas la ressource de quelque manière que (seulement comment elle est), et vous pouvez utiliser l' une des disponibles variables de propriété sprite au lieu des valeurs directes pour tous les arguments de la fonction. L'image ci-dessous illustre comment différentes valeurs affectent le dessin du sprite: REMARQUE: La fusion des couleurs n'est recommandée pour la cible HTML5 que lorsque WebGL est activé, bien que vous puissiez toujours définir la couleur de fusion si elle n'est pas activée et elle mélangera l'image-objet comme d'habitude. Cependant, tout mélange de cette manière crée un sprite en double qui est ensuite stocké dans le cache et utilisé lorsque cela est nécessaire. Ceci est loin d'être optimal et si vous utilisez plusieurs changements de couleur, cela ralentira les performances de vos jeux à moins que vous n'activiez WebGL. si vous ne souhaitez pas utiliser WebGL, vous pouvez définir la taille du cache de polices pour essayer de limiter cela si nécessaire en utilisant la fonction sprite_set_cache_size(). REMARQUE: cette action est uniquement destinée à être utilisée dans les divers événements de tirage au sort et ne tirera rien si elle est utilisée ailleurs. Syntaxe de l'action: Arguments: Argument Description Sprite Le sprite à dessiner Frame Le cadre de sprite à dessiner (utilisez la variable intégrée image_index pour l'image courante) X La position x à dessiner dans la pièce Y La position y à dessiner dans la pièce Scale X Le facteur d'échelle pour l'axe X (utilisez la variable intégrée image_xscale pour l'échelle X actuelle) Scale Y Le facteur d'échelle pour l'axe Y (utilisez la variable intégrée image_yscale pour l'échelle Y actuelle) Rotation L'angle de dessin du sprite (de 0 à 360, rotation anti-horaire où 0 est à droite - utilisez la variable intégrée image_angle pour la rotation actuelle) Colour La couleur à mélanger avec le sprite (la valeur par défaut est le blanc, et vous pouvez utiliser la variable intégrée image_blend pour la couleur actuelle) Exemple: Le code de bloc d'action ci-dessus vérifie une variable et si elle est définie sur true alors le sprite est dessiné avec une mise à l'échelle aléatoire le long des axes x et y et mélangé avec la couleur rouge, sinon le sprite est dessiné en utilisant les transformations par défaut.
