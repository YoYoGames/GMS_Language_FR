Exécuter le script Cette action peut être utilisée pour appeler une fonction de script ou une fonction définie par l'utilisateur. En général, cette action n'est utilisée que pour le support de l'héritage et ne devrait pas vraiment être utilisée dans les projets actuels ou nouveaux. Pour utiliser l'action vous donnez l'actif du script à appeler - ou le nom de la variable de la fonction définie par l'utilisateur à appeler - puis vous fournissez tous les arguments nécessaires, c'est-à-dire toutes les valeurs (chaînes, entiers, indices de ressources, etc...) que vous souhaitez entrer dans le script ou la fonction afin qu'il puisse les utiliser. Vous pouvez ajouter des arguments supplémentaires si nécessaire en cliquant sur l'icône plus à côté de l'action. Certains scripts et fonctions définies par l'utilisateur peuvent également renvoyer une valeur qui sera utilisée ultérieurement dans la liste des blocs d'action. Vous pouvez donc spécifier la variable cible qui doit stocker cette valeur de retour. Celle-ci peut avoir été créée précédemment ou peut être une nouvelle variable locale temporaire (si vous cochez la case " Temp "). locale (si vous cochez la case "Temp"). Les champs "cible" et "argument" peuvent être laissés vides s'ils ne sont pas nécessaires. Notez qu'il est généralement préférable d'utiliser l'action Function Call pour appeler des fonctions définies par l'utilisateur, et que vous ne devriez généralement pas avoir besoin d'utiliser cette action, ou d'appeler une ressource de script directement dans le code. Syntaxe de l'action : Arguments : Arguments Description Script Le nom du script ou de la fonction définie par l'utilisateur à appeler. Argument0 ... Argument3 Les différents arguments (valeurs) qui doivent être transmis au script ou à la fonction (les arguments non utilisés peuvent être laissés vides). Cible Le nom de la variable qui doit être ciblée pour toutes les valeurs retournées (peut être laissé vide).
