Si indéfini Cette action est utilisée pour vérifier si une variable ou une expression retourne une valeur "indéfinie", et si c'est le cas, cette action retournera true et si ce n'est pas le cas, elle retournera false . Undefined" peut être retourné pour un certain nombre de raisons, et une valeur indéfinie peut provoquer un crash ou une erreur sérieuse dans votre projet, vous pouvez donc utiliser cette action pour vérifier cela et l'attraper avant qu'il ne se produise. Si vous marquez l'argument "Not", alors vous vérifiez si une valeur n'est pas pas "indéfinie". Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme le montre l'image ci-dessous : Ces actions seront maintenant exécutées si le "if" vaut true, tandis que toute action déposée ailleurs sera exécutée après le bloc "if". Notez que vous pouvez "empiler" des contrôles non définis dans la même action en cliquant sur l'icône plus à côté de l'action, et en donnant une autre variable à vérifier. Cela permettra de vérifier si toutes les variables données sont indéfinies en une seule fois et de n'exécuter le code d'action ajouté que si elles le sont.
