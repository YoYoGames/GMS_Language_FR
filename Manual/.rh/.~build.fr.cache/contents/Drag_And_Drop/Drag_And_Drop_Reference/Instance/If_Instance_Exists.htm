Si l'instance existe Cette action peut être utilisée pour vérifier si une instance d'un objet spécifique existe dans la salle de jeu, ou si une instance spécifique elle-même existe. Vous donnez l'objet à partir de l'explorateur d'actifs, et si une instance existe dans la salle, l'action reviendra true sinon il reviendra false. Notez que vous pouvez utiliser ici une variable avec l'ID unique d'une instance spécifique (par exemple, tel que renvoyé par l'action Créer une instance d'objet ) ou même la valeur d'ID d'instance telle que renvoyée par l'éditeur de salle lorsque vous avez ajouté l'instance. Si vous marquez l'argument "Non", l'action vérifiera si aucune instance n'existe et si aucune n'est trouvée, elle retournera true et si même un existe il reviendra false. Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme indiqué dans l'image ci-dessous: Ces actions seront désormais exécutées si le "si" est évalué à true, tandis que toutes les actions déposées ailleurs seront exécutées après le bloc "if". Syntaxe de l'action: Arguments: Argument Description Object Le nom de l'objet pour lequel rechercher une instance (ou un ID d'instance). Exemple: Le code de bloc d'action ci-dessus vérifie la présence d'une collision et renvoie la valeur d'ID unique de l'instance avec laquelle il entre en collision. Cette valeur est stockée dans une variable temporaire, puis vérifiée pour voir si la valeur correspond à une instance qui se trouve actuellement dans la pièce (si aucune collision n'est trouvée, la variable contiendra le mot-clé "noone "), et si c'est le cas, alors une variable est définie dans l'instance avec laquelle il est entré en collision.
