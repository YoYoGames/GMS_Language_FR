Définir la variable d'instance Avec cette action, vous pouvez définir l'une des variables intégrées. Toutes les instances de tous les objets ont un certain nombre de variables déjà créées en leur sein pour contenir des informations essentielles telles que la position x / y actuelle, la vitesse, la direction, la visibilité et bien d'autres choses. Les variables d'instance que vous pouvez utiliser sont: Coordonnée X - La position le long de l'axe x dans la pièce. Coordonnée Y - La position le long de l'axe y dans la pièce. Direction - La direction du mouvement de l'instance (la valeur par défaut est 0, où droite est 0 °, haut est 90 °, gauche est 180 ° et bas est 270 °). Vitesse - La vitesse (en pixels par image de jeu) à laquelle l'instance se déplace. Vitesse horizontale - La vitesse horizontale (en pixels par image de jeu) à laquelle l'instance se déplace. Une vitesse négative signifie qu'elle se déplace vers la gauche, une vitesse positive est à droite et la valeur par défaut est 0. Vitesse verticale - La vitesse verticale (en pixels par image de jeu) à laquelle l'instance se déplace. Une vitesse négative signifie qu'elle monte, une vitesse positive est réduite et la valeur par défaut est 0. Visible - La variable de bascule de visibilité (la valeur par défaut est true sauf indication contraire dans l' éditeur d'objets, définissez sur false rendre invisible). Notez qu'une instance invisible exécute toujours tous ses événements à l'exception de l'événement Draw. Persistent - La variable à bascule de persistance (la valeur par défaut est false sauf indication contraire dans l' éditeur d'objets, définissez sur true rendre persistant). Cela rendra une instance "persistante" d'une pièce à l'autre, c'est-à-dire: l'instance ne sera pas nettoyée du côté de la pièce mais sera transférée dans la pièce suivante. Notez que lorsqu'une instance est persistante, elle n'appellera pas son événement Create ni Destroy, mais appellera les événements Clean Up, Room End et Room Start. Calque - Le calque auquel l'instance a été affectée. Profondeur - La valeur de profondeur affectée au calque. Sprite - La ressource de sprite affectée à l'instance. Index d'image - L'index d'image du sprite affecté. Cette valeur définit l'image de l'image-objet affectée à l'occurrence, et si la valeur est supérieure au nombre d'images de l'animation, elle "boucle" de telle sorte qu'une image-objet avec 5 images si elle est définie sur l'image 7 affiche réellement l'image 2. Angle d'image - L'angle sur lequel l'instance doit être définie (la valeur par défaut est 0, où droite est 0 °, haut 90 °, gauche 180 ° et bas 270 °). Image Alpha - La valeur alpha de l'occurrence (la valeur par défaut est 1) de 0 à 1, où 0 est totalement transparent et 1 est totalement opaque. Fusion d'image - La couleur à mélanger avec l'image-objet d'instance (la valeur par défaut est le blanc). Vitesse d'image - La vitesse d'animation de l'instance. Échelle horizontale - L'échelle horizontale à laquelle l'image-objet d'instance doit être dessinée (cela affectera également le cadre de délimitation de l'instance et les collisions). Une échelle de 1 indique qu'il n'y a pas de mise à l'échelle (1: 1), les valeurs plus petites seront réduites (0,5, par exemple, la moitié de la largeur du sprite), les valeurs plus grandes augmenteront et les valeurs négatives refléteront le sprite et le redimensionneront à moins que la valeur utilisée est exactement -1 (auquel cas le sprite est simplement mis en miroir le long de l'axe y sans mise à l'échelle). Échelle verticale - Échelle verticale à laquelle l'image-objet d'instance doit être dessinée (cela affectera également le cadre de délimitation de l'instance et les collisions). Une échelle de 1 indique aucune mise à l'échelle (1: 1), des valeurs plus petites seront réduites (0,5, par exemple, la moitié de la hauteur de l'image-objet), des valeurs plus grandes seront mises à l'échelle et des valeurs négatives inverseront l'image-objet et la mettront à l'échelle à moins que le la valeur utilisée est exactement -1 (auquel cas le sprite est simplement retourné le long de l'axe y sans mise à l'échelle). Mask - L'index de masque de l'instance. La valeur par défaut est -1 (ce qui signifie que l'instance utilisera le masque de collision pour l'index de sprite actuellement assigné), mais s'il est défini sur une autre ressource de sprite, le masque de collision de cette instance sera utilisé à la place du sprite assigné au sprite indice. Friction - La friction pour utiliser chaque image de jeu du jeu pour ralentir l'instance. Cette valeur sera simplement déduite de la valeur de vitesse actuelle à chaque image de jeu (ou ajoutée si la valeur de vitesse est négative). Gravité - La gravité appliquera une force constante dans la direction de la gravité (voir ci-dessous) de l'instance, influençant sa vitesse (la valeur par défaut est 0). Notez que la gravité est une force cumulative et accélérera l'objet si vous choisissez de ne pas plafonner la vitesse finale. Direction de la gravité - Cela peut être utilisé pour définir la direction du mouvement lorsque la gravité des instances est supérieure à 0 (la valeur par défaut est 270). Notez que les directions dans GameMaker Studio 2 sont calculées comme 0 ° étant à droite, 90 ° vers le haut, 180 ° vers la gauche et 270 ° vers le bas. Score - Définissez le score variable. Vies - Définissez le lives variable. Santé - Réglez le health variable. Une fois que vous avez sélectionné la variable intégrée à utiliser, vous définissez la valeur de cette variable. La valeur que vous utilisez dépendra de la variable sélectionnée (voir la liste ci-dessus pour les valeurs par défaut et les plages) et vous pouvez marquer la valeur comme étant relative à la valeur actuelle de la variable. Vous pouvez également ajouter des variables supplémentaires à définir dans la même action en cliquant sur l'icône plus à côté de l'action, et en sélectionnant une autre variable d'instance et en lui attribuant une valeur.
