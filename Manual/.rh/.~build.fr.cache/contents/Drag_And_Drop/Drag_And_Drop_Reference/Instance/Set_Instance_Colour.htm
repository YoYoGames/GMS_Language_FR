Définir la couleur de l'instance Ce bloc d'action définit le image_blend couleur pour "mélanger" avec le sprite d'instance. La valeur par défaut est le code hexadécimal $FFFFFFFF (qui représente la couleur blanche en utilisant le format ARVB ) et cela dessinera le sprite sans mélange ni alpha complet. Toute autre valeur (y compris les constantes de couleur internes comme c_red, ou c_aqua ) mélangera la couleur spécifiée avec le sprite original et définira le image_alpha composant pour l'instance, mais uniquement lorsque la case Utiliser Alpha est cochée. Cela écrasera toute valeur alpha définie précédemment à l'aide de l'action Set Instance Alpha. Veuillez noter que pour que les modifications apportées à cette action soient visibles, l'instance doit avoir soit aucun événement de dessin (et donc GameMaker Studio 2 dessine par défaut le sprite) soit être dessinée à l'aide de l'action Draw Self. Il est également important de noter que vous devriez essayer de limiter la fusion sur la plate-forme HTML5 (sauf si vous utilisez WebGL), car chaque sprite mélangé doit être mis en cache séparément et ainsi avoir de nombreux sprites mélangés peut affecter les performances. Syntaxe de l'action: Arguments: Argument Description Colour La nouvelle couleur de fusion à utiliser (cliquer sur l'échantillon de couleur ouvrira la fenêtre du sélecteur de couleur). Exemple: Le code de bloc d'action ci-dessus définit un nouveau sprite ainsi qu'un certain nombre d'autres propriétés pour la façon dont ce sprite doit être affiché, y compris la définition de la couleur de fusion sur le rose.
