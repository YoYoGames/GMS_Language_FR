Définir le compte à rebours de l'alarme Ce bloc d'action définit une alarme donnée sur le compte à rebours, et lorsque l'alarme atteint zéro, l' événement d'alarme associé sera déclenché. Vous fournissez le numéro d'alarme de 0 à 11, puis la valeur sur laquelle définir l'alarme. La valeur utilisée est le nombre de "pas de jeu" que vous souhaitez passer avant que l'événement d'alarme ne soit déclenché, et vous pouvez le définir par rapport à la valeur d'alarme actuelle (en ajoutant ou en soustrayant la valeur actuelle). Notez que la valeur doit être une valeur entière et que vous pouvez la définir sur -1 pour arrêter l'alarme. Notez que lorsqu'une alarme compte à rebours, l'événement est exécuté lorsqu'elle atteint 0, mais l'alarme compte à rebours jusqu'à -1 pour indiquer qu'elle est terminée. Définir une alarme sur 0 ne lancera pas l'événement, car il comptera à rebours jusqu'à -1 étape suivante, donc pour exécuter une alarme immédiatement, elle doit être définie sur 1. Notez que vous pouvez définir des alarmes supplémentaires dans la même action en cliquant sur le bouton icône plus à côté de l'action. Syntaxe de l'action: Arguments: Argument Description Alarm L'alarme à régler (l'événement d'alarme correspondant sera déclenché lorsqu'il atteindra 0). Countdown La valeur du compte à rebours en étapes de jeu. Exemple: Le code de bloc d'action ci-dessus crée une variable locale (temporaire) et stocke la valeur actuelle de l'alarme [0]. Ceci est alors vérifié et s'il est inférieur à 0, l'alarme est mise à 60.
