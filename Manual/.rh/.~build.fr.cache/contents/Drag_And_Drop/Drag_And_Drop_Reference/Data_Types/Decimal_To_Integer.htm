Décimal en entier Cette action prendra un nombre décimal et le convertira en un entier. Vous donnez la valeur à convertir, puis sélectionnez le type d'arrondi à effectuer pour le transformer en une valeur entière. Les types d'arrondis disponibles sont: Round: Cela prend un nombre décimal et arrondit vers le haut ou vers le bas le plus proche nombre entier. Dans le cas particulier où le nombre fourni est exactement un demi-entier (par exemple, 1,5, 17,5, -2,5, etc...) le nombre sera arrondi à la valeur paire la plus proche, par exemple, 2,5 serait arrondi à 2, tandis que 3,5 serait arrondi à 4. Ce type d'arrondi est appelé arrondi des banquiers et sur un grand nombre d'itérations ou lors de l'utilisation de calculs en virgule flottante, il donne un arrondi statistiquement meilleur que le plus traditionnel "arrondi supérieur si supérieur à 0,5 et arrondi inférieur sinon " approcher. Ceil: Cela prendra une valeur décimale et arrondir au nombre entier le plus proche. Contrairement à round (ci-dessus), cela ne prend pas en compte la partie décimale de la valeur, donc un nombre comme 5,0000001 serait toujours arrondi à 6. Il convient de noter que si vous utilisez l'action aléatoire Get Random Number pour générer un nombre aléatoire décimale, puis en appelant cette action pour l'arrondir, vous pouvez alors obtenir des résultats inattendus. L'action aléatoire pourrait potentiellement renvoyer 0, et utiliser "Ceil" sur cela ne donnera pas 1 mais 0, donc si vous obtenez une décimale aléatoire de 0 à 9 et utilisez cette fonction, vous pouvez vous attendre à obtenir des entiers de 1 à 10, mais en En réalité, vous pouvez obtenir des nombres entiers de 0 à 10. C'est une possibilité éloignée mais qui doit être prise en compte lors de l'utilisation de cette action. Etage: Cela prendra une valeur décimale et arrondir vers le bas à la plus proche valeur entière. Contrairement à round (ci-dessus), cela ne prend pas en compte la partie décimale de la valeur, donc un nombre comme 5.9999999 serait toujours réduit à 5. Une fois que vous avez sélectionné le type d'arrondi, vous devez fournir le nombre à arrondir (cela peut être une variable), puis donner une variable cible à laquelle renvoyer la nouvelle valeur entière (qui peut être marquée comme une variable locale temporaire). Syntaxe de l'action: Arguments: Argument Description Rounding Le type d'arrondi à utiliser pour obtenir l'entier (voir description ci-dessus) Number Le nombre décimal à arrondir à un entier Target La variable cible pour stocker l'entier renvoyé Exemple: Le code de bloc d'action ci-dessus prend une valeur décimale et la convertit en entier, en stockant la valeur renvoyée dans une variable locale temporaire.
