Créer un système de particules Cette action va initialiser le système de particules pour que vous puissiez créer et utiliser des types de particules et des émetteurs avec lui. Vous donnez le nom de la couche sous forme de chaîne - par exemple "Particle Layer" (qui doit exister dans l'éditeur de pièce) - puis vous marquez le système de particules comme étant persistant ou non. système comme étant Persistant ou non. Si le système est marqué comme Persistant, il restera actif dans toutes les pièces après sa création, et une couche gérée lui sera attribuée si la couche initiale n'existe pas dans les pièces suivantes. Si la persistance n' est pas marquée, le système sera automatiquement détruit à la fin de la pièce dans laquelle il a été créé, tout comme les émetteurs associés au système. Si vous souhaitez supprimer manuellement le système de la mémoire, vous devez utiliser l'action Destroy Particle System. Vous devez également fournir une variable cible qui contiendra la valeur d'identification unique du système de particules. Cette valeur ID sera utilisée dans d'autres fonctions pour définir les propriétés du système ou pour indiquer à GameMaker Studio 2 quel système utiliser lors de la création de particules. Vous pouvez choisir de marquer cette variable comme étant une variable locale temporaire, mais cela signifie qu'elle sera supprimée à la fin du script d'action ou de l'événement, ce qui signifie que le système existera toujours mais vous laissera sans aucun moyen d'y accéder. (en général, les systèmes n'auront jamais besoin d'être stockés dans une variable locale temporaire, mais pour certaines choses, comme l'attraction de particules sur une surface, cela peut être nécessaire). REMARQUE: une fois créé, un système de particules persistant reste résident dans la mémoire du système pendant toute la durée du jeu. Par conséquent, si vous devez le recréer à un moment donné (par exemple, si vous appelez l'action Redémarrer le jeu ), vous devez d'abord détruire le système actuel, sinon vous obtiendrez une fuite de mémoire. Une fuite de mémoire se produit lorsqu'une ressource est ajoutée à la mémoire, puis "déréférencée", ce qui signifie que GameMaker Studio 2 n'a plus aucun moyen d'interagir avec cette ressource. cette ressource. La mémoire est alors encombrée de ressources inaccessibles, ce qui peut entraîner des problèmes dans votre jeu. Pour détruire un système de particules persistant lorsqu'il n'est plus nécessaire, utilisez l'action Détruire le système de particules. Syntaxe de l'action : Arguments : Arguments Description Couche La couche où les particules de ce système seront dessinées. Persistant Si le système de particules doit rester actif après la fin de la pièce actuelle. Cible La variable cible qui contient la valeur d'identification unique du système créé. Exemple : Le bloc d'action ci-dessus crée une variable globale, puis crée un nouveau système de particules et attribue sa valeur d'identification unique à la variable globale. Le système de particules créé est marqué comme persistant, il restera donc actif même après la fin de la pièce actuelle.
