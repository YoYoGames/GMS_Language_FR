Créer un émetteur de particules Pour être en mesure de voir vos particules, vous devez d'abord créer un système de particules, puis définir certains types de particules et enfin créer un émetteur de particules en utilisant cette action. Pour créer l'émetteur, vous appelez cette action et assignez l'émetteur à un système de particules que vous avez précédemment créé (voir Système de particules), puis vous donnez une variable cible pour stocker sa valeur d'identification unique pour une utilisation dans d'autres actions. valeur d'identification unique pour l'utiliser dans d'autres actions. Vous pouvez choisir de marquer cette variable comme étant une variable locale temporaire, mais cela signifie qu'elle sera supprimée à la fin du script de l'action ou de l'événement, ce qui signifie que l'émetteur existera toujours mais vous laissera sans aucun moyen d'y accéder. Vous devez donc faire attention lorsque vous l'utilisez (en général, les emitters n'auront jamais besoin d'être stockés dans une variable locale temporaire, mais pour certaines choses, comme l'attraction de particules sur une surface, cela peut être nécessaire). IMPORTANT ! Si votre système de particules est persistant, vous devez soit détruire tous les émetteurs à la fin de chaque pièce en utilisant Destroy Particle Emitter et les recréer dans la pièce suivante si nécessaire, soit stocker leurs valeurs d'identification dans des variables globales afin qu'ils soient accessibles à tout moment dans n'importe quelle pièce, soit les faire créer par un objet persistant, afin que l'objet soit persistant et puisse y accéder via des variables d'instance. Notez également que si vous appelez l'action Redémarrer le jeu, vous devez détruire le système de particules au préalable afin que le système et les émetteurs assignés soient supprimés de la mémoire avant de redémarrer. Si vous ne supprimez pas correctement les émetteurs de la mémoire du système comme indiqué ci-dessus entraînera une fuite de mémoire. Cela se produit lorsqu'une ressource est ajoutée à la mémoire, puis "déréférencée", ce qui signifie que GameMaker Studio 2 n'a plus aucun moyen d'interagir avec cette ressource. Cela conduit la mémoire est encombrée par des ressources inaccessibles, ce qui peut entraîner des problèmes dans votre jeu.
