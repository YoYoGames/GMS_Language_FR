Définir la chronologie de l'instance Cette action attribue une chronologie du navigateur d'actifs à l'instance. Vous donnez la chronologie à attribuer, puis vous pouvez choisir de faire en sorte que la chronologie soit en boucle lorsqu'elle atteint le dernier moment (l'image suivante après l'exécution du dernier moment sera de nouveau en boucle au premier moment de la chronologie) ainsi que de savoir s'il faut démarrer la chronologie immédiatement ou non. Si vous ne démarrez pas la chronologie immédiatement, vous pouvez utiliser l'action Définir l'état de la chronologie pour la démarrer ultérieurement dans un autre événement ou script. Syntaxe de l'action: Arguments: Argument Description Timeline La ressource de chronologie à affecter à une instance Loop Indicateur pour avoir la boucle de la chronologie après le dernier moment (la valeur par défaut est marquée) Start Timeline Indicateur pour que la chronologie démarre immédiatement ou non (la valeur par défaut est marquée) Exemple: Le code de bloc d'action ci-dessus attribuera une chronologie à l'instance appelant les actions et définira le moment initial sur 10, puis la vitesse de la chronologie sur 2.
