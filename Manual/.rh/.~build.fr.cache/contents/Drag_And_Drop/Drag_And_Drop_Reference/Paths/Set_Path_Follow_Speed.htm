Vous pouvez utiliser des valeurs négatives pour signifier que l'instance doit suivre le chemin en sens inverse et il est intéressant de noter que la vitesse que vous définissez ici peut être modifiée par le modificateur de vitesse défini dans l'éditeur de chemin pour chaque point du chemin, donc si vous Ainsi, si vous définissez une vitesse de chemin de 2 et que l'un des points du chemin a un modificateur de vitesse de 200%, la vitesse réelle des pixels par image de jeu à ce point sera de 4. Exemple : Le code du bloc d'action ci-dessus démarre une instance qui suit un chemin, puis génère une valeur aléatoire entre 2 et 4. entre 2 et 4 avant d'utiliser cette valeur pour définir la vitesse à laquelle l'instance se déplace le long du chemin.
