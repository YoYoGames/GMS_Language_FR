Chargement du jeu Cette action chargera une partie précédemment enregistrée à l'aide de l'action Enregistrer la partie. Vous donnez le nom de fichier du fichier à charger (sous forme de chaîne, y compris l'extension) et le jeu sera chargé et continuera à fonctionner à partir du point enregistré. Notez que cela n'est pas conçu comme un système universel de sauvegarde / chargement, et essayer de charger un jeu sauvegardé à partir d'une précédente exécution du projet peut donner des erreurs (surtout si vous avez utilisé des éléments tels que des structures de données ou des particules ), et en tant que tel, vous devriez essayez uniquement de charger les jeux qui ont été sauvegardés au cours d'une seule partie (pour des choses comme les points de contrôle). Notez également que le fichier ne sera pas chargé avant la fin de l'événement ou du script en cours, de sorte que toutes les actions après l'appel de l'action de chargement seront toujours exécutées. Pour une approche plus complète du chargement des données de jeu sauvegardées, consultez les Actions sur les fichiers. Syntaxe de l'action: Arguments: Argument Description Timeline La ressource de chronologie à affecter à une instance Exemple: Le code de bloc d'action ci-dessus vérifiera une variable globale et si elle est inférieure ou égale à 0, il chargera une partie précédemment sauvegardée.
