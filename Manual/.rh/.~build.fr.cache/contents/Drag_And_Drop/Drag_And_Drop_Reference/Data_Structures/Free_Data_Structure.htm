Structure de données libre Cette action libérera la structure de données de la mémoire et devrait être appelée chaque fois qu'une structure de données n'est plus nécessaire, par exemple dans le même Destroy d'une instance ou l'événement Room End, etc... La libération d'une structure de données signifie qu'aucune autre action La libération d'une structure de données signifie qu'aucune autre action ne peut être utilisée pour la référencer puisqu'elle n'existe plus, bien que vous puissiez également le vérifier en utilisant l'action If Data Structure Exists. Notez que si la structure de données n'a pas d'index assigné à une variable de portée globale, elle ne sera pas reportée d'une instance à l'autre. Par exemple, si vous créez une instance de lecteur et qu'une structure de données de type liste est affectée à une variable dans cette instance, la liste devra être libérée si le joueur est détruit ou si la room change avec l'instance du joueur (à moins que l'instance du joueur ne soit marquée comme persistante), donc l'instance du joueur aura deux actions Free Data Structure pour ces deux événements. ces deux événements. Il est important que toutes les structures de données créées soient détruites de cette manière lorsqu'elles ne sont plus nécessaires afin d'éviter les fuites de mémoire qui peuvent causer des problèmes de performance ou même faire planter votre jeu. Variable La variable contenant l'index de la structure de données à libérer.
