Bibliothèque d'actions de manette de jeu GameMaker Studio 2 dispose d'un certain nombre d'actions dédiées qui peuvent être utilisées pour détecter à la fois les commandes analogiques et numériques de plusieurs manettes de jeu connectées. Ces actions vous obligent à spécifier une valeur d'index de la manette de jeu, qui est un nombre (à partir de 0) qui représente la manette de jeu sélectionnée. Notez que lorsqu'une manette de jeu est branchée sur votre appareil (ou qu'elle est supprimée), un événement système asynchrone est déclenché (cependant, pour gérer cela, vous devrez peut-être utiliser du code). Les "slots" de manette de jeu sont indexés à partir de 0, les slots 0 - 3 inclus étant uniquement pour les manettes Xinput, c'est-à-dire: les contrôleurs Xbox360 et compatibles. Cependant, vous pouvez également vérifier les emplacements 4 à 11 inclus pour les manettes de jeu DirectInput, ce qui signifie que vous pouvez détecter de nombreux autres modèles de contrôleur lorsque vous êtes connecté via ces emplacements. Lorsque vous travaillez avec les actions de la manette de jeu, les entrées peuvent provenir d' axes, de boutons et / ou de déclencheurs, que GameMaker Studio 2 attribuera à certaines ou à toutes les Notez que sous Windows, les 4 premiers emplacements de manette de jeu (0 - 3) sont gérés à l'aide de la dll XInput, ce qui signifie que seuls les contrôleurs XBox peuvent être 100% compatibles et que pour les autres types de manettes, vous devez vérifier le reste des emplacements de manette de jeu (4 - 11). macOS est pris en charge avec un maximum de 4 appareils connectés autorisés à la fois, et ces appareils peuvent UNIQUEMENT être du type Playstation 3 ou Xbox 360. Veuillez noter que l' option «Créer pour Mac AppStore » dans les options de jeu Mac doit être désactivée pour que le support de coussin fonctionne. Ubuntu prend également en charge l'entrée GamePad, mais vous devrez peut-être installer des bibliothèques supplémentaires à partir du référentiel Ubuntu Vous pouvez le faire facilement en ouvrant un terminal de ligne de commande et en tapant ce qui suit, ce qui installera la prise en charge de l'interface graphique pour le joystick ainsi que le joystick lui-même:sudo apt-get install jstest-gtk sudo apt-get install joystick. Les jeux HTML5 prennent en charge les manettes de jeu sur la plupart des principaux navigateurs, à l'exception de Safari. La prise en charge de la manette de jeu iOS avec l'armoire iCade. L'axe gauche correspond au contrôleur de bâton (bien que l'entrée soit numérique, pas analogique), les quatre boutons «face» correspondent aux boutons avant de l'armoire, et les quatre boutons d'épaule correspondent à ceux à l'arrière de l'armoire. Android exportation Android prend en charge les contrôleurs NYKO et les contrôleurs Bluetooth génériques, mais uniquement lorsqu'ils sont activés, ce qui signifie que vous devrez cocher l'option iCade / Bluetooth dans la section Général des options de jeu Android. Ils nécessitent le niveau d'API 12 pour fonctionner pleinement et il convient de noter que GameMaker Studio 2 enregistrera comme connecté tous les appareils Bluetooth avec lesquels votre appareil est couplé, qu'il soit réellement connecté ou non. Par conséquent, cela doit être pris en compte lors de l'attribution et de la vérification des "slots". Sur PS4, si vous souhaitez utiliser le suivi du pavé tactile, vous devez utiliser les fonctions du device_mouse_* boutons (il n'y a pas d' DnD™ pour cela).
