Si le bouton du Gamepad est pressé Avec cette action, vous pouvez interroger le gamepad pour voir si un bouton est actuellement pressé. L'action renverra vrai à chaque étape où le bouton du gamepad est pressé (et maintenu), ou faux sinon, bien que si vous cliquez sur le modificateur not, vous pouvez alors vérifier si le bouton n'est pas enfoncé, c'est-à-dire que l'action renverra vrai si aucun bouton n'est enfoncé et faux s'il l'est. Si vous n'avez besoin de vérifier qu'une seule pression sur un bouton, alors utilisez l'action if gamepad button down. Si le bouton vérifié est un bouton analogique, la pression ne sera pas détectée tant que la valeur du bouton n'aura pas dépassé le seuil défini à l'aide de l'action Set Gamepad Button Threshold. Notez que pour ajouter des actions dans le bloc "if", elles doivent être déposées sur le côté de l'action, comme le montre l'image ci-dessous : Ces actions seront désormais exécutées si le bloc "if" vaut true, tandis que toute action déposée ailleurs sera exécutée après le bloc "if". Syntaxe de l'action : Arguments : Arguments Description Manette de jeu L'index du gamepad. Bouton Le nom du bouton à vérifier. Non Négation de la vérification (vrai devient faux et vice versa) Exemple : Le code du bloc d'action ci-dessus interroge l'état du bouton du gamepad 0 à chaque étape et si un bouton est maintenu enfoncé, il vérifie l'appui initial sur le bouton. Si la vérification est vraie lors de l'appui initial sur le bouton, un son est joué, puis, tant que le bouton est maintenu enfoncé, la vitesse de l'instance est fixée à 10. Si le bouton n'est pas le bouton n'est pas maintenu enfoncé, une vérification est effectuée au moment du relâchement du bouton pour remettre la vitesse à 0.
