Options de jeu demacOS Cette section présente les différentes options disponibles qui contrôlent la façon dont vos projets de jeux macOS seront compilés. Les différentes sections sont les suivantes : Général Dans la section Informations sur le produit , la première chose à faire est de remplir le nom d'affichage de votre jeu. puis indiquez son numéro de version, son ID d'application, son fichier de sortie d'application et les détails du copyright. Le fichier de sortie de l'application est l'emplacement vers lequel les fichiers d'installation de votre jeu vont sur votre machine de développement Mac. de développement Mac. Lorsque vous créez votre .app.zip, ce fichier peut aller n'importe où (l'emplacement que vous spécifiez lors du dialogue de sauvegarde n'est pas la seule copie faite de ce .app.zip), mais le .pkg et les autres fichiers sont laissés dans ce dossier, ce qui vous permet de tester les installations locales via le Terminal de manière à simuler l'achat dans le Mac Store, etc... Vous devez également indiquer ici l'identifiant d'équipe que vous souhaitez utiliser pour signer l'application finale que GameMaker Studio 2 crée pour vous. Ce paramètre remplacera l'identifiant d'équipe que vous avez fourni dans les préférences demacOS. Après l'identifiant d'équipe, vous avez également la possibilité de fournir l'identifiant de signature, qui est requis par Apple pour toutes les applications ne provenant pas de la boutique (comme les jeux Steam, par exemple). Si vous activez cette option, assurez-vous que le SDK Steamworksest installé (voir ici), et que vous avez ajouté l'ID de l'applicationSteam dans les options générales du jeu. Enfin, vous avez la possibilité de désactiver le sandbox du système de fichiers. Si vous cochez cette option, le bac à sable de GameMaker Studio 2 sera désactivé pour l'enregistrement et le chargement des fichiers, ce qui permettra à vos jeux d'accéder aux fichiers à partir de n'importe quel système exécutant le jeu. Cette option n'est pas cochée par défaut, car nous vous recommandons de maintenir le sandbox de GameMaker Studio 2 pour les fichiers sur tous les systèmes et, lorsque cette option est cochée, certaines zones d'enregistrement/de chargement peuvent ne pas être disponibles en fonction des autorisations du système d'exploitation. Pour plus d'informations sur le bac à sable, consultez les pages consacrées au système de fichiers de GameMaker Studio 2. Graphiques Les options graphiques sont celles que vous devez configurer pour déterminer comment votre jeu utilisera la carte graphique de votre Mac cible. Les options suivantes sont incluses pour que vous puissiez les modifier : Autoriser le menu et le dock en plein écran: Lorsqu'elle est cochée, cette option permet d'afficher le menu et le dock du système d'exploitation si le jeu est en mode plein écran. Cette option est désactivée par défaut. Afficher le curseur: Lorsque cette option est cochée, le curseur normal de macOS sera affiché. Si vous la décochez, aucun curseur ne sera affiché, à moins que vous n'en ayez créé un dans le code de votre jeu. Cette option est désactivée par défaut. Démarrer en plein écran: Si cette option est cochée, le jeu démarrera en mode plein écran, sinon il démarrera en mode fenêtré. Cette option est désactivée par défaut. Autoriser le passage en plein écran: si cette option est cochée, l'utilisateur peut passer du plein écran au mode fenêtré et inversement en utilisant les raccourcis standard de macOS. Cette option est désactivée par défaut. Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques de pixels nets, il est préférable de désactiver cette fonction. mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de l'activer. Cette option est désactivée par défaut. Utiliser la synchronisation pour éviter le déchirement: Ceci permet d'activer ou de désactiver v-sync. Notez que si vous avez un jeu avec une vitesse de 120 et que le joueur a un moniteur avec un taux de rafraîchissement de 60, activer cette option verrouillera votre vitesse de jeu à 60 également. de 60, l'activation de cette option bloquera la vitesse du jeu à 60 également. Cette option est désactivée par défaut. Autoriser le redimensionnement de la fenêtre: En cochant cette case, l'utilisateur peut modifier la taille de la fenêtre du jeu (l'option Fenêtre sans bordure doit être désactivée pour que cela fonctionne). Cette option est désactivée par défaut. Activer la rétine: si vous cochez cette case, le tampon arrière sera à la résolution la plus élevée (réelle) lorsque le jeu est exécuté sur un moniteur compatible avec la rétine, tandis que si vous la décochez, le tampon arrière sera à la résolution apparente (inférieure). Cela signifie que la fenêtre de jeu initiale aura une largeur et une hauteur deux fois supérieures à celles de la pièce/du port de vue de la première pièce. Cependant, la surface de l'application n'est PAS mise à l'échelle en conséquence. Si vous voulez que votre jeu tire le meilleur parti de l'écran rétina, vous devez également définir la surface de l'application de manière à ce qu'elle corresponde à la taille de la fenêtre (notez que cela doublera les pixels sur la largeur et la hauteur, donc n'activez cette option et ne mettez la surface de l'application à l'échelle que si votre jeu bénéficiera de la résolution accrue). de l'augmentation de la résolution). Images La section des images est l'endroit où vous fournissez les images dont votre jeu a besoin. Pour Mac, vous devez fournir un fichier d'icône (qui doit être au format .png et 1024x1024px) et un écran d'accueil (qui peut être au format .png, .bmp, .jpg ou .gif ). L'écran d'accueil s'affichera pendant le chargement du jeu. Pour les paquets DMG, vous pouvez fournir une image d'arrière-plan de l'installateur qui apparaîtra comme arrière-plan de l'installateur DMG. Il convient de noter que GameMaker Studio 2 dispose d'un outil Project Image Generator qui peut être utilisé pour créer automatiquement toutes les images requises pour les différentes plates-formes cibles sur lesquelles votre jeu est compilé. est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu'elles correspondent à vos besoins.
