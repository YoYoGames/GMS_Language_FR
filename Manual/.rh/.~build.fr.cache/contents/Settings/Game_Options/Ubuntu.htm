Options de jeu d'Ubuntu Cette section présente les différentes options disponibles qui contrôlent la façon dont vos projets de jeux Ubuntu(Linux) seront compilés. Les différentes sections sont les suivantes : Général C'est dans cet onglet que vous définissez le nom de votre jeu tel qu'il doit être affiché pendant son exécution. (courte et longue), ainsi qu'une URL pour la page d'accueil du jeu ou du développeur. Ici, vous avez également la possibilité d'activer Vapeur dans votre jeu. Si vous activez cette option, assurez-vous que vous avez installé le SDK Steamworks(voir ici), et que vous avez ajouté l'ID de l'applicationSteam dans les options générales du jeu. Enfin, vous avez la possibilité de désactiver le sandbox du système de fichiers. Si vous cochez cette option, le bac à sable de GameMaker Studio 2 sera désactivé pour l'enregistrement et le chargement des fichiers, ce qui permettra à vos jeux d'accéder aux fichiers à partir de n'importe quel système exécutant le jeu. Cette option n'est pas cochée par défaut, car nous vous recommandons de maintenir le sandbox de GameMaker Studio 2 pour les fichiers sur tous les systèmes et, lorsque cette option est cochée, certaines zones d'enregistrement/de chargement peuvent ne pas être disponibles en fonction des autorisations du système d'exploitation. Pour plus d'informations sur le bac à sable, consultez les pages consacrées au système de fichiers de GameMaker Studio 2. Graphiques Les options graphiques sont celles que vous devez configurer pour déterminer comment votre jeu utilisera la carte graphique de votre machine Ubuntu cible. Les options suivantes sont incluses pour que vous puissiez les modifier : Démarrer en plein écran: Si cette option est cochée, le jeu démarrera en mode plein écran, sinon il démarrera en mode fenêtré. Cette option est désactivée par défaut. Autoriser le passage en plein écran: si cette option est cochée, l'utilisateur peut passer du plein écran au fenêtré et inversement en utilisant les raccourcis standard du système d'exploitation Ubuntu. Cette option est désactivée par défaut. Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques de pixels nets, il est préférable de le désactiver, mais si vous avez de jolis mélanges alpha et des graphiques aux bords lissés, il vaut mieux le laisser activé. mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de l'activer. Afficher le curseur: Lorsque cette option est activée, le curseur par défaut du système d'exploitation sera affiché dans votre jeu. Si cette option est désactivée, lorsque la souris est au-dessus de la fenêtre du jeu, aucun curseur ne sera visible, à moins que vous n'en ayez créé un dans le cadre du projet de jeu (cette option est activée par défaut). du projet de jeu (cette option est activée par défaut). Utiliser la synchronisation pour éviter le déchirement: Ceci permet d'activer ou de désactiver v-sync. Notez que si vous avez un jeu avec une vitesse de 120 et que le joueur a un moniteur avec un taux de rafraîchissement de 60, activer cette option verrouillera votre vitesse de jeu à 60 également. de 60, l'activation de cette option bloquera la vitesse du jeu à 60 également. Cette option est désactivée par défaut. Autoriser le redimensionnement de la fenêtre: En cochant cette case, l'utilisateur peut modifier la taille de la fenêtre du jeu (l'option Fenêtre sans bordure doit être désactivée pour que cela fonctionne). Cette option est désactivée par défaut. Mise à l'échelle: Vous pouvez choisir de conserver le rapport hauteur/largeur (une pièce 4:3 sera donc "letter boxée" sur un écran 16:9) ou de mettre l'image à l'échelle (en l'étirant pour l'adapter au plein écran). Voir La surface d'application pour plus de détails. Enfin, il est possible de définir la taille de la page de texture. La taille par défaut (et la plus compatible) est de 2048x2048, mais vous pouvez choisir entre 256x256 et 8192x8192. Le site est également un bouton marqué Prévisualisation qui génère les pages de texture pour cette plate-forme et ouvre une fenêtre pour que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d'avoir des pages de texture plus grandes. structurées et éviter d'avoir des pages de texture plus grandes (ou plus petites) que nécessaire. Pour plus d'informations sur les pages de texture, voir ici. REMARQUE: plus la taille de la page de texture est grande, moins votre jeu sera compatible avec les PC ayant des spécifications inférieures. Images Vous pouvez ici ajouter un écran de démarrage pour votre jeu et définir l'icône de jeu qui sera utilisée. L'icône doit être une image de 64x64 pixels au format .png. L'écran de démarrage doit avoir au moins la même taille que la première pièce de votre jeu (ou son port d'affichage) et être également au format .png. L'écran d'accueil s'affiche pendant le chargement du jeu et peut être activé ou désactivé en cochant l'option Use Splash Screen (désactivée par défaut). Il convient de noter que GameMaker Studio 2 dispose d'un outil Project Image Generator qui peut être utilisé pour créer automatiquement toutes les images requises pour les différentes plates-formes cibles sur lesquelles votre jeu est compilé. est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu'elles correspondent à vos besoins.
