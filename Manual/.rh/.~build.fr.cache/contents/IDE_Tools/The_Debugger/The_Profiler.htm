Le profileur Le débogueur vous permet de parcourir votre code et de trouver tout problème avec la base de code, mais vous devez parfois vérifier des détails plus subtils afin d'optimiser correctement vos jeux. L' option Profil vous aide à faire exactement cela en vous donnant un aperçu des performances, en indiquant le temps nécessaire pour appeler des fonctions, des scripts et même le temps qu'il a fallu à un événement spécifique pour se terminer. Outre la fenêtre principale affichant tous les différents événements, scripts et fonctions appelés, vous disposez également des options suivantes pour utiliser le Profiler: Activer désactiver C'est le bouton pour activer ou désactiver le Profiler. Lorsqu'il est activé, le profileur démarre et vous commencez à voir les événements, les scripts et les fonctions répertoriés dans la fenêtre principale avec différentes valeurs de données pour chacun d'eux. La façon dont ces données sont affichées dépendra de l' affichage de l'heure et du mode d' affichage (tous deux expliqués plus en détail ci-dessous). Affichage de l'heure Cette case à cocher vous permet de basculer entre le temps moyen et le temps absolu du profileur. Lorsqu'elle n'est pas cochée, les valeurs affichées dans la fenêtre principale seront le nombre total d'appels effectués à la fonction, le script ou l'événement et le temps total pris (les temps sont en microsecondes ) au cours du profil. Cependant, si vous activez cette option, ces colonnes affichent le temps moyen nécessaire par étape pour chaque fonction, script ou événement, et le nombre moyen de fois qu'il a été appelé. Mode d'affichage Ce menu vous permet de choisir le mode d'affichage, qui peut être de haut en bas ou de bas en haut. La sélection du mode de vue de haut en bas affichera le profil de haut en bas (plus ou moins identique à la hiérarchie de la pile d'appels) afin d'afficher l'événement, puis le script et toutes les fonctions utilisées. Notez que vous pouvez double-cliquer sur l'une des entrées pour l'ouvrir dans la fenêtre de débogage Source si vous en avez une. Si vous voyez un icône à côté d'un nom, puis il y a d'autres appels de script ou de fonction dans cette section, et en cliquant sur le élargira l'arborescence pour les afficher. La sélection du mode de vue de bas en haut affichera les choses de bas en haut afin que vous puissiez voir toutes les fonctions et les appels de script individuellement. Lors de l'utilisation de ce mode d'affichage, cliquez sur le développera la fonction ou le script pour montrer ce qui l'a réellement appelé. Afficher la cible La vue cible vous permet de choisir entre l'affichage du GML du projet (données d'événement et de code), les processus du moteur d'arrière-plan ou les deux. Si vous choisissez " GML ", vous obtiendrez les événements, les fonctions et les scripts appelés à chaque étape tandis que la vue "Moteur" affichera uniquement les appels de moteur requis par le projet, vous permettant de voir les goulots d'étranglement dans la manière dont votre projet gère les choses. Si vous sélectionnez "Les deux", les deux ensembles de données seront affichés, mais notez que lorsqu'ils sont combinés avec la vue "Bottom Up", les différents appels de script et de fonction seront imbriqués dans les appels du moteur et vous devrez cliquer sur le bouton pour les développer pour les voir. Les données de profil sont affichées dans quatre colonnes, et en cliquant sur l'une d'elles, les données seront triées en fonction des critères de colonne. Nom: il s'agit du nom de l'événement, du script ou de la fonction appelé, et peut avoir un à côté de lui pour indiquer d'autres appels vers lui ou depuis celui-ci (selon le mode d'affichage que vous avez sélectionné). Vous pouvez double-cliquer sur l'un des noms pour qu'il ouvre le bloc de code ou le script dans la fenêtre Source. Cliquez sur le haut de cette colonne pour trier les noms par ordre alphabétique. Temps: Cela affichera soit le temps total qu'une fonction (ou un script ou un événement) a pris pendant le profilage, soit le temps moyen qu'il a pris à chaque étape pour être appelé (en fonction de l' option d'affichage du temps sélectionnée) et il est affiché dans microsecondes. Appels: Ici, vous pouvez voir soit le nombre total d'appels à l'événement, la fonction ou le script donné, soit le nombre moyen d'appels à celui-ci en une seule étape (encore une fois, en fonction de l' option d'affichage de l'heure sélectionnée).
