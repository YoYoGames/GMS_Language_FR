Montres L'idée même du débogueur est de vous permettre de "regarder" chaque détail de votre jeu. Cela se fait en utilisant des "fenêtres de surveillance" qui, par défaut, sont ouvertes dans la fenêtre Sortie ancrée au bas de l'EDI. Vous pouvez cliquer sur chacun des différents onglets pour voir les montres qu'ils contiennent, dont certaines peuvent être mises à jour en temps réel (lors du débogage en temps réel est activé à partir de la barre d'outils du débogueur) tandis que d'autres nécessiteront que le débogueur ait mis le jeu en pause. Lorsque vous regardez des variables dans l'une des fenêtres, il peut s'agir de l'un des multiples types de données. Cependant, en raison de la façon dont GameMaker Studio 2 stocke certaines choses, le débogueur ne sait pas toujours exactement ce que contient la variable. Cela est dû au fait que les valeurs d'ID internes pour les structures de données ou les instances sont des entiers et que le débogueur ne sait donc pas toujours si la valeur entière stockée dans une variable est pour une structure de données ou une instance ou est simplement une valeur entière. Pour cette raison, vous pouvez faire un clic droit sur n'importe quelle valeur de variable et obtenez une fenêtre contextuelle qui répertorie les types de données disponibles que cette variable pourrait être: Les chaînes et les tableaux ne permettent pas cette option, et il convient de noter que même si une structure de données est stockée dans une variable d'instance, son numéro d'identification est global, donc si vous avez quatre instances et que chacune crée une ds_list (par exemple), chaque liste aura un ID de 0 à 3, et elles ne seront pas toutes 0. Une fois que vous avez sélectionné votre type de données, la variable affichera alors les entrées dans la structure en cliquant simplement sur le petit à côté de son nom. Cela élargira le contenu des structures et vous pourrez également modifier certaines des valeurs des variables affichées en double-cliquant simplement sur la valeur et lui donner un nouveau. REMARQUE: vous ne pouvez pas réattribuer des tableaux ou des structures de données de cette manière, mais vous pouvez modifier une partie de leur contenu (vous ne pouvez pas modifier les valeurs «clés» dans les mappes, ni la valeur de priorité pour les files d'attente prioritaires). Vous trouverez ci-dessous une liste de chacune des différentes fenêtres de surveillance ainsi qu'une explication de leur utilité et de leur utilisation: des locaux La fenêtre Locals affiche les propriétés de l'étape en cours, qui comprend le nom de l'événement (ou du script) en cours d'exécution, le décalage dans le code de la machine virtuelle (VM) pour le script en cours (l'acronyme «PC» signifie «program counter» ), l'instance "self" exécutant le bloc de code et l'instance "autre" de l'événement. Cette dernière partie n'est toujours valide que lorsque l'événement est un événement de collision ou qu'il y a un "with "utilisé dans le code. Si l'instance est en collision," other "affichera l'autre instance (et ses variables) dans la collision, mais s'il s'agit d'un with alors l'instance "Self" devient celle qui exécute le code, tandis que "Other" est alors l'instance qui contient réellement le with. À tous les autres moments, «autre» affichera simplement les mêmes données que «Soi». Notez que les instances "Self" et "Other" peuvent être développées en cliquant sur pour afficher une liste de toutes les variables locales et d'instance actuellement déclarées pour le bloc de code parcouru. Si vous faites un clic droit sur une valeur, vous pouvez définir son type pour voir les détails des structures de données. Globales La fenêtre Globals affichera une liste de toutes les variables globales actuellement déclarées et leurs valeurs. Si vous faites un clic droit sur une valeur, vous pouvez définir son type pour voir les détails des structures de données. Montres Cette fenêtre vous permet d'ajouter une variable spécifique dont vous souhaitez effectuer le suivi. Cela peut être une variable de portée globale ou d' instance , une variable intégrée ou même un tableau ou une structure de données. Si le code d'instance parcouru a la variable surveillée, elle sera affichée ici, ce qui signifie que vous pouvez facilement suivre une valeur sans avoir à la rechercher dans les fenêtres Locals ou Globals. Si vous faites un clic droit sur une valeur, vous pouvez définir son type, la supprimer ou effacer toutes les montres de la fenêtre. Si vous faites un clic droit sur une valeur, vous pouvez définir son type, la supprimer ou effacer toutes les montres de la fenêtre. Toutes les instances Si vous avez besoin de voir toutes les instances qui se trouvent actuellement dans la salle, vous pouvez sélectionner cette fenêtre et elles seront répertoriées. Vous pouvez également voir l'état de toutes les variables intégrées, les variables d' instance ou les structures qu'elles ont, et vous pouvez cliquer avec le bouton droit de la souris sur une valeur pour définir son type. Instance sélectionnée Cette fenêtre est pour lorsque vous sélectionnez une instance de votre jeu. Cela se fait simplement en mettant le jeu en pause, puis en cliquant sur n'importe quelle instance visible dans la fenêtre du jeu. L'instance trouvée sous le curseur de la souris verra alors ses informations affichées dans la fenêtre Instance sélectionnée, où vous pouvez voir ses variables intégrées, ses variables d'instance et ses structures identiques à celles de la fenêtre Instances. États de rendu Cette montre vous montre toutes les informations disponibles sur l'état de l'image en cours de rendu, comme l'alpha de dessin ou la couleur de remplissage. Cela ne peut pas être mis à jour en temps réel, vous devrez donc suspendre le jeu pour mettre à jour et obtenir les détails du cadre. Surfaces / Textures Cette montre vous permet de voir exactement ce qui est attirée sur toutes les surfaces que vous avez dans votre jeu (y compris la surface d'application) ainsi que de voir les pages de texture s qui sont en mémoire. Vous pouvez sélectionner des textures ou des surfaces dans le menu en bas de la montre, puis passer la souris sur la texture ou l'ID de surface pour l'afficher dans une fenêtre contextuelle: Notez que le débogueur doit être mis en pause avant la mise à jour. Profil Cette fenêtre est l'endroit où vous pouvez voir le profil système actuel du jeu testé et c'est là que vous pouvez tout vérifier dans votre jeu, des événements entiers aux appels de fonction individuels pour voir comment ils fonctionnent et repérer les zones problématiques ou les goulots d'étranglement. Pour plus d'informations sur le profilage, consultez la page suivante: Le profileur Amortir Si vous avez initialisé des tampons dans votre jeu, cette fenêtre vous montrera les données du tampon. Vous pouvez sélectionner n'importe quel tampon créé en définissant la valeur de l'ID de tampon, qui commencera à 0 pour le premier tampon créé dans votre jeu, et incrémentera de 1 pour chaque tampon consécutif: Si vous faites un clic droit dans la fenêtre de données ou cliquez sur le bouton Affichage des données, vous pouvez sélectionner le mode d'affichage des données, définir l' alignement et le type, et vous pouvez également définir la disposition des données à partir de la zone de sélection de colonne. Il y a une fenêtre déroulante ici où vous pouvez sélectionner dans une liste par défaut de mises en page.
