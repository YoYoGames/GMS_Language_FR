Travailleurs à distance Lorsque vous compilez votre jeu, vous utilisez généralement le worker "Local", qui est l'ordinateur sur lequel GameMaker Studio 2 a été installé. Cependant, dans certaines circonstances, vous devrez peut-être utiliser un autre ordinateur pour créer vos projets. Par exemple, si vous travaillez sur un Mac et que vous souhaitez créer un Windows, vous voudrez alors configurer un télétravailleur et demander à un Windows faire la construction. Essentiellement, lorsque vous utilisez un travailleur distant, GameMaker Studio 2 enverra tous les fichiers requis de la machine (locale) sur laquelle il s'exécute vers la machine de travail distant, puis utilisera cette machine pour créer l'exécutable final du projet (soit en enregistrant sur la machine distante, ou le renvoyer à la machine locale par la suite). Cependant, avant de pouvoir utiliser cette fonctionnalité, vous devrez avoir installé certains outils sur la machine de construction et effectué une configuration, que nous expliquerons ici. IMPORTANT! Vous pouvez configurer un travailleur distant à l'aide d'un Windows local et d'un Windows distant, ou entre un PC Mac local et un Windows distant, mais actuellement vous ne pouvez pas configurer un Windows (ou Mac) local et un PC Mac distant, c.-à-d.: vous ne pouvez pas avoir de télétravailleur Mac, seulement un Windows. La configuration initiale Pour configurer un travailleur distant, vous devez d'abord installer l' application de l'outil de génération Remote Worker sur le PC à utiliser. Vous pouvez trouver cet outil en accédant au menu Construire dans GameMaker Studio 2 sur la machine locale et en sélectionnant l'option " Afficher le programme d'installation de Remote Worker ": Vous pouvez également cliquer sur l'icône «œil» dans Target Manager et obtenir le programme d'installation de Remote Worker de cette façon: Le programme d'installation s'appellera quelque chose comme "GameMaker Studio-Remote-Installer-XXXXXX.exe "et vous devez le copier sur le PC que vous souhaitez utiliser en tant que travailleur distant (vous n'avez pas besoin d' GameMaker Studio 2 sur le travailleur distant, juste ce programme). Une fois que vous l'avez copié sur le PC de construction, installez-le en suivant les invites à l'écran et lorsque le programme d'installation a terminé, le système d'exploitation peut vous demander d'accéder via le pare-feu à votre réseau. Une fois que vous avez accordé l'accès, le programme d'installation ouvrira alors une page dans votre navigateur Web et vous devez noter l' adresse IP et le numéro de port indiqués sur cette page car vous en aurez besoin pour l'étape suivante. Vous devez maintenant revenir à la machine sur laquelle GameMaker Studio 2 est installé et cliquer sur le bouton Target Manager pour ouvrir la fenêtre Target Manager, puis allez dans la section Worker et cliquez sur le bouton Add Worker. Cela ouvrira alors la fenêtre suivante: Ici, vous devez donner un nom à votre Remote Worker (c'est ce qui sera affiché dans la liste Worker dans le Target Manager) puis dans la section Host Name, ajoutez l'adresse avec le port qui vous a été montré et noté précédemment à partir de la machine ouvrière. Si tout s'est bien passé, vous devriez maintenant recevoir un message indiquant que vous pouvez vous connecter avec succès à l'ordinateur de travail distant. Une fois votre Remote Worker installé et connecté à votre ordinateur IDE principal, vous devrez configurer les paramètres de construction locaux pour chaque plate-forme (voir la section ci-dessous pour plus de détails) avant de pouvoir compiler votre projet. Paramètres à distance La page des paramètres affiche les différents chemins et paramètres de construction pour l'application Remote Worker. En haut des paramètres, vous avez les différents chemins qui seront utilisés par l'application Remote Worker et avant de faire quoi que ce soit, ils doivent être révisés et définis sur des chemins alternatifs si nécessaire (en général, les chemins par défaut devraient bien fonctionner). Vous pouvez accéder à la page des paramètres de Remote Worker à l'aide de l'une des méthodes suivantes: Cliquez sur le lien "Paramètres" en haut de la page Remote Worker qui s'ouvre dans le navigateur de votre machine distante: Utilisez le RMB sur l'icône de la barre d'état de Remote Worker pour ouvrir les paramètres dans le navigateur de votre machine distante: Sur votre ordinateur local, ouvrez le Gestionnaire de cibles et cliquez sur l'icône «rouage» en haut pour ouvrir les paramètres du travailleur dans une fenêtre sur votre ordinateur local: Cela affichera la page des paramètres où vous pouvez voir (et définir) les différentes options qui vous permettent de définir les chemins pour la construction de votre projet: Paramètres de construction locaux Après avoir vérifié les chemins d'accès de l'application Remote Worker à utiliser, vous pouvez ensuite définir (si nécessaire) les différents paramètres de construction locale. Différentes plates-formes nécessiteront l'installation de différents outils de construction sur le PC de construction et le travailleur distant devra être en mesure de les utiliser pour créer vos jeux via la page Paramètres. Les premiers paramètres ici sont les paramètres généraux: Ici, vous pouvez configurer le port pour le serveur d'aide et le port que le débogueur doit utiliser. Ceux-ci peuvent être modifiés si vous rencontrez des problèmes de connexion lors de l'utilisation du module de débogage, mais en général, ils doivent rester à leurs valeurs par défaut. Vous pouvez maintenant continuer et configurer les différentes options pour les plates-formes pour lesquelles le télétravailleur peut créer: Si vous avez accès à la Windows, vous devriez être en mesure de créer immédiatement à l'aide du Windows à l'aide de l'option Machine virtuelle (VM). Cependant, si vous souhaitez utiliser le YoYo (YYC), vous devrez également définir le chemin d'accès à Visual Studio sur le PC de construction. Pour plus de détails sur la configuration d'un Windows pour compiler vos projets, veuillez consulter l'article suivant du service d'assistance: Configuration pour Windows Compiler Une fois que vous avez configuré votre Remote Worker et que vous avez terminé les paramètres de construction locaux (voir la section ci-dessus), vous pouvez commencer à l'utiliser pour compiler vos jeux. Pour cela, il suffit de sélectionner Remote Worker dans la liste Target Manager, de sélectionner la plate-forme à tester / compiler, ainsi que toute autre option (comme le type de compilation ou la configuration), puis d'appuyer sur Lecture ou Créer un exécutable. REMARQUE: les plates-formes disponibles dans Target Manager changeront en fonction des capacités de l'ordinateur Remote Worker et toutes les cibles ne seront pas disponibles. Notez également qu'il n'y a pas de cible «Test» pour les travailleurs distants. Si vous lisez simplement le projet, la machine Remote Worker le compilera et l'exécutera, mais si vous créez un exécutable, alors Remote Worker compilera le projet puis retournera un *.zip fichier sur la machine locale pour enregistrement. Il convient de noter que ni le bouton Déboguer ni le bouton Clean sera disponible lors de l'utilisation d'un télétravailleur. Notez également que vous ne pouvez actuellement effectuer qu'une seule compilation distante à la fois. Une fois que vous avez installé et exécuté Remote Worker sur votre machine distante, il apparaîtra dans la barre des tâches sous forme d'icône, et vous pouvez cliquer dessus avec le bouton droit de la souris pour afficher les options suivantes: Ouvrir: ouvrez l'application Remote Worker dans votre navigateur par défaut. Afficher le journal: ouvrez le fichier journal du télétravailleur dans votre navigateur. Paramètres réseau: ici, vous pouvez filtrer les machines sur lesquelles vous souhaitez pouvoir envoyer des builds à la machine Remote Worker. Chaque filtre peut être un filtre de style CIDR ou un nom d'hôte d'une machine à autoriser. Vous devez être conscient que si vous souhaitez que plusieurs machines se connectent à un travailleur distant, ou si vous mélangez les types de connexion (par exemple: filaire et wifi), vous devrez peut-être modifier la plage IP autorisée ici. Modifier les paramètres: ouvrez le menu des paramètres de Remote Worker (expliqué dans la section Paramètres ci-dessus). Quitter: quittez l'application Remote Worker. Cela signifie que vous ne pourrez plus vous connecter (ni compiler) avec l'application à partir d'un ordinateur local tant que vous n'aurez pas redémarré l'application Remote Worker.
