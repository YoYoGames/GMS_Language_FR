Contrôle de la source Source Control Management (SCM) est le nom donné à la méthode de travail avec des sous-versions ou des sauvegardes de vos projets via un référentiel et une source locale. Fondamentalement, une solution SCM est un progiciel indépendant qui contrôle tous les aspects de la maintenance, de la modification et de la comparaison des versions de votre projet au fur et à mesure que vous y travaillez. Ceci est particulièrement utile pour ceux qui travaillent en équipe et doivent pouvoir contrôler qui fait quoi et ne pas craindre de perdre des données ou d'apporter des modifications qui devront peut-être être annulées ultérieurement, mais les individus peuvent bénéficier de ce puissant mais flexible système aussi. Il existe différents outils SCM à votre disposition, et GameMaker Studio 2 est livré avec un Git inclus qui peut être utilisé immédiatement, et vous n'avez pas besoin d'installer de packages supplémentaires car ils sont tous fournis avec le plugin lui-même. Vous trouverez ci-dessous des liens vers un petit tutoriel sur la configuration de ce plugin et l'utilisation des outils SCM avec un projet. IMPORTANT! Nous vous recommandons d'avoir une certaine connaissance pratique du fonctionnement du contrôle de code source avant d'essayer de l'utiliser dans GameMaker Studio 2. Configuration du plug-in Git contrôle de source Options du référentiel Flux de travail standard Rétablissement de fichiers Cloner un référentiel Conflits Configuration des outils de fusion / comparaison externes Lorsque vous avez activé le contrôle de la source pour un projet (voir la section Configuration du plug Git contrôle de la source ci-dessus pour plus de détails), GameMaker Studio 2 affichera le menu contextuel suivant en haut de l'EDI: Nous décrivons ici chacune des options disponibles (la plupart d'entre elles sont expliquées plus en détail dans les sections du didacticiel ci-dessus): Créer un référentiel de projet: cette option vous permet de créer un référentiel local dans le même répertoire que vos fichiers de projet. Importer un projet dans un référentiel: cette option vous permet d'importer un projet dans un référentiel externe. Cloner le référentiel: Cette option vous permettra de cloner un référentiel d'une source vers une nouvelle destination. Valider les modifications: avec cette option, vous pouvez organiser les fichiers modifiés dans votre projet, puis les valider dans le référentiel. Push Changes: Après avoir effectué une validation ou une fusion, cette option est utilisée pour pousser les modifications vers le référentiel maître. Extraire les modifications: avec cette option, vous pouvez mettre à jour le référentiel local en extrayant les fichiers modifiés du référentiel maître. Afficher l'historique: Cela ouvrira la fenêtre d'historique dans laquelle vous pourrez afficher tout l'historique des versions du projet et choisir de restaurer des chemins de fichiers spécifiques ou des révisions entières. Afficher les conflits: Cela ouvrira la fenêtre Conflits et listera tous les fichiers en conflit qui peuvent exister dans le projet, vous permettant de les traiter via l' GameMaker Studio 2 ou à l'aide d'un outil de fusion spécifique.
