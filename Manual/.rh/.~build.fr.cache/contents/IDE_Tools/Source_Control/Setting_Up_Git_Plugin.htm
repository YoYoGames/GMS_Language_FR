Configuration du plugin GIT Lorsque nous commençons à utiliser le plugin GIT, nous devons configurer une identité pour que GIT puisse commiter, ce qui signifie que nous devons aller dans la section Plugins - Source Control (Git ) des Préférences et ajouter vos détails d'authentification. Cette identité d'authentification sera utilisée pour tous les projets à venir, et GameMaker Studio 2 vous propose deux façons différentes de la configurer : 1. l'installer : Nom d'utilisateur et mot de passe: En cliquant sur le bouton "Ajouterun nouvel utilisateur/une nouvelle authentification de passe", vous obtiendrez la fenêtre suivante où vous pouvez saisir l'URL du référentiel , le nom d'utilisateur et le mot de passe. le mot de passe: Si vous voulez que cela n'affecte qu'un référentiel spécifique, placez l'URL du référentiel dans le champ supérieur. Lorsque quelque chose a besoin de vérifier l'authentification, il cherchera d'abord une correspondance spécifique au référentiel, puis une correspondance au domaine. Cela vous permettra d'avoir une authentification par défaut pour un domaine, et de la remplacer plus tard par des détails spécifiques pour certains référentiels. NOTE : Si vous utilisez un référentiel externe, vous devez utiliser le nom d'utilisateur et le mot de passe associés au compte configuré pour ce référentiel, mais si vous voulez utiliser un référentiel local, vous pouvez utiliser n'importe quel nom d'utilisateur et mot de passe. Paires de clés SSH: En cliquant sur " AddNew SSH Keypair Authentication ", vous obtiendrez la fenêtre suivante où vous pouvez saisir l'URL du référentiel et le mot de passe avant d' indiquer les chemins d'accès aux fichiers de la clé publique et de la clé privée requises : . chemins d'accès aux fichiers de clé publique et de clé privée requis : Comme pour les authentifications par nom d'utilisateur/mot de passe, vous pouvez avoir des authentifications spécifiques au domaine et au référentiel lorsque vous utilisez des paires de clés SSH. Si vous avez déjà généré une paires de clés SSH, vous pouvez configurer les chemins ici, ainsi que tout mot de passe requis pour y accéder. Notez que si vous avez besoin d'un accès rapide à la clé publique, le bouton de copie copiera automatiquement le contenu dans le presse-papiers pour vous. Si vous n'avez pas encore de paire de clés SSH, cliquez sur le bouton Create KeyPair pour faire apparaître une boîte de dialogue de fichier indiquant où placer la clé privée, et la clé publique sera créée à côté - les chemins d'accès aux clés publique et privée seront remplis automatiquement pour vous. Ces paramètres seront maintenant utilisés pour communiquer avec le dépôt (IE : Git Hub, Bit Bucket, etc.) lors de la gestion des demandes de clonage, de poussée et de traction.
