Configurer le plugin GIT Lorsque vous commencez à utiliser le GIT, nous devons configurer une identité pour que GIT s'engage en tant que, ce qui signifie que nous devons aller dans la section Plugins - Source Control ( Git ) des Préférences et ajouter vos détails d'authentification. Cette identité d'authentification sera utilisée pour tous les projets futurs, et GameMaker Studio 2 vous propose deux manières différentes de la configurer: Nom d'utilisateur et mot de passe: Cliquez sur le bouton "Ajouter un nouvel utilisateur / authentification de passe " pour afficher la fenêtre suivante où vous pouvez saisir l' URL du référentiel, le nom d' utilisateur et le mot de passe: Si vous souhaitez que cela affecte uniquement un référentiel spécifique, placez l'URL du référentiel dans le champ supérieur. Cependant, si vous souhaitez une authentification à l'échelle du domaine (par exemple: "bitbucket.org " ou "github.com ") puis placez simplement le domaine et rien d'autre. Lorsque quelque chose doit vérifier l'authentification, il recherchera d'abord une correspondance de référentiel spécifique, puis une correspondance de domaine. Cela vous permettra d'avoir une authentification par défaut pour un domaine et de remplacer avec des détails spécifiques pour certains référentiels ultérieurement. REMARQUE: Si vous utilisez un référentiel externe, vous devez utiliser le nom d'utilisateur et le mot de passe associés au compte configuré pour ce référentiel, mais si vous souhaitez utiliser un référentiel local, vous pouvez utiliser n'importe quel nom d'utilisateur et mot de passe. Paires de clés SSH: Cliquez sur " Ajouter une nouvelle authentification de paire de clés SSH " pour afficher la fenêtre suivante dans laquelle vous pouvez saisir l' URL et le mot de passe du référentiel avant de donner les chemins d'accès aux fichiers de clé publique et de clé privée requis: Comme pour les authentifications par nom d'utilisateur / mot de passe, vous pouvez avoir des authentifications spécifiques au domaine et au référentiel lorsque vous utilisez des paires de clés SSH. Si vous avez déjà généré une paire de clés SSH, vous pouvez définir les chemins ici, ainsi que tout mot de passe requis pour y accéder. Notez que si vous avez besoin d'un accès rapide à la clé publique, le bouton Copier copiera automatiquement le contenu dans le presse-papiers pour vous. Si vous ne possédez pas encore de paire de clés SSH, cliquez sur le bouton Créer une paire de clés pour afficher une boîte de dialogue de fichier indiquant où placer la clé privée, et la clé publique sera créée à côté de celle-ci - cela remplira les chemins vers le clés publiques et privées automatiquement pour vous. Ces paramètres seront désormais utilisés pour communiquer avec le référentiel (IE: Git Hub, Bit Bucket, etc.) lors du traitement des demandes de clonage, push et pull. Cependant, vous devez toujours configurer les options par projet, ce qui peut être fait en ouvrant les Options principales et en cliquant sur l'option Activer le contrôle de la source. Cela activera SCM pour le projet en cours. Une fois que vous avez activé le contrôle de code source (et l'avez appliqué), un nouveau menu contextuel apparaîtra en haut de l'EDI avec l'option suivante (dont chacune est expliquée plus en détail dans le reste des sections de ce didacticiel): REMARQUE: Si vous avez besoin de modifier à nouveau les détails d'authentification, double-cliquez simplement sur l'entrée et la fenêtre de détails s'ouvrira pour que vous puissiez la modifier, mais vous ne pourrez pas renommer l'URL. Si vous devez modifier l'URL, vous devrez supprimer les détails d'authentification et les rajouter. Pour supprimer les détails d'authentification, cliquez sur le bouton de fermeture à gauche de la ligne dans la fenêtre principale des Préférences.
