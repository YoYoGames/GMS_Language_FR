Options du référentiel Après avoir configuré les GIT et activé le contrôle de code source pour votre projet, vous devez maintenant lier l' GameMaker Studio 2 à un référentiel, ce qui se fait de trois manières: Créer un référentiel de projet Pousser le projet actuel dans un référentiel externe Cloner un référentiel existant Dans le premier cas, la création d'un référentiel de projet, cela créera un GIT directement là où se trouve votre projet. Dans le GIT il s'agit d'un " référentiel non nu ", donc Push / Pull n'a aucun sens dans ce cas, mais vous pouvez toujours valider et garder une trace des modifications du projet, ainsi qu'annuler les modifications et les révisions si nécessaire. Pour configurer cela, il vous suffit de cliquer sur Contrôle de la source > Créer un référentiel de projet dans le menu principal. Cependant, vous avez peut-être déjà un projet existant et souhaitez le placer dans un référentiel externe pour le suivi et le partage au sein de votre équipe. Cela nécessite qu'un référentiel externe ait déjà été configuré, ainsi que le nom d'utilisateur et le mot de passe corrects définis dans vos préférences d'identité pour y accéder. Si vous l'avez prêt, cliquez sur Contrôle de la source > Importer le projet dans le référentiel et remplissez l'URL: Enfin, pour des raisons pratiques, vous pouvez cloner un référentiel externe via l'EDI. Encore une fois, vous aurez besoin du nom d'utilisateur et du mot de passe du référentiel auquel vous allez vous connecter dans vos préférences d'identité, puis il vous suffit de cliquer sur Contrôle de la source > Référentiel de clonage. Remplissez l'URL (en utilisant la variante HTTPS plutôt que SSH) et dites où vous voulez qu'elle aille: Une fois le clonage terminé, il ouvrira automatiquement un navigateur de fichiers pour que vous puissiez ouvrir le projet si vous le souhaitez.
