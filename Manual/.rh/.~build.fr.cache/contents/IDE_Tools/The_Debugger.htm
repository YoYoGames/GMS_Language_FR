Le débogueur Le débogueur est un outil puissant pour vérifier votre jeu, et est particulièrement utile pour traquer les erreurs et les bogues dans votre code, ainsi que pour vérifier que les scripts sont exécutés quand ils le doivent, et que les variables et les tableaux contiennent les valeurs attendues, etc. valeurs que vous attendez, etc... Vous pouvez lancer le débogueur en exécutant votre jeu depuis l'IDE GameMaker Studio 2 à l'aide du bouton Debug en haut de l'IDE, ou en allant dans le menu Build et en sélectionnant " Debug". REMARQUE: le pare-feu Windows (ou tout autre programme de pare-feu que vous exécutez) peut vous demander de créer une exception de sécurité pour ce module. Vous devez le faire, sinon il ne fonctionnera pas correctement. Ceci est dû à la façon dont le débogueur fonctionne et est essentiel pour le débogage des plates-formes mobiles. Lorsque vous exécutez un jeu en mode débogage, l'IDE affiche les différents outils disponibles pour le débogueur : L'image ci-dessus montre la disposition par défaut du débogueur, mais comme pour d'autres aspects de l'IDE, vous pouvez la personnaliser en fonction de vos besoins, en fermant les fenêtres dont vous n'avez pas besoin ou en modifiant la taille des documents et des fenêtres dont vous avez besoin. fenêtres dont vous n'avez pas besoin ou en modifiant la taille des documents et des fenêtres dont vous avez besoin. Vous pouvez réinitialiser la disposition du débogueur à tout moment - ou rouvrir les fenêtres fermées - à partir du menu contextuel du débogueur en haut de l'IDE : Nous expliquons ci-dessous à quoi sert chaque section du débogueur : Vue graphique La vue Graphique vous montre une représentation graphique de l'utilisation de la mémoire et du taux de rafraîchissement de votre projet au fil du temps : Le graphique indique l'utilisation actuelle et moyenne de la mémoire ainsi que le nombre de FPS actuel et moyen dans le temps. Vous pouvez effectuer un zoom avant ou arrière à l'aide des boutons / et de la molette de la souris , et si vous placez le curseur de la souris sur l'un des éléments du graphique, la valeur de cet élément s'affiche au point sélectionné. La vue Graphique affiche également certains événements de débogage sous forme de points sur la longueur du graphique. Ces événements sont générés automatiquement par GameMaker Studio 2 à certains moments de l'exécution d'un jeu pour montrer des événements importants spécifiques, mais vous pouvez également ajouter vos propres éléments graphiques à l'aide des fonctions show_debug_message() et debug_event(). Les événements système seront marqués d'une couleur différente de celle des événements déclenchés par le code : Lorsque vous exécutez le jeu en mode débogage et que vous passez la souris sur ces événements, une liste de tous les événements qui ont été déclenchés à ce moment-là dans le jeu s'affiche, Vous avez la possibilité d'ouvrir la liste dans une fenêtre séparée pour la réviser en cliquant sur le texte " Cliquerpour ouvrir dans une nouvelle fenêtre ": Vue des ressources La vue des ressources vous montre une liste de toutes les ressources utilisées dans le projet qui peuvent contenir du code ou des actions DnD™. Vous pouvez ensuite double-cliquer sur sur l'une des ressources pour ouvrir le code correspondant dans la fenêtre Vue du code, où vous pouvez vérifier les valeurs et ajouter ou supprimer des points d'arrêt. Pile d'appels Cette fenêtre affiche la pile d'événements en cours, ainsi que le numéro de ligne du code en cours d'exécution. La pile d'appels est essentiellement une structure de données "dernier entré - premier sorti" qui montre l'ordre d'exécution de tout le code du jeu à la fois à partir du programme d'exécution de GameMaker Studio 2 (puisque le débogage est effectué à l'aide de la machine virtuelle (VM) ) et à partir de ce que vous avez écrit pour le jeu. Le débogueur devra avoir été mis en pause pour que des informations soient affichées ici. Les trois premiers boutons sont utilisés pour contrôler votre jeu, vous permettant de le démarrer, de le mettre en pause, de le redémarrer ou de l'arrêter à tout moment. Lorsque le jeu est en pause (soit en cliquant sur le bouton Pause ou à partir d'un point d'arrêt), vous pouvez alors utiliser les trois boutons d'étape pour parcourir votre code. Lorsque nous parlons de "pas à pas" dans le code, cela signifie simplement que vous pouvez demander à votre jeu d'exécuter la ligne de code en cours, puis de passer à la suivante, ce qui vous permet d'exécuter l'ensemble de votre jeu une ligne de code à la fois si vous le souhaitez. Normalement, vous utilisez simplement le bouton Step In pour faire cela, mais vous avez également un bouton pour passer au-dessus du code et un autre pour sortir du code. L'étape suivante est utile lorsque vous l'ensemble du script ou du bloc de code en une seule étape et s'arrêtera ensuite à la ligne suivante, tandis que l'étape de sortie vous fera sortir d'un script (il s'exécutera, mais en une seule fois) et s'arrêtera ensuite à l'étape suivante lorsque la portée changera. l'étape suivante lorsque la portée change. La dernière icône pour les mises à jour en temps réel permet à la fenêtre de débogage de mettre à jour ses données en temps réel, de sorte que vous pouvez voir comment les variables changent ou les instances sont créées et détruites au fur et à mesure. Notez que cela peut affecter les performances du jeu et que toutes les informations ne sont pas disponibles en temps réel. Par exemple, la montre Render State (voir la section sur les montres) ne peut pas être visualisée en temps réel. Vue du code La vue du code est l'endroit où nous pouvons voir et revoir les différents scripts ou le code d'événement de notre projet pendant son exécution. Par défaut, cette vue est vide, mais vous pouvez double-cliquer sur sur n'importe quelle ressource de la vue des ressources pour l'ouvrir ici. Vous pouvez également ouvrir plusieurs événements ou scripts différents, et ils seront ajoutés à l'espace de travail de la vue Code sous forme d'onglets en haut. Lorsqu'une fenêtre de code est ouverte dans la vue Code, vous pouvez cliquer avec le bouton gauche de la souris sur dans la gouttière (où se trouvent les numéros de ligne) pour ajouter ou supprimer des points d'arrêt (voir plus bas dans cette page pour plus d'informations sur les points d'arrêt). Vous pouvez également utiliser le bouton droit de la souris pour ouvrir le menu contextuel, qui vous permet également d'ajouter ou de supprimer des points d'arrêt et de copier le code sélectionné dans le presse-papiers. code sélectionné dans le presse-papiers. Montres Cette section est consacrée à l'observation de différents aspects de votre jeu afin de permettre un débogage plus approfondi et détaillé. Ce sujet est traité en profondeur à la page suivante : Montres Il est important de noter que lorsque le jeu est en cours d'exécution, il ne met pas à jour les fenêtres de débogage avec les valeurs des variables et autres détails par défaut, vous devez donc d'abord utiliser le bouton Pause dans le débogueur pour mettre le jeu en pause avant que les Locals, Globals et autres données soient disponibles, ou activer la mise à jour en temps réel à partir des icônes en haut . Le site seule exception à cette règle est la vue Profil , qui se met à jour en temps réel lorsqu'elle est activée indépendamment du commutateur temps réel, car elle possède son propre bouton on/off. Vous trouverez des informations complètes sur cette vue à la page suivante : Le Profiler Une autre partie importante du débogage de votre jeu est l'utilisation de points d'arrêt. Un point d'arrêt est un endroit du code de votre projet où vous souhaitez qu'il s'arrête pour que vous puissiez voir ce qui se passe. Lorsque le débogueur fonctionne, vous pouvez ajouter ou supprimer des points d'arrêt à tout moment et dans n'importe quel bloc de code ou de script en cliquant à gauche sur dans la gouttière (où se trouvent les numéros de ligne), ou vous pouvez utiliser le bouton droit de la souris pour pour ouvrir le menu contextuel, qui vous permet également d'ajouter ou de supprimer des points d'arrêt : Les points d'arrêt peuvent également être définis depuis l'IDE GameMaker Studio 2. Lors de l'écriture de votre projet, ils peuvent être ajoutés ou supprimés à tout moment à l'aide de la commande f9 (ou en utilisant le bouton droit de la souris et en sélectionnant " togglebreakpoint "), soit sur une action dans la fenêtre d'objet (pour les utilisateurs de DnD™), soit sur une ligne dans une fenêtre d'éditeur de code. Désormais, lorsque vous exécutez votre jeu en mode débogage, il se met en pause au(x) point(s) que vous avez défini(s). Les points d'arrêt définis à partir de l'IDE sont sauvegardés avec votre projet et persistent au cours des différentes exécutions jusqu'à ce qu'ils soient supprimés. Cependant, si vous ajoutez un point d'arrêt à votre code dans le débogueur, il sera perdu après la fermeture du module ou le lancement d'un nouveau test. Notez que tous les points d'arrêt apparaissent dans la fenêtre de sortie où ils peuvent être activés, désactivés ou même supprimés.
