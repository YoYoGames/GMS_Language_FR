Packages d'actifs locaux Dans le menu Outils, vous avez deux options pour créer et importer des packages d'actifs locaux. Un package d'actifs locaux est un groupe de ressources enregistrées à l'aide de *.yymp format de fichier qui peut être facilement installé dans des projets existants. Cela signifie que vous pouvez créer votre propre bibliothèque de packages pour les ressources couramment utilisées, telles que les bibliothèques de scripts, les intros d'écran de démarrage, les systèmes de menus, etc. sans avoir à utiliser le YoYo Games Marketplace. Créer un package d'actifs Pour créer un package d'actifs, vous devez d'abord avoir créé les ressources que vous souhaitez enregistrer dans un projet, puis dans le menu Outils, sélectionnez l'option Créer un package local. Cela ouvrira la fenêtre suivante: Dans cette fenêtre, vous pouvez fournir un nom d'éditeur facultatif et un nom d'affichage et un ID de package requis. L'ID du package sera utilisé pour générer le nom de fichier avec lequel le package sera enregistré et sera automatiquement ajouté avec le *.yymp extension de fichier lors de la création. Vous pouvez également définir le numéro de version du package et choisir de le signer ou non (ce paramètre est désactivé par défaut). La signature d'un package nécessite que vous ayez créé un certificat de signature (vous pouvez le faire à partir des préférences de Marketplace ), et signifie que tout élément qui a été signé ne pourra pas être importé s'il a été falsifié après sa création. Une fois que vous avez rempli les détails du package, vous pouvez alors continuer et sélectionner les actifs que vous souhaitez inclure dans la section "Projet " sur la droite (en utilisant le bouton gauche de la souris ou une combinaison de / et avec le bouton gauche de la souris). Les ressources sélectionnées peuvent être déplacées vers la vue "Ensemble d'actifs " à l'aide des boutons Ajouter ou Ajouter tout et peuvent être supprimées à nouveau à l'aide des boutons Supprimer et Tout supprimer. Une fois que vous êtes satisfait de la sélection des ressources et que vous avez rempli les détails requis, cliquez sur le bouton OK pour ouvrir un explorateur de fichiers dans lequel vous pouvez choisir l'emplacement où enregistrer le package. Une fois sélectionné, le package sera créé et une fenêtre de confirmation sera affichée, avec une option pour ouvrir l'emplacement du package dans l'explorateur de fichiers du système d'exploitation. Importation d'un package d'actifs Il existe deux façons d'importer un package d'actifs locaux: la première consiste simplement à faire glisser le *.yymp fichier d'une fenêtre d'explorateur sur l'IDE GameMaker Studio 2 la seconde consiste à sélectionner l' option "Importer le package local " dans le menu Outils. Lorsque vous utilisez le menu Outils, une fenêtre d'explorateur s'ouvre et vous pouvez naviguer jusqu'à l'emplacement du package, puis le sélectionner pour l'importation. Dans les deux cas, la fenêtre suivante GameMaker Studio 2: Sur la gauche se trouvent le contenu du package et sur la droite les parties que vous souhaitez importer. Vous pouvez sélectionner une ou plusieurs ressources du package (en utilisant le bouton gauche de la souris ou une combinaison de / et avec le bouton gauche de la souris) puis utilisez le bouton Ajouter pour les ajouter à l' importation, ou utilisez le bouton Ajouter tout pour ajouter tout pour l' importation. Lorsque tout est prêt, vous pouvez cliquer sur le bouton Importer pour ajouter les ressources sélectionnées à votre projet. Notez que vous avez également la possibilité d' importer toutes les ressources dans un nouveau projet, ce qui créera un nouveau projet pour vous (et vous invitera à donner un emplacement pour l'enregistrer), puis y ajoutera les ressources du package. Veuillez noter que les packages d'actifs créés à l'aide d'une version antérieure à GameMaker Studio 2 nécessiteront une conversion car le format de fichier de projet a changé avec la mise à jour 2.3. Cela signifie qu'il peut être préférable d'importer le projet entier et de tout convertir, puis de l'exporter en tant que nouveau package vers un nouvel emplacement si les actifs sont importés fréquemment dans d'autres projets.
