Le micro serveur Web La fonction principale de ces petits outils est de se connecter et de servir des fichiers à certaines des plates-formes cibles disponibles pour GameMaker Studio 2 (comme la cible HTML5, par exemple). Lorsque vous exécutez un jeu sur une cible qui nécessite le Micro Web Server, une nouvelle fenêtre s'ouvre lorsque le jeu est servi sur l'appareil choisi: REMARQUE: la première fois que vous exécutez un jeu sur une cible qui nécessite le Micro Web Server, il est possible que vous receviez une alerte de pare-feu. Vous devez autoriser le serveur Web à communiquer via le pare-feu si cela se produit. Le serveur Web affichera une certaine quantité d'informations de débogage liées aux fichiers qui ont été servis, et dispose d'un bouton pour effacer cela en bas. À côté du bouton "Effacer" se trouve le chemin d'accès au fichier JS racine du projet, tandis qu'en dessous vous avez l'option Vitesse du réseau. Ce paramètre est défini sur "Aucune limite" par défaut, mais en cliquant dessus, vous ouvrirez une liste de différentes options réseau: La modification de ce paramètre modifiera également l' option Taille du paquet vers la droite. Vous pouvez également le modifier manuellement, auquel cas la vitesse du réseau sera réglée sur "Personnalisé". Le paramétrer sur autre chose que la valeur maximale ralentira efficacement la vitesse de transfert du réseau et vous permettra de tester des jeux dans un environnement à faible bande passante. Vous pouvez également voir la liste des adresses IP autorisées ainsi que le numéro de port utilisé, mais ceux-ci ne peuvent pas être modifiés dans le Micro Web Server et doivent être définis à la place dans les paramètres du Gestionnaire de périphériques pour le périphérique cible donné.
