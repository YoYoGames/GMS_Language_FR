Examinons quelques exemples : Modification de l'alpha (transparence) La valeur alpha est ce qui contrôle la transparence de ce qui est dessiné, et dans GameMaker Studio 2, vous pouvez utiliser la variable intégrée image_alpha pour modifier le degré de transparence du sprite attribué. Dans l'événement de création, ajoutez simplement le code DnD™ou GML suivant : var _val = random(1) ; image_alpha = _val ; L'alpha de l'image est calculé comme une valeur comprise entre 0 et 1, où 0 est totalement transparent et 1 est totalement opaque (par défaut, il est défini sur 1). Ainsi, dans cet exemple, tout ce que nous faisons est de définir l'alpha de l'image à une valeur décimale aléatoire de 0 à 1. Par défaut, cette couleur est le blanc, ce qui signifie essentiellement qu'aucune couleur ne sera ajoutée au sprite lorsqu'il sera affiché à l'écran. Cependant, vous pouvez utiliser d'autres couleurs pour obtenir des effets spéciaux, par exemple, utiliser du rouge pour montrer que l'instance a reçu des dommages. des dommages. Dans cet exemple, nous allons mélanger différentes couleurs avec le sprite pendant qu'une touche est pressée et maintenue enfoncée, et vous devrez donc ouvrir (ou créer) un objet, lui attribuer un sprite, puis donner à l'objet un événement Key Down . Dans ce Key Down Event, ajoutez le DnD™ou GML suivant : var _col = choose(c_red, c_green, c_blue, c_yellow, c_fuchsia, c_orange) ; image_blend = _col ; Placez quelques instances de cet objet dans une pièce, puis cliquez sur le bouton Play en haut de l'IDE, et faites un essai en maintenant la touche Espace enfoncée et en la relâchant. Vous devriez voir que chaque instance change rapidement de couleur lorsque la touche est maintenue enfoncée, et cesse de changer lorsqu'elle est relâchée : Changement d'échelle Une autre des propriétés que nous pouvons modifier pour notre sprite est la valeur de l'échelle , ce qui nous permet de le dessiner plus grand ou plus petit quand nous le voulons. L'échelle est calculée indépendamment le long des axes X et Y par deux variables distinctes, les variables image_xscale et la variable image_yscale. Par défaut, elles sont définies sur 1, et agissent comme des multiplicateurs, de sorte qu'une valeur de 0,5 correspond à la moitié de l'échelle et qu'une valeur de 2 correspond au double de l'échelle. IMPORTANT ! Si vous modifiez l'échelle du sprite à l'aide de ces variables , la taille de la boîte englobante sera également modifiée en conséquence, ce qui signifie que la zone de détection des collisions du sprite sera également modifiée. Dans cet exemple, nous allons utiliser quelques mathématiques simples pour faire en sorte qu'une instance mette à l'échelle le sprite de haut en bas dans une boucle. Dans cet événement, ajoutez ce qui suit : timer = 0 ; Maintenant, ajoutez un événement d'étape à l'objet avec ceci : timer = timer + 1 ; var _val = dsin(timer) ; image_xscale = 1 + _val ; image_yscale = 1 + _val ; Ici, nous utilisons la fonction mathématique dsin() pour générer une valeur entre -1 et 1 en utilisant la variable timer, puis nous l'appliquons aux variables d'échelle. Après avoir placé quelques instances dans une pièce et appuyé sur le bouton Play , vous devriez voir comment les instances passent d'une échelle de 0 à une échelle de 2 et inversement. d'une échelle de 0 à une échelle de 2 et inversement. Une dernière chose... changez la section " image_yscale" en " 1 - _val" et voyez ce qui se passe !
