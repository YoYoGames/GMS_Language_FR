ExtensionsAndroid Pour créer une extension pour Android, vous devez le faire en deux parties. La première partie consiste à ajouter l'extension elle-même, avec les fichiers requis, etc... et la seconde consiste à créer les fonctions et les macros/constantes dont l'extension a besoin. Les fonctions et les constantes sont ajoutées en utilisant des fichiers de type placeholder pour les regrouper, donc vous ajoutez un placeholder et définissez ensuite les fonctions et les macros comme expliqué dans la section ci-dessus. Pour ajouter le reste des fichiers, vous devez devez d'abord cocher la Android dans la section Fonctionnalités supplémentaires de l'éditeur : Ici, vous donnez les détails suivants : Nom de la classe: Votre extension peut avoir plusieurs classes, chaque classe ayant ses propres fonctions et constantes. Vous devez donc lui donner un nom qui reflète son objectif. AutorisationsAndroid: Vous pouvez ajouter ici toutes les autorisations supplémentaires dont votre extension a besoin. Ces autorisations dépendent entièrement de l'utilisation de l'extension. Vous devez donc consulter la documentation fournie par Google pour la plate-forme Android ou, si vous utilisez un SDK tiers, la documentation fournie avec le SDK. Pour ajouter une nouvelle autorisation, vous devez cliquer sur le bouton pour ajouter une autorisation fictive, puis cliquer lentement sur pour la modifier en fonction de vos besoins. Vous pouvez supprimer des autorisations à l'aide du bouton . Injecter aux dépendances de Gradle: Ici, vous pouvez ajouter tout code supplémentaire qui doit être injecté (ajouté) dans les dépendances de construction de Gradle. Injection dans le manifeste AndroidManifest.xml: Vous définissez ici tout code supplémentaire à injecter (ajouter) dans le fichier XML Android Manifest lorsque votre jeu est construit pour les tests ou la version finale. Veillez à réviser soigneusement ce fichier (et vos autorisations) avant de soumettre un jeu à la boutique, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Injecter dans l'application AndroidManifest.xml: Vous définissez ici tout code supplémentaire à injecter (ajouter) au fichier XML Android Manifest sous le titre Application lorsque votre jeu est construit pour les tests ou la version finale. Assurez-vous de réviser (et vos autorisations) avec soin avant de soumettre des jeux à la boutique, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Injection dans AndroidManifest.xml RunnerActivity: Vous définissez ici tout code supplémentaire à injecter (ajouter) au fichier XML Android Manifest sous le titre RunnerActivity lorsque votre jeu est construit pour les tests ou la version finale. Veillez à révisez soigneusement ces paramètres (et vos autorisations) avant de soumettre des jeux à la boutique, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Une fois que vous l'avez configuré correctement, vous devez ajouter les fichiers requis pour que votre paquet d'extension fonctionne. Pour ce faire, vous devez cliquer sur les boutons en bas de la page, soit Ajouter SDK, soit Ajouter Source, puis naviguer vers les fichiers que vous souhaitez ajouter. souhaitez ajouter. Les fichiers ajoutés seront stockés dans le répertoire Source de Androidavec votre extension. Vous pouvez ouvrir cet emplacement à tout moment en faisant un clic droit sur sur l'extension et en sélectionnant Ouvrir le répertoire d'extension.
