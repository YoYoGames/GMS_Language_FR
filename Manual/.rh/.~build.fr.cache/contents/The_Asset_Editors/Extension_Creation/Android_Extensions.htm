Extensions Android Pour créer une extension pour Android vous devez le faire en deux parties. La première partie serait d'ajouter l'extension elle-même, ainsi que les fichiers requis, etc... et la seconde consiste à créer les fonctions et les macros / constantes dont l'extension a besoin. Les fonctions et les constantes sont ajoutées à l'aide de fichiers d'espace réservé pour les regrouper, vous devez donc ajouter un espace réservé, puis définir les fonctions et les macros comme expliqué dans la section ci-dessus. Pour ajouter le reste des fichiers, vous devez d'abord cocher la case Android dans la section Fonctionnalités supplémentaires de l'éditeur: Ici, vous donnez les détails suivants: ClassName: votre extension peut avoir plusieurs classes, chaque classe ayant ses propres fonctions et constantes, vous devez donc lui donner un nom qui reflète son objectif. Autorisations Android: ici, vous pouvez ajouter toutes les autorisations supplémentaires requises par votre extension. La nature de ces autorisations dépendra entièrement de l'utilisation de l'extension.Vous devez donc consulter la documentation fournie par Google pour la Android forme Android ou, si vous utilisez un SDK tiers, la documentation fournie avec le SDK. Pour ajouter une nouvelle autorisation, vous devez cliquer sur le bouton pour ajouter une autorisation d'espace réservé, puis effectuez une lente cliquez dessus pour le modifier selon ce qui est requis. Vous pouvez supprimer des autorisations à l'aide du bouton. Injecter dans les dépendances Gradle: Ici, vous pouvez ajouter tout code supplémentaire qui doit être injecté (ajouté) dans les dépendances de construction Gradle. Injecter dans le manifeste Android Manifest.xml: Ici, vous définissez tout code supplémentaire à injecter (ajouté) dans le Android lorsque votre jeu est conçu pour le test ou la version finale. Assurez-vous de réviser soigneusement ceci (et vos autorisations) avant de soumettre des jeux au magasin, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Injecter dans l'application Android Manifest.xml: Ici, vous définissez tout code supplémentaire à injecter (ajouté) dans le Android sous l'en-tête Application lorsque votre jeu est conçu pour le test ou la version finale. Assurez-vous de réviser soigneusement ceci (et vos autorisations) avant de soumettre des jeux au magasin, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Injecter dans Android Manifest.xml RunnerActivity: Ici, vous définissez tout code supplémentaire à injecter (ajouté) dans le Android sous l'en-tête RunnerActivity lorsque votre jeu est conçu pour le test ou la version finale. Assurez-vous de réviser soigneusement ceci (et vos autorisations) avant de soumettre des jeux au magasin, car des paramètres incorrects entraîneront l'échec de la soumission de votre jeu. Une fois que vous avez configuré cela correctement, vous devrez ajouter les fichiers requis pour que votre package d'extension fonctionne. Pour ce faire, vous devez cliquer sur les boutons en bas, soit Ajouter SDK ou Ajouter une source, puis parcourir les fichiers que vous souhaitez ajouter. Les fichiers ajoutés seront stockés dans le Android Répertoire source avec votre extension. Vous pouvez ouvrir cet emplacement à tout moment en faisant un clic droit sur l'extension et en sélectionnant Open Extension Directory. Pour plus d'informations sur la création d' Android, veuillez consulter la base de connaissances YoYo Games.
