Les événements asynchrones Un événement asynchrone est un événement qui est déclenché lorsque GameMaker Studio 2 reçoit un rappel d' une source externe, qui peut provenir du Web ou de l'appareil exécutant votre jeu. En résumé, vous demandez à GameMaker Studio 2 de faire quelque chose, comme charger une image, et il commence à le faire tout en continuant à faire ce qu'il doit faire pendant ce temps. Ensuite, lorsque la requête est terminée, un callback est envoyé à GameMaker Studio 2 et tous les événements asynchrones définis pour ce type de callback seront déclenchés. Veuillez noter que les événements asynchrones sont déclenchés pour toutes les instances qui les possèdent, tout comme les événements clavier, de sorte que vous pouvez, par exemple, effectuer un appel http_get() dans une instance, mais avoir l'événement HTTP asynchrone dans une autre pour gérer le rappel. REMARQUE: Vous devez savoir qu'en raison de la protection XSS des navigateurs, les demandes et les tentatives de chargement de ressources provenant d'autres domaines sont bloquées et peuvent donner lieu à des résultats vides lorsque vous utilisez l'un des événements suivants. Il existe différents types d'événements associés à la catégorie des événements asynchrones, et ils sont tous expliqués dans les sections ci-dessous : NOTE: La variable async_load qui est mentionnée dans les sections suivantes n'est valable que dans ces événements, car la carte DS vers laquelle elle pointe est créée au début de l'événement, puis supprimée à la fin, cette variable étant réinitialisée à la valeur -1 à tout autre moment.
