Les événements Async Un événement asynchrone est un événement qui est déclenché lorsque GameMaker Studio 2 reçoit un rappel d'une source externe, qui peut provenir du Web ou de l'appareil exécutant votre jeu. Fondamentalement, vous dites à GameMaker Studio 2 de faire quelque chose - comme charger une image - et il commencera à le faire mais continuera à faire tout ce qu'il a à faire pendant ce temps. Ensuite, une fois la demande terminée, un rappel sera envoyé à GameMaker Studio 2 et tous les événements asynchrones définis pour ce type de rappel seront déclenchés. Veuillez noter que les événements asynchrones sont déclenchés pour toutes les instances qui les ont, tout comme les événements de clavier, de sorte que vous pouvez - par exemple - faire un http_get() appeler dans une instance, mais avoir l' événement HTTP asynchrone dans une autre pour gérer le rappel. REMARQUE: vous devez savoir qu'en raison de la protection XSS dans les navigateurs, les demandes et les tentatives de chargement de ressources à partir de plusieurs domaines sont bloquées et peuvent sembler renvoyer des résultats vides lors de l'utilisation de l'un des événements suivants. Il existe différents types d'événements associés à la catégorie d'événements asynchrones, et ils sont tous expliqués dans les sections ci-dessous: REMARQUE: la variable async_load qui est mentionnée dans les sections suivantes n'est valide que dans ces événements, car la carte DS vers laquelle elle pointe est créée au début de l'événement, puis supprimée à nouveau à la fin, cette variable étant réinitialisée à un valeur de -1 à tous les autres moments.
