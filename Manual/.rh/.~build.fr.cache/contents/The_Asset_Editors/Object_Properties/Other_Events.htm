Les autres événements Cette page explique chacun des différents événements collectés dans la catégorie Autres événements de l'éditeur d'objets: Salle extérieure Cet événement est déclenché lorsqu'une instance sort de la pièce et est basé sur une vérification effectuée par rapport au sprite affecté (et ses propriétés) de l'instance, de sorte que même si vous avez défini l'échelle x ou y de l'image sur une valeur autre que Premièrement, cet événement ne sera déclenché que lorsque le sprite entier sera hors de l'écran. Si l'instance n'a pas de sprite, alors la position de l'instance est utilisée et au moment où sa position x ou y est en dehors de la pièce, elle déclenchera également l'événement. Cet événement est généralement utilisé pour des choses comme les balles, où elles sont détruites une fois qu'elles quittent la pièce afin que vous ne vous retrouviez pas avec des millions de balles qui s'envolent à l'infini et ralentissent votre jeu. Notez que cet événement n'est déclenché qu'une seule fois lorsque l'instance quitte initialement la salle. Limite d'intersection Cet événement est déclenché lorsqu'une instance "touche" le bord intérieur de la salle et, comme l'événement de la salle extérieure, est basé sur une vérification effectuée par rapport au sprite attribué à l'instance, de sorte que même si vous avez défini l'image x ou y redimensionnée à une valeur différente de un, cet événement est toujours déclenché au moment où une partie de l'image-objet croise la limite de la pièce. Si l'instance n'a pas de sprite, alors la position de l'instance est utilisée et au moment où sa position x ou y croise la limite de la pièce, elle déclenchera également l'événement. Vous pouvez utiliser cet événement pour faire des choses comme dire à une instance de "rebondir" vers la pièce à nouveau plutôt que de sortir. Vues Les vues de caméra sont normalement définies dans l'éditeur de pièce et utilisées pour afficher uniquement une petite zone d'une grande pièce à la fois. Cette catégorie d'événements comporte deux sous-catégories distinctes ( vue extérieure et frontière d'intersection ), avec huit événements différents correspondant chacun aux huit ports d'affichage disponibles. Ces deux catégories fonctionnent exactement de la même manière que les événements de salle respectifs, en prenant uniquement la limite de la vue de la caméra comme la chose par rapport à l'instance à vérifier plutôt que la salle. Début du jeu Cet événement spécial n'est déclenché qu'une seule fois dans tout le jeu et uniquement pour les instances avec des actions ou du code placés dans l'événement. Ces instances doivent être présentes dans la première salle du jeu et y avoir été placées dans l'éditeur de salle, plutôt que d'avoir été créées dynamiquement à l'aide de code ou d'actions. Il convient de noter que cet événement se produit après l'événement create de toutes les instances et peut donc contenir du code ou des actions avec des variables qui ont été précédemment définies dans cet événement. Cet événement est généralement défini dans un seul objet "contrôleur" et est généralement utilisé pour initialiser des variables globales, démarrer la musique, lire des fichiers et toute autre chose qui n'est généralement effectuée qu'une seule fois au début d'une partie. Fin du jeu Semblable à l'événement Game Start mentionné ci-dessus, cet événement n'est déclenché qu'une seule fois dans tout le jeu et c'est juste avant la fermeture de la fenêtre de jeu. Encore une fois, pour une instance avec cet événement pour faire quoi que ce soit, il doit être dans la salle lorsque le jeu est fermé. Veuillez noter que cet événement n'est vraiment viable que pour les Windows, Ubuntu ( Linux ) et macOS, car les appareils mobiles peuvent simplement fermer le jeu sans donner à GameMaker Studio 2 la possibilité d'appeler cet événement. Pour les jeux par navigateur créés avec le module HTML5, cet événement devrait fonctionner lorsque l'onglet de jeu ou le navigateur est fermé, mais tous les navigateurs ne prennent pas en charge cette fonctionnalité (Firefox, Chrome et Safari le font au moment de la rédaction), il faut donc faire attention lors de son utilisation.. En règle générale, vous utiliserez cet événement pour enregistrer des données. Début de la salle Cet événement se produit pour toutes les instances initialement dans une salle lorsque la salle démarre. Veuillez noter que cela se produit après l'événement de création et peut donc dépendre des variables et des éléments définis précédemment dans cet événement. Fin de la chambre Cet événement arrive à toutes les instances qui se trouvent dans la salle lorsque la salle se termine. Cet événement est très utile pour "nettoyer" après la fin d'un niveau, par exemple vous pouvez supprimer les ressources chargées ici, ou supprimer un système de particules pour éviter les fuites de mémoire etc... Fin de l'animation Dans GameMaker Studio 2 sprites ne sont pas des choses statiques car ils peuvent avoir des sous-images et être animés à des vitesses différentes. Chaque image d'une animation (appelée sous-image) a son propre numéro, commençant à 0, qui peut être vérifié dans le code ou via des actions, mais parfois tout ce que vous devez vraiment savoir, c'est quand l'animation est terminée. C'est alors que cet événement est déclenché, juste à la fin de l'animation lorsque l'index de la sous-image montre que la dernière image a été atteinte. Cet événement est vraiment utile pour beaucoup de choses, par exemple un objet d'explosion où vous pouvez définir l'instance pour qu'elle se détruise lorsque la dernière image de l'animation est atteinte. Mise à jour de l'animation Cet événement est conçu uniquement pour être utilisé avec les fonctions d'animation squelettique. Il s'agit d'un événement spécial qui est déclenché à chaque étape d'une instance qui utilise un sprite d'animation squelettique, et est conçu pour "intercepter" les données osseuses une fois que l'orientation des structures a été calculée pour l'état d'animation actuel mais avant que ces données ne soient validées. à utiliser pour dessiner. Cela vous permet de modifier les données osseuses à l'aide des fonctions appropriées. Événement d'animation Cet événement est conçu uniquement pour être utilisé avec les fonctions d'animation squelettique. Cet événement est déclenché dans un objet auquel est assigné un sprite d'animation squelettique, et où l'objet utilise le dessin par défaut pour le sprite affecté, ou le sprite est dessiné via draw_self (). Il ne se déclenche actuellement pas sur les sprites d'animation squelette dessinés d'une autre manière. L'événement est conçu pour capturer et transmettre des données qui ont été affectées à des événements particuliers dans l'animation de sprite comme défini dans l'éditeur utilisé pour les créer. Lorsque l'événement d'animation est déclenché, une carte DS spéciale sera créée, appelée carte event_data (à peu près la même chose que nous avons la carte async_load pour un événement async). Cette carte aura toutes les paires clé / valeur suivantes que vous pouvez analyser pour obtenir des informations sur l'événement: "name ": le nom de l'événement (tel que défini dans le programme d'animation) "track ": l'index de la piste sur laquelle l'animation est en cours de lecture (la valeur par défaut est 0) "integer ": une valeur entière associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est 0) "float ": une valeur flottante associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est 0.0) "string ": une valeur de chaîne associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est une chaîne vide" ") Chemin terminé Les chemins peuvent être une partie importante de la création de n'importe quel jeu, que ce soit pour un jeu de type tower defense, ou un shoot-em-up ou autre, et il est souvent très important qu'une instance fasse une action particulière lorsqu'elle atteint la fin de son chemin. Cet événement détecte cela et est déclenché lorsqu'il se produit, ce qui vous permet d'ajouter du code ou des actions pour gérer les circonstances, par exemple dans un jeu de tower defense, vous l'utiliseriez pour détruire l'instance et supprimer un point de santé de l'objet de but. Evénements utilisateur Il s'agit d'événements spéciaux qui ne sont pas déclenchés GameMaker Studio 2, mais qui doivent être appelés implicitement par vous à l'aide de code ou d'actions pendant que le jeu est en cours d'exécution. Il y a 12 événements utilisateur au total et ils peuvent contenir du code ou des actions et ils peuvent être appelés à tout moment dans la boucle de jeu, ce qui les rend très utiles pour créer vos propres événements qui se produisent lorsque vous le décidez. Message de diffusion Cet événement est déclenché chaque fois qu'un sprite ou une séquence génère une chaîne de message de diffusion.
