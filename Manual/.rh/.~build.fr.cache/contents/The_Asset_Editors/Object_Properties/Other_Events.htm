Fin de l'animation Dans GameMaker Studio 2, les sprites ne sont pas des objets statiques, car ils peuvent avoir des sous-images et être animés à différentes vitesses. Chaque image d'une animation (appelée sous-image) possède son propre numéro, commençant à 0, qui peut être vérifié dans le code ou par des actions, Mais parfois, tout ce dont vous avez vraiment besoin est de savoir quand l'animation est terminée. C'est à ce moment-là que cet événement est déclenché, juste à la fin de l'animation, lorsque l'index de la sous-image indique que la dernière image a été atteinte. Cet événement est vraiment utile pour de nombreuses choses, par exemple un objet d'explosion où vous pouvez configurer l'instance pour qu'elle se détruise elle-même lorsque la dernière image de l'animation est atteinte. Il est conçu pour "intercepter" les données osseuses après que l'orientation des os ait été calculée pour l'état d'animation actuel, mais avant que ces données ne soient utilisées pour le dessin. Cela vous permet d'apporter des modifications aux données des os en utilisant les fonctions appropriées. Cette carte contient toutes les paires clé/valeur suivantes, que vous pouvez analyser pour obtenir des informations sur l'événement : "name" : le nom de l'événement (tel que défini dans le programme d'animation) "track: l'index de la piste sur laquelle l'animation est jouée (0 par défaut).
