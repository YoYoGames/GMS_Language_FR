Variables d'objet Lorsque vous cliquez sur le bouton Variables , la fenêtre Variables d'objet s'ouvre. Vous pouvez alors l'utiliser pour générer un nombre quelconque de variables avant l'événement de création de l'objet : Dans cette fenêtre, vous pouvez cliquer sur le bouton Ajouter pour ajouter une nouvelle variable à la liste. Vous pouvez alors nommer la variable ainsi que définir son type et ajuster ses propriétés: Le nom de la variable doit commencer par une lettre et ne peut contenir que des lettres, des chiffres et le symbole de soulignement '_'. avec une longueur maximale de 64 symboles. Ainsi, les variables valides sont des choses comme fish, foo_bar, num1, et les variables invalides seraient 6fish, foo bar , ou *num. Une fois que vous avez donné le nom de la variable, vous devez définir son type, qui peut être l'un des suivants : Il convient de noter que cette fonctionnalité est particulièrement utile lorsque vous travaillez avec des instances parent/enfant - puisque vous pouvez avoir un objet parent avec un ensemble de variables définies et les modifier simplement dans l'objet enfant (voir la section Variables d'objet et objets enfants ci-dessous pour plus de détails) - et lorsque vous ajoutez des instances dans l'éditeur de salle (voir la section Layers > Instance Layer), puisque vous pouvez également modifier ces variables pour les instances individuelles qui ont été placées dans la salle. individuelles qui ont été placées dans la room. Les différents types de variables que vous pouvez créer sont énumérés ci-dessous : Réel: Un nombre réel est un nombre qui n'est pas un nombre entier et qui peut être positif ou négatif. Ainsi, 124.5, 45639.566546456, 0.9, -45.5 etc... sont tous des exemples de nombres réels. Tous les nombres réels sont stockés sous forme de valeurs à virgule flottante de 64 bits (doubles). Les types de données réelles (et entières) disposent d'une option permettant de définir une plage de valeurs. Les types de données réelles (et entières) disposent d'une option permettant de définir une plage de valeurs à laquelle on accède en cliquant sur le bouton Options :Lorsque vous cochez cette case, vous pouvez alors saisir une valeur de départ et une valeur finale et, au lieu d'avoir une valeur fixe affichée pour la variable dans la fenêtre Variables d'objet, vous aurez un curseur qui sera fixé sur ces valeurs : Entier: Un entier est un nombre entier et peut être positif ou négatif, par exemple 30004, 19, 0, -300. Notez que vous pouvez définir une plage de valeurs pour le nombre entier de sortie, comme expliqué ci-dessus pour les nombres réels. Chaîne de caractères: Une chaîne de caractères est tout ce qui a été placé entre guillemets, par exemple "poisson", "Hello World" ou "12345". Booléen: Un booléen est une valeur qui est soit vraie, soit fausse. Dans la fenêtre Variables de l'éditeur d'objets, il s'agit simplement d'une case que vous cochez pour vrai et décochez pour faux. Expression: Une expression est une phrase mathématique qui peut contenir des nombres ordinaires, des variables, des chaînes de caractères ou des fonctions GML, ainsi qu'un ou plusieurs opérateurs. Par exemple, sqrt(85 * 6) + 5.5 est une expression. Ressource: Une ressource est simplement l'une des ressources que vous avez définies dans l'explorateur de biens. Lorsque vous sélectionnez ce type de ressource, vous pouvez ensuite cliquer sur le bouton Ouvrir l'explorateur d'actifs pour ouvrir l'explorateur d'actifs et sélectionner la ressource dont vous avez besoin. Liste: La sélection d'un type d'entrée de type liste signifie que vous pouvez alors créer une sélection de valeurs (il peut s'agir de chaînes de caractères, de réels, d'expressions, etc...) dont vous pouvez ensuite choisir une ou plusieurs pour la variable à retourner. Pour définir les éléments qui dans la liste, vous devez d'abord les ajouter dans les Options (cliquez sur le bouton ) : Vous pourrez choisir l'élément comme valeur par défaut pour la variable dans la fenêtre principale à partir du menu déroulant : Note z que si vou s avez coché Multi-select dans les Options de la liste, la variable deviendra un tableau de toutes les options sélectionnées : . options sélectionnées : Couleur: le type de couleur vous permet de définir une valeur de couleur à stocker dans la variable. Vous pouvez saisir un nombre réel (de 0 à 16777216, qui aura une valeur alpha de 255), ou une valeur hexadécimale (au format $RRGGBBAA). double-cliquer sur sur la palette de couleurs pour ouvrir le sélecteur de couleurs et y définir la couleur. Variables d'objet et objets enfants Une caractéristique importante des variables d'objet est qu'elles sont héritées par tous les objets enfants que vous avez dans le navigateur d'actifs (voir la section sur les objets parents pour plus d'informations). ce qui signifie que vous pouvez choisir de remplacer ou de modifier tout ou partie de ces variables si vous le souhaitez. Lorsque vous créez un objet enfant d'une autre ressource objet pour laquelle des variables ont été définies, celles-ci apparaissent dans la fenêtre Variables d'objet comme suit : Vous pouvez voir dans l'image du haut que l'objet Parent a quatre Variables d'Objet et dans l'image du bas, celles-ci sont également affichées, mais en "grisé", et l'icône "Inherited From Parent" indique qu'elles ont été héritées. Ces variables héritées peuvent ensuite être modifiées si vous cliquez sur le bouton Override Variable . Vous pouvez alors ajuster le curseur de plage ou les valeurs, ou sélectionner différents éléments dans les listes, etc... Notez que lorsque vous modifiez une variable parent, vous pouvez uniquement changer la valeur définie, mais pas le nom, ni les options de la variable. Notez également que vous pouvez ajouter de nouvelles variables d'objet aux objets enfants également (et dans l'image d'exemple ci-dessus vous pouvez voir que "Character Type" est une nouvelle variable d'objet seulement pour l'objet enfant). Si vous avez modifié une variable d'objet héritée, vous pouvez cliquer sur le bouton Delete pour supprimer la modification, mais pas la variable. La variable sera La variable sera toujours héritée du parent, mais elle utilisera à nouveau la valeur par défaut du parent. Si vous souhaitez supprimer complètement la variable, vous devez le faire à partir de l'objet parent.
