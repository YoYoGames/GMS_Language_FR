Ordre des événements Lors de l'examen des événements dans GameMaker Studio 2, il convient de noter que l'ordre exact dans lequel TOUS les événements vont se produire à chaque étape ne peut pas être clairement indiqué, simplement parce que cela dépend du fonctionnement interne de GameMaker Studio 2 et que cela est sujet à changement. au fur et à mesure que le logiciel se développe. Cependant, certains événements se dérouleront toujours dans le même ordre. Le premier ensemble d'événements qui se produiront toujours de la même manière sont ceux qui se produisent lors de la première entrée dans une pièce. L'ordre dans lequel les différents événements se déclencheront est: Les variables d'objet / variables d'instance sont initialisées (ce sont les variables que vous pouvez définir dans l' EDI de l' éditeur d'objets ou de l' éditeur de pièce ). Si vous avez un objet avec des variables attachées, elles seront créées avant toute autre chose, puis toutes les instances auxquelles des variables sont attachées seront définies par la suite afin que vous puissiez remplacer les variables d'objet par des variables spécifiques à l'instance dans l'éditeur de salle. Code de création de pièce - il s'agit du code unique écrit dans l'éditeur de pièce pour la première entrée dans une pièce Événement de début de salle de toutes les instances - une des «autres» catégories d'événements et se déclenchera pour toutes les instances, persistantes ou non Il convient également de noter que vous pouvez également définir l'ordre dans lequel des instances spécifiques sont créées dans l'éditeur de salle lui-même en les déplaçant vers le haut ou vers le bas dans la liste de la fenêtre Propriétés de la couche d'instance. En dehors de ces événements spécifiques, le seul ordre connu qui se produira toujours de la même manière, peu importe ce qui appartient aux événements en trois étapes et aux différents événements de tirage au sort.
