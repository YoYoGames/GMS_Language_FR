Ordre des événements Lorsque l'on considère les événements dans GameMaker Studio 2, il convient de noter que l'ordre exact dans lequel TOUS les événements vont se produire à chaque étape ne peut pas être clairement établi, tout simplement parce qu'il dépend du fonctionnement interne de GameMaker Studio 2 et cela peut changer au fur et à mesure du développement du logiciel. Cependant, certains événements se dérouleront toujours dans le même ordre. La première série d'événements qui se produiront toujours de la même manière est celle qui se produit lorsqu'on entre dans une pièce pour la première fois. L'ordre dans lequel les différents événements se déclenchent est le suivant : Les variables d'objet / d'instance sont initialisées (ce sont les variables que vous pouvez définir dans l'IDE de l'éditeur d'objet ou de l'éditeur de salle). Si vous avez un objet auquel sont attachées des variables, celles-ci seront créées avant toute autre chose, puis toutes les instances auxquelles sont attachées des variables seront définies après, de sorte que vous pouvez remplacer les variables d'objet par des variables spécifiques à l'instance dans l'éditeur de salle. éditeur. Create Event suivi du Code de Création d'Instance de chaque instance - ainsi, lorsque chaque instance est créée, elle exécutera d'abord son Create Event puis son Code de Création d'Instance avant de passer à l'instance suivante à créer. Événement de début de jeu - cet événement est déclenché une fois dans la toute première salle du jeu pour toutes les instances placées dans la salle à partir de l'éditeur de salle (notez que l'appel de game_restart() déclenchera à nouveau cet événement. Code de création de la pièce - il s'agit du code unique écrit dans l'éditeur de pièce lors de la première entrée dans une pièce. Room Start Event of all instances - l'un des événements de la catégorie "autres" qui se déclenchera pour toutes les instances, persistantes ou non. Il est également intéressant de noter que vous pouvez également définir l'ordre dans lequel les instances spécifiques sont créées dans l'éditeur de salle lui-même en les déplaçant vers le haut ou vers le bas de la liste de la fenêtre Propriétés du calque d'instance. En dehors de ces événements spécifiques, le seul ordre connu qui se produira toujours de la même manière, quoi qu'il arrive, appartient aux événements des trois pas et aux différents événements de tirage . Ceux-ci resteront toujours cohérents, donc si vous avez un code qui dépend d'un timing spécifique à chaque étape de votre jeu, vous devriez utiliser : Début de l'événement d'étape Événement step (notez que l'événement step est exécuté juste avant que les instances ne soient placées dans leurs nouvelles positions) Événement de fin d'étape Tous les événements de dessin sont également toujours traités dans l'ordre suivant (à l'exception de l'événement Window Resize , qui est déclenché différemment) : Événement avant le tirage au sort Événement Draw Begin Événement de tirage au sort Événement de fin de tirage Événement post-tirage Événement de début du dessin de l'interface graphique Événement "Draw GUI Draw GUI End Event Retour : Événements relatifs aux objets Suivant : Autres événements Copyright YoYo Games Ltd. 2021 Tous droits réservés
