Système Cet événement ne peut être déclenché que par un rappel d'un événement au niveau du système (comme la détection d'une manette de jeu ou la connexion automatique à XBox Live) et il renverra une DS Map stockée dans la variable async_load, contenant différentes paires clé / valeur en fonction de l'événement au niveau du système qui a déclenché le rappel. Ces événements sont: Manettes de jeu Lorsque cet événement est déclenché pour une manette de jeu en cours de connexion ou de déconnexion, il renverra l'une des paires clé / valeur suivantes dans le async_load carte: "event_type "- le type d'événement système reçu, qui sera l'une des chaînes suivantes: "gamepad discovered "- se produit lorsque le système signale qu’une nouvelle manette de jeu a été connectée "gamepad lost "- se produit lorsque le système a perdu la connexion à une manette de jeu "pad_index "- l'index du pad qui a été ajouté ou supprimé Cet événement vous permet désormais de déplacer toute la logique de vérification de votre manette de jeu de l'événement Step ou d'un événement d'alarme dans l'événement système et de ne l'exécuter que lorsque cela est réellement nécessaire. IMPORTANT! Cet événement ne sera PAS déclenché à moins que vous n'en ayez au moins un gamepad_* fonction ou action DnD™ dans votre code de jeu. Le coureur initialisera le sous-système de la manette de jeu uniquement lorsque les fonctions sont utilisées dans le projet, donc si elles ne sont pas présentes, l'ajout / la suppression de manettes de jeu ne déclenchera pas l'événement système. Claviers virtuels Lorsque cet événement est déclenché pour un clavier virtuel ouvert ou fermé, il renverra les paires clé / valeur suivantes dans le async_load carte: "event_type "- le type d'événement système reçu, qui sera"virtual keyboard status "pour les claviers virtuels. "screen_height "- la hauteur du clavier virtuel (en pixels). Ce sera 0 si le clavier est invisible. "keyboard_status "- l'état actuel du clavier, renvoyé par l'une des chaînes suivantes: "cache" "caché" "montrant" "visible" Voir ici pour plus d'informations sur le clavier virtuel. Xbox Live L'événement système asynchrone peut être déclenché lors du ciblage de la XBox One à l'aide de l'exportation UWP et en cochant l'option Activer XBox Live dans les options de jeu UWP. Lorsque vous démarrez un projet GameMaker Studio 2 UWP sur lequel Xbox Live est activé, le projet essaie automatiquement de se connecter en mode silencieux à Xbox Live. Les résultats de cette tentative de connexion seront renvoyés sous la forme d'une des paires clé / valeur suivantes dans le async_load carte: "event_type "- le type d'événement système reçu, qui sera l'une des chaînes suivantes: "user signed in "- la connexion de l'utilisateur silencieux a été effectuée avec succès "user sign in failed "- la connexion de l'utilisateur silencieux a échoué (lorsque cela se produit, vous pouvez utiliser la fonction xboxlive_show_account_picker() pour que l'utilisateur choisisse un compte avec lequel se connecter) "user signed out "- l'utilisateur s'est déconnecté D'autres fonctions de XBox Live peuvent également déclencher cet événement, mais les différents rappels de ces fonctions sont détaillés sur les pages de fonctions correspondantes. Pour plus d'informations sur les fonctions spécifiques disponibles pour XBox Live, veuillez cliquer ici. État du système audio Ce type d'événement est spécifiquement destiné à être utilisé lorsque vous travaillez avec la cible HTML5, car il sera déclenché chaque fois que l'état du contexte Web Audio change. Cela signifie que si vous avez, par exemple, un morceau de musique de fond en boucle, vous pouvez le mettre en pause ou l'arrêter et le redémarrer, en fonction du déclenchement de cet événement. Cela peut être vérifié en recherchant la paire clé / valeur suivante dans le async_load carte: "event_type "- le type d'événement système reçu, qui sera la chaîne"audio_system_status "si le système audio s'est initialisé ou si le contexte a changé. Si ce type d'événement est déclenché, il y aura une clé supplémentaire dans le async_load carte: "status "- L'état du système audio, qui sera l'une des deux chaînes suivantes "available "- Le système audio est initialisé et disponible pour lire des sons "unavailable "- Le système audio n'est pas initialisé ou le contexte n'est pas en cours d'exécution et ne peut donc pas lire de sons (toutes les fonctions de lecture de sons renverront -1) Pour plus d'informations sur les fonctions spécifiques disponibles pour l'audio, cliquez ici. REMARQUE: Bien que cet événement soit conçu pour être utilisé avec la cible HTML5, il sera également déclenché sur toutes les autres plates-formes, mais sur tout (sauf HTML5), il ne sera déclenché qu'une seule fois au démarrage du jeu lorsque le moteur audio est initialisé pour la première fois.
