HTTP L'événement HTTP est celui qui est déclenché par le rappel de l'une des fonctions dehttp_*(), comme http_post_string(). Il génère en fait une carte DS qui qui est exclusive à cet événement et qui est stockée dans la variable spéciale async_load (veuillez vous reporter aux fonctions individuelles pour exemples de code qui expliquent l'utilisation de cet événement de manière plus détaillée). Cette carte DS a la structure générale suivante : "id" : L'identifiant qui a été renvoyé par la commande. Si vous envoyez une série de demandes à http_, vous devez savoir à laquelle vous recevez une réponse. pour la comparer à la valeur que vous avez stockée lors de l'envoi initial de la demande afin de trouver la bonne. "status" : Renvoie une valeur inférieure à 0 en cas d'erreur, 0 en cas de succès et 1 si le contenu est en cours de téléchargement. "result" : Les données reçues (chaîne uniquement), ou le chemin d'accès au fichier téléchargé si vous avez utilisé l'option http_get_file(). "url" : L'URL complète que vous avez demandée. "http_status" : Le code d'état http brut (si disponible). Cela renvoie le code d'état standard pour la plupart des navigateurs, par exemple : 304 pour "Not Modified" ou 204 pour "No Content", etc... L'exemple ci-dessus montre ce que vous obtenez lorsque vous utilisez la fonction http_post_string() mais chacune des fonctions de http_ peut renvoyer une carte légèrement différente. Veuillez donc vous reporter à l'entrée du manuel de chaque fonction pour connaître les données précises qui sont renvoyées. NOTE: Comme async_load crée une carte DS, ces fonctions sont particulièrement utiles lorsqu'elles sont couplées avec les fonctions json_encode() et json_decode() et les fonctions Des données supplémentaires peuvent également être fournies par cette carte si vous avez demandé des fichiers à télécharger. Dans ce cas, le "statut" aura la valeur 1 et la carte DS contiendra ces clés supplémentaires : "contentLength" : Il s'agit de la taille du fichier que vous devez vous attendre à recevoir selon le serveur web (peut être -1 si le serveur ne renvoie pas ces données). "sizeDownloaded" : La taille des données qui ont déjà été téléchargées. Notez que l'événement ne sera pas déclenché pour chaque paquet de données reçu, mais sera plutôt mis à jour à tout moment pendant le téléchargement dans la boucle principale du jeu. Notez également que cette fonctionnalité n'est actuellement disponible que pour les plateformes cibles Windows ordinaires.
