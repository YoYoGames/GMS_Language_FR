L'éditeur de script Cette section traite de l'éditeur de script (également appelé éditeur de texte), dans lequel vous saisissez tout le code dont votre jeu a besoin pour fonctionner. votre jeu aura besoin pour fonctionner. Cet éditeur est utilisé pour créer des actifs de script personnalisés, ainsi que pour coder les événements d'objets, pour ajouter du code de création de salle, et pour beaucoup d'autres choses, et en tant que tel, cette section détaille simplement le fonctionnement de l'éditeur de script, et non pas comment créer des actifs de script ni comment utiliser le Drag and Drop™pour créer des scripts. Lorsque vous vous serez familiarisé avec GameMaker Studio 2 et que vous souhaiterez l'utiliser au maximum, il est conseillé de commencer à apprendre à utiliser ce langage, car il élargit considérablement vos possibilités lors de la création de jeux. Un actif de script peut être nommé en cliquant avec le bouton droit de la souris sur dans le navigateur d'actifs et en sélectionnant Renommer, mais notez que le nom du script doit se conformer aux règles de script pour les fonctions, c'est-à-dire qu'il ne peut être qu'alphanumérique, doit commencer par une lettre et le seul symbole qu'il peut contenir est le symbole "_" sous la barre. L'éditeur de script dans d'autres endroits ne peut pas être renommé et aura un nom spécifique à ce qui est édité, comme un événement d'objet par exemple. Lorsque vous ouvrez la fenêtre de l'éditeur de script, elle présente les options et la disposition suivantes : Onglet Script / Événement L'éditeur de script s'ouvre dans une fenêtre avec des onglets en haut pour vous permettre d'avoir plusieurs scripts ou événements dans une seule fenêtre (bien que ce comportement puisse être modifié à partir des préférences pour donner une nouvelle fenêtre à chaque script ou événement). Vous pouvez cliquer sur sur un onglet et le faire glisser pour changer l'ordre des onglets, ou - si vous préférez - vous pouvez le sortir de la fenêtre actuelle et le placer dans une nouvelle fenêtre. Si vous préférez, vous pouvez le sortir de la fenêtre actuelle et le placer sur l'espace de travail pour créer une nouvelle fenêtre pour ce script (ou l'ajouter à une autre fenêtre). Si vous faites glisser un onglet de script hors de la fenêtre de l IDE, un nouvel IDE sera créé pour contenir cette ressource de script, et il peut être utilisé comme vous le feriez avec la fenêtre principale. Notez que si vous modifiez le code d'un événement d'objet dans l'éditeur de script et que vous avez maximisé l'éditeur de script ou que vous l'avez placé dans une fenêtre ou un espace de travail séparé, le menu de l'éditeur accessible par un clic droit comportera quelques options supplémentaires : Restaurer l'espace de travail: Ceci fera sortir l'éditeur de script d'un état maximisé/espace de travail et le relier à l'objet dans l'espace de travail. Allez à l'objet: Cela vous amènera à l'espace de travail où se trouve l'objet contenant le code et vous permettra de vous concentrer sur cet objet. Ajouter /Ouvrir un événement: Ceci vous permet d'ajouter un nouvel événement à l'objet auquel le script actuel appartient, et ouvrira un nouvel onglet de code dans l'éditeur de script pour l'événement ajouté. Plus important encore, si vous faites une faute de frappe ou si vous construisez le code de manière incorrecte, l'IDE GameMaker Studio 2 vous en informera en signalant la ligne de code par un point d'exclamation rouge indiquant une erreur de syntaxe. Vous pouvez modifier la façon dont l'opération de recherche est effectuée en basculant les boutons suivants : Lorsque vous activez cette option, la fonction de recherche ne met en évidence que les chaînes de caractères qui correspondent à l'ensemble de la chaîne de caractères saisie. Par exemple, si cette option n'est pas activée, une recherche de "random " affichera tous les mots qui contiennent cette chaîne - comme irandom(), ou randomise() - alors que si vous l'activez, vous n'obtiendrez que la fonction random(). Lorsque cette option est activée, vous demandez à GameMaker Studio 2 de vérifier non seulement le contenu de la chaîne de recherche, mais aussi la casse. Par exemple, si vous avez un sprite appelé "spr_Dog" et que vous faites une recherche pour "chien" si cette option est désactivée, la chaîne de caractères du sprite sera mise en évidence, mais si l'option est activée, elle ne le sera pas puisque "chien" n'est plus considéré comme identique à "chien". Notez que le terme "suivant" est considéré comme le suivant après la position actuelle du curseur, et que vous pouvez passer à des termes différents en utilisant les flèches en haut à droite de la fenêtre de recherche. En cliquant sur cette option, tous les exemples de la chaîne de recherche dans le script seront remplacés par la chaîne de remplacement indiquée. Notez que si vous souhaitez effectuer une recherche globale (c'est-à-dire une recherche dans l'ensemble du projet plutôt que dans le script ou l'événement en cours), vous pouvez appuyer sur / + + "F".
