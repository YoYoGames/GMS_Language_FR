async_load Cette variable a une portée globale et est utilisée pour contenir une carte DS lorsqu'elle est utilisée dans les événements asynchrones, et -1 à tous les autres moments. Le contenu réel de la carte DS dépendra du type de rappel de l'événement asynchrone, ainsi que de la fonction utilisée pour déclencher l'événement. les pages individuelles de ces événements pour obtenir des détails complets sur tous les contenus possibles de la carte DS. Syntaxe : async_load ; Les retours : Réel (valeur entière) Exemple étendu : sprite = sprite_add("site.com/path/image.png", 16, true, true, 0, 0) ; Le code ci-dessus serait appelé dans un événement pour charger un sprite à partir d'une URL externe. Cela déclencherait alors l'événement asynchrone Image Loaded, où vous analyseriez la carte async_load: if ds_map_find_value(async_load, "id") == sprite { if (ds_map_find_value(async_load, "status") >= 0) { sprite_index = sprite ; } } Le code ci-dessus va d'abord vérifier l'ID de la carte async_load, puis le statut de la callback. Si la valeur est supérieure ou égale à 0 (signalant un succès), le résultat de la callback sera alors utilisé pour définir l'index du sprite de l'instance à l'image nouvellement chargée.
