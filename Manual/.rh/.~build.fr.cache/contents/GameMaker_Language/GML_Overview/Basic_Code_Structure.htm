Structure de base du code Le code est écrit en blocs et un bloc de code typique consiste en un ensemble d'instructions, appeléesinstructions, qui sont ensuite interprétées par GameMaker Studio 2 et utilisés pour faire en sorte que quelque chose se produise dans votre jeu. Ce "quelque chose" peut être aussi simple que d'ajouter 2 et 2 pour obtenir 4, ou aussi complexe que de faire fuir un ennemi lorsque sa santé passe en dessous d'une certaine valeur. La structure réelle du programme peut varier considérablement, en fonction des fonctions qu'il utilise, mais pour l'essentiel, elle ressemble à ceci : ; ; ... Les déclarations doivent être séparées par un symbole " ;" afin d'éviter les erreurs avec les déclarations de variables et de garder votre code propre et ordonné, et elles peuvent être constituées de déclarations de variables, desexpressions et des appels à des fonctions spécifiques. Vous pouvez également "regrouper" des instructions sous forme de bloc à l'aide des accolades {} afin qu'elles se suivent, comme dans l'exemple suivant exemple conditionnel: if () { ; ; ... } REMARQUE: le langage GameMaker accepte également le début et la fin à la place des accolades {}, bien que ce ne soit pas la façon la plus courante de procéder : if () begin ; ; ... end Voici une représentation plus visuelle de l'aspect d'un bloc de code, cette fois créé sous forme de script dans l'éditeur de script de GameMaker Studio 2: Il existe un certain nombre de types d'instructions, d'expressions, de conditionnels et de fonctions, qui sont tous abordés en détail dans les sections suivantes du manuel.
