exit Le exit l'instruction a la syntaxe suivante:exit;exit termine simplement l'exécution de la fonction de script courante,method, ouevent. Notez qu'il y a une légère différence d'utilisation ici en fonction de la portée: si tu utilises exit dans une fonction ou une méthode de script, il quittera simplement la fonction et retournera au code qui a appelé la fonction si tu utilises exit dans un bloc de code à partir d'un objet, il quittera l'événement entier même s'il y a plusieurs blocs de code séparés après exit a été appelé. Lorsqu'il est utilisé dans un événement, exit est généralement utilisé pour éviter qu'une instance exécute du code supplémentaire lorsqu'une condition spécifique a été remplie (ou non). Le code ci-dessous présente un exemple de la façon dont il serait utilisé, dans ce cas dans un événement de collision, bien qu'il puisse être utilisé dans tous les cas.if (!visible)     {     exit;     } while (place_meeting(x, y))     {     x -= lengthdir_x(1, direction - 180);     y -= lengthdir_y(1, direction - 180);     } Le code ci-dessus vérifie une variable et si elle se résout en true, puis il quitte le bloc de code, sinon il continue et exécute le reste du code. REMARQUE: cela ne met pas fin à l'exécution du jeu. Pour cela, vous devez utiliser la fonction game_end ().
