xboxlive_stats_get_social_leaderboard Cette fonction peut être utilisée pour récupérer un classement social des classements pour une statistique donnée. Vous fournissez l'ID utilisateur (tel que renvoyé, par exemple, par la fonction xboxlive_get_user() ), la chaîne de statistiques (telle que définie lorsque vous l'avez créée à l'aide des fonctions xboxlive_stats_set_stat_ *), puis vous spécifiez un certain nombre de détails sur les informations de classement que vous souhaitez récupérer. Notez que vous ne pouvez récupérer un classement social que pour les statistiques int ou réelles, mais pas pour les statistiques de chaîne, et que si vous marquez l'argument "favourites_only" comme vrai, les résultats ne contiendront que des données pour les amis marqués par l'utilisateur. comme "favoris". IMPORTANT! Les statistiques utilisées dans les classements sociaux n'ont pas besoin d'être enregistrées en tant que "Statistiques en vedette" dans le XDP / Windows Dev Center. La fonction générera un rappel qui déclenchera un événement asynchrone du système. Cet événement aura la carte spéciale DS async_load qui doit ensuite être analysé pour les clés suivantes: " id " - Tiendra la constante achievement_stat_event " event " - Contiendra la chaîne "GetLeaderboardComplete" " userid " - L'ID utilisateur associé à la demande " error " - 0 en cas de succès, une autre valeur en cas d'erreur. Voici l'erreur la plus courante renvoyée: -2145844848: Mauvaise requête (la statistique n'est pas un type de classement valide, c'est-à-dire: une chaîne) " message d'erreur " - Une chaîne avec un message d'erreur, le cas échéant " display_name " - L'ID unique du classement tel que défini sur le tableau de bord du fournisseur. " numentries " - Le nombre d'entrées dans le classement que vous avez reçues. Le reste de la carte DS contiendra également les données du classement au format suivant (où "N" est la position dans les données du classement, de 0 à "numentries"): " Player N " - Le nom du joueur, où "N" est une valeur entière correspondant à sa position dans la liste des entrées du classement. " Playerid N " - L'identifiant unique du joueur, "N". « Rank N » - Le rang du joueur «N» dans le classement. " Score N " - Le score du joueur "N". Syntaxe:xboxlive_stats_get_social_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, favourites_only); Argument Description user_id L'ID utilisateur de l'utilisateur pour lequel obtenir le classement stat La statistique (sous forme de chaîne) pour créer le classement mondial à partir de num_entries Le nombre d'entrées du classement mondial à récupérer start_rank Le rang dans le classement à partir duquel commencer (utilisez 0 si l'argument "start_at_user" est défini sur true ) start_at_user Mis à true pour commencer au rang de l'ID utilisateur, false sinon (défini sur false si l'argument "start_rank" est autre que 0) favourites_only Mis à true pour afficher uniquement les amis marqués comme "favoris" ou false autrement Retour:N/A Exemple étendu: Voici un exemple détaillé de la façon dont cette fonction peut être utilisée.
