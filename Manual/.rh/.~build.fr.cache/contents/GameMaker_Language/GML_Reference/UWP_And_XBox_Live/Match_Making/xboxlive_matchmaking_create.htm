xboxlive_matchmaking_create Cette fonction va créer une session multi-joueurs à l'aide du modèle de session, de la trémie de matchmaking et du modèle d'association de dispositifs sécurisés, que vous devriez avoir créés au préalable dans le tableau de bord XDP. L'identifiant de l'utilisateur que vous pouvez récupérer pour une tablette de jeu particulière pad en appelant la fonction xboxlive_user_for_pad() et la visibilité sera l'une des constantes suivantes : Constant Description xboxlive_match_visibilité_ouverte spécifie que la session peut être rejointe par les autres. xboxlive_match_visibilité_privée spécifie que la session n'est pas visible pour les utilisateurs qui ne sont pas membres de la session. xboxlive_match_visibility_usetemplate spécifie que la valeur utilisée dans le modèle de session doit être utilisée (ceci devrait normalement être utilisé car vous ne pouvez pas remplacer la valeur d'un modèle si une valeur a été définie dans XDP). La fonction renvoie une valeur unique d'identification de la demande, qui peut ensuite être utilisée pour identifier l'événement asynchrone social correct pour cette fonction. Cet événement sera déclenché lorsque la session aura été créée et contiendra un DS Map dans la variable async_load avec les paires clé/valeur suivantes : "requestid" - contient l'ID de la demande tel que renvoyé par la fonction appelante. "status" - contient la chaîne "session_created" pour vous informer du type d'événement qui a été déclenché. "sessionid" - contient l'ID de la session, ou -1 en cas d'échec "error" - contient 0 en cas de succès, ou -1 en cas d'échec "correlationid" - le handle de la session pour la corrélation avec la fonction xboxlive_matchmaking_join_session() Syntaxe : xboxlive_matchmaking_create(user_id, visibility, template, hopper, sdatemplate, [matchattributes]) ; Arguments Description nom d'utilisateur Le pointeur d'identification de l'utilisateur à utiliser visibilité Une des constantes énumérées ci-dessus modèle Le nom du modèle de session Trémie Le nom de la trémie d'allumage sdatemplate Le nom du modèle d'association de dispositifs sécurisés matchattributes [OPTIONNEL] Les retours : ID de la demande (nombre entier) Exemple : var userID = xboxlive_user_for_pad(global.PadIndex) ; result = xboxlive_matchmaking_create(userID, xboxlive_match_visibility_usetemplate, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic") ; L'exemple ci-dessus récupère l'ID de l'utilisateur sur la manette donnée et initialise ensuite une session de matchmaking pour lui.
