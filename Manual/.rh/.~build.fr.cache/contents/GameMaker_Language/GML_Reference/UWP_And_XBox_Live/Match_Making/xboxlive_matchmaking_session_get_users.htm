xboxlive_matchmaking_session_get_users Cette fonction va créer et remplir une carte DS avec les détails des utilisateurs de la session spécifiée, ou -1 s'il y a une erreur. Vous pouvez obtenir le pointeur d'identification de la session à partir de la carte DS async_load DS map qui est renvoyée dans l'événement social asynchrone lorsque vous avez créé ou trouvé une session (cf. xboxlive_matchmaking_create() pour plus de détails). La carte DS retournée contient les paires clé/valeur suivantes : " num_results" - contient le nombre d'utilisateurs dans la session "userId "- contient l'ID de l'utilisateur (< index> est une valeurde 0 à "num_results" - 1) " userIsOwner "- contient 1 si l'utilisateur est l'hôte, sinon 0 (< index> est une valeurde 0 à "num_results" - 1). Syntaxe : xboxlive_matchmaking_session_get_users(session_id) ; Arguments Description identifiant de la session Le pointeur d'ID de session à utiliser Les retours : ID de la demande (nombre entier) Exemple : var session_map = xboxlive_matchmaking_session_get_users(global.SessionID) ; if (session_map != -1) { for (var i = 0 ; i < session_map[ ? "num_results"] ; i++) { if (session_map[ ? "userIsOwner" + string(i)] == 1) { global.OwnerID = session_map[ ? "userId" + string(i)] ; } } } L'exemple ci-dessus récupère les données utilisateur de tous les utilisateurs participant à la session de jumelage et définit une variable globale avec l'ID du propriétaire de la session.
