steam_get_quota_total Lorsque vous utilisez le Steam Cloud pour stocker et synchroniser des fichiers, vous devez définir le quota d'espace dont votre jeu aura besoin. Ce quota est appliqué à chaque jeu compatible avec le nuage, par utilisateur et par jeu. Ainsi, par exemple, si le quota pour le jeu X est de 1 mégaoctet, chaque compte Steam propriétaire du jeu X peut stocker, au maximum, 1 mégaoctet de données associées à ce jeu dans le nuage. Les données stockées par le jeu X n'affecteront pas les autres jeux compatibles avec le cloud que l'utilisateur possède (par exemple, le jeu Y). Le quota par défaut pour les nouveaux jeux Steamworksest d'un gigaoctet, mais vous pouvez le modifier à partir du panneau de configuration Steamworksde votre jeu. NOTE: Une fois le quota épuisé , l'écriture de fichiers échouera. Si vous pensez qu'il est possible que le quota soit épuisé pour l'utilisateur de votre jeu, vous devez créer un code pour le gérer, car en ne faisant rien vous laissez les utilisateurs dans une situation où en ne faisant rien, vous laissez les utilisateurs dans une situation où ils sont incapables de réparer les choses, ce qui entraînera une mauvaise expérience de votre jeu. Syntaxe : steam_get_quota_total() ; Les retours : Real Exemple : if steam_is_cloud_enabled_for_app() { quota = steam_get_quota_total() ; } Le code ci-dessus vérifie si le nuage de vapeur est activé pour le jeu et si oui, il obtient la taille du quota de stockage et la stocke dans une variable.
