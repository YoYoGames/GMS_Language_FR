steam_ugc_subscribe_item Cette fonction peut être utilisée pour s'abonner à un élément UGC. Vous fournissez l'ID du fichier publié, puis la fonction renvoie une valeur d'ID asynchrone unique qui peut être utilisée pour vérifier si l'élément souscrit a été téléchargé dans l'événement asynchroneSteam. Cet événement contiendra la async_load DS map qui sera remplie avec les paires clé/valeur suivantes : "id" - L'ID asynchrone renvoyé par la fonction qui a déclenché l'événement. "resultat" - Le résultat de l'opération (une valeur réelle). Ce sera soit la constante GML ugc_result_success ou un autre nombre réel. Vous devez donc vérifier cette constante pour vous assurer que l'appel a réussi, et si sinon, quelque chose n'a pas fonctionné correctement. Le reste des valeurs possibles retournées sont affichées comme le résultat de la valeur "EResult" de Steam et vous devriez voir steamclientpublic .h dans les en-têtes du SDK pour les 89 valeurs possibles. "event_type" - La chaîne "ugc_subscribe_item". "published_file_id " - L'identifiant unique de l'élément publié. Syntaxe : steam_ugc_subscribe_item(published_file_id) ; Arguments Description fichier_publié_id L'identifiant unique du fichier auquel l'UGC doit s'abonner. Les retours : ID asynchrone Exemple : steam_sub= steam_ugc_subscribe_item(global.pubFileID) ; Le code ci-dessus s'abonne (et télécharge) l'élément dont l'ID de fichier est stocké dans la variable globale "pubFileID".
