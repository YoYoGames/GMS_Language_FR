steam _ugc_start_item_update Cette fonction doit être appelée avant d'ajouter ou de mettre à jour des informations sur un élément UGC. Vous devez fournir l'ID d'application unique pour votre jeu sur Steam, ainsi que l'ID de fichier publié unique qui a été renvoyé pour l'élément lorsque vous l'avez créé à l'aide de la fonction steam_ugc_create_item(). La fonction renverra un descripteur de mise à jour unique pour l'élément, que vous pouvez ensuite utiliser dans les fonctions d'élément UGC pour mettre à jour (ou ajouter) des informations à télécharger. Syntaxe:steam_ugc_start_item_update(consumer_app_id, published_file_id); Argument Description consumer_app_id L'ID d'application unique pour votre jeu sur Steam. published_file_id La valeur d'ID de fichier publié unique pour l'élément. Retour:Handle Exemple:var app_id = steam_get_app_id(); var updateHandle = steam_ugc_start_item_update(app_id, global.Publish_ID); steam_ugc_set_item_title(updateHandle, "My workshop item(3)!"); steam_ugc_set_item_description( updateHandle, "testing workshop..."); steam_ugc_set_item_visibility(updateHandle, ugc_visibility_public); var tagArray; tagArray[0] = "Test"; tagArray[1] = "New"; steam_ugc_set_item_tags(updateHandle, tagArray); steam_ugc_set_item_preview(updateHandle, "promo.jpg"); steam_ugc_set_item_content(updateHandle, "WorkshopContent1"); requestId = steam_ugc_submit_item_update(updateHandle, "Version 1.2"); Le code ci-dessus obtient l'ID du jeu, puis l'utilise avec un ID de fichier publié précédemment stocké pour générer une poignée de mise à jour pour l'élément. Ce descripteur est ensuite utilisé pour mettre à jour diverses informations avant que la mise à jour ne soit envoyée aux serveurs de l'Atelier.
