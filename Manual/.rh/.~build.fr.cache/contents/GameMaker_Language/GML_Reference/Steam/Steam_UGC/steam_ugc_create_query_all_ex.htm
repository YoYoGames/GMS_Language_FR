steam_ugc_create_query_all_ex Cette fonction peut être utilisée pour interroger la base de données UGC. La fonction requiert l'ID du jeu qui va consommer l'élément et/ou l'ID du jeu qui a créé l'élément, et vous devez utiliser les constantes suivantes pour le type de requête à créer, le type d'élément à interroger et le numéro de page à interroger. Notez qu'une requête retournera un nombre maximum de 50 éléments. Type de requête Constant Description ugc_query_RankedByVote Voir la documentation deSteam. ugc_recherche_RankedByPublicationDate - ugc_query_AcceptedForGameRankedByAcceptanceDate - ugc_query_RankedByTrend - ugc_query_FavoritedByFriendsRankedByPublicationDate - ugc_recherche_CreatedByFriendsRankedByPublicationDate - ugc_query_RankedByNumTimesReported - ugc_recherche_CreatedByFollowedUsersRankedByPublicationDate - ugc_query_NotYetRated - ugc_query_RankedByTotalVotesAsc - ugc_query_RankedByVotesUp - ugc_query_RankedByTextSearch (recherche de texte classé) - Type de correspondance Constant Description ugc_match_Items Voir la documentation deSteam. ugc_match_Items_Mtx - ugc_match_Items_ReadyToUse - ugc_match_Collections - ugc_match_Artwork - ugc_match_Videos - ugc_match_Screenshots (captures d'écran) - ugc_match_AllGuides - ugc_match_WebGuides - ugc_match_Guides intégrés - ugc_match_UsableInGame - ugc_match_ControllerBindings (liens avec les contrôleurs) - La fonction renvoie une valeur unique de poignée de requête qui doit être stockée dans une variable pour être utilisée dans les autres fonctions de requête. Notez que cette fonction ne fait que préparer la requête mais ne l'envoie pas réellement - pour cela, vous devez appeler la fonction steam_ugc_send_query.htm">steam_ugc_send_query() - et vous pouvez utiliser d'autres fonctions steam_ugc_query_* () pour affiner la demande de recherche avant qu'elle ne soit réellement envoyée. Syntaxe : steam_ugc_create_query_all_ex(query_type, match_type, page, creator_app_id, consumer_app_id) ; Arguments Description type de requête Le type de requête de données à créer (voir les constantes listées ci-dessous). type de correspondance Le type d'éléments UGC à interroger (voir les constantes énumérées ci-dessous). page Le numéro de la page à interroger. creator_app_id L'identifiant de l'application du créateur de l'élément. consumer_app_id L'identifiant de l'application du consommateur. Les retours : Traitement des requêtes Exemple : query_handle = steam_ugc_create_query_all_ex(ugc_query_RankedByVote, page, global.AccountID, 0, global.GameID) ; Le code ci-dessus crée une demande de requête et stocke son handle dans une variable pour une utilisation future.
