gc_get_stats Avec cette fonction, vous pouvez récupérer des informations sur l'état actuel du garbage collector. La fonction renverra une structure qui aura les variables de membre suivantes (notez que «objets» fait ici référence à tout ce qui peut être récupéré et non à des instances d'objet générales telles que définies dans le navigateur d'actifs):objects_touched - c'est le nombre d'objets actifs que le ramasse-miettes a trouvés dans l'image précédente. Cela variera en fonction de la génération collectée.objects_collected - le nombre d'objets dont le ramasse-miettes a déterminé qu'ils n'étaient pas actifs dans l'image précédente, et qui pourraient donc être supprimés.traversal_time - c'est le temps en microsecondes (sur le thread principal) que le garbage collector a pris pour déterminer quels objets étaient actifs.collection_time - c'est le temps en microsecondes (sur un thread séparé) que le garbage collector a mis pour nettoyer les objets jugés inactifs.gc_frame - c'est un compteur qui est incrémenté à chaque fois qu'une passe de garbage collection se produit. Si le garbage collection est désactivé, cela n'augmentera pas.generation_collected - il s'agit de l'indice de la dernière génération collectée. 0 est la plus jeune génération et 3 est actuellement la plus âgée.num_generations - c'est le nombre total de générations de garbage collection.num_objects_in_generation - ceci est un tableau (de taille num_generations ) contenant le nombre d'objets dans chaque génération. REMARQUE: sur la plate-forme cible HTML5, le garbage collection est géré par le moteur JavaScript et par conséquent, les variables membres de la structure ci-dessus renverront toutes 0 si cette fonction est utilisée sur cette plate-forme. Lorsque vous utilisez cette fonction, veuillez noter que les informations affichées pour les objets ne sont mises à jour que lorsqu'une génération complète est terminée, ce qui peut prendre plusieurs images en fonction du réglage de la durée d'image (voir ici pour plus d'informations sur la synchronisation des images). Syntaxe:gc_get_stats(); Retour:Struct Exemple:if (global.debug == true)     {     var _s = gc_get_stats();     var _t = _s.traversal_time;     var _c = _s.collection_time;     show_debug_message("Traversal time = " + string(_t))     show_debug_message("Collection time = " + string(_c))     } Le code ci-dessus vérifie une variable globale et si elle est true il obtient des informations du collecteur garabge et les envoie à la console en tant que messages de débogage.
