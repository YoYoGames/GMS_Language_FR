tampon_sha1 En cryptographie, SHA-1 est une fonction de hachage cryptographique conçue par l'Agence de sécurité nationale des États-Unis et est employée dans plusieurs applications et protocoles largement utilisés, comme le populaire Git où elle est utilisée pour vérifier les changements de fichiers. Cette fonction prend des données d'entrée dans un tampon et renvoie un condensé de message de 160 bits au format ASCII. De cette façon, vous pouvez générer une clé sécurisée qui peut être stockée et utilisée pour vérifier l'intégrité de l'information envoyée à (ou à l'étranger). de l'information envoyée à (ou reçue de) un serveur externe (par exemple). Lorsque vous appliquez cette fonction à des tampons, vous devez spécifier l'identifiant du tampon à utiliser, puis une valeur de décalage (en octets) pour savoir où commencer, et enfin une taille (toujours en octets) pour la région à hacher. NOTE: SHA-1 n'est pas complètement sécurisé et peut être cassé. Voir cette page pour plus d'informations. Syntaxe : buffer_sha1(buffer, offset, size) ; Arguments Description tampon L'index du tampon à utiliser. décalage La valeur du décalage des données. taille La taille du tampon. Les retours : Chaîne de caractères Exemple : check_string = buffer_sha1(buff, 0, buffer_get_size(buff)) ; Le code ci-dessus va créer un hachage sha1 pour les données complètes stockées dans le tampon indexé par la variable "buff", et stocker le hachage renvoyé dans la variable "check_string".
