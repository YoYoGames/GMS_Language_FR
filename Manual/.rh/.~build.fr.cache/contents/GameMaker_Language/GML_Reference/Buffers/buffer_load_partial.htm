buffer_load_partial Cette fonction chargera certaines des données de tampon qui ont été précédemment sauvegardées à l'aide du buffer_save() fonctionne dans un tampon déjà créé. Vous donnez l'id du tampon précédemment créé à charger, puis le fichier de tampon enregistré à charger, puis le décalage depuis le début du tampon (en octets) à partir duquel vous souhaitez charger les données. Les arguments suivants permettent de définir la longueur des données du tampon (en octets) à partir du point de décalage initial que vous souhaitez charger et du point de décalage vers lequel charger les données dans le tampon (encore une fois, en octets). Syntaxe:buffer_load_partial(buffer, filename, offset, src_len, dest_offset); Argument Description buffer L'index du tampon à charger. filename Le nom du fichier à charger. offset Décalage dans la mémoire tampon vers lequel charger (en octets). src_len La longueur de la partie du tampon à charger (en octets). dest_offset Le décalage où commencer à placer les données partielles dans le nouveau tampon (en octets). Retour:Real Exemple:buff = buffer_create(256, buffer_grow, 1); var _file = "save.dat"; var _so = 6; var _sl = 5; var _do= 0; buffer_load_partial(buff, _file, _so, _sl, _do); Le code ci-dessus créera un nouveau tampon de «croissance» puis chargera une partie des données enregistrées dans le fichier «save.dat» sur celui-ci.
