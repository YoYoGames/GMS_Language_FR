buffer_tell Lorsque vous lisez ou écrivez des données dans une mémoire tampon à l'aide du buffer_read() ou buffer_write() la position actuelle de «recherche» est avancée par les octets écrits ou lus, et avec cette fonction, vous pouvez obtenir la position actuelle de «recherche» pour une utilisation dans d'autres fonctions de tampon. Par exemple, si votre alignement de tampon est défini sur 4 octets et que vous écrivez une seule donnée d'une taille de 1 octet, effectuez un buffer_tell(), vous obtiendrez une valeur de retour de 1. Cependant, si vous écrivez un autre élément de données, également d'une taille de 1 octet, effectuez un buffer_tell(), vous obtiendrez une valeur de retour de 5 car l'alignement a "rempli" les données à cette position. Syntaxe:buffer_tell(buffer); Argument Description buffer L'index du tampon à utiliser. Retour:Real Exemple:var pos = buffer_tell(buff); buffer_seek(buff, buffer_seek_start, 0); val[0] = buffer_read(buff, buffer_S16); val[1] = buffer_read(buff, buffer_S16); val[2] = buffer_read(buff, buffer_S16); buffer_seek(buff, buffer_seek_start, pos); Le code ci-dessus stockera la position de recherche actuelle dans le tampon indexé dans la variable "buff" vers la variable locale "pos". La position de recherche de tampon sera alors définie au début du tampon, et trois éléments de données sont lus dans un tableau, avant de finalement réinitialiser la position de recherche de tampon à l'endroit où elle se trouvait auparavant.
