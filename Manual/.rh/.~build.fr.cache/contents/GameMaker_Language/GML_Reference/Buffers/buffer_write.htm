buffer_write Cette fonction peut être utilisée pour écrire des données dans un tampon préalablement créé. Les données que vous écrivez doivent être en accord avec l'argument "type" de cette fonction, ce qui signifie que vous ne pouvez pas essayer d'écrire une chaîne de caractères comme un entier 16bit non signé, par exemple. Les constantes suivantes peuvent être utilisées pour définir le type de données : Constant Description tampon_u8 Un nombre entier non signé de 8 bits. Il s'agit d'une valeur positive comprise entre 0 et 255. tampon_s8 Un nombre entier signé de 8 bits. Il peut s'agir d'une valeur positive ou négative comprise entre -128 et 127 (0 est considéré comme positif). tampon_u16 Un nombre entier non signé de 16 bits. Il s'agit d'une valeur positive comprise entre 0 et 65 535. tampon_s16 Un nombre entier signé de 16 bits. Il peut s'agir d'une valeur positive ou négative comprise entre -32 768 et 32 767 (0 est considéré comme positif). tampon_u32 Un nombre entier non signé de 32 bits. Il s'agit d'une valeur positive comprise entre 0 et 4 294 967 295. tampon_s32 Un nombre entier signé de 32 bits. Il peut s'agir d'une valeur positive ou négative comprise entre -2 147 483 648 et 2 147 483 647 (0 étant considéré comme positif). tampon_u64 Un nombre entier non signé de 64 bits. tampon_f16 Un flottant de 16 bits. Il peut s'agir d'une valeur positive ou négative dans la même plage qu'un entier signé de 16 bits. tampon_f32 Un flottant de 32 bits. Il peut s'agir d'une valeur positive ou négative dans la même plage qu'un nombre entier signé de 32 bits. tampon_f64 Un flotteur 64 bits. tampon_bool Une valeur booléenne. Elle ne peut être que 1 ou 0(vrai ou faux). chaîne_tampon Une chaîne de caractères de n'importe quelle taille, finalisée par un caractère de terminaison nul. Texte tampon Une chaîne de caractères de n'importe quelle taille, sans le caractère final nul. La fonction renvoie 0 si elle réussit ou -1 si elle échoue. Syntaxe : buffer_write(buffer, type, valeur) Arguments Description tampon L'index du tampon dans lequel écrire. type Le type de données qui doit être écrit dans le tampon (voir la liste des constantes ci-dessus). valeur Les données à écrire. Les retours : 0 en cas de succès, ou -1 en cas d'échec Exemple : buffer_seek(buff, buffer_seek_start, 0) ; buffer_write(buff, buffer_s16, 0) ; buffer_write(buff, buffer_s16, x) ; buffer_write(buff, buffer_s16, y) ; Le code ci-dessus trouve le début du tampon avec l'identifiant stocké dans la variable "buff" et y écrit une série de valeurs entières signées de 16 bits.
