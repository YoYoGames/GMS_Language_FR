tampon_get_surface Avec cette fonction, vous pouvez écrire les informations d'une surface dans un tampon donné. Le tampon doit avoir été créé précédemment et doit être un tampon de croissance aligné sur 1 octet d'une taille approximative (en octets) de la surface que vous allez écrire. Les informations de la surface seront toujours écrites au début du tampon (indépendamment de la position de recherche) mais vous pouvez choisir une valeur de décalage (en octets) pour déplacer la position d'écriture à l'endroit où vous le souhaitez par rapport au début du tampon. Notez que la fonction écrit chaque pixel de la surface dans le tampon en utilisant un formatage BGRA sur la cible Windows, mais sur d'autres cibles, cela peut être différent en fonction du système d'exploitation ou même du périphérique. REMARQUE: cette fonction a été modifiée à partir de la mise à jour 2.3.1 de GameMaker Studio. Les versions précédentes utilisaient la constante buffer_surface_copy, qui est désormais maintenant dépréciée. Syntaxe : buffer_get_surface(buffer, surface, offset) ; Arguments Description tampon L'index du tampon à utiliser. surface L'indice de la surface à utiliser. décalage La valeur du décalage des données (en octets). Les retours : N/A Exemple : buffer_get_surface(buff, application_surface, 0) ; Le code ci-dessus copiera les données stockées dans la surface de l'application dans le tampon indexé dans la variable "buff" sans décalage.
