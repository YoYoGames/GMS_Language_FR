buffer_peek Avec le buffer_read() fonction, vous pouvez lire des données à partir du tampon donné à la position actuelle de «recherche», chaque élément de données en cours de lecture faisant avancer cette position par les octets en cours de lecture ou d'écriture. Cependant, il peut être nécessaire pour vous de lire une donnée donnée sans vouloir changer la position de recherche actuelle, et c'est à ce moment-là que vous utiliserez cette fonction. Vous fournissez simplement à la fonction un identifiant de tampon, puis une position de décalage (à partir du début du tampon) dans ce tampon à lire, ainsi que le type de données que vous souhaitez lire. REMARQUE: l'utilisation d'un type de données incorrect pour les données lues entraînera des valeurs erronées! Si la fonction réussit, une valeur (Real, string, int, etc...) sera renvoyée en fonction du type de données, tandis qu'un échec renverra simplement 0. Syntaxe:buffer_peek(buffer, offset, type); Argument Description buffer L'index du tampon à utiliser. offset La position de décalage (en octets) dans la mémoire tampon à partir de laquelle lire les données données. type Le type de données à lire dans le tampon (voir la liste des constantes ici ). Retour:Real/String or 0 if it fails. Exemple:var red = buffer_peek(buff, 1, buffer_u8); var green = buffer_peek(buff, 2, buffer_u8); var blue = buffer_peek(buff, 3, buffer_u8); image_blend = make_colour_rgb(red, green, blue); Le code ci-dessus obtiendra trois valeurs à partir de trois positions différentes dans le tampon indexé dans la variable "buff" et utilisera ensuite ces valeurs pour définir le mélange d'image de l'instance.
