buffer_read Cette fonction peut être utilisée pour lire des données à partir d'un tampon créé précédemment. La valeur de retour dépendra du type de données que vous lisez, qui en lui-même est défini par les constantes suivantes: Constant Description buffer_u8 Un entier 8 bits non signé. Il s'agit d'une valeur positive comprise entre 0 et 255. buffer_s8 Un entier signé de 8 bits. Cela peut être une valeur positive ou négative de -128 à 127 (0 est classé comme positif). buffer_u16 Un entier 16 bits non signé. Il s'agit d'une valeur positive comprise entre 0 et 65 535. buffer_s16 Un entier 16 bits signé. Cela peut être une valeur positive ou négative de -32 768 à 32 767 (0 est considéré comme positif). buffer_u32 Un entier 32 bits non signé. Il s'agit d'une valeur positive comprise entre 0 et 4 294 967 295. buffer_s32 Un entier 32 bits signé. Cela peut être une valeur positive ou négative de -2 147 483 648 à 2 147 483 647 (0 est considéré comme positif). buffer_u64 Un entier 64 bits non signé. buffer_f16 Un flotteur 16 bits. Cela peut être une valeur positive ou négative dans la plage de +/- 65504. (Pas actuellement pris en charge!) buffer_f32 Un flotteur 32 bits. Cela peut être une valeur positive ou négative dans la plage de +/- 16777216. buffer_f64 Un flotteur 64 bits. buffer_bool Une valeur booléenne. Ne peut être que 1 ou 0 (true ou false ) buffer_string Une chaîne de n'importe quelle taille. buffer_text Une chaîne de n'importe quelle taille, sans le dernier caractère de fin nul. Si la fonction réussit, elle renverra une valeur du type donné, mais si elle échoue, cela provoquera une erreur du runner. REMARQUE: l'utilisation d'un type de données incorrect pour les données en cours de lecture entraînera le renvoi de valeurs erronées. Syntaxe:buffer_read(buffer, type) Argument Description buffer L'index de la mémoire tampon à partir de laquelle lire. type Le type de données à lire dans le tampon (voir la liste des constantes ci-dessus). Retour:Real/String Exemple:var cmd = buffer_read(buff, buffer_s16); Le code ci-dessus lit à partir du tampon avec l'ID stocké dans la variable "buff" une valeur 16 bits signée dans la variable locale "cmd".
