gif_add_surface Cette fonction permet d'enregistrer une image dans un fichier GIF à partir d'une surface. Vous fournissez l'ID du fichier GIF (tel que renvoyé par la fonction gif_open()) et un ID de surface (tel que retourné par la fonction surface_create()), ainsi qu'un temps de retard entre les images. Le délai est calculé en 1/100e de seconde, mais notez qu'une fois le GIF créé, si vous l'utilisez à des fins promotionnelles par exemple, les différents navigateurs peuvent interpréter le délai entre les images de manière légèrement différente. légèrement différent - voir ici pour plus de détails. Vous pouvez (facultativement) fournir des valeurs de décalage X et Y pour la surface utilisée pour ajouter au GIF. Vous pouvez (facultativement) fournir des valeurs de décalage X et Y pour la surface utilisée pour ajouter au GIF, auquel cas la partie dessinée au GIF commencera à partir du décalage et non de la position par défaut (0,0), ce qui vous permet de sélectionner uniquement une section de la surface à ajouter. Vous pouvez également fournir une valeur de quantification, qui est un nombre entier compris entre 0 et 3, où 0 correspond à une quantification complète et 3 à une quantification nulle (les valeurs inférieures entraînent une dégradation de la qualité, mais créent un GIF plus petit). Notez qu'il n'y a pas beaucoup de différence entre la qualité par défaut de 2, et la plus élevée de 3, et l'utilisation de 3 ralentira de manière significative la création du GIF, il faut donc faire attention à l'utilisation de la valeur maximale. Notez qu'il n'y a pas de limites intégrées sur le nombre d'images ou sur la taille de ces images dans votre GIF, mais si vous utilisez trop de mémoire, la fonction échouera et retournera -1, sinon elle retournera 0. Syntaxe : gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]) ; Arguments Description gif_index L'ID du gif auquel ajouter les données de surface surface L'ID de la surface à utiliser comme cadre ajouté retard_temps Le temps de retard pour la trame (en centièmes de seconde) [xoffset] Facultatif] Le décalage le long de l'axe X de la surface à ajouter à partir de [yoffset] Facultatif] Le décalage le long de l'axe Y de la surface à ajouter à partir de [quantification] Facultatif] Quantité de quantification de 0 à 3 (0 pour la qualité la plus faible, 3 pour la qualité la plus élevée, 2 par défaut) Les retours : Entier Exemple : si save_gif == true { if count == 0 { gif_image = gif_open(room_width, room_height) ; } else if count < 30 { gif_add_surface(gif_image, application_surface, 6/100) ; } else { gif_save(gif_image, "GameCapture.gif") ; count = 0 ; save_gif = false ; } count++ ; } Le code ci-dessus va créer un fichier image GIF avec 30 images prises sur la surface de l'application, puis l'enregistrer.
