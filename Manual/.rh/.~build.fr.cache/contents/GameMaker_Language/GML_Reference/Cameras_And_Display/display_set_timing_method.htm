Méthode d'affichage de la durée de vie des produits Cette fonction peut être utilisée pour définir la méthode de timing à utiliser pour le rendu de votre jeu. La méthode peut être l'une des constantes suivantes : Constant Description tm_sleep Utiliser la valeur de la marge de sommeil comme méthode principale de chronométrage tm_countvsyncs Utiliser la synchronisation vsync comme méthode principale de synchronisation (par défaut pour toutes les plateformes supportées) La méthode de synchronisation vsync utilise la prise en charge de la synchronisation verticale par la plate-forme cible pour fournir un point d'ancrage pour les calculs de temps de rendu du jeu, tandis que le réglage du mode sur la marge de sommeil essaiera simplement de s'assurer que chaque image dure le temps correct (disons 1/30e ou 1/60e de seconde) en attendant ou en dormant. temps (disons 1/30e ou 1/60e de seconde) en attendant ou en dormant. En général, le timing vsync par défaut donnera les résultats les plus fluides, mais notez que lorsque vous utilisez la méthode vsync, les valeurs de la marge de sommeil sont toujours pertinentes, bien qu'elles aient un impact réduit. Nous vous recommandons donc de conserver la valeur par défaut lorsque vous utilisez cette méthode. Par défaut, sur toutes les plates-formes, à l'exception de PS4, Ubuntu et HTML5, GameMaker Studio 2 utilisera la méthode de synchronisation vsync, tandis que sur les plates-formes non prises en charge, seule la synchronisation par marge de sommeil peut être utilisée. Si vous souhaitez définir la marge de sommeil, vous pouvez le faire à l'aide de la fonction display_set_sleep_margin() et vous pouvez trouver quelle méthode de chronométrage est actuellement utilisée avec la fonction display_get_timing_method(). Syntaxe : display_set_timing_method(méthode) ; Arguments Description méthode La méthode de chronométrage à utiliser (voir la liste des constantes, ci-dessus). Les retours : N/A Exemple : if display_get_timing_method() != tm_sleep { display_set_timing_method(tm_sleep) ; if display_get_sleep_margin() != 20 { display_set_sleep_margin(20) ; } } Le code ci-dessus vérifiera la méthode de synchronisation et si elle n'est pas définie sur tm_sleep , elle sera définie sur cette méthode et la marge de sommeil sera définie sur 20.
