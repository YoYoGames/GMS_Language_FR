get_open_filename Cette fonction ouvre un dialogue et demande au joueur un nom de fichier à ouvrir avec le filtre donné. Le filtre a la forme " nom1 | masque1 | nom2 | masque2 |... ", où le masque peut contenir les différentes options avec un point-virgule entre elles et vous pouvez également utiliser un "*" pour désigner n'importe quelle chaîne. Par exemple: " bitmaps | *.bmp; *. Wmf ", demanderait à l'utilisateur de sélectionner le nom d'un fichier bitmap dans le système, mais uniquement ceux qui ont les deux extensions spécifiées (bmp et png ). cette fonction n'ouvre pas le fichier lui-même! Il renvoie uniquement une chaîne avec le nom complet et le chemin du fichier. Si l'utilisateur appuie sur " Annuler ", une chaîne vide "" est renvoyée. Une chose importante à noter lors de l'utilisation de cette fonction est qu'elle vous accorde certaines autorisations pour ce fichier, pour la durée de votre partie. Ainsi, une fois que vous avez obtenu le chemin du fichier de l'utilisateur, vous pouvez y accéder encore et encore sans avoir à le demander. Cependant, comme cette fonction est pour la lecture d'un fichier, les autorisations accordées sont uniquement pour la lecture et l'enregistrement ne sera pas autorisé par le système d'exploitation. Si vous avez besoin de lire et d'écrire dans le fichier, vous pouvez obtenir l'autorisation en utilisant la fonction get_save_filename(). REMARQUE: cette fonction n'est valide que sur les cibles Windows et macOS Syntaxe:get_open_filename(filter, fname); Argument Description filter Le filtre et / ou le nom du type de fichier. fname Le nom de fichier suggéré à utiliser. Retour:String Exemple:var file; file = get_open_filename("text file|*.txt", ""); if file != ""     {     file_text_open_read(file);     } Le code ci-dessus demandera à l'utilisateur de sélectionner un fichier texte à ouvrir, puis de vérifier le nom et le chemin du fichier retourné, et s'il existe, il ouvrira le fichier pour la lecture.
