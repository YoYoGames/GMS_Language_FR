get_save_filename_ext Cette fonction ouvre un dialogue et demande au joueur un nom de fichier à sauvegarder avec le filtre donné. Le filtre a la forme " name1|mask1|name2|mask2|...", où le masque peut contenir les différentes options avec un point-virgule entre elles. et vous pouvez également utiliser un "*" pour signifier n'importe quelle chaîne. Par exemple : " bitmaps|*.bmp;*.wmf", demanderait à l'utilisateur de sélectionner le nom d'un fichier bitmap du système, mais seulement ceux qui ont les deux extensions spécifiées (bmp et png). cette fonction ne sauvegarde pas le fichier lui-même ! Elle renvoie seulement une chaîne avec le nom complet et le chemin d'accès du fichier. Si l'utilisateur appuie sur "Annuler", une chaîne vide "" est renvoyée. Vous pouvez également ajouter un pour lancer la recherche, et une légende à afficher en haut de la fenêtre au lieu de celle par défaut. Une chose importante à noter lorsque vous utilisez cette fonction est qu'elle vous accorde certaines permissions pour ce fichier, pour la durée de votre jeu. Ainsi, une fois que vous avez obtenu le chemin d'accès au fichier de la part de l'utilisateur, vous pouvez y accéder encore et encore sans avoir à le demander. permission s'applique aussi bien à la lecture qu'à l' écriture. REMARQUE: Cette fonction n'est valable que sur les cibles Windows et macOS. Syntaxe : get_save_filename_ext(filter, fname, directory, caption) ; Arguments Description filtre Le filtre et/ou le nom du type de fichier. fname Le nom de fichier suggéré à utiliser. répertoire Le répertoire à partir duquel la recherche doit commencer. légende La légende de la fenêtre. Les retours : Chaîne de caractères Exemple : var file ; file = get_save_filename_ext("screenshot|*.png", "", working_directory, "Save a level here") ; if file != "" { screen_save(file) ; } Le code ci-dessus demandera à l'utilisateur de sélectionner un nom de fichier image pour le sauvegarder dans le répertoire de travail, puis vérifiera le nom et le chemin du fichier retourné. Si ce n'est pas une chaîne vide, il prendra une capture d'écran et la sauvegardera en utilisant le nom donné.
