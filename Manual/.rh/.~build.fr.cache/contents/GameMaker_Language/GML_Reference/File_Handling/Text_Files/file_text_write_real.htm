fichier_texte_écrit_réel Cette fonction permet d'écrire un nombre dans le fichier texte précédemment ouvert. Notez que comme la valeur à écrire peut être un nombre réel, toutes les décimales seront écrites avec un point "." comme séparateur. Si le fichier contient déjà des informations, ces informations seront effacées et la chaîne de caractères sera écrite au début du fichier, à moins que vous n'ayez ouvert le fichier avec la commande file_text_open_append(). Vous pouvez également éviter cela en utilisant la fonction file_text_readln() avec la fonction fichier_texte_eof() pour parcourir en boucle le contenu du fichier jusqu'à la fin, puis commencer à écrire. Il est important de noter que lorsque vous écrivez de très grands nombres dans un fichier texte à l'aide de cette fonction, ils peuvent être convertis dans un format standard simplifié, comme "6.6624e+003", qui ne peut pas être relu correctement dans GameMaker Studio 2. Pour éviter ce genre de problème, vous devez plutôt convertir la valeur en une chaîne de caractères et utiliser la fonction file_text_write_string() à la place. Syntaxe : file_text_write_real(fileid, val) ; Arguments Description fileid L'identifiant du fichier à modifier. val La valeur réelle à écrire dans le fichier. Les retours : Real Exemple : var file = file_text_open_write(working_directory + "hiscore.txt") ; for (var i = 0 ; i < 10 ; ++i ;) { file_text_write_real(file, scr[i]) ; file_text_writeln(file) ; file_text_write_string(file, scr_name[i]) ; file_text_writeln(file) ; } file_text_close(file) ; Le code ci-dessus ouvre un fichier pour l'écriture et boucle ensuite sur deux tableaux, en écrivant chaque valeur de tableau sur une nouvelle ligne du fichier.
