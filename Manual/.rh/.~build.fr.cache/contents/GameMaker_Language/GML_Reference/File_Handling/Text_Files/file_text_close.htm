fermeture du texte du fichier Une fois que vous avez fini de travailler avec un fichier donné (que ce soit en le lisant ou en l'écrivant), vous devez refermer le fichier, sinon vous risquez de perdre les informations qu'il contient. Cela permet également d'éviter les fuites de mémoire et de s'assurer que vous ne dépassez jamais la limite de fichiers en ayant plus de 32 fichiers ouverts. la limite de fichiers en ayant plus de 32 fichiers ouverts. Syntaxe : file_text_close(fileid) ; Arguments Description fileid L'identifiant du fichier à fermer. Les retours : N/A Exemple : var file = file_text_open_write(working_directory + "Game_Data.txt") ; while (!file_text_eof(file)) { file_text_readln(file) ; } file_text_write_string(file, level_data) ; file_text_close(file) ; Le code ci-dessus ouvre un fichier pour l'écrire, puis passe en boucle les lignes de texte déjà écrites dans le fichier jusqu'à ce qu'il atteigne la fin. À ce stade, il écrit une chaîne de caractères, puis referme le fichier. Retour : Fichiers texte Suivant : file_text_open_read Copyright YoYo Games Ltd. 2021 Tous droits réservés
