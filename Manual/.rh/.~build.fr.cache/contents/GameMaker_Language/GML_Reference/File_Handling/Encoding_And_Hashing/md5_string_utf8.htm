md5_string_utf8 En cryptographie, MD5 (Message-Digest algorithm 5) est une fonction de hachage cryptographique largement utilisée avec une valeur de hachage de 128 bits et a été employée dans une grande variété d'applications de sécurité. Elle est également couramment utilisée pour vérifier l'intégrité des fichiers et des chaînes de caractères. Cette fonction prend une chaîne UTF-8 en entrée (qui a un nombre variable d'octets par caractère) et renvoie le hachage MD5 hexadécimal de 32 caractères qui est unique à cette chaîne. De cette façon, vous pouvez générer une clé sécurisée qui peut être stockée et utilisée pour vérifier l'intégrité de l'information en cours de traitement. pour vérifier l'intégrité des informations envoyées à (ou reçues de) un serveur externe (par exemple). REMARQUE: il existe deux formats pour l'encodage MD5, UTF-8 et unicode. Les deux sont fournis pour faciliter la communication avec différentes configurations de serveur, mais le plus courant à utiliser est unicode. REMARQUE: MD5 n'est pas complètement sécurisé et peut être cassé. Voir cette page pour plus d'informations. Syntaxe : md5_string_utf8(string) Arguments Description chaîne de caractères La chaîne à hacher. Les retours : Chaîne de caractères Exemple : var hash, str ; str = json_encode(hiscore_map) ; hash = md5_string_utf8(str) ; ini_open("local.ini") ; ini_write_string("info", "0", hash) ; ini_close() ; get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str) Le code ci-dessus va encoder une carte DS dans une chaîne JSON. Un hachage MD5 est ensuite généré et stocké dans un fichier ini afin qu'il puisse être utilisé ultérieurement pour vérifier l'intégrité du JSON si les mêmes informations sont reçues ultérieurement du serveur.
