La gestion des fichiers À un moment donné, lors de la création de votre jeu, il sera nécessaire de stocker des informations en externe et de pouvoir récupérer ces informations. Par exemple, vous souhaiterez peut-être stocker des informations de connexion, des scores élevés, ou peut-être la progression du joueur à travers vos niveaux. Quelle que soit la raison, vous aurez besoin de savoir comment utiliser les différentes fonctions de fichiers disponibles lorsque vous travaillez avec GameMaker Studio 2. Il est très important de réaliser que GameMaker Studio 2 est sandbox par défaut, ce qui signifie qu'il ya un certain nombre de limites et les complications mineures lors de l' utilisation des fonctions du système de fichiers qui sont disponibles, et il convient de noter aussi que le système de fichiers fonctionnera un peu différemment en fonction de la plate-forme que vous ciblez. Pour plus d'informations, veuillez consulter la section suivante: Le système de fichiers Vous pouvez désactiver le sandbox sur les cibles du bureau ( Windows, macOS et Ubuntu ( Linux )) à partir des options de jeu pour la plate-forme choisie, ce qui signifie que vous pouvez ensuite enregistrer et charger de n'importe où, dans les limites des autorisations du système d'exploitation. Steam vous a accordé la permission d'utiliser leur API dédiée avec votre jeu, vous constaterez qu'un nouvel ensemble de fonctions de fichiers est également disponible. Celles-ci sont expliquées dans la section API Steam de ce manuel. Enfin, vous devez également noter qu'il existe des différences distinctes entre certaines plates-formes cibles qui seraient prises en compte lors de l'utilisation des fonctions de fichiers. Ceux-ci sont: iOS: sur iOS, si vous chargez un fichier inclus à partir d'un dossier, c'est-à-dire: avec un chemin quelque chose comme "Backgrounds/background1.png ", vous n'avez PAS besoin d'inclure le dossier dans le chemin et il suffit de fournir le nom du fichier. Sur toutes les autres plates-formes, vous devez fournir le (s) dossier (s) dans le cadre du chemin d'accès au fichier. macOS et Ubuntu ( Linux ): sur macOS et Ubuntu, les noms de fichiers et les chemins sont sensibles à la casse sur ces plates-formes, et donc pour éviter tout problème potentiel, GameMaker Studio 2 fera automatiquement en sorte que tous les noms / chemins utilisent par défaut des minuscules. HTML5: sur HTML5, l'enregistrement et le chargement vers et depuis le stockage local fonctionneront comme expliqué dans les différentes pages de fonction, cependant si vous chargez un fichier depuis un serveur, vous ne devez utiliser que la fonction buffer_load_async() car le chargement synchrone est obsolète sur la plupart des navigateurs et sera finalement obsolète. Notez également que vous ne pouvez pas créer ou détruire des répertoires dans le stockage local du navigateur.
