network_create_server_raw Cette fonction permet de créer un nouveau serveur réseau pour votre jeu, en utilisant l'un des protocoles de connexion autorisés (voir les constantes listées ci-dessous). Vous indiquez le type de serveur, puis lui donnez un port à utiliser, et enfin le nombre maximum de connexions qui devraient être autorisées à tout moment au serveur (notez que ce nombre dépend de vous, mais trop de clients connectés saturera le réseau ou le processeur de l'appareil ne sera pas en mesure de gérer le traitement de ce nombre de joueurs, alors utilisez-le avec précaution). La fonction renvoie un identifiant unique qui doit être utilisé stocké dans une variable et utilisé pour identifier le serveur dans toutes les autres fonctions réseau, ou une valeur inférieure à 0 si la connexion échoue. Constant Description network_socket_tcp Créez un socket en utilisant TCP. network_socket_udp Créez un socket en utilisant UDP. network_socket_ws Créez un socket Web ( uniquement pour la connexion à des projets HTML5), à l'aide de TCP. Comme cela crée un serveur "brut", il n'acceptera ni n'utilisera aucun des en- GameMaker communication, et vous devriez donc utiliser les fonctions network_send_raw() et network_send_udp_raw() pour envoyer des données non formatées au serveur créé. Syntaxe:network_create_server_raw(type, port, max_client); Argument Description type Le type de serveur à créer (voir les constantes ci-dessous). port Le port que le serveur utilisera. max_client Le nombre maximum de clients pouvant se connecter à la fois. Retour:Real Exemple:var port = 6510; server = network_create_server_raw(network_socket_tcp, port, 32); while (server < 0 && port < 65535)     {     port++     server = network_create_server(network_socket_tcp, port, 32);     } Le code ci-dessus essaiera de créer un serveur utilisant TCP via le port 6510.
