La mise en réseau Les fonctions de mise en réseau de GameMaker Studio 2 sont basées sur des sockets, ce qui est une norme de l'industrie sur toutes les plates-formes. Avec les sockets, vous avez la possibilité de créer à la fois des clients et des serveurs en utilisant un simple GML qui vous permet de créer même une partie solo en utilisant le modèle client / serveur (quelque chose qui permettra d'ajouter facilement le multijoueur plus tard). Fondamentalement, une socket est un objet qui peut envoyer, recevoir, se connecter et écouter des ports sur le réseau. Ces fonctions ne concernent que les connexions TCP / IP, sur lesquelles Internet est basé (IPv4 pour être plus précis). En substance, cela vous permet de connecter un socket en utilisant une adresse IP (disons 192.168.1.10) à un autre socket sur une autre adresse IP (par exemple 192.168.1.11). Une fois connectés, ces deux sockets peuvent envoyer des données dans les deux sens en utilisant le network_send_* fonctions (comme network_send_packet() ) et l'événement asynchrone dédié qui est déclenché par eux. Cet événement asynchrone générera une carte DS contenant les données reçues ainsi que d'autres détails, et vous pouvez trouver les détails complets dans la section sur l' événement asynchrone réseau. Au lieu que les programmes aient à lire et à gérer chaque bit de trafic réseau entrant dans une machine, les adresses IP traitent également des ports, qui sont numérotés de 0 à 65535. Cela permet à chaque programme d'obtenir uniquement les paquets de données qui l'intéressent, et pas tout ce qui a été envoyé par chaque programme. Cela permet également de gagner un temps CPU considérable, car cela réduit considérablement le traitement des données, donc avec GameMaker Studio 2 vous pouvez également «écouter» les ports. Ainsi, un serveur créera un socket, puis le liera à un port spécifique pour l'écouter. Il sera alors informé des connexions, des déconnexions et des transferts de données à partir de ce socket. Les "paquets" de données eux-mêmes sont constitués de données binaires extraites d'un tampon. Ce tampon peut être créé et manipulé à l'aide des fonctions de tampon de GameMaker Studio 2. Il est à noter que si vous créez un projet pour la cible HTML5, vous ne pourrez pas utiliser les fonctions de création du serveur en raison des restrictions du navigateur et devrez écrire votre propre serveur pour cette plate-forme en utilisant quelque chose comme Electron ou NodeJS. Notez également que si votre jeu est multiplateforme et que vous prévoyez de vous connecter à une instance HTML5 du jeu à partir d'un serveur fonctionnant sur un appareil mobile ou de bureau, alors votre serveur ne peut communiquer avec l'instance HTML5 qu'en utilisant le protocole Web Socket (ceci est fait en utilisant la constante network_socket_ws - voir network_create_server() pour plus d'informations). Cela signifie que votre serveur devra peut-être créer deux sockets sur des ports différents, un pour se connecter à des cibles non HTML5 ( Android, Windows, etc...) et un pour se connecter à la cible HTML5.
