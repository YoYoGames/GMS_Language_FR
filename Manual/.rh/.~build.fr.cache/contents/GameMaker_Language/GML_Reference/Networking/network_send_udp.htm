réseau_send_udp Avec cette fonction, vous pouvez envoyer des données sur le réseau en utilisant UDP à un serveur. La fonction prend l'identifiant de la prise par laquelle se connecter, l'URL à laquelle se connecter et le port à utiliser. Vous devez ensuite fournir l'identifiant du tampon qui contient les données à être envoyées (pour plus d'informations sur les tampons, voir Référence - Tampons) et enfin la taille (en octets) des données. UDP est "sans connexion" dans le sens où vous n'effectuez pas de connexion, vous envoyez juste un paquet directement à une IP. directement à une IP, et le serveur reçoit les données entrantes d'une adresse IP et doit les traiter "telles quelles". La fonction renvoie le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué. Il est important de noter que la taille finale des données envoyées qui est renvoyée par cette fonction comprendra également les informations d'en-tête GameMaker, qui représentent 12 octets supplémentaires. NOTE: Cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet fonctionnant sur la cible HTML5, et les projets HTML5 ne pourront pas non plus recevoir d'UDP. Syntaxe : network_send_udp(socket, url, port, buffer, size) ; Arguments Description prise de courant L'identifiant du socket à utiliser. url L'url ou l'IP à laquelle se connecter (une chaîne). port Le port auquel se connecter. tampon L'identifiant du tampon à partir duquel les données sont récupérées. taille La taille (en octets) des données. Les retours : Real Exemple : network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff)) ; Le code ci-dessus va envoyer un paquet UDP au serveur défini par l'URL sur le port 6510.
