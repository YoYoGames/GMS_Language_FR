network_send_udp_raw Avec cette fonction, vous pouvez envoyer des données sur le réseau en utilisant UDP à un serveur. La fonction prend l' identifiant de socket pour se connecter, l'URL à laquelle se connecter et le port à utiliser. Vous devez ensuite fournir le buffer id qui contient les données à envoyer (pour plus d'informations sur les buffers voir Référence - Buffers ) et enfin la taille (en octets) des données. UDP est "sans connexion" en ce sens que vous ne faites pas réellement de connexion, vous envoyez simplement un paquet directement à une adresse IP, et le serveur reçoit les données entrantes d'une adresse IP et doit les traiter "telles quelles". Les données envoyées ne sont en aucun cas formatées par GameMaker Studio 2 et les appareils récepteurs les recevront sous forme de flux, ce qui signifie que vous devrez les gérer vous-même. La fonction renverra le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué. REMARQUE: Cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet exécuté sur la cible HTML5, et les projets HTML5 ne pourront pas non plus recevoir UDP. Syntaxe:network_send_udp_raw(socket, url, port, buffer, size); Argument Description socket L'identifiant du socket à utiliser. url L'url ou l'adresse IP à laquelle se connecter (une chaîne). port Le port auquel se connecter. buffer L'identifiant du tampon à partir duquel obtenir les données. size La taille (en octets) des données. Retour:Real Exemple:network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff)); Le code ci-dessus enverra un paquet UDP brut au serveur défini par l'URL sur le port 6510.
