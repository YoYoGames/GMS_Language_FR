physics_fixture_set_awake Lorsque vous démarrez une room avec de la physique, ou que vous liez des appareils à des instances, les appareils sont toujours considérés comme étant "éveillés"... c'est-à-dire qu'ils traitent des événements et interagissent avec les instances environnantes. Cependant, cela peut parfois conduire à des problèmes, surtout si vous avez un certain nombre d'instances avec des appareils qui sont côte à côte lorsqu'une pièce commence (pensez à une tour faite de diverses instances avec des appareils) car, étant donné qu'ils sont "éveillés", ils vont bouger et éventuellement changer de position en raison de l'apparition soudaine de la gravité et des collisions qui les affectent. Dans ce cas, vous pouvez utiliser cette fonction pour signaler que l'appareil est "endormi", auquel cas il ne traitera rien lorsqu'il sera créé dans la room. Syntaxe : physics_fixture_set_awake(fixture, state) Arguments Description L'appareil l'indice du dispositif drapeau si un luminaire est éveillé(true) ou non(false) Les retours : N/A Exemple : physics_fixture_set_awake(fix_Cloud, true) ; Le code ci-dessus signale le dispositif comme étant "éveillé" lorsqu'il est lié à une instance.
