phy_rotation Cette variable peut être utilisée pour obtenir (ou définir) l'angle du projecteur d'instances en degrés, de la même manière que pour définir ou obtenir la variable angle_image. Notez toutefois que dans le monde physique, les rotations sont calculées de manière opposée au monde normal du jeu GameMaker Studio 2, ce qui signifie que les fonctions vectorielles telles que point_direction()doivent voient leurs valeurs de retour modifiées (il suffit de transformer le positif en négatif pour résoudre ce problème). Syntaxe : phy_rotation ; Les retours : Réel (valeur à virgule flottante de simple précision, ou indéfini si l'instance n'est pas activée pour la physique) Exemple : if (phy_speed_x > 0) || (phy_speed_y > 0) { phy_rotation += sqrt(sqr(phy_speed_x) + sqr(phy_speed_y)) /10 ; } Le code ci-dessus vérifie la vitesse linéaire et si l'un ou l'autre vecteur est différent de 0, il calcule alors la vitesse réelle et l'utilise pour définir la rotation.
