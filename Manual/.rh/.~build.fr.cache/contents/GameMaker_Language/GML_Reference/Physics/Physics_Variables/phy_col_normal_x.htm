phy_col_normal_x Cette variable en lecture seule renvoie la composante x de la normale de la collision correspondant à la phy_collision_x valeur du tableau. Pour chaque point de contact, il existe une normale de contact associée (qui est généralement la même pour tous les points de contact de la collision). la même normale pour tous les points de contact de la collision). Cette normale de contact est un vecteur unitaire qui pointe d'une instance de la collision à l'autre, et peut être utilisée, par exemple, pour calculer la direction de "poussée" correcte pour résoudre les cas de collisions REMARQUE: Cette variable n'est disponible que dans l'événement de collision d'une instance activée pour la physique. Syntaxe : phy_col_normal_x ; Les retours : Réel (valeur à virgule flottante de simple précision, ou indéfini si l'instance n'est pas activée pour la physique) Exemple : repeat(5 + irandom(5)) { avec (instance_create_layer(x, y, "Effects", obj_Debris) { physics_apply_local_impulse(0, 0, other.phy_col_normal_x[0], other.phy_col_normal_y[0]) ; } } Le code ci-dessus utilise la normale de contact pour définir la direction du mouvement d'une instance créée dans l'événement de collision entre deux instances activées par la physique.
