Variables de physique Comme avec le système de collisions et de mouvement «traditionnel», le système physique a également une série de variables d'instance affectées à chaque instance d'un objet lors de sa création. Certaines de ces propriétés peuvent être définies par l'utilisateur dans GML, mais beaucoup sont en lecture seule et d'autres encore ne sont applicables qu'en cas de collision. Les variables suivantes existent dans tous les objets activés pour la physique et elles peuvent être lues et écrites (c'est-à-dire modifiées) à l'aide du code des instances et des fonctions de script: phy_activephy_angular_velocityphy_angular_dampingphy_linear_velocity_xphy_linear_velocity_yphy_linear_dampingphy_speed_xphy_speed_yphy_position_xphy_position_yphy_position_xpreviousphy_position_ypreviousphy_rotationphy_fixed_rotationphy_bullet Certaines variables sont des variables en lecture seule, ce qui signifie que les informations qu'elles contiennent ne peuvent pas être modifiées via le code d'instance ou les fonctions de script (mais elles peuvent être modifiées en interne par GameMaker Studio 2 c'est pourquoi il s'agit de variables et non de constantes ). La liste suivante répertorie les variables en lecture seule qui existent dans tous les objets activés pour la physique: phy_speedphy_com_xphy_com_yphy_dynamicphy_kinematicphy_inertiaphy_massphy_sleeping Enfin, il existe un certain nombre de variables temporaires qui existent dans les objets activés pour la physique et qui ne sont créées que lors de l'événement de collision.
