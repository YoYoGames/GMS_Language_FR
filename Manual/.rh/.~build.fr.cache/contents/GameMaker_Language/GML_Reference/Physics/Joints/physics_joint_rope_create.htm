physics_joint_rope_create Une articulation à corde est utilisée pour joindre deux instances que l'on souhaite maintenir à une distance constante, quelles que soient les autres forces qui agissent sur elles. Avec un joint de distance, vous pouvez obtenir un "étirement du joint" où les deux montages se sépareront et se comporteront de manière étrange si une trop grande tension est exercée sur le joint, alors que le joint en corde ne le fait pas et ne s'étendra pas plus loin que la longueur maximale définie. Lorsque vous créez un joint en corde, les deux instances doivent déjà être créées et avoir une fixation assignée. Ensuite, vous définissez les deux points d'ancrage en coordonnées de pièce. Le premier point d'ancrage est connecté à l'instance 1, le second point d'ancrage est connecté à l'instance 2 et la distance et l'argument maxlength définissent la contrainte de longueur maximale sur le joint. la contrainte de longueur maximale sur le joint. L'image ci-dessous montre comment cela fonctionne : Comme vous pouvez le constater, les points d'ancrage sont spécifiés en tant que coordonnées de la pièce. surtout si les instances sont créées en même temps que les articulations plutôt que d'être placées dans la room via l'éditeur de room. Vous devez également savoir que les articulations sont créées indépendamment de la taille du sprite des instances ou des fixtures qu'elles ont attachées. Ainsi, si vous créez un joint de corde ailleurs qu'à l'origine de l'instance, il est toujours valide et contraindra les deux instances par rapport à la position à laquelle il a été créé. Si vous définissez la valeur "col" à true (vrai), les deux instances peuvent interagir et entrer en collision l'une avec l'autre, mais seulement si elles ont des événements de collision, alors que si elle est réglée à false (faux), elles n'entreront jamais en collision. Syntaxe : physics_joint_rope_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, maxlength, col) Arguments Description inst1 La première instance à se connecter avec l'articulation inst2 La deuxième instance pour se connecter avec le joint w_anchor1_x La première coordonnée x de l'articulation, dans l'univers du jeu. w_anchor1_y La première coordonnée y de l'articulation, dans l'univers du jeu. w_anchor2_x La deuxième coordonnée x de l'articulation, dans l'univers du jeu. w_anchor2_y la deuxième coordonnée y de l'articulation, dans l'univers de jeu longueur maximale La longueur maximale que l'articulation peut "s'étirer". col Si les deux instances peuvent entrer en collision (vrai) ou non (faux). Les retours : Nombre entier (indice de l'articulation) Exemple : var mainFixture, o_id ; mainFixture = physics_fixture_create() ; physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2) ; o_id=instance_create_layer(x+300, y, "Instances", obj_Rudder) ; physics_fixture_bind(mainFixture, id) ; physics_fixture_bind(mainFixture, o_id) ; physics_joint_rope_create(id, o_id, x + 50, y, o_id.x - 50, o_id.y, 300, 0) ; physics_fixture_delete(mainFixture) ; Le code ci-dessus crée et définit un nouveau dispositif, puis crée une instance de "obj_Rudder". Le dispositif est ensuite assigné à l'instance qui exécute le code ainsi qu'à l'instance nouvellement créée et une corde est créée entre elles.
