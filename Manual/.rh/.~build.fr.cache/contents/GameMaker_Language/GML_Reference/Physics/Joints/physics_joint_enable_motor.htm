physics_joint_enable_motor Lorsque vous avez une articulation avec un moteur(prismatique ou rotatif), vous pouvez vouloir être en mesure d'allumer ou d'éteindre le moteur en fonction de variables et conditions dans le jeu. Pour cela, vous devez avoir stocké l'index de l'articulation dans une variable et ensuite vous pouvez activer ou désactiver le moteur en utilisant cette fonction et en réglant l'argument "motor" sur true ou false. Syntaxe : physics_joint_enable_motor(joint, moteur) Arguments Description joint L'articulation sur laquelle vous souhaitez activer ou désactiver le moteur. moteur Si vous souhaitez activer (true) ou désactiver (false) le moteur. Les retours : N/A Exemple : var t_fix = physics_fixture_create() ; physics_fixture_set_circle_shape(t_fix, sprite_get_width(sprite_index) / 2) ; var o_id=instance_create_layer(x+300, y, "Instances", obj_Door) ; physics_fixture_bind(t_fix, id) ; physics_fixture_bind(t_fix, o_id) ; perma_joint = physics_joint_revolute_create(id, o_id, x+25, y, -90, 90, 1, 10, 2, 0, 0) ; physics_joint_enable(perma_joint, 1) ; physics_fixture_delete(t_fix) ; Le code ci-dessus crée et définit un nouveau dispositif et crée ensuite une instance de "obj_Door", liant le dispositif créé aux deux instances. Elles sont ensuite reliées par un joint tournant sans moteur et les angles sont limités à une oscillation de +/- 90 degrés. et nous stockons l'index du joint dans la variable "perma_joint". Nous activons ensuite le moteur à l'aide de cette variable, avant de supprimer finalement le dispositif de fixation de la mémoire car il n'est plus nécessaire.
