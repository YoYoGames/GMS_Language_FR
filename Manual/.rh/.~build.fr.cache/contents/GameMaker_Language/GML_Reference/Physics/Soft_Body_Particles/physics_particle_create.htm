physics_particle_create Avec cette fonction, vous pouvez créer une seule particule n'importe où dans votre salle de jeu, en définissant certains drapeaux et propriétés. Les "indicateurs" sont la valeur de retour d'une combinaison des constantes suivantes: Constant Description phy_particle_flag_water Les propriétés par défaut d'une particule de corps mou. phy_particle_flag_zombie Une particule zombie est une particule qui sera détruite après une seule étape avec toutes les autres marquées de cette manière. phy_particle_flag_wall Cela définit la particule comme statique, la créant essentiellement comme un objet immobile dans la simulation physique, car elles resteront dans une position fixe, peu importe ce qui les heurte. Vous devez utiliser cet indicateur plutôt que de définir la densité sur 0. phy_particle_flag_spring Les particules de ressort produisent l'effet d'être attachées les unes aux autres, comme par un ressort. Les particules créées avec ce drapeau sont "connectées" par paires, chaque particule étant connectée à celle qui en était la plus proche au moment de la création. Une fois appariées, les particules ne changent pas de «partenaires», et plus une force externe les tire les unes des autres, plus la puissance avec laquelle elles entreront en collision lorsque cette force externe est supprimée est grande. Notez que quelle que soit la distance entre les particules appariées, la connexion entre elles ne se cassera pas. phy_particle_flag_elastic Les particules élastiques se déforment et peuvent également rebondir lorsqu'elles entrent en collision avec d'autres corps rigides dans la simulation physique. phy_particle_flag_viscous Une particule visqueuse est une particule qui présente une «adhérence» ou une «adhérence», comme l'huile. Les particules visqueuses s'agglutineront et se colleront davantage. phy_particle_flag_powder Les particules de poudre produisent un effet de dispersion tel que vous pourriez le voir avec du sable ou de la poussière. phy_particle_flag_tensile Les particules de traction sont utilisées pour produire l'effet de la tension superficielle, ou la courbure tendue sur la surface d'un corps de liquide. Ils peuvent être utilisés, par exemple, pour créer la tension superficielle que vous verriez sur une goutte d'eau. Une fois la tension rompue, les particules rebondissent comme si elles étaient élastiques, mais continuent également à s'attirer les unes les autres. En conséquence, les particules ont tendance à former des grappes lorsqu'elles rebondissent. phy_particle_flag_colourmixing Les particules de mélange de couleurs prennent une partie de la couleur des autres particules avec lesquelles elles entrent en collision. Notez que si une seule des deux particules en collision est une particule mélangeant des couleurs, l'autre particule conserve sa couleur d'avant la collision. Ces indicateurs utilisent le masquage de bits pour créer une valeur de sortie finale qui est ensuite vérifiée pour définir les différentes propriétés de base de la particule (la propriété de base étant toujours celle de phy_particle_flag_water ). Par exemple, si vous souhaitez simuler un liquide visqueux avec une tension superficielle, vous utiliserez le bit au niveau du bit ou "|" pour marquer les bits appropriés comme indiqué dans l'exemple ci-dessous. De cette façon, vous pouvez définir différentes propriétés (autres que les propriétés globales) pour chaque particule individuelle créée. Outre les drapeaux, vous pouvez également définir la position dans la pièce à laquelle créer la particule, la vitesse horizontale et verticale initiale, la couleur initiale et l'alpha, ainsi que la catégorie utilisateur. Les valeurs de couleur et alpha ne sont utilisées que lors du dessin de la particule à l'aide de la fonction physics_particle_draw(), mais la valeur de catégorie est utilisée dans plusieurs autres fonctions. Cette valeur est une valeur entière arbitraire que vous donnez à chaque particule dont vous souhaitez avoir des propriétés similaires, et elle peut être utilisée pour définir des indicateurs ultérieurement pour cette catégorie spécifique ainsi que pour un certain nombre d'autres choses. Par exemple, si vous avez défini une particule «eau» et une particule «huile», vous donneriez à toutes ces particules marquées comme «eau» la catégorie utilisateur de 1 et à toutes les particules créées comme «huile» la catégorie utilisateur de 2, ce qui permet vous pourrez ensuite cibler une seule catégorie pour modifier ses indicateurs ou obtenir des données spécifiques. REMARQUE: vous pouvez utiliser n'importe quelle valeur entière pour la catégorie sauf 0 qui est réservé par la simulation et est utilisé pour sélectionner toutes les catégories dans d'autres fonctions (comme pour le dessin). La fonction renvoie une valeur d'index (ou ID) unique pour la particule qui peut être stockée dans une variable pour être utilisée dans d'autres fonctions qui référencent directement la particule. Syntaxe:physics_particle_create(flags, x, y, xv, yv, col, alpha, category) Argument Description flags Les drapeaux à placer sur la particule. x La position x à laquelle créer la particule. y La position y à laquelle créer la particule. xv La vitesse horizontale initiale. yv La vitesse verticale initiale. col La couleur de base à utiliser pour la particule. alpha Alpha de base à utiliser pour la particule. category La catégorie définie par l'utilisateur à laquelle appartient la particule. Retour:Real Exemple:var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile; var x_vel = lengthdir_x(5, image_angle); var y_vel = lengthdir_y(5, image_angle); var p = physics_particle_create(flags, x, y, x_vel, y_vel, c_white, 1, 1); Le code ci-dessus créera des variables pour définir le type de particule et sa vitesse initiale, puis les utilisera pour créer une particule à la position x / y des instances.
