physique_monde_créer Même si vous avez défini la physique de vos objets, sans un monde dans lequel les placer, ils utiliseront par défaut le système de collision traditionnel de GameMaker Studio 2. Vous avez donc besoin de cette fonction pour créer le système physique de la pièce spécifiée soit dans un événement d'objet, ou dans le code de création de la pièce elle-même (il peut également être défini sans utiliser de code dans l'éditeur de pièce). Cette fonction associera un "monde" physique à la room et toutes les instances créées dans la room se comporteront en utilisant la physique qui a été définie pour la room. la physique qui a été définie pour elles et le monde lui-même. Vous remarquerez que nous avons un argument qui définit un rapport entre la taille des pixels et les mètres. En effet, les fonctions physiques utilisent des mesures du monde réel et nous devons donc traduire les coordonnées et la taille des pixels dans ces mesures du monde réel en utilisant une "échelle pixel-mètre". Vous voudrez ajuster le paramètre pixeltometrescale jusqu'à ce que la taille moyenne des pixels des objets que vous utilisez se traduise approximativement en objets physiques simulés d'une taille appropriée. Ci-dessus, vous pouvez voir que nous l'avons réglé sur 32 pixels, soit l'équivalent d'un mètre. Il convient de noter que que le système physique fonctionnera avec des rapports plus grands ou plus petits, mais que les meilleures performances sont obtenues lorsque les mesures réelles de vos objets ne sont pas inférieures à 0,1 mètre et ne dépassent pas 50 mètres. Syntaxe : physics_world_create(pixeltometrescale) ; Arguments Description pixeltométrieéchelle définit le rapport entre les pixels à l'écran et les mètres dans le monde réel. Un rapport de 32:1 sera spécifié comme 1/32 (ou 0,03125). Les retours : N/A Exemple : physics_world_create(1/32) ; Le code ci-dessus va créer un monde physique dans la pièce actuelle avec un rapport d'échelle de 32 pixels, soit l'équivalent d'un mètre.
