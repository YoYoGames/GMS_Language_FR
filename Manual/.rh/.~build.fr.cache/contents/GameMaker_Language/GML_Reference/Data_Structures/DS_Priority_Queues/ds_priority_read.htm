ds_priority_read Avec cette fonction, vous pouvez recréer une priorité DS sauvegardée (une priorité qui a été précédemment écrite sous forme de chaîne de caractères à l'aide de la fonction ds_priority_write()). Vous devez d'abord créer une nouvelle priorité DS pour y lire la chaîne de caractères. la priorité DS existe déjà et contient des informations, celles-ci seront effacées avant la lecture. Cette fonction est d'une importance vitale lors de la création de mécanismes de sauvegarde/chargement pour votre jeu. Notez que si la structure de données a été créée avec les précédentes version de GameMaker vous devez ajouter l'argument optionnel "legacy", en le mettant à true car le format de la chaîne a changé avec cette version. Syntaxe : ds_priority_read(id, str [, legacy]) ; Arguments Description id L'identifiant de la structure de données à écrire. str La chaîne de caractères à partir de laquelle écrire. héritage(facultatif) Peut être soit vrai, soit faux, soit complètement omis. Les retours : N/A Exemple : p_queue = ds_priority_create() ; ini_open("save.ini") ; var str = ini_read_string("P_Queues", "0", "") ; if (str != "") { ds_priority_read(p_queue, str) ; } ini_close() ; Le code ci-dessus crée une file d'attente prioritaire et stocke l'index dans la variable "p_queue". Il ouvre ensuite un fichier ini et lit une chaîne de caractères à partir de celui-ci, en vérifiant d'abord que la chaîne n'est pas retournée comme vide. Cette chaîne est ensuite lue dans la priorité DS nouvellement créée. Retour : Files d'attente prioritaires DS Suivant : ds_priority_write Copyright YoYo Games Ltd. 2021 Tous droits réservés
