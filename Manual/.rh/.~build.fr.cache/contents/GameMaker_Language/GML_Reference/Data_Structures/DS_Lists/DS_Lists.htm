Listes DS Une liste DS est une structure de données qui stocke les informations de manière séquentielle au fur et à mesure de leur ajout (un peu comme un tableau ). Les listes DS sont des structures de données très flexibles qui vous permettent d'ajouter des valeurs à la fin ou de les insérer quelque part au milieu de la liste, ainsi que de vous donner la possibilité de mélanger l'ordre (pour randomiser les valeurs) ou de les trier par ordre croissant ou ordre décroissant. Lorsque vous accédez aux structures de données de liste DS, vous devez toujours essayer d'utiliser des valeurs entières pour la position de la liste, et tous les indices non entiers seront GameMaker Studio 2 si vous ne le faites pas. Si ce n'est pas ce dont vous avez besoin, vous devrez faire l'arrondi vous-même avant de passer l'index que vous souhaitez vérifier. REMARQUE: comme pour toutes les ressources dynamiques, les structures de données occupent de la mémoire et doivent donc toujours être détruites lorsqu'elles ne sont plus nécessaires pour éviter les fuites de mémoire qui ralentiront et finiront par planter votre jeu. NOTE: si vous avez besoin de vérifier si une structure de données existe, vous pouvez utiliser la fonction ds_exists (). Outre les fonctions spécifiques répertoriées ci-dessous, vous pouvez tous utiliser une expression (appelée accesseur ) pour ajouter ou modifier le contenu de votre liste DS. Cet accesseur ressemble à un tableau 1D avec la syntaxe suivante:list_index[| index] Vous pouvez trouver plus d'informations, ainsi que des exemples, à partir de la GML Overview sur Accessors. En général, cela n'est pas obligatoire, mais cela présente l'avantage de collecter automatiquement (détruire) les structures de données marquées lorsque la liste parente est détruite, et il est également nécessaire lorsqu'il est utilisé en conjonction avec le json_encode() fonction afin que la liste et ses données soient correctement codées.
