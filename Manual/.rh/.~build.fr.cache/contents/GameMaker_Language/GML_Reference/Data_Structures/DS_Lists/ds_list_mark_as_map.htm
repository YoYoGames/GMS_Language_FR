ds_list_mark_as_map Cette fonction "marquera" (ou "marquera") une position donnée dans une liste DS précédemment créée comme contenant une carte DS. Cette fonctionnalité est requise lors de l'encodage de chaînes JSON (que vous pouvez créer à l'aide de json_encode() ), mais peut également être utile lors de l'imbrication de structures de données, car les éléments marqués de cette manière seront automatiquement récupérés (détruits) lorsque la liste DS parent est détruite. Cela signifie que vous n'avez pas à parcourir manuellement le contenu de la liste et à détruire individuellement les structures de données marquées avant de détruire la liste "parente". Toutefois, si vous supprimez individuellement la position de la liste, la structure de données qu'elle contenait ne sera pas récupérée et vous devez appeler la fonction de destruction de carte DS appropriée avant de supprimer la position de la liste parente. Notez également que si vous appelez la fonction ds_list_clear() sur une liste, tous les éléments marqués comme des cartes seront également détruits lorsque la liste sera effacée. Il est également intéressant de noter que si la position qui a été marquée change dans la liste, la "marque" se déplacera avec elle, donc si vous avez marqué la position 3 dans la liste (par exemple), puis insérez 2 autres éléments en dessous de sorte il se déplace jusqu'à la position 5, il sera toujours marqué comme une carte. Syntaxe:ds_list_mark_as_map(id, pos); Argument Description id L'identifiant de la liste à marquer. pos La position dans la liste à marquer. Retour:Real (integer - ID of the DS map if successful or -1 if it fails) Exemple:var sub_map = ds_map_create(); ds_map_add(sub_map, "player", player_array); ds_map_add(sub_map, "enemy", enemy_array); ds_list_add(j_list, sub_map); ds_list_mark_as_map(j_list, 0); Le code ci-dessus crée une carte DS, puis la remplit avec deux clés, chacune contenant un tableau de valeurs. Cette carte est ensuite ajoutée dans la liste DS donnée, et la position "marquée" comme telle afin qu'elle puisse être correctement codée ultérieurement.
