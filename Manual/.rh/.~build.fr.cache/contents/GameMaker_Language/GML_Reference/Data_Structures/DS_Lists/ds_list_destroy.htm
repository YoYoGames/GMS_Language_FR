ds_list_destroy Cette fonction supprimera la structure de données de la liste donnée de la mémoire, libérant les ressources qu'elle utilisait et supprimant toutes les valeurs qu'elle contenait. Cette fonction doit toujours être utilisée lorsque vous avez fini d'utiliser ds_list pour éviter les fuites de mémoire qui peuvent ralentir et planter votre jeu. Notez que la destruction d'une liste dé-référencera toutes les structures de données qui y sont stockées, ce qui entraînera une fuite de mémoire, vous devrez donc parcourir la liste et détruire tous les éléments de la structure de données manuellement avant de la détruire pour éviter cela. Le seul moment où cela n'est pas nécessaire est lorsque vous avez marqué des éléments de la liste comme une autre liste DS ou comme une carte DS, auquel cas ces éléments seront détruits et leur mémoire nettoyée automatiquement également. IMPORTANT! Lorsque vous créez une structure de données, la valeur d'index pour l'identifier est une valeur entière commençant à 0. Cela signifie que les structures de données de différents types peuvent avoir la même valeur d'index, donc en cas de doute, vous devriez utiliser la fonction ds_exists () avant y accéder. Notez également que les index sont réutilisés, donc une valeur d'index de structure de données détruite peut être utilisée par une valeur nouvellement créée par la suite, nous vous recommandons donc de toujours définir la variable qui contenait l'index DS à -1 après la destruction. Syntaxe:ds_list_destroy(id); Argument Description id L'identifiant de la structure de données à supprimer. Retour:N/A Exemple:if (lives == 0)     {     ds_list_destroy(AI_list);     AI_list = -1;     room_goto(rm_Menu);     } Le code ci-dessus vérifiera la valeur de la variable globale intégrée "lives" et si elle vaut 0, il détruit la liste DS indexée dans la variable "AI_list" puis change de pièce.
