ds_list_destroy Cette fonction supprime de la mémoire la structure de données de la liste donnée, libérant les ressources qu'elle utilisait et supprimant toutes les valeurs qu'elle contenait. Cette fonction doit toujours être utilisée lorsque vous avez fini d'utiliser la ds_list pour éviter les fuites de mémoire qui peuvent ralentir et faire planter votre jeu. qui peuvent ralentir et faire planter votre jeu. Notez que la destruction d'une liste déréférencera toutes les structures de données qui y sont stockées, ce qui entraînera une fuite de mémoire. Vous devrez donc parcourir la liste et détruire manuellement tous les éléments de structure de données avant de la détruire pour éviter cela. La seule fois où cela n'est pas nécessaire est lorsque vous avez marqué des éléments de la liste comme étant une autre liste DS ou une carte DS, auquel cas ces éléments seront détruits et leur mémoire sera également nettoyée automatiquement. et leur mémoire sera également nettoyée automatiquement. IMPORTANT ! Lorsque vous créez une structure de données, la valeur de l'index pour l'identifier est une valeur entière commençant à 0. Cela signifie que des structures de données de types différents peuvent avoir la même valeur d'index, donc en cas de doute, vous devriez utiliser la fonction ds_exists() avant d'y accéder. Notez également que les indices sont réutilisés, donc une valeur d'indice de structure de données détruite peut être utilisée par une structure nouvellement créée par la suite ; nous recommandons donc de de toujours donner la valeur -1 à la variable qui contenait l'indice DS après la destruction. Syntaxe : ds_list_destroy(id) ; Arguments Description id L'identifiant de la structure de données à supprimer. Les retours : N/A Exemple : if (lives == 0) { ds_list_destroy(AI_list) ; AI_list = -1 ; room_goto(rm_Menu) ; } Le code ci-dessus vérifie la valeur de la variable globale intégrée "lives" et si elle est égale à 0, il détruit la liste DS indexée dans la variable "AI_list" et change ensuite de pièce.
