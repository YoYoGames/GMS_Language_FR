ds_stack_read Avec cette fonction, vous pouvez recréer une pile DS sauvegardée (une pile qui a été précédemment écrite sous forme de chaîne de caractères à l'aide de la fonction ds_stack_write()). Vous devez d'abord créer une nouvelle pile DS pour y lire la chaîne, et si la pile DS existe déjà et contient des informations, celles-ci seront effacées avant la lecture. Cette fonction est d'une importance vitale lors de la création de mécanismes de sauvegarde/chargement pour votre jeu. Notez que si la structure de données a été créée avec des versions précédentes de GameMaker vous devez ajouter l'argument optionnel "legacy", en le mettant à true car le format de la chaîne de caractères a changé avec cette version. Syntaxe : ds_stack_read(id, str [, legacy] ) ; Arguments Description id L'identifiant de la structure de données à lire. str La chaîne de caractères à lire. héritage(facultatif) Peut être soit vrai, soit faux, soit complètement omis. Les retours : N/A Exemple : stack = ds_stack_create() ; ini_open("save.ini") ; var str = ini_read_string("Stacks", "0", "") ; if (str != "") { ds_stack_read(stack, str) ; } ini_close() ; Le code ci-dessus crée une pile et stocke l'index dans la variable "stack". Il ouvre ensuite un fichier ini et lit une chaîne de caractères à partir de celui-ci, en vérifiant d'abord que la chaîne n'est pas retournée comme vide. Cette chaîne est ensuite lue dans la pile DS nouvellement créée. Retour : ds_stack_create Suivant : ds_stack_write Copyright YoYo Games Ltd. 2021 Tous droits réservés
