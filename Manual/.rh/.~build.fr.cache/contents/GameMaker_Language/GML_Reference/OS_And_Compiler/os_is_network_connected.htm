os_is_network_connected Avec cette fonction, vous pouvez vérifier et voir si votre appareil dispose actuellement d'une connexion Internet et il reviendra true si c'est le cas, ou false si ce n'est pas le cas, et selon le système d'exploitation, il peut tenter d'établir une connexion avant de renvoyer une valeur. La fonction a un argument facultatif pour l'activer / la désactiver en essayant d'établir une connexion lorsqu'elle est appelée, et cette option n'est actuellement que pour la cible Nintendo Switch Lorsqu'il est réglé sur true la fonction tentera d'établir une connexion, et lorsqu'elle est définie sur false il se limitera simplement à vérifier la connexion. Notez que toute tentative de connexion peut entraîner le blocage du système d'exploitation Switch REMARQUE: Cette fonction vérifie l'API interne du périphérique qui contrôle les connexions et peut donc renvoyer true s'il existe une connexion Bluetooth, une connexion Wi-Fi ou même simplement une connexion réseau normale permettant l'accès à Internet. Syntaxe:os_is_network_connected(attempt_connection); Argument Description attempt_connection OPTIONNEL! Pour Nintendo Switch Target uniquement. Mis à true pour tenter une connexion au niveau du système d'exploitation lors de l'appel. Retour:Boolean Exemple:if os_is_network_connected()    {    global.connected = true;    } Le code ci-dessus vérifie si l'appareil est connecté à Internet et si c'est le cas, il définit une variable globale.
