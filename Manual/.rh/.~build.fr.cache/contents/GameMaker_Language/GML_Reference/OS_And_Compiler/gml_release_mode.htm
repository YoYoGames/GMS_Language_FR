gml_release_mode Lorsque votre jeu est compilé, il inclut également certaines fonctions internes pour vérifier les erreurs lors de son exécution. Ces contrôles internes couvrent de nombreux aspects du code d'un jeu, notamment les plages, les paramètres et certains aspects généraux du compilateur GML interne. et sont souvent à l'origine des messages d'erreur finaux. Cependant, cette fonction peut être utilisée pour désactiver ces contrôles internes (c'est-à-dire les supprimer complètement du jeu compilé), ce qui donne un gain de vitesse à votre projet final. Veuillez noter que cela signifie que si vous faites quelque chose d'incorrect dans le code de votre jeu, le jeu compilé peut tout simplement se planter ou avoir un comportement très inattendu. Par exemple, si l'on accède à une grille de DS à une en dehors de la taille définie, en mode développeur, vous obtiendrez un message d'erreur dans la fenêtre du compilateur, mais les contrôles en place permettront au jeu de continuer à fonctionner. Cependant, en mode release, ces contrôles n'existeront pas et le code pourrait donc retourner soit un pointeur NULL, soit un mauvais pointeur qui à son tour pointera vers une mémoire aléatoire. Ainsi, au mieux, vous pourriez obtenir les mauvaises données renvoyées, ou au pire, vous pourriez écraser quelque chose en mémoire - données, graphiques, ou même le code lui-même. Un autre exemple si vous fournissez les données de couleur, de position, de texture, etc... dans un ordre différent de celui que vous avez défini lors de la création du format de sommet, le jeu peut se planter ou produire des résultats inattendus lorsque le mode release est activé. Syntaxe : gml_release_mode(flag) ; Arguments Description drapeau Soit true pour définir le mode de publication, soit false pour le développement général. Les retours : N/A Exemple : gml_release_mode(true) ; L'exemple de code ci-dessus configurera le jeu pour qu'il utilise le mode "release" lors de la compilation.
