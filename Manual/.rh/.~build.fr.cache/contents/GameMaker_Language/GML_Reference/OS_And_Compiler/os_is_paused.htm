os_is_paused Lors du développement pour des appareils mobiles, vous devez avoir un moyen de vérifier que l'appareil est toujours concentré sur votre jeu ou non, d'autant plus que si votre jeu est sur un appareil iOS, Android ou Windows Phone, il peut être interrompu par des appels téléphoniques et autres applications. Vous pouvez utiliser cette fonction pour vérifier si l'appareil est en pause ou non. Il est à noter que cette fonction exécutera tout code contenu dans le bloc de code sur les appareils, mais que GameMaker Studio 2 sera mis en pause immédiatement après que l'appareil mettra le jeu en cache en arrière-plan. Donc, si vous avez quelque chose comme l'exemple de code fourni ci-dessous, l'instance à créer n'apparaîtra pas ou n'exécutera aucun de ses événements tant que le jeu n'aura pas été redémarré à partir de l'état mis en cache. Cela signifie que vous préférerez peut-être avoir le os_is_paused fonction appeler une alarme, puis dans l'événement d'alarme (qui sera défini lorsque l'appareil est mis en pause mais ne fonctionnera pas tant qu'il ne sera pas mis en pause), créez un menu de pause, redémarrez la musique, etc. Si vous utilisez cette fonction sur Windows, Ubuntu ( Linux ), MacOS ou HTML5, la fonction n'est activée que pendant une étape pour true lorsque le jeu perd le focus, puis il reviendra false à nouveau jusqu'à ce que le jeu soit remis au point (et le perde à nouveau). En effet, ces plates-formes ne mettent pas en cache les applications et les jeux et continueront donc à fonctionner en arrière-plan. REMARQUE: sur les appareils mobiles, les sons et la musique seront arrêtés lorsque le système d'exploitation est mis en pause. Ils ne redémarreront pas à moins que vous ne les lanciez spécifiquement avec la fonction appropriée. Syntaxe:os_is_paused() Retour:Boolean Exemple:if os_is_paused()     {     if !instance_exists(obj_PauseMenu)         {         instance_create_layer(0, 0, "Controllers", obj_PauseMenu);         }     } Le code ci-dessus vérifie si le jeu a été mis en pause, et si c'est le cas, il crée une instance de l'objet "obj_PauseMenu" s'il n'en existe pas déjà.
