clé_virtuelle_add Cette fonction vous permet d'associer les "touches" d'une zone de l'écran à des événements clavier. Cela signifie qu'une fois que vous avez assigné une zone à une touche virtuelle, tous les contacts sur cette zone déclencheront l'événement clavier correspondant à la touche que vous avez assignée à la zone. à cette zone. Vous pouvez également affecter chaque touche virtuelle que vous définissez à une variable, qui peut ensuite être utilisée dans les autres fonctions de touches virtuelles pour les afficher, les masquer et les supprimer. Ces clés sont attribuées à chaque pièce et seront automatiquement supprimées par GameMaker Studio 2 lors d'un changement de pièce. automatiquement par GameMaker Studio 2 lors d'un changement de pièce. La position réelle de la touche virtuelle est basée sur la position de l'écran plutôt que sur la position de la pièce. Les valeurs x/y sont donc absolues à l'écran. Cela signifie que vous n'avez pas à vous soucier de l'utilisation des vues ou des coordonnées relatives de la pièce, Vous pouvez simplement définir les clés dans l'événement Create d'un objet (vous ne devez définir une clé virtuelle qu'une seule fois par pièce), puis dessiner vos sprites de clés dans l'événement Draw GUI, en dimensionnant la couche GUI pour qu'elle soit identique à l'écran. Syntaxe virtual_key_add(x, y, w, h, keycode) ; Arguments Description x La coordonnée x (côté gauche) de la touche virtuelle sur l'écran y La coordonnée y (côté supérieur) de la touche virtuelle sur l'écran w La largeur de la clé virtuelle h La hauteur de la clé virtuelle code clé Quelle touche du clavier doit être déclenchée en touchant cette zone ? Les retours : index de la clé virtuelle Exemple : global.Left = virtual_key_add(32, 32, 64, 64, vk_left) ; Le code ci-dessus crée une clé virtuelle carrée de 64x64 pixels, positionnée sur l'écran à (32, 32) qui déclenchera l'événement vk_left lorsqu'elle sera touchée et assigne l'index de cette clé virtuelle à une variable globale.
