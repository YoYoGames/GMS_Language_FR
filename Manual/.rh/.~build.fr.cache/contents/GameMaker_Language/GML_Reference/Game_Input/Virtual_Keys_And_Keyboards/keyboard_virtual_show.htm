keyboard_virtual_show Cette fonction peut être utilisée pour afficher le clavier virtuel sur l'appareil exécutant le jeu. Lorsque vous appelez cette fonction, vous devez fournir l'une des constantes suivantes pour chacun des trois premiers arguments: Type de clavier: cette constante est utilisée pour définir le jeu de touches qui sera disponible sur le clavier virtuel. Les types de clavier disponibles sont: Type de clavier Description kbv_type_default Le type de clavier par défaut pour le système actuel. kbv_type_ascii Un clavier ASCII uniquement. kbv_type_url Un clavier normal optimisé pour la saisie d'URL. Comprend généralement un ".com" ou d'autres clés de domaine, ainsi que "/" et "." clés. kbv_type_email Un clavier normal optimisé pour la saisie des e-mails. Comprend généralement "@" et "." personnages. kbv_type_numbers Un clavier composé uniquement de chiffres, généralement affiché sous forme de pavé numérique. kbv_type_phone Un clavier de téléphone. Généralement des chiffres uniquement avec les touches "*" et "#". kbv_type_phone_name Un clavier optimisé pour saisir à la fois un numéro de téléphone et un nom. Généralement similaire à un clavier ASCII, mais avec une sélection de caractères spéciaux limitée. Type de retour: Cette constante est utilisée pour définir ce qui est affiché sur la touche retour / action du clavier virtuel. Les types de retour disponibles sont: Type de retour Description kbv_returnkey_default Titre de la clé de retour par défaut pour le système actuel. kbv_returnkey_go Définit le titre de la clé de retour sur "Go". kbv_returnkey_google Définit le titre de la clé de retour sur "Google" ou sur une icône de recherche générique dans certains cas. kbv_returnkey_join Définit le titre de la clé de retour sur "Go". kbv_returnkey_next Définit le titre de la clé de retour sur "Suivant". kbv_returnkey_route Définit le titre de la touche de retour sur "Route". kbv_returnkey_search Définit le titre de la clé de retour sur "Rechercher" ou sur une icône de recherche générique dans certains cas. kbv_returnkey_send Définit le titre de la touche de retour sur «Envoyer». kbv_returnkey_yahoo Définit le titre de la clé de retour sur "Yahoo" ou sur une icône de recherche générique dans certains cas. kbv_returnkey_done Définit le titre de la clé de retour sur "Terminé". kbv_returnkey_continue Définit le titre de la clé de retour sur "Continuer". kbv_returnkey_emergency Définit le titre de la touche de retour sur "Appel d'urgence". Type d'autocapitalisation: Cette constante est utilisée pour déterminer comment les mots saisis via le clavier virtuel doivent être autocapitalisés. Les types d'autocapitalisation disponibles sont: Type d'autocapitalisation Description kbv_autocapitalize_none L'autocapitalisation est désactivée. kbv_autocapitalize_words Les mots seront automatiquement mis en majuscules. kbv_autocapitalize_sentences Les phrases seront automatiquement mises en majuscules. kbv_autocapitalize_characters Tous les caractères seront en majuscules. Le dernier argument consiste à activer / désactiver le texte prédictif, et ce serait défini sur true pour le permettre, et false sinon, mais notez que ce n'est pas parce qu'il est autorisé qu'il sera utilisé car cela dépendra des préférences de l'utilisateur sur l'appareil. En mode texte prédictif, le clavier virtuel ne génère pas d'événements de pression de touche GameMaker Studio 2 Au lieu de cela, il ne mettra à jour que le dernier caractère appuyé et les variables de chaîne du clavier. Cela est dû à l'incapacité de détecter si un changement dans le champ de texte interne utilisé pour détecter les pressions sur les touches provenait d'une touche réelle du clavier virtuel ou d'une suggestion de texte. Dans ces cas, vous voudrez lire le keyboard_string par opposition à la lecture de tout type d'entrée de clé brute. Il est également important de noter que l'utilisateur obtiendra différents claviers avec des capacités différentes selon le système d'exploitation de la plate-forme, avec les mises en garde suivantes pour une utilisation par cible: UWP: l'implémentation UWP ne prend pas en charge return_key_type, autocapitalization_type ou predictive_text_enabled. Passer ces paramètres dans une fonction ne causera aucune erreur, ils n'auront tout simplement aucun effet sur le clavier. Android: sur Android, le type de clavier kbv_type_phone_name n'est pas pris en charge et affichera un clavier ASCII à la place, et le type de touche de retour ne peut être kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send ou kbv_returnkey_done. Si d'autres types de clés de retour sont utilisés sur cette plate-forme, le système utilisera par défaut les clés de retour de remplacement répertoriées ci-dessous: Clé de retour indisponible Clé de remplacement kbv_returnkey_google kbv_returnkey_yahoo kbv_returnkey_search kbv_returnkey_join kbv_returnkey_route kbv_returnkey_emergency kbv_returnkey_go kbv_returnkey_continue kbv_returnkey_next Android TV / FireTV: les types de touches de retour personnalisées ne sont pas pris en charge sur les claviers ASCII - la touche de retour par défaut sera toujours affichée. tvOS: le texte prédictif et la mise en majuscule automatique ne sont pas pris en charge sur tvOS, et la hauteur du clavier est renvoyée par les événements système et la fonction keyboard_virtual_height() sera par défaut à la hauteur de l'écran car elle couvre tout l'écran et en raison de la désactivation des fonctions de dimension droite du clavier sur tvOS. Notez également que les événements de clavier physique (Bluetooth) ne seront pas diffusés tant que le clavier virtuel n'a pas été ouvert. L'appel de cette fonction générera un événement asynchrone système, dans lequel le async_load La carte DS sera remplie avec les paires clé / valeur suivantes: " event_type " - ce sera "l' état du clavier virtuel " lorsqu'il est déclenché par les fonctions du clavier virtuel. " screen_height " - la hauteur du clavier virtuel (en pixels). Ce sera 0 si le clavier est invisible. " keyboard_status " - l'état actuel du clavier, renvoyé par l'une des chaînes suivantes: "cache" "caché" "montrant" "visible" Syntaxekeyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled); Argument Description keyboard_type Détermine le jeu de touches disponible sur le clavier virtuel. return_key_type Détermine ce qui est affiché sur la touche retour / action du clavier virtuel. autocapitalization_type Détermine comment / si les mots saisis via le clavier virtuel seront autocapitalisés. predictive_text_enabled Mis à true /false pour activer / désactiver la saisie de texte prédictive. Retour:N/A Exemple:if input == false     {     input = true;     keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);     } Le code ci-dessus affichera le clavier virtuel du système d'exploitation si la variable donnée n'est pas définie sur true.
