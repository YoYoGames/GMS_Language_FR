Entrée du clavier Lorsque vous utilisez le clavier dans GameMaker Studio 2, vous disposez d'un grand nombre de fonctions qui peuvent être utilisées pour reconnaître les différents états du clavier, comme enfoncé ou relâché. Il y en a également qui stockent toutes les pressions sur les touches sous forme de chaîne ou qui peuvent vous dire quelle était la dernière touche enfoncée. ou qui peuvent vous dire quelle était la dernière touche enfoncée, ainsi que d'autres qui vous permettent d'effacer complètement l'état du clavier. NOTE : Ces fonctions sont conçues pour les plateformes de bureau Windows/Mac/Ubuntu uniquement. Il se peut que certaines des variables et constantes intégrées ne soient pas valables sur d'autres plateformes et que de nombreuses fonctions ne fonctionnent pas sur les mobiles. Chaque caractère d'entrée d'une touche (ou de plusieurs touches) est défini par son code UTF8, qui est une valeur numérique. Cette valeur peut être récupérée pour n'importe quel caractère à l'aide de la fonction ord() mais, GameMaker Studio 2 possède également une série de constantes pour les touches spéciales du clavier les plus utilisées et une fonction spéciale. Typiquement, vous utiliserez une combinaison de ord( ) avec les fonctions keyboard_check*(), quelque chose comme ça : if keyboard_check(ord("A")) { hspeed = -5 ; } Ainsi, l'exemple ci-dessus vérifiera la touche "A" et si elle est enfoncée, la vitesse horizontale de l'objet sera fixée à -5. Notez que l'utilisation de ord( ) de cette manière ne fonctionnera correctement que si la chaîne d'entrée n'a qu'un caractère et est un nombre de 0 à 9 ou un caractère romain majuscule de A à Z. La fonction ord() retournera une valeur UTF8 complète, mais les fonctions keyboard_check*() ne détecteront que A - Z et 0 - 9. Mais que se passe-t-il si vous voulez utiliser les touches fléchées ? Ou si vous voulez modifier une action en utilisant la touche "shift" ? Eh bien, pour cela, GameMaker Studio 2 dispose d'une série de constantes vk_(vk_ signifie touche virtuelle) que vous pouvez utiliser à la place de de ord(): Constant Description vk_nokey keycode représentant qu'aucune touche n'est pressée vk_anykey code de touche représentant l'appui sur une touche quelconque vk_gauche code de la touche flèche gauche vk_right code de la touche flèche droite vk_up code de la touche flèche vers le haut vk_down code de la touche flèche vers le bas vk_enter touche Entrée vk_escape touche d'échappement vk_space touche espace vk_shift l'une ou l'autre des touches majuscules vk_control l'une des touches de contrôle vk_alt touche alt vk_backspace touche retour arrière vk_tab touche de tabulation vk_home touche d'accueil vk_end touche finale vk_suppression touche de suppression vk_insert insérer la clé vk_pageup touche "pageup vk_pagedown touche pagedown vk_pause touche pause/rupture vk_printscreen touche printscreen/sysrq vk_f1 ... vk_f12 code des touches de fonction F1 à F12 vk_numpad0 ... vk_numpad9 les touches numériques du clavier numérique vk_multiplier touche multiplicatrice du clavier numérique vk_divide touche de division sur le clavier numérique vk_add touche d'ajout sur le clavier numérique vk_subtract touche de soustraction du clavier numérique vk_decimal les touches de point décimal du clavier numérique Les constantes suivantes ne peuvent être utilisées qu'avec keyboard_check_direct(): Constant Description vk_lshift touche shift gauche vk_lcontrol touche de commande gauche vk_lalt touche alt gauche vk_rshift touche shift droite vk_rcontrol touche de contrôle droite vk_ralt touche alt droite Voici un petit exemple de l'utilisation de la fonction vk_ : if keyboard_check_pressed(vk_tab) { instance_create_layer(x, y, "Controllers", obj_Menu) ; } Le code ci-dessus détectera si la touche "Tab" est pressée et créera une instance de l'objet "obj_Menu" si c'est le cas.
