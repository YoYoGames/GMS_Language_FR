keyboard_set_map Parfois, lors de la création d'un jeu, vous souhaiterez peut-être qu'une clé fasse la même chose qu'une autre. Par exemple, de nombreuses personnes utilisent les touches WASD pour le mouvement, mais beaucoup de personnes utilisent également les touches fléchées! Alors que faire? Eh bien, vous pouvez coder le système de mouvement deux fois, mais c'est un peu compliqué et heureusement redondant car cette fonction vous permet de "mapper" une touche à une autre et donc toute entrée de l'une ou l'autre des touches sera interprétée comme la même chose. Pour ce faire, vous choisissez une clé que vous souhaitez mapper (key2 - ce sera la clé que vous écrivez le code) et une clé que vous voulez qu'il soit mis en correspondance (key1). Après cela, les pressions sur l'une ou l'autre des touches seront interprétées par GameMaker Studio 2 comme provenant de key2. Vous pouvez également utiliser cette fonction pour concevoir un système dans lequel l'utilisateur peut définir ses propres touches pour jouer en mappant simplement la touche d'entrée utilisateur à la touche que vous avez codée dans le jeu. La fonction prend une valeur de code clé retournée par la fonction ord() (uniquement les lettres majuscules de A à Z ou les chiffres de 0 à 9), ou l'un des vk_* constantes répertoriées sur la page principale de saisie au clavier. IMPORTANT! La clé à laquelle vous mappez (key1) ne sera plus utilisable car c'est le symbole de clé réel une fois que vous utilisez cette fonction. Par exemple, si vous mappez la flèche vers le haut à "W", vous ne pourrez plus détecter la pression de la touche "W" comme un "W", elle sera toujours considérée comme la flèche vers le haut. Pour annuler cela, mappez la clé sur elle-même - de sorte que key1 et key2 seraient ord("W") - ou utilisez la fonction keyboard_unset_map(). Syntaxe:keyboard_set_map(key1, key2); Argument Description key1 C'est la clé à laquelle key1 doit être mappée key2 C'est la clé qui doit être mappée Retour:Boolean Exemple:keyboard_set_map(ord("A"), vk_left); L'exemple de code ci-dessus mappera la touche «A» à la touche fléchée gauche. Cela signifie que le joueur peut utiliser la touche «A» ou la touche fléchée gauche, et que tout le code écrit pour la flèche gauche répondra également à la touche «A» utilisée à la place.
