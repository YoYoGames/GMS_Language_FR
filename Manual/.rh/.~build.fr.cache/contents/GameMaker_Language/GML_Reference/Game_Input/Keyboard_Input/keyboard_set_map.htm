clavier_set_map Parfois, lorsque vous créez un jeu, vous souhaitez qu'une touche ait la même fonction qu'une autre. Par exemple, beaucoup de gens utilisent les touches WASD pour se déplacer, mais beaucoup de gens utilisent aussi les touches fléchées ! Alors, que faire ? Eh bien, vous pourriez coder le système de mouvement deux fois, mais c'est un peu compliqué et heureusement redondant, car cette fonction vous permet de "mapper" une touche sur une autre, de sorte que toute entrée provenant de l'une ou l'autre touche sera interprétée de la même manière. Pour ce faire, vous choisissez une touche que vous voulez faire correspondre (key2 - ce sera la touche pour laquelle vous écrivez le code) et une touche à laquelle vous voulez qu'elle soit associée (key1). Ensuite, les pressions sur l'une ou l'autre touche seront interprétées par GameMaker Studio 2 comme provenant de la touche 2. Vous pouvez également utiliser cette fonction pour concevoir un système dans lequel l'utilisateur peut définir ses propres touches pour jouer en faisant simplement correspondre la touche d'entrée de l'utilisateur à la touche que vous avez codée dans le jeu. La fonction prend une valeur de code de touche telle que retournée par la fonction ord() (uniquement les lettres majuscules de A à Z ou les chiffres de 0 à 9), ou l'une des constantes vk_* listées sur la page principale de saisie du clavier. IMPORTANT ! La touche que vous associez à (touche 1) ne sera plus utilisable comme symbole de touche réel une fois que vous aurez utilisé cette fonction. Par exemple, si vous associez la flèche vers le haut à "W", vous ne pourrez plus détecter l'appui sur la touche "W" comme un "W". de la touche "W" comme un "W", elle sera toujours considérée comme la flèche du haut. Pour annuler cela, il faut soit affecter la touche à elle-même - ainsi la touche 1 et la touche 2 seront toutes deux ord("W") - soit utiliser la fonction keyboard_unset_map(). Syntaxe : clavier_set_map(key1, key2) ; Arguments Description Clé1 Il s'agit de la clé à laquelle la clé 1 doit être associée. clé2 Voici la clé qui doit être cartographiée Les retours : Booléen Exemple : keyboard_set_map(ord("A"), vk_left) ; L'exemple de code ci-dessus associe la touche "A" à la touche flèche gauche. Cela signifie que le joueur peut utiliser soit la touche "A", soit la flèche gauche, et que tout code écrit pour la flèche gauche répondra également à l'utilisation de la touche "A". à la place.
