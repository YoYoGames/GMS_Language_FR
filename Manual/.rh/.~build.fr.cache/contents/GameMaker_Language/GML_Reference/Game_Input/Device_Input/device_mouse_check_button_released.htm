device_mouse_check_button_released Cette fonction renvoie vrai ou faux selon que l'appareil tactile que vous spécifiez a été libéré ou non. L'argument périphérique fait référence au numéro de contact de l'appareil, qui peut être compris entre 0 et n et notez que le nombre maximum de touches pouvant être détectées dépendra beaucoup de l'appareil utilisé et du système d'exploitation qu'il exécute (la plupart des 4 touches simultanées). Cette fonction n'est déclenchée qu'une seule fois par l'action de libération réelle et les constantes standard de mb_left et mb_right peut être utilisé pour vérifier l'un ou l'autre des boutons. Notez que mb_right n'aura été détecté que si une double pression est détectée et maintenue lors de la deuxième pression avant la libération (ce comportement peut être désactivé à l'aide de la fonction device_mouse_dbclick_enable(). Syntaxe:device_mouse_check_button_released(device, button); Argument Description device Le périphérique (de 0 à n ) qui est vérifié. button Le bouton de l'appareil en cours de vérification. Retour:Boolean Exemple:if device_mouse_check_button_released(0, mb_left) { press=false; } Le code ci-dessus vérifie si le périphérique a été pressé et définit une variable sur false si c'est le cas.
