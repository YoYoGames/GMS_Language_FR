push_cancel_local_notification Cette fonction annule la notification push donnée. Vous pouvez obtenir l'ID pour l'annulation de la notification à l'aide des fonctions push_get_first_local_notification() et push_get_next_local_notification() et la fonction retournera true en cas de succès ou false sinon (par exemple si la notification n'existe pas ou a déjà été déclenchée). REMARQUE: cette fonction est limitée aux modules cibles iOS et Android Syntaxe:push_cancel_local_notification(ID); Argument Description ID L'ID de la notification à annuler. Retour:Real Exemple:var map = ds_map_create(); var ntf = push_get_first_local_notification(map); while(ntf >= 0)     {     var data = ds_map_find_value(map, "data");     if data == "Daily_Reward"         {         push_cancel_local_notification(ntf);         }     ntf = push_get_next_local_notification(map);     } ds_map_destroy(map); Le code ci-dessus crée un DS Map puis demande des données pour la première notification disponible dans la file d'attente. Si une notification existe, la carte est remplie et le code vérifie alors la clé "données" pour une chaîne spécifique. Si la chaîne est trouvée, la notification est annulée et passe à la vérification de la notification suivante jusqu'à ce que tous les éléments de la file d'attente aient été vérifiés.
