push_get_next_local_notification Cette fonction va remplir une carte DS prédéfinie avec une série de paires clé/valeur pour la prochaine notification push locale dans la file d'attente à afficher. Elle renvoie -1 s'il n'y a pas d'autres notifications en file d'attente, ou sinon une valeur réelle représentant l'ID de la notification. Cet ID peut ensuite être utilisé pour annuler la notification à l'aide de la fonction push_cancel_local_notification(). Vous devez appeler la fonction push_get_first_local_notification() pour obtenir la première notification dans la file d'attente, puis utiliser cette fonction pour la parcourir. NOTE: Cette fonction est limitée aux iOS et Android modules cibles. a carte DS contiendra les paires clé/valeur suivantes : "title" - Le titre de la notification (disponible uniquement sur la plateforme Android plateforme car iOS ne stocke pas les informations relatives au titre) "message" - Le texte du corps du message "data"- La chaîne du paquet de données Syntaxe : push_get_next_local_notification(map) ; Arguments Description carte L'ID de carte DS à utiliser. Les retours : Real Exemple : var map = ds_map_create() ; var ntf = push_get_first_local_notification(map) ; while(ntf >= 0) { var data = ds_map_find_value(map, "data") ; if data == "Daily_Reward" { push_cancel_local_notification(ntf) ; } ntf = push_get_next_local_notification(map) ; } ds_map_destroy(map) ; Le code ci-dessus crée une carte DS et demande ensuite des données pour la première notification disponible dans la file d'attente. Si une notification existe, la carte est remplie et le code vérifie ensuite la clé "data" pour une chaîne spécifique. Si la chaîne est trouvée, la notification est annulée et passe à la notification suivante jusqu'à ce que toutes les notifications de la file d'attente aient été vérifiées.
