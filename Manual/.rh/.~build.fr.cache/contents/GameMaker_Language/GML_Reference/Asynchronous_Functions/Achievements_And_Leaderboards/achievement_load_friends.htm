réalisation_chargement_amis Cette fonction enverra une requête au serveur pour obtenir des informations sur tous les amis des utilisateurs connectés et déclenchera un callback Événement social asynchrone qui contient le site async_load map remplie avec les paires clé/valeur pertinentes. La clé id de cette carte DS est utilisée pour identifier le callback correct (il peut y avoir plus d'une fonction de déclenchement pour un même événement). (il peut y avoir plus d'une fonction de déclenchement pour un événement asynchrone donné), et sera associée à la constante achievement_friends_info ainsi qu'à un certain nombre d'autres paires clé/valeur pour chaque ami. Le contenu exact de la carte est ci-dessous : REMARQUE: Cette fonction est destinée à iOS seulement. "id" - Pour cette fonction, il s'agit de achievement_friends_info. "FriendN" - Le nom de l'ami, où "N" est une valeur entière correspondant à sa position dans la liste d'amis. "FriendidN" - L'identifiant unique de l'ami, "N". Syntaxe : achievement_load_friends() Les retours : N/A Exemple étendu : Le code suivant serait probablement appelé après que le joueur se soit connecté à son compte de jeu à l'aide de la fonction achievement_login() pour obtenir une liste de tous les amis de l'utilisateur : achievement_friends_info() ; Ceci enverra une requête pour les informations sur les amis de l'utilisateur et générera un callback asynchrone avec la carte spéciale async_load DS contenant les données suivantes : var ident = ds_map_find_value(async_load, "id") ; if ident == achievement_friends_info { var numfriends = ds_map_find_value(async_load, "numfriends") ; global.numfriends = numfriends ; for(var i=0 ; i < numfriends ; i++ ;) { global.friendname[i] = ds_map_find_value(async_load, "Friend" + string(i)) ; global.friendid[i] = ds_map_find_value(async_load, "Friendid" + string(i)) ; achievement_get_pic(global.friendid[i]) ; } } Le code ci-dessus vérifie la carte DS retournée dans l'événement social asynchrone et si son "id" correspond à la constante vérifiée, il boucle ensuite dans la carte en stockant toutes les différentes valeurs dans un certain nombre de tableaux et demande une image pour chacune des entrées de la carte (cf. achievement_get_pic()).
