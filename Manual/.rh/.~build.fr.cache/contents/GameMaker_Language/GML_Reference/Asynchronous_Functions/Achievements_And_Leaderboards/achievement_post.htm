achievement_post Vous pouvez utiliser cette fonction pour envoyer vos réalisations au classement et au service de réussite choisis. Sur Android vous aurez besoin de l'ID de la réussite sous forme de chaîne (il s'agit de l'ID de réussite unique qui a été attribué lors de la configuration de la réussite), tandis que sur toutes les autres plates-formes, vous fournissez le nom de la réussite sous forme de chaîne, puis vous donnez le pourcentage que vous avez accompli pour obtenir le succès (0 - Aucun, 100 - Terminé). La fonction déclenchera un événement asynchrone social où le retour async_load DS map aura les paires clé / valeur suivantes: " type " - C'est le type d'événement qui a été déclenché, qui sera la chaîne "achievement_post_result "pour cette fonction. " id " - Il s'agit de l'ID de l'événement et, pour cette fonction, il renverra une constante GML GooglePlayServices_EVENT_ID_POST_ACHIEVEMENT. " achievement_id " - Il s'agit de l'ID unique de la réussite qui a été mis à jour. " status " - Ce sera 1 si la réussite a été mise à jour avec succès, ou 0 si la demande a échoué Syntaxe:achievement_post(achievement, percent) Argument Description achievement Le nom / l'ID de la réalisation. percent Le pourcentage de la réussite accomplie. Retour:N/A Exemple:if achievement_available()     {     achievement_post(global.Achievement[0], 100); } Le code ci-dessus vérifie si le classement et le service de réussite choisis sont disponibles et, le cas échéant, publie une réussite.
