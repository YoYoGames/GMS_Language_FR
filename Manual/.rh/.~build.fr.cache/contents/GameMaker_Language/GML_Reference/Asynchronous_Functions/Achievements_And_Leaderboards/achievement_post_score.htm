achievement_post_score Vous pouvez utiliser cette fonction pour envoyer votre score au classement et au service de réussite choisis. Sur Android, vous envoyez l'ID du tableau de score sous forme de chaîne (il s'agit de l'ID de classement unique qui a été attribué lorsque vous configurez le classement), tandis que sur toutes les autres plates-formes, vous fournissez le nom du classement défini sous forme de chaîne, puis vous donnez la valeur réelle du score. La fonction déclenchera un événement asynchrone social où le retour async_load DS map aura les paires clé / valeur suivantes: " type " - C'est le type d'événement qui a été déclenché, qui sera la chaîne "achievement_post_score_result "pour cette fonction. " id " - Il s'agit de l'ID de l'événement et, pour cette fonction, il renverra une constante GML GooglePlayServices_EVENT_ID_POST_SCORE. " leaderboard_id " - Il s'agit de l'ID unique du classement qui a été mis à jour. " status " - Ce sera 1 si les scores ont été mis à jour avec succès, ou 0 si la demande a échoué " daily_new_best " - Ce sera 1 si le score soumis était le meilleur de l'utilisateur au cours de la dernière journée, ou 0 dans le cas contraire. " daily_best " - Cela contiendra le meilleur score de l'utilisateur du dernier jour de jeu. " week_new_best " - Ce sera 1 si le score soumis était le meilleur de l'utilisateur au cours de la semaine dernière, ou 0 dans le cas contraire. " week_best " - Ceci contient le meilleur score de l'utilisateur de la semaine dernière. " all_time_new_best " - Ce sera 1 si le score soumis était le meilleur de tous les temps de l'utilisateur, ou 0 sinon. " all_time_best " - Cela détient le meilleur score de l'utilisateur de tous les temps. Syntaxe:achievement_post_score(leaderboard, score) Argument Description leaderboard Le nom / ID du classement. score Le score à envoyer. Retour:N/A Exemple étendu:if achievement_available()     {     achievement_post_score(global.LeaderboardID, 1000);     } Le code ci-dessus vérifie si le classement et le service de réussite choisis sont disponibles et si c'est le cas, il publie un score dans le tableau des scores appelé "Classement". Cela déclenchera un événement asynchrone social qui peut être analysé comme ceci:if async_load[? "id"] == GooglePlayServices_EVENT_ID_POST_SCORE     {     if async_load[? "type"] == "achievement_post_score_result"         {         if async_load[? "status"] == 1             {             global.Best_Day = async_load[? "daily_best"];             global.Best_Week = async_load[? "weekly_best"];             global.Best = async_load[? "all_time_best"];             }         }     } Le code ci-dessus vérifie le async_load DS map pour les touches «id» et «type» qui correspondent à cette fonction, puis il vérifie l'état de l'événement. Si l'état est 1, certaines valeurs de la carte DS sont stockées dans des variables globales.
