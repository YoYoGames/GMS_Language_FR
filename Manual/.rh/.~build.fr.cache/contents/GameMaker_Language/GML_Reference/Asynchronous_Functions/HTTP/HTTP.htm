HTTP Cette section répertorie toutes les différentes fonctions HTTP asynchrones disponibles dans GameMaker Studio 2. Ces fonctions généreront un événement HTTP asynchrone dans toutes les instances qui en ont: http_requesthttp_gethttp_get_filehttp_post_string Veuillez noter que ce qui précède http_* les fonctions peuvent ne pas fonctionner comme prévu en raison de problèmes de sécurité entre domaines. Cela signifie que les demandes adressées à votre serveur ou les tentatives de chargement de ressources à partir de différents domaines sont bloquées et peuvent sembler renvoyer des résultats vides ou des erreurs 404. L'un des moyens de contourner ce problème est d'avoir du PHP côté serveur qui permet à certains domaines d'accéder à votre serveur (c'est aussi un moyen de protéger vos ressources et de bloquer les serveurs qui ne sont pas inclus dans la liste d'autorisation PHP). Voici un exemple du code que vous pouvez utiliser pour cela:$http_origin = $_SERVER['HTTP_ORIGIN']; if ($http_origin == "http://127.0.0.1:51268")    {    header("Access-Control-Allow-Origin: *");    } Pour les demandes de chargement d'image où la détermination ou la définition de leur type d'origine croisée est importante, les fonctions suivantes existent:http_get_request_crossoriginhttp_set_request_crossorigin Dans la plupart des cas, ces fonctions ne devraient pas avoir besoin d'être utilisées, mais si le jeu est stocké sur un serveur sécurisé - où certains actifs peuvent nécessiter une authentification de base pour être accédés et génèrent des erreurs de sécurité lors du chargement - définir le type d'origine croisée pour les demandes d'image à "use-credentials" peut être nécessaire (par opposition au paramètre "anonyme" par défaut).
