http_set_request_crossorigin Cette fonction permet de définir le type d'origine croisée à utiliser lors du chargement d'images à partir d'un fichier. Cette fonction est exclusivement destinée à la plate-forme HTML5 et n'est utile que lors du chargement de sprites depuis un fichier à l'aide de la fonction sprite_add() à partir d'un fichier. Notez que lorsque cette fonction est définie sur "use-credentials", il n'est plus nécessaire d'indiquer un chemin d'accès absolu au sprite chargé, mais plutôt un chemin d'accès relatif (comme indiqué dans l'exemple ci-dessous). Par défaut, le type d'origine croisée est défini sur "anonymous". Syntaxe : http_set_request_crossorigin(origin_type) ; Arguments Description type_origine Le type d'origine croisée à utiliser (une chaîne de caractères) Les retours : N/A Exemple : if string_lower(http_get_request_crossorigin()) != "use-credentials" { http_set_request_crossorigin("use-credentials") ; } sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0) ; Le code ci-dessus vérifie d'abord le type d'origine croisée actuellement défini et s'il n'est pas "use-credentials", il est défini comme "use-credentials", puis un sprite est ajouté à partir d'un fichier.
