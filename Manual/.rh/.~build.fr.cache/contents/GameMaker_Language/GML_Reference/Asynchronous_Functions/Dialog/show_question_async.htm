show_question_async Cette fonction ouvre une fenêtre et affiche la question que vous définissez dans la fonction à l'utilisateur. Il s'agit d'une fonction asynchrone et, en tant que telle, GameMaker Studio 2 ne bloque pas le périphérique sur lequel il est exécuté en attendant une réponse, mais continue plutôt à exécuter les événements normalement. La fonction a deux boutons qui indiquent "Oui" et "Non", et une fois que l'utilisateur a appuyé sur un, un événement de dialogue asynchrone est déclenché qui, pour la durée de cet événement uniquement, aura une carte DS stockée dans la variable async_load. Cette carte contiendra les deux clés, " id " et " status ". "id" est la valeur qui a été renvoyée par la fonction lorsqu'elle est appelée, tandis que "status" sera soit true ou false pour "Oui" et "Non" respectivement. Syntaxe:show_question_async(string); Argument Description String La question à poser à l'utilisateur. Retour:Real Exemple étendu: L' événement de pression gauche de la souris (par exemple) de l'objet qui affiche le message aurait le code suivant:msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?"); Ce qui précède affichera une question avec la chaîne donnée, demandant à l'utilisateur d'appuyer sur «oui» ou «non». L'identifiant de la fonction est stocké dans la variable "msg" et sera utilisé dans l' événement de dialogue asynchrone comme indiqué ci-dessous:var i_d, stat; i_d = ds_map_find_value(async_load, "id"); if i_d == msg     {     if ds_map_find_value(async_load, "status")         {         coins -= armour[0,5];         global.protection += armour[0,0];         }     } Le code ci-dessus vérifie la clé "id" du retour DS Map par rapport à la valeur stockée dans la variable "msg". S'ils sont identiques, il vérifie alors si l'un des deux boutons a été enfoncé et s'il revient true il déduira alors une valeur d'une variable et ajoutera également une valeur à une variable globale.
