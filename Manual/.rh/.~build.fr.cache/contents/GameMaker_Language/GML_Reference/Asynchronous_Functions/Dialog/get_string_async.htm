get_string_async Cette fonction ouvre une fenêtre et affiche un message ainsi qu'un espace pour que l'utilisateur saisisse une chaîne (qui contiendra la chaîne par défaut fournie pour commencer). Il s'agit d'une fonction asynchrone et, en tant que telle, GameMaker Studio 2 ne bloque pas le périphérique sur lequel il est exécuté en attendant une réponse, mais continue plutôt à exécuter les événements normalement. Une fois que l'utilisateur a tapé sa chaîne et appuyé sur le bouton "OK", un événement de dialogue asynchrone est déclenché qui, pour la durée de cet événement uniquement, aura une carte DS stockée dans la variable async_load. Cette carte contiendra les trois clés, " id ", " status " et " result ". "id" est la valeur qui a été retournée par la fonction lorsqu'elle est appelée, le "statut" sera soit true pour le bouton "OK" enfoncé, ou false si le message a été annulé (le cas échéant, car toutes les plates-formes cibles ne permettent pas l'annulation du message). Enfin, "result" renverra la chaîne saisie par l'utilisateur (ou une chaîne vide "" si aucune n'a été fournie). Syntaxe:get_string_async(string, default); Argument Description String Le message à afficher dans la boîte de dialogue. default La chaîne par défaut. Retour:Real Exemple étendu: L' événement de pression gauche de la souris (par exemple) de l'objet qui affiche le message aurait le code suivant:msg = get_string_async("What's your name?","Anon"); Ce qui précède affichera un message demandant à l'utilisateur de saisir une chaîne et d'appuyer sur "OK". L'identifiant de la fonction est stocké dans la variable "msg" et sera utilisé dans l'événement Dialogues asynchrones comme indiqué ci-dessous:var i_d = ds_map_find_value(async_load, "id"); if i_d == msg     {     if ds_map_find_value(async_load, "status")         {         if ds_map_find_value(async_load, "result") != ""             {             global.Name = ds_map_find_value(async_load, "result");             }         }     } Le code ci-dessus vérifie la clé "id" du retour DS Map par rapport à la valeur stockée dans la variable "msg". S'ils sont identiques, il vérifie alors si "OK" a été appuyé (plutôt que la fenêtre étant fermée / annulée) et si elle revient true il vérifie ensuite le "résultat" d'une chaîne pour s'assurer qu'aucune chaîne vide n'a été renvoyée avant de définir une variable globale.
