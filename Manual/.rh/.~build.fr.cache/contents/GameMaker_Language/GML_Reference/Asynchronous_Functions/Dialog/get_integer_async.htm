get_integer_async Cette fonction ouvre une fenêtre et affiche un message ainsi qu'un espace pour que l'utilisateur puisse saisir une valeur (qui contiendra la valeur par défaut fournie pour commencer). Il s'agit d'une fonction asynchrone, et en tant que telle, GameMaker Studio 2 ne bloque pas le périphérique sur lequel elle est exécutée pendant qu'elle attend le chargement du message. le périphérique sur lequel elle est exécutée en attendant une réponse, mais continue à exécuter les événements normalement. Une fois que l'utilisateur a tapé sa chaîne de caractères et appuyé sur le bouton "OK", un événement asynchrone est déclenché. Dialogue est déclenché et, pendant la durée de cet événement uniquement, une carte DS est stockée dans la variable async_load. Cette carte contiendra les trois clés, " id "," status " et "value ". " id " est lavaleur renvoyéepar la fonction lorsqu'elle est appelée, le " status " sera soit vrai si le bouton " Okay " a été pressé, soit faux si le message a été annulé (le cas échéant, car toutes les plateformes cibles ne permettent pas d'annuler les messages). Enfin, "value" renverra l'entier que l'utilisateur a entré sous forme de chaîne (une chaîne vide "..."). sous la forme d'une chaîne (une chaîne vide "" sera renvoyée si aucune entrée n'a été donnée). NOTE : Cette fonction est réservée à un usage de débogage et de testet ne doit pas être utilisée dans les jeux publiés. Dans ce cas, vous devez créer une interface utilisateur personnalisée pour recevoir les entrées des joueurs, de manière à avoir un contrôle total sur l'apparence et le comportement des boîtes de dialogue. Syntaxe : get_integer_async(string, default) ; Arguments Description Chaîne de caractères Le message à afficher dans la boîte de dialogue. par défaut La valeur par défaut.
