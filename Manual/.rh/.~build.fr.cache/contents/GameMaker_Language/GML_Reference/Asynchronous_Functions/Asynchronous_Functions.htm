Fonctions asynchrones Dans le contexte de GameMaker Studio 2, une fonction asynchrone est une fonction qui envoie à un serveur Web, ou demande une entrée utilisateur, ou même diffuse des données, tout en laissant GameMaker Studio 2 continuer à fonctionner sans bloquer le jeu en cours d'exécution. Lorsque le rappel d' informations est reçu, un événement asynchrone spécial est déclenché qui, à son tour, permet à toute instance avec un événement asynchrone qui lui est assigné d'exécuter d'autres fonctions (bien que cela ne soit pas obligatoire). Donc, en gros, c'est un moyen de communiquer avec une ressource externe (serveur, utilisateur ou appareil) sans que le jeu ou le projet ne s'arrête en attendant une réponse. Pourquoi est-ce important? Eh bien, puisqu'il permet à GameMaker Studio 2 de continuer à fonctionner tout en envoyant ou en recevant des données, cela signifie que vous pouvez faire beaucoup de choses sans bloquer la boucle de jeu comme: diffuser les données de l'appareil dans votre jeu sans que le joueur n'attende que les choses se chargent avoir des événements de rappel pour faire des choses uniquement lorsque les informations correctes sont reçues communiquer et échanger des données avec un serveur Web Cela fait que tout fonctionne de manière beaucoup plus fluide et discrète. Il convient de noter que les événements asynchrones seront déclenchés pour toutes les instances qui les ont, vous pouvez donc utiliser une fonction asynchrone dans une instance et avoir l'événement asynchrone qui traite la réponse dans une autre, ou même plusieurs autres. Vous pouvez aller ici pour plus d'informations sur le fonctionnement de l'événement asynchrone - Evénements asynchrones.
