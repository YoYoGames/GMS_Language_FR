draw_sprite Cette fonction dessine le sprite et la sous-image donnés à une position dans la salle de jeu. Pour le sprite, vous pouvez utiliser la variable d'instance sprite_index pour obtenir le sprite courant qui est assigné à l'instance qui exécute le code, ou vous pouvez utiliser n'importe quel autre actif de sprite. Il en va de même pour la sous-image, qui peut également être définie par la variable d'instance image_index qui définira la sous-image à celle sélectionnée pour le sprite de l'instance actuelle (notez que vous pouvez dessiner un sprite différent tout en utilisant la valeur de la sous-image pour l'instance actuelle), ou vous pouvez utiliser toute autre valeur pour dessiner une sous-image spécifique du sprite choisi. Si la valeur est supérieure au nombre de sous-images, GameMaker Studio 2 bouclera automatiquement le numéro pour sélectionner l'image correspondante (par exemple, si le sprite en cours de dessin possède 5 sous-images numérotées de 0 à 4 et que nous fixons la valeur de l'index à 7, la fonction dessinera la sous-image 3, numérotée 0). Enfin, la position x et y est la position dans la pièce où le sprite sera dessiné, et elle est centrée sur le décalage x et le décalage y du sprite. REMARQUE: cette fonction peut ne pas fonctionner comme prévu lorsque vous utilisez des sprites d'animation de squelette, et vous pouvez constater que la fonction ne dessine que la première image de la pose par défaut. Vous devriez utiliser les fonctions draw_skeleton_* à la place. Syntaxe : draw_sprite(sprite, subimg, x, y) ; Arguments Description sprite L'index du sprite à dessiner. sous-image La sous-image (frame) du sprite à dessiner (image_index ou -1 correspondent à la frame actuelle de l'animation dans l'objet). x La coordonnée x de l'endroit où dessiner le sprite. y La coordonnée y de l'endroit où dessiner le sprite. Les retours : N/A Exemple : draw_sprite(sprite_index, image_index, x, y) ; draw_sprite(spr_Halo, 0, x, y-32) ; Ceci dessinera le sprite assigné à l'instance (sprite_index) avec la sous-image courante à la position x et y de l'instance dans la room. Elle dessinera ensuite la première image du sprite indexé par "spr_Halo" à la même position x et y mais 32 pixels "au-dessus".
