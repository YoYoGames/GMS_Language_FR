Sprites et tuiles Même si vous pouvez configurer des arrière-plans dans l'éditeur de salle et attribuer des sprites à des instances, cela n'est souvent pas assez flexible pour le jeu que vous créez. Par exemple, si vous avez une instance qui doit dessiner un menu, vous voudrez peut-être afficher différents sprites de bouton à la fois, ou si vous avez un jeu avec des arrière-plans de parallaxe défilants, vous devrez être en mesure de les définir via le code dans un instance de contrôleur d'un certain type. Dans ces situations, vous devez utiliser les fonctions suivantes qui peuvent dessiner des éléments de sprite et d'arrière-plan n'importe où dans la pièce, certaines des fonctions vous permettant également de manipuler la forme, la taille, la couleur et l'alpha du sprite pour obtenir des effets spéciaux. Vous pouvez attribuer un sprite à une instance et par défaut GameMaker Studio 2 le dessinera avec les propriétés données par les différentes variables de sprite, cependant si vous ajoutez quelque chose dans l'événement draw, vous «prenez le contrôle» et devez dire GameMaker Studio 2 quoi dessiner. Cela signifie que si vous devez utiliser les fonctions suivantes pour définir exactement de quoi il s'agit, l'instance doit dessiner et où dans la pièce elle doit être dessinée: draw_selfdraw_spritedraw_sprite_extdraw_sprite_generaldraw_sprite_partdraw_sprite_part_extdraw_sprite_stretcheddraw_sprite_stretched_extdraw_sprite_posdraw_sprite_tileddraw_sprite_tiled_ext Si vous utilisez des sprites au format SWF, il existe également des options supplémentaires pour contrôler l'anticrénelage utilisé lors du dessin (voir Importation de sprites non bitmap pour plus d'informations sur ces types de sprites). Notez que ces fonctions fonctionnent à un niveau global et définiront l'AA sur tous les sprites vectoriels SWF dessinés après leur appel. draw_enable_swf_aadraw_set_swf_aa_leveldraw_get_swf_aa_level Outre les fonctions répertoriées ci-dessus, il existe également deux fonctions spéciales conçues pour être utilisées avec des animations squelettiques (voir Importer des sprites non bitmap pour plus d'informations sur ces types de sprites, et la section Animations squelettiques pour des fonctions spécifiques liées à ces types de sprites): draw_skeletondraw_skeleton_instancedraw_skeleton_collisiondraw_skeleton_time Les tuiles sont dessinées automatiquement en fonction des données qu'elles contiennent, du jeu de tuiles utilisé et des propriétés de la carte de tuiles.
