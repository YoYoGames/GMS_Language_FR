draw_tile Avec cette fonction, vous pouvez dessiner n'importe quelle tuile donnée à partir d'un jeu de tuiles - rivaliser avec les rotations, les retournements et les miroirs - en définissant le cadre (s'il est animé) et la position dans la pièce. La valeur de l'ID du jeu de tuiles est l'index du jeu de tuiles tel que défini dans l'EDI et les données de tuiles sont le "blob" de données qui contient toutes les informations sur l'index de tuiles et les transformations qui ont été appliquées. Vous pouvez utiliser la fonction tilemap_get() pour obtenir les données de mosaïque qui peuvent ensuite être manipulées à l'aide des fonctions de mosaïque, ou vous pouvez les construire vous-même en utilisant les constantes de mosaïque suivantes ainsi que la valeur d'index de la mosaïque à créer: Constant Description tile_rotate Utilisé pour définir / obtenir le bit de rotation d'un objet blob de données de mosaïque. tile_mirror Utilisé pour définir / obtenir le bit miroir d'un objet blob de données de tuile. tile_flip Utilisé pour définir / obtenir le bit de retournement d'un blob de données de tuile. tile_index_mask Une constante spéciale qui est pour "et" -ing avec le blob de données de tuile pour extraire l'index de tuile. Vous utiliseriez au niveau du bit "ou" ces constantes avec la valeur d'index de tuile pour obtenir les données de tuile finales:var tiledata = 8 | tile_rotate | tile_flip; Le code ci-dessus crée simplement une variable locale pour stocker les données de tuile pour la tuile avec l'index de 8 sur l'ensemble de tuiles, et il est tourné et retourné. Cela peut ensuite être utilisé pour dessiner la vignette à cet index à partir du jeu de vignettes que vous spécifiez. ceci n'est lié à aucune couche de tuile map et la tuile sera dessinée à la profondeur de couche de l'occurrence faisant le dessin, à la position x / y que vous spécifiez dans la pièce. Notez que vous pouvez également définir le cadre pour dessiner. Si la vignette indexée fait partie d'un ensemble d'animations, vous pouvez définir le numéro d'image ici (de 0 au nombre d'images - 1) et la vignette indexée sera simplement l'indicateur pour quel ensemble de tuiles animées vous utilisez pendant qu'elle le sera. être le cadre donné qui est dessiné. Si la vignette ne fait pas partie d'un ensemble animé, le cadre doit être défini sur 0. Syntaxe:draw_tile(tile set, tiledata, frame, x, y) Argument Description tile set L'ID du jeu de tuiles à utiliser tiledata Les données de tuiles à utiliser frame Le numéro d'image de la vignette animée à utiliser (0 par défaut pour les vignettes non animées) x La position x dans la pièce à laquelle dessiner y La position y dans la pièce à laquelle dessiner Retour:Real Exemple:var lay_id = layer_get_id("Tiles_sky"); var map_id = layer_tilemap_get_id(lay_id); var data = tilemap_get(map_id, 0, 0); tile_set_flip(data, true); draw_tile(set_Wall, data, 0, x, y); Le code ci-dessus obtiendra les données sur une tuile à partir du tilemap donné, puis définira le drapeau "flip" et utilisera ces données pour dessiner la tuile retournée à la position de l'instance.
