dessiner_carreau Avec cette fonction, vous pouvez dessiner n'importe quelle tuile d'un jeu de tuiles - avec des rotations, des renversements et des miroirs - en définissant le cadre (si animé) et la position dans la pièce. La valeur ID de l'ensemble de tuiles est l'index de l'ensemble de tuiles tel qu'il est défini dans l'IDE et les données de tuiles sont le "blob" de données. est le "blob" de données qui contient toutes les informations sur l'indice de la tuile et les transformations qui ont été appliquées. Vous pouvez utiliser la fonction tilemap_get() pour obtenir les données de tuiles qui peuvent ensuite être manipulées à l'aide des fonctions de tuiles, ou vous pouvez les construire vous-même en utilisant les constantes de tuiles suivantes avec la valeur de l'indice de la tuile à créer : Constant Description rotation des tuiles Utilisé pour définir/obtenir le bit de rotation d'un blob de données de tuiles. tuile_mirror Utilisé pour définir/obtenir le bit miroir d'un blob de données de tuile. tuile_flip Utilisé pour définir/obtenir le bit de retournement d'un blob de données de tuile. masque d'indexation des tuiles Une constante spéciale qui est utilisée pour "et" avec le blob de données de la tuile pour extraire l'index de la tuile. Vous devez effectuer un "or" bit à bit de ces constantes avec la valeur de l'indice de la tuile pour obtenir les données de la tuile finale : var tiledata = 8 | tile_rotate | tile_flip ; Le code ci-dessus crée simplement une variable locale pour stocker les données de la tuile avec l'index 8 sur le jeu de tuiles, et il est tourné et retourné. Ceci peut ensuite être utilisé pour dessiner la tuile à cet index à partir du jeu de tuiles que vous spécifiez. La tuile sera dessinée à la profondeur de couche de l'instance effectuant le dessin, à la position x/y que vous spécifiez dans la room. Notez que vous pouvez également définir le cadre à dessiner. Si la tuile indexée fait partie d'une animation vous pouvez définir ici le numéro de l'image (de 0 au nombre d'images - 1) et la tuile indexée sera simplement l'indicateur de l'ensemble de tuiles animées que vous utilisez, tandis que c'est l'image donnée qui sera dessinée. Si la tuile ne fait pas partie d'un d'un jeu de tuiles animées, le cadre doit être défini à 0. Syntaxe : draw_tile(ensemble de tuiles, données de tuiles, cadre, x, y) Arguments Description jeu de carreaux L'ID du jeu de tuiles à utiliser données en mosaïque Les données de la tuile à utiliser cadre Le numéro d'image de la tuile animée à utiliser (0 par défaut pour les tuiles non animées). x La position x dans la pièce à dessiner y La position y dans la pièce à dessiner Les retours : Real Exemple : var lay_id = layer_get_id("Tiles_sky") ; var map_id = layer_tilemap_get_id(lay_id) ; var data = tilemap_get(map_id, 0, 0) ; tile_set_flip(data, true) ; draw_tile(set_Wall, data, 0, x, y) ; Le code ci-dessus récupère les données sur une tuile à partir de la carte de tuiles donnée, puis active l'indicateur "flip" et utilise ces données pour dessiner la tuile retournée à la position de l'instance.
