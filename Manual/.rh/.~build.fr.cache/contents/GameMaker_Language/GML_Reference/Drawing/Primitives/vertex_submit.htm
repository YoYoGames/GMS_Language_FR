vertex_submit Vous pouvez utiliser cette fonction pour soumettre le contenu d'un tampon de sommets au pipeline graphique pour une utilisation avec un shader. Vous fournissez l'index de tampon à utiliser, le type primitif de base à utiliser (voir les constantes ci-dessous) et la texture à utiliser. Le type primitif de base n'est utilisé que pour affecter l'ordre dans lequel les sommets que vous avez stockés dans le tampon sont dessinés et connectés, mais les données réelles utilisées pour chacun des sommets seront celles que vous avez définies lors de la création du tampon de sommets. REMARQUE: cette fonction ne peut être utilisée que dans l' événement Draw. Pour un exemple visuel des différentes primitives de base disponibles, voir l'image ci-dessous: Constant Description pr_pointlist Une primitive constituée d'une liste de points. pr_linelist Une primitive composée de lignes individuelles dans une liste. pr_linestrip Une primitive composée d'une bande consécutive de lignes. pr_trianglelist Une primitive composée de triangles individuels dans une liste. pr_trianglestrip Une primitive composée d'une bande consécutive de triangles. Syntaxe:vertex_submit(buffer, primitive, texture); Argument Description buffer Le tampon à utiliser. primitive Le type de base primitif. texture La texture à utiliser (ou -1 pour aucune texture). Retour:N/A Exemple:shader_set(shader_prim); vertex_submit(buff, pr_trianglelist, sprite_get_texture(sprite_index)); shader_reset(); Le code ci-dessus soumettra le tampon de vertex indexé dans la variable "buff" pour le dessin avec le shader " shader _prim", en utilisant une liste de triangles comme primitive de base et la texture du sprite pour l'instance exécutant le code.
