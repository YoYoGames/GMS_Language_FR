orientation_type_partie Cette fonction est utilisée pour déterminer l'orientation du sprite de la particule lors de sa création et peut également être utilisée pour que l'orientation de la particule augmente ou diminue au cours de sa vie. Les valeurs d'orientation minimale et maximale sont par défaut à 0, mais ces valeurs peuvent être modifiées pour rendre l'orientation aléatoire en suivant la procédure standard de GameMaker. peuvent être modifiées pour rendre l'orientation aléatoire en suivant les directions standard de GameMaker Studio 2: 0 degré pour la droite, 90 degrés pour le haut, 180 degrés pour la gauche et 270 degrés pour le bas. Si vous les réglez sur la même valeur, les particules seront toutes créées avec la même orientation. Vous pouvez également définir une valeur d'incrémentation qui ajoutera (s'il s'agit d'un nombre positif) ou soustraira (s'il s'agit d'un nombre négatif) une quantité de degrés à l'orientation pendant sa durée de vie. Cette valeur peut être au minimum de (+/-) 0,01. Vous pouvez également définir le facteur "wiggle", comme pour les autres fonctions de particules. Il s'agit d'une valeur qui sera ajoutée ou soustraite aléatoirement à l'orientation à chaque étape de la vie des particules. Évidemment, les valeurs les plus grandes sont plus prononcées que les plus petites. plus petites, et cette valeur peut même être négative, la plage maximale étant comprise entre -20 et 20. Enfin, vous pouvez choisir que l'orientation soit relative ou non, ce qui signifie que si la particule a une direction (et une vitesse), le sprite de la particule sera orienté autour de ce vecteur. Gardez à l'esprit que si vous avez réglé la vitesse de la particule sur réduire et qu'elle atteint 0, le manque de vitesse fixe la direction à la valeur par défaut de 0° et donc une orientation relative fera "sauter" le sprite de la particule à un angle différent. Syntaxe : part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative ) ; Arguments Description ind L'indice du type de particule à modifier. ang_min L'angle minimal de départ des particules. ang_max L'angle de départ maximal des particules. ang_incr L'augmentation (ou la diminution si elle est négative) de l'angle en degrés par pas. ang_wiggle La quantité d'oscillation de l'angle dans le sens des aiguilles d'une montre ou dans le sens inverse des aiguilles d'une montre par étape. ang_relatif Il faut définir son angle par rapport à la direction du mouvement de la particule (true) ou non (false). Les retours : N/A Exemple : part_type_shape(particle2, pt_shape_spark) ; part_type_size(particle2, 0.10, 0.50, 0.01, 0) ; part_type_scale(particle2, 0.30, 0.30) ; part_type_colour1(particle2, 8454143) ; part_type_alpha1(particle2, 0.50) ; part_type_speed(particle2, 4, 4, -0.07, 1) ; part_type_direction(particle2, 0, 359, 0, 20) ; part_type_orientation(particle2, 0, 359, 0, 20, 1) ; part_type_blend(particle2, 1) ; part_type_life(particle2, 1, 5) ; Le code ci-dessus va définir diverses valeurs de particules, y compris l'orientation qui sera une valeur aléatoire entre 0º et 359º. Un montant aléatoire sera également ajouté à chaque étape, entre 0 et 20, et l'orientation sera relative à la direction du mouvement. direction du mouvement.
