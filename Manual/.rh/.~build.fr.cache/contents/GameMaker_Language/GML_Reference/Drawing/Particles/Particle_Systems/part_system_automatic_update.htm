mise à jour automatique du système partiel Cette fonction contrôle si GameMaker Studio 2 doit mettre à jour le système de particules automatiquement ou non. Normalement, vous n'avez pas besoin d'utiliser cette fonction car la valeur par défaut de true (la mise à jour automatique est activée) correspond à ce que vous souhaitez qu'il se passe. ce que vous souhaitez. Cependant, pour des effets spéciaux ou pour mettre le jeu en pause, vous pouvez définir cette valeur sur false et cela empêchera toute mise à jour du système de particules donné, à moins qu'elle ne soit explicitement commandée par l'utilisation de la fonction part_system_update() ou si vous utilisez à nouveau cette fonction pour activer les mises à jour automatiques. L'absence de mise à jour signifie que les particules créées dans ce système ne changeront plus de position, de couleur ou de tout autre paramètre et que les émetteurs cesseront également de fonctionner, ce qui aura pour effet de "geler" le système à l'endroit exact où la mise à jour automatique a été réglée sur false. Une chose à noter est que si vous utilisez les effets simples créés par les fonctions effet_créer_supérieur() ou effet_create_below() alors vous pouvez utiliser les valeurs 0 (pour les effets inférieurs) ou 1 (pour les effets supérieurs) comme index du système de particules et ainsi activer la mise à jour automatique pour ceux-ci également (cela fonctionnera également pour activer la mise à jour pour les effets de particules DnD™). Syntaxe : part_system_automatic_update(ind, automatic) ; Arguments Description ind L'indice du système de particules à modifier. automatique Si la mise à jour automatique est activée(true) ou non (false). Les retours : N/A Exemple : if global.Pause { part_system_automatic_update(global.Sname, false) ; } Le code ci-dessus désactivera les mises à jour des particules si la variable globale "Paused" est vraie.
