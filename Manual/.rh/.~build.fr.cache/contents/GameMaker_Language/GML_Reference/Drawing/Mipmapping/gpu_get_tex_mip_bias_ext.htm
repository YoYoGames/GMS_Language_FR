gpu_get_tex_mip_bias_ext Avec cette fonction, vous pouvez récupérer la valeur du biais mipmap pour un échantillonneur de shader Vous fournissez la valeur d'index pour l' shader (comme renvoyé par la fonction shader_get_sampler_index(), et la fonction renverra une valeur de 0,0 pour aucun biais, ou une valeur plus grande où 1 est égal au premier mipmap, 2 est égal au deuxième mipmap etc... Cela contrôle la vitesse à laquelle le mip map est permuté et fera généralement le shader textures de shader sont plus floues plus la valeur est élevée et plus la "distance" visualisée est grande. Notez que cela peut également renvoyer des valeurs négatives, auquel cas shader textures de shader seront plus nettes sur une plus grande distance, plus la valeur est basse. Syntaxe:gpu_get_tex_mip_bias_ext(sampler_index); Argument Description sampler_index L'indice de l'échantillonneur de shader Retour:Real (default: 0) Exemple:var _sampleIndex = shader_get_sampler_index(shd_Glass, "s_Background"); var _spriteTex = sprite_get_texture(sprite_index, 0); shader_set(shd_Glass); if gpu_get_tex_mip_bias_ext(_sampleIndex) != 0     {     gpu_set_tex_mip_bias_ext(_sampleIndex, 0);     } texture_set_stage(_sampleIndex , _spriteTex); shader_reset(); Le code ci-dessus définit le biais du filtre mip sur 0 pour l' shader donné s'il n'a pas déjà été défini sur 0.
