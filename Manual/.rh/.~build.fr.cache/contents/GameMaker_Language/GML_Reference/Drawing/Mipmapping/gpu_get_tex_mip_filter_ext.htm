gpu_get_tex_mip_filter_ext Avec cette fonction, vous pouvez obtenir le mode de filtre mip actuel pour un échantillonneur de shader donné. Vous fournissez la valeur de l'index de l'échantillonneur de shader (tel que retourné par la fonction shader_get_sampler_index(), et la fonction renvoie l'une des constantes de valeur de mode énumérées ci-dessous. Syntaxe : gpu_get_tex_mip_filter_ext(sampler_index) ; Arguments Description sampler_index L'index de l'échantillonneur de shader à obtenir Les retours : Constantes (énumérées ci-dessous) : Constant Description tf_point Cela signifie que le mélange entre les niveaux de mipmap est désactivé, ce qui peut provoquer des transitions de texture visibles, mais donne les meilleures performances. tf_linear Cela signifie que le mélange entre les niveaux de mipmap est activé (c'est également connu sous le nom de filtrage trilinéaire), ce qui lisse les transitions de texture, mais entraîne une légère baisse des performances. tf_anisotropique Cela signifie que le filtrage anisotropique est activé, ce qui améliore considérablement la qualité de la transition des textures et peut réduire le flou visible avec d'autres modes de filtrage, mais il a le plus grand impact sur les performances. Exemple : var _sampleIndex = shader_get_sampler_index(shd_Glass, "s_Background") ; var _spriteTex = sprite_get_texture(sprite_index, 0) ; shader_set (shd_Glass) ; if gpu_get_tex_mip_filter_ext(_sampleIndex) != tf_point { gpu_set_tex_mip_filter_ext(_sampleIndex, tf_point) ; } texture_set_stage(_sampleIndex , _spriteTex) ; shader_reset() ; Le code ci-dessus définit le mode de filtre mip à tf_point (désactivant le mipmapping) pour l'échantillonneur de texture du shader donné s'il n'a pas déjà été défini.
