gpu_set_tex_mip_filter Cette fonction permet de définir le mode de filtrage actuel de l'ipm sur l'un des trois types pris en charge. Vous donnez la constante qui se réfère au mode de filtrage de l'IAM dont vous avez besoin, où vous pouvez choisir entre un filtrage ponctuel (paramètre par défaut, signifiant aucun filtrage), le filtrage linéaire (également appelé filtrage trilinéaire) ou le filtrage aniostrope (voir le tableau des constantes ci-dessous). Constant Description tf_point Cela désactivera le filtrage entre les niveaux de mipmap, ce qui peut provoquer des transitions de texture visibles, mais donne les meilleures performances. tf_linear Cela permettra d'activer le filtrage linéaire entre les niveaux de mipmap (également connu sous le nom de filtrage trilinéaire), qui lisse les transitions de texture, mais cela aura un impact mineur sur les performances. tf_anisotropique Cela permet d'activer le filtrage anisotropique entre les niveaux de mipmap, ce qui améliore grandement la qualité de la transition des textures et peut réduire le flou visible avec les autres modes de filtrage, mais cela a le plus grand impact sur les performances. Syntaxe : gpu_get_tex_mip_filter(filtre) ; Arguments Description filtre Le mode de filtre mip à utiliser (une constante, par défaut : tf_point). Les retours : N/A Exemple : if keyboard_check(ck_enter) { switch(gpu_get_tex_mip_filter()) { case tf_point : gpu_set_tex_mip_filter(tf_linear) ; break ; case tf_linear : gpu_set_tex_mip_filter(tf_anisotropic) ; break ; case tf_anisotropic : gpu_set_tex_mip_filter(tf_point) ; break ; } } Le code ci-dessus vérifie le clavier et si la touche spécifiée est pressée, il récupère le filtre mip actuel et passe à la valeur suivante, en passant par les différents modes.
