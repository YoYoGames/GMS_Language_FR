surface_free Lorsque vous travaillez avec des surfaces, vous devez toujours utiliser cette fonction lorsque vous avez fini de les utiliser. Les surfaces occupent de l'espace dans la mémoire et doivent donc être supprimées, normalement à la fin d'une pièce, mais cela peut être fait à tout moment selon l'utilisation que vous en faites. l'utilisation que vous en faites. Si vous ne le faites pas, vous risquez de provoquer des fuites de mémoire qui finiront par ralentir et faire planter votre jeu. NOTE: Lorsque vous travaillez avec des surfaces, il est possible qu'elles cessent d'exister à tout moment, car elles sont stockées dans la mémoire de texture. Vous devez TOUJOURS vérifier qu'une surface existe en utilisant surface_exists() avant d'y faire référence directement. Syntaxe : surface_free(surface_id) ; Arguments Description surface_id L'ID de la surface à libérer. Les retours : N/A Exemple : if keyboard_check_pressed(vk_escape) { surface_free(surf) ; room_goto(rm_Menu) ; } Le code ci-dessus vérifie si une touche a été pressée et s'il en détecte une, il libère la mémoire réservée à la surface indexée dans la variable "surf" et change ensuite de pièce.
