curseur_sprite La définition de cette variable demandera à GameMaker Studio 2 d'utiliser le sprite désigné comme curseur (en fait, il sera réglé sur la position x/y actuelle de la souris à chaque étape). La valeur par défaut est -1, ce qui signifie qu'il n'y a pas de sprite pour le curseur. mais vous pouvez attribuer n'importe quel index de sprite provenant des ressources du jeu ou importé d'une ressource externe. Veuillez noter qu'il n'y a aucun moyen de contrôler la vitesse d'animation ou l'index d'image, donc si le sprite a des sous-images, celles-ci seront cycliques à la même vitesse que la vitesse de la pièce. à la même vitesse que celle de la pièce. Pour supprimer le sprite du curseur, vous pouvez redonner à cette variable la valeur -1. Il convient également de noter que cette variable ne remplace pas le curseur de la fenêtre de jeu, et qu'il sera toujours dessiné normalement. Pour éviter cela, vous pouvez utiliser la fonction window_set_cursor() et et lui attribuer la constante cr_none qui rendra le curseur standard invisible. Syntaxe : curseur_sprite ; Les retours : Real Exemple : curseur_sprite = spr_CustomCursor ; Le code ci-dessus va définir le sprite indexé dans la variable "spr_CustomCursor" comme étant le sprite du curseur du jeu.
