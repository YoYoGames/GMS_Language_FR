cursor_sprite La définition de cette variable indiquera à GameMaker Studio 2 d'utiliser le sprite désigné comme curseur (en le définissant essentiellement sur la position x / y actuelle de la souris à chaque étape). La valeur par défaut est -1, ce qui n'est pas un sprite pour le curseur, mais vous pouvez attribuer n'importe quel index de sprite à partir des ressources du jeu ou qui a été importé à partir d'une ressource externe. Veuillez noter qu'il n'y a aucun moyen de contrôler la vitesse de l'animation ou image_index, donc si le sprite a des sous-images, celles-ci seront cyclées à la même vitesse que la vitesse de la salle. Pour supprimer l'image-objet du curseur, vous pouvez redéfinir cette variable sur -1. Il est également intéressant de noter que cette variable ne remplace pas le curseur de la fenêtre de jeu, et qu'elle sera toujours dessinée normalement. Pour éviter cela, vous pouvez utiliser la fonction window_set_cursor() et réglez-le sur la constante cr_none ce qui rendra le curseur standard invisible. Syntaxe:cursor_sprite; Retour:Real Exemple:cursor_sprite = spr_CustomCursor; Le code ci-dessus définira le sprite indexé dans la variable "spr_CustomCursor" pour être le sprite curseur pour le jeu.
