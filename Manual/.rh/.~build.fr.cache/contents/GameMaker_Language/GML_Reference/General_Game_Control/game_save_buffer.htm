tampon_sauvegarde_jeu Il s'agit d'une variante de la fonction game_save() veuillez donc lire sa page en premier car elle contient des informations importantes relatives à son utilisation et à celle de cette fonction. Cette fonction permet de sauvegarder l'état actuel du jeu dans un tampon "grow" préalablement créé (voir Tampons) qui peut ensuite être rechargé à tout moment à l'aide de la fonction game_load_buffer() fonction. Cette fonction est conçue pour être utilisée dans une seule pièce à la fois (c'est-à-dire qu'il n'est pas conseillé de sauvegarder un tampon dans une pièce, puis d'essayer de charger le tampon depuis une autre pièce) et ne devrait idéalement être utilisée que pour les points de contrôle ou les redémarrages de niveau. NOTE: Cette fonction est très limitée et est conçue pour les débutants afin de mettre en place un système de checkpoint rapidement, mais les utilisateurs plus avancés peuvent préférer coder leur propre système en utilisant les fonctions du fichier, car le jeu ne sauvegardera aucune des ressources dynamiques telles que les structures de données, les surfaces, les sprites ajoutés, etc. Syntaxe : game_save_buffer(buffer) ; Arguments Description tampon L'identifiant du tampon à sauvegarder. Les retours : N/A Exemple : if keyboard_check_pressed(ord("S")) { global.Checkpoint = true ; game_save_buffer(save_buff) ; } Le code ci-dessus va définir une variable globale à true et ensuite sauvegarder l'état actuel du jeu dans le tampon indexé dans la variable "save_buff" lorsque la touche "S" est pressée.
