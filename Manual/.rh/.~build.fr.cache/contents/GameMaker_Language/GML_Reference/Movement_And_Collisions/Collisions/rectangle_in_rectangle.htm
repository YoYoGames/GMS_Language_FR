rectangle_in_rectangle Cette fonction vérifie deux zones rectangulaires que vous définissez pour voir si le rectangle source n'est pas en collision, complètement dans les limites des rectangles de destination, ou s'ils se touchent simplement. S'ils ne touchent pas du tout, la fonction retournera 0, si la source est complètement dans la destination, elle retournera 1, et si elles se chevauchent simplement, elle retournera 2. L'image ci-dessous illustre ceci: Syntaxe:rectangle_in_rectangle(sx1, sy1, sx2, sy2, dx1, dy1, dx2, dy2); Argument Description sx1 Coordonnée x du côté gauche du rectangle source. sy1 Coordonnée y du côté supérieur du rectangle source. sx2 Coordonnée x du côté droit du rectangle source. sy2 Coordonnée y du côté inférieur du rectangle source. dx1 Coordonnée x du côté gauche du rectangle de destination à vérifier. dy1 Coordonnée y du côté supérieur du rectangle de destination à vérifier. dx2 Coordonnée x du côté droit du rectangle de destination à vérifier. dy2 Coordonnée y du côté inférieur du rectangle de destination à vérifier. Retour:Real Exemple:switch(rectangle_in_rectangle(x - 50, y - 50, x + 50, y + 50, mouse_x - 50, mouse_y - 50, mouse_x + 50, mouse_y + 50))    {    case 0: image_blend = c_yellow; break;    case 1: image_blend = c_red; break;    case 2: image_blend = c_orange; break;    } Le code ci-dessus vérifie la collision de deux zones rectangulaires et définit l'instance image_blend basé sur la valeur de retour.
