mp_grid_path Avec cette fonction, vous pouvez créer un chemin qui naviguera d'un point de départ à un point d'arrivée à l'aide d'un mp_grid que vous avez préalablement défini, en évitant les obstacles qui ont déjà été ajoutés dans la grille. Les arguments xstart et ystart indiquent le début du chemin en coordonnées de la pièce, tandis que les arguments xgoal, ygoal indiquent la destination. Vous pouvez également sélectionner le mouvement horizontal / vertical uniquement ou autoriser les mouvements diagonaux complets en spécifiant true dans l'argument allowdiag. La fonction renvoie soit vrai (elle a réussi à trouver un chemin), soit faux (elle a échoué) ainsi que la définition du chemin choisi. Dans l'image suivante, vous pouvez voir comment tout cela fonctionne ensemble: Comme vous pouvez le voir, les objets "pipe" ont été ajoutés dans la grille, ce qui signifie que tout chemin créé doit les contourner. Il y a deux chemins créés dans l'image, l'un (vert) a été fait avec les diagonales autorisées et l'autre (rouge) sans. La différence entre les deux est assez évidente avec le chemin vert paraissant beaucoup plus "élégant" et direct, mais tout dépend de l'usage que vous allez leur faire de celui que vous préférez. Notez que le chemin est indépendant de l'instance actuelle - c'est un chemin à travers la grille, pas un chemin pour une instance spécifique, même si une instance spécifique peut avoir la variable qui stocke l'index du chemin. Vous devrez peut-être également déboguer ces chemins pour voir comment ils sont créés et interagissent dans l'environnement de jeu, auquel cas vous devriez utiliser le draw_path() fonction. REMARQUE: le chemin doit avoir été créé au préalable (soit dans le code avec path_add () soit en tant que ressource) et sera remplacé par le chemin généré par cette fonction. Syntaxe:mp_grid_path(id, path, xstart, ystart, xgoal, ygoal, allowdiag); Argument Description id Index du mp_grid à utiliser path index du chemin à utiliser par la fonction x start Coordonnée x de départ du nouveau chemin y start Coordonnée y de départ du nouveau chemin xgoal Finition de la coordonnée x du nouveau chemin ygoal Finition de la coordonnée y du nouveau chemin allowdiag Indique si les déplacements en diagonale sont autorisés au lieu de simplement horizontaux ou verticaux Retour:Boolean Exemple:global.grid = mp_grid_create(0, 0, room_width div 32, room_height div 32, 32, 32); mp_grid_add_instances(global.grid, obj_wall, false); with (obj_Enemy)     {     path = path_add();     if mp_grid_path(global.grid, path, x, y, obj_Player.x, obj_Player.y, 1)         {         path_start(path, 0, 3, 0);         }     } Le code ci-dessus crée une variable globale "global.grid", puis génère un mp_grid et assigne son index (id) à cette variable pour une utilisation dans tous les autres appels de fonction mp_grid. Il ajoute ensuite toutes les instances de "obj_Wall" dans la grille avant d'obtenir toutes les instances de "obj_Enemy" pour créer un chemin puis utiliser mp_grid_path pour calculer une déroute de leur position à la position de "obj_Player".
