mp_potential_path_object Cette fonction calcule un chemin pour l'instance depuis sa position et son orientation actuelles jusqu'à la position spécifiée par les arguments xgoal, ygoal. Elle utilise la taille de pas indiquée tout en essayant d'éviter toute collision avec les instances de l'objet spécifié par l'argument "obj". Comme dans la fonction mp_potential_step() elle utilise des étapes de champ potentiel pour y parvenir et la manière dont elle le fait peut être modifiée par la fonction mp_potential_settings(). Le chemin indiqué doit déjà exister et il sera écrasé par le nouveau chemin. La fonction renvoie si un chemin a été trouvé (true) ou non (false). Pour éviter que la fonction ne continue à calculer indéfiniment, vous devez fournir un facteur supérieur à 1. La fonction s'arrêtera et signalera un échec si elle ne peut pas trouver un chemin plus court que ce facteur multiplié par la distance entre le départ et le but. Un facteur de 4 est normalement suffisant, mais si vous prévoyez de longs détours, vous pouvez l'augmenter. En cas d'échec, un chemin est tout de même créé dans la direction approximative du but, mais il ne l'atteindra pas. NOTE: Cette fonction ne déplace pas l'instance. Elle définit uniquement un chemin, et vous devez utiliser les fonctions Path pour le déplacement. Syntaxe : mp_potential_path_object(path, xgoal, ygoal, stepsize, factor, obj) Arguments Description chemin indice du chemin qui doit être utilisé par la fonction xgoal La position x de la cible. ygoal La position y de la cible. taille d'étape La vitesse de déplacement de l'instance en pixels par pas. facteur Ce nombre est utilisé pour éviter une boucle infinie. Il doit être supérieur à 1. Voir la description de la fonction pour plus d'informations. obj. L'objet qui doit bloquer le chemin. Peut être un index d'objet, un id d'instance ou le mot-clé spécial,all Les retours : Booléen Exemple : path = path_add() ; mp_potential_path_object(path, obj_Player.x, obj_Player.y, 3, 4, obj_Wall) ; path_start(path, 3, 0, 0) ; Le code ci-dessus crée une nouvelle ressource path et stocke son index dans la variable "path". Il génère ensuite un chemin en utilisant mp_potential_path entre l'objet avec ce code et l'objet "obj_Player", en vérifiant qu'il n'y a pas de collision avec l'objet "obj_Wall". avec l'objet "obj_Wall". Enfin, elle lance l'objet le long de ce chemin, même si la fonction n'a pas réussi à renvoyer un chemin complet vers l'objet "obj_Player".
