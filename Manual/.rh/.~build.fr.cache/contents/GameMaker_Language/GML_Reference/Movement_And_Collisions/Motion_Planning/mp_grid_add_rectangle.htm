mp_grid_add_rectangle Cette fonction vous demande de définir un rectangle dans la pièce, puis elle marque toutes les cellules de la grille MP "touchant" ce rectangle comme étant interdites, ce qui signifie que les fonctions de recherche de chemin ne peuvent pas les traverser. L'image ci-dessous illustre le fonctionnement de cette fonction : Comme vous pouvez le constater, le rectangle défini par (50, 90) à (200, 180) marque toutes les cellules équivalentes de la grille MP qu'il touche comme étant interdites. cellules de la grille MP qu'il touche comme étant interdites. Syntaxe : mp_grid_add_rectangle(id, x1, y1, x2, y2) ; Arguments Description id Index de la grille mp_ qui doit être utilisée x1 La coordonnée x du côté gauche du rectangle à vérifier. y1 La coordonnée y du côté supérieur du rectangle à vérifier. x2 La coordonnée x du côté droit du rectangle à vérifier. y2 La coordonnée x du côté inférieur du rectangle à vérifier. Les retours : N/A Exemple : mp_grid_add_rectangle(grid, 0, 0, 100, 200) ; Le code ci-dessus marquera comme interdites toutes les cellules de la grille mp_grid indexées dans la variable "grid" qui se trouvent dans la zone 0,0 à 100,200.
