asset_has_tags Avec cette fonction, vous pouvez vérifier si une ou plusieurs chaînes d'étiquettes sont affectées à un actif à partir du navigateur d'actifs. Vous fournissez le nom de l'actif (sous forme de chaîne) ou son index d'actif, ainsi qu'une seule chaîne de balise ou un tableau où chaque élément est une seule chaîne de balise. Si vous fournissez une valeur d'index d'actif, vous devrez alors fournir l'argument de type d'actif facultatif (une constante), car les actifs de types différents peuvent avoir le même index, même s'ils ne peuvent pas avoir le même nom. Les types d'actifs disponibles sont répertoriés dans le tableau ci-dessous: Constant Description asset_object Le nom donné fait référence à un objet. asset_sprite Le nom donné fait référence à un sprite. asset_sound Le prénom fait référence à un son. asset_room Le prénom fait référence à unroom. asset_tiles Le nom donné fait référence à un jeu de tuiles. asset_path Le nom donné fait référence à un chemin. asset_script Le prénom fait référence à unscript. asset_font Le prénom fait référence à une police. asset_timeline Le prénom fait référence à une chronologie. asset_ shader Le prénom fait référence à un shader. asset_animationcurve Le nom donné fait référence à une courbe d'animation. asset_sequence Le prénom fait référence à une séquence. Si la fonction réussit et qu'une ou plusieurs balises sont présentes pour l'actif, elle retournera true sinon il reviendra false. Si vous avez besoin de vérifier une correspondance précise avec une balise ou un ensemble de balises donné, utilisez la fonction asset_has_tags(). Syntaxe:asset_has_any_tag(name_or_index, tags, [asset_type]); Argument Description name_or_index Le nom de l'actif (une chaîne) ou sa valeur d'index (un entier). tags Une seule chaîne d'étiquettes d'inventaire ou un tableau avec diverses étiquettes d'inventaire. [asset_type] OPTIONNEL! Type d'actif pour lequel vérifier les balises, utilisé uniquement lors de la fourniture d'un index pour le premier argument. Retour:Boolean Exemple:var _a = array_create(3); _a[0] = "boss"; _a[1] = "main_boss"; _a[2] = "final_boss"; if asset_has_any_tag(object_index, _a, asset_object) { instance_create_layer(0, 0, "Overlay", obj_Boss_Text); } Le code ci-dessus créera un tableau de balises, puis vérifiera si l'une d'entre elles est appliquée à l'objet donné, et si elles le sont, il créera une instance d'un autre objet.
