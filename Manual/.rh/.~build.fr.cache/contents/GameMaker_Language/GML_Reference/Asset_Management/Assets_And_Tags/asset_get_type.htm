type d'actif_get Cette fonction permet d'obtenir le type de poste référencé à partir de son nom (une chaîne de caractères). L'une des constantes listées ci-dessous sera retournée. Syntaxe : asset_get_type(name) ; Arguments Description nom Le nom de la ressource du jeu dont il faut obtenir le type (une chaîne). Les retours : Constant Constant Description objet_actif Le nom donné fait référence à un objet. actif_sprite Le nom donné fait référence à un sprite. actif_son Le prénom fait référence à un son. salle des actifs Le prénom fait référence à une pièce. piliers_de_l'actif Le nom donné fait référence à un ensemble de carreaux. chemin_de_l'actif Le nom donné fait référence à un chemin. script_de_l'actif Le nom donné fait référence à un script. asset_font Le nom donné fait référence à une police de caractères. chronologie des actifs Le prénom fait référence à une ligne de temps. asset_shader Le nom donné fait référence à un shader. courbe d'animation de l'actif Le nom donné fait référence à une courbe d'animation. séquence_d'actifs Le nom donné fait référence à une Séquence. actif_inconnu Le nom donné fait référence à un actif qui n'existe pas ou qui n'est pas l'un de ceux énumérés ci-dessus. Exemple : if asset_get_type("pth_Path_" + string(global.Game)) == asset_unknown { show_debug_message("Le chemin n'existe pas ! !!") ; } else { path_index = asset_get_index("pth_Path_" + string(global.Game)) ; } Le code ci-dessus vérifie un nom d'asset créé dynamiquement pour voir si l'asset est du bon type. Si ce n'est pas le cas, un message de débogage est affiché, sinon le nom de l'actif est utilisé pour affecter l'actif à l'instance.
