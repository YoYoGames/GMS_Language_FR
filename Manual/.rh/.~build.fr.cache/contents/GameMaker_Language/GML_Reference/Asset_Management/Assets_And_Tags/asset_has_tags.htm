actif_has_tags Avec cette fonction, vous pouvez vérifier si une ou plusieurs chaînes de balises sont affectées à un actif du navigateur d'actifs. Vous fournissez soit le nom du poste (sous forme de chaîne), soit son index, ainsi qu'une chaîne de balises unique ou un tableau où chaque élément est une chaîne de balises unique. Si vous fournissez une valeur d'index d'asset, vous devrez fournir l'argument facultatif de type d'asset (une constante), car des assets de types différents peuvent avoir le même index, même s'ils ne peuvent pas avoir le même nom. Les types d'actifs types d'actifs disponibles sont répertoriés dans le tableau ci-dessous : Constant Description objet_actif Le nom donné fait référence à un objet. actif_sprite Le nom donné fait référence à un sprite. actif_son Le prénom fait référence à un son. salle des actifs Le prénom fait référence à une pièce. piliers_de_l'actif Le nom donné fait référence à un ensemble de carreaux. chemin_de_l'actif Le nom donné fait référence à un chemin. script_de_l'actif Le nom donné fait référence à un script. asset_font Le nom donné fait référence à une police de caractères. chronologie des actifs Le prénom fait référence à une ligne de temps. asset_shader Le prénom fait référence à un shader. courbe d'animation de l'actif Le nom donné fait référence à une courbe d'animation. séquence_d'actifs Le nom donné fait référence à une Séquence. Si la fonction réussit et que toutes les balises données sont présentes pour le poste, elle renvoie true, sinon elle renvoie false. Si vous avez besoin de vérifier l'une des balises d'une sélection plutôt que toutes les balises, utilisez la fonction fonction asset_has_any_tag(). Syntaxe : asset_has_tags(name_or_index, tags, [asset_type]) ; Arguments Description nom_ou_index Le nom du poste (une chaîne de caractères) ou sa valeur d'index (un nombre entier). tags Une seule chaîne de tags d'actifs ou un tableau avec plusieurs tags d'actifs. [type d'actif] FACULTATIF ! Le type de bien pour lequel il faut vérifier les balises, utilisé uniquement lorsqu'un index est fourni pour le premier argument. Les retours : Booléen Exemple : var _a = array_create(3) ; _a[0] = "enemy" ; _a[1] = "level_" + string(global.Level) ; _a[2] = "boss" ; if asset_has_tags(object_index, _a, asset_object) { instance_create_layer(0, 0, "Overlay", obj_Boss_Text) ; } Le code ci-dessus crée un tableau de balises et vérifie ensuite si elles sont toutes appliquées à l'objet donné. Si c'est le cas, il crée une instance d'un autre objet. Retour : Actifs et étiquettes Suivant : asset_has_any_tag Copyright YoYo Games Ltd. 2021 Tous droits réservés
