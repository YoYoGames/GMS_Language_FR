asset_get_tags Avec cette fonction, vous pouvez récupérer toutes les balises affectées à un actif à partir du navigateur d'actifs. Vous fournissez soit le nom de l'actif (sous forme de chaîne), soit son index d'actif, et la fonction renverra un tableau de balises pour cet actif. Si aucune balise n'est trouvée ou s'il y a une erreur (ie: la chaîne de nom donnée n'existe pas) alors le tableau retourné sera vide. Si vous fournissez une valeur d'index d'actif, vous devrez alors fournir l'argument de type d'actif facultatif (une constante), car les actifs de types différents peuvent avoir le même index, même s'ils ne peuvent pas avoir le même nom. Les types d'actifs disponibles sont répertoriés dans le tableau ci-dessous: Constant Description asset_object Le nom donné fait référence à un objet. asset_sprite Le nom donné fait référence à un sprite. asset_sound Le prénom fait référence à un son. asset_room Le prénom fait référence à unroom. asset_tiles Le nom donné fait référence à un jeu de tuiles. asset_path Le nom donné fait référence à un chemin. asset_script Le prénom fait référence à unscript. asset_font Le prénom fait référence à une police. asset_timeline Le prénom fait référence à une chronologie. asset_ shader Le prénom fait référence à un shader. asset_animationcurve Le nom donné fait référence à une courbe d'animation. asset_sequence Le prénom fait référence à une séquence. Syntaxe:asset_get_tags(name_or_index, [asset_type]); Argument Description name_or_index Le nom de l'actif (une chaîne) ou sa valeur d'index (un entier). [asset_type] OPTIONNEL! Type d'actif pour lequel obtenir les balises, utilisé uniquement lors de la fourniture d'un index pour le premier argument. Retour:Array Exemple:my_tags = asset_get_tags(object_get_name(object_index)); Le code ci-dessus récupérera toutes les balises affectées à l'objet à partir duquel l'instance exécutant le code a été créée.
