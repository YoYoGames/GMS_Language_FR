tag_get_asset_ids Avec cette fonction, vous pouvez obtenir tous les biens d'un type donné auxquels sont attribuées les balises données. Vous fournissez soit une seule balise (sous forme de chaîne), soit un tableau, où chaque élément du tableau est une balise (sous forme de chaîne), ainsi que le type de bien à vérifier. Le type de bien doit être l'une des constantes suivantes : Constant Description objet_actif Le nom donné fait référence à un objet. actif_sprite Le nom donné fait référence à un sprite. actif_son Le prénom fait référence à un son. salle des actifs Le prénom fait référence à une pièce. piliers_de_l'actif Le nom donné fait référence à un ensemble de carreaux. chemin_de_l'actif Le nom donné fait référence à un chemin. script_de_l'actif Le nom donné fait référence à un script. asset_font Le nom donné fait référence à une police de caractères. chronologie des actifs Le prénom fait référence à une ligne de temps. asset_shader Le prénom fait référence à un shader. courbe d'animation de l'actif Le nom donné fait référence à une courbe d'animation. séquence_d'actifs Le nom donné fait référence à une Séquence. La fonction renvoie un tableau, où chaque élément du tableau est une valeur d'index de bien unique. S'il n'y a pas de biens du type ayant la ou les étiquettes données, un tableau vide sera renvoyé. Syntaxe : tag_get_asset_ids(tags, asset_type) ; Arguments Description tags Une seule chaîne de tags d'actifs ou un tableau avec plusieurs tags d'actifs. type d'actif Une constante de type d'actif (liste ci-dessus) Les retours : Array Exemple : var _paths = tag_get_asset_ids("enemy", asset_path) ; var _num = irandom(array_length(_paths) - 1) ; path_start(_paths[_num], 1, path_action_reverse, false) ; Le code ci-dessus utilise la balise "enemy" pour trouver toutes les ressources de chemin avec cette balise, avant d'en choisir une au hasard et de l'attribuer à l'instance qui exécute le code.
