chemin_décalage Avec cette fonction, vous pouvez déplacer un chemin le long de l'axe horizontal et vertical. Veuillez noter que cela n'aura aucun effet visible dans le jeu si l'instance qui suit le chemin décalé le suit de manière relative (plutôt qu'absolue) car la position du chemin n'est alors pas prise en compte. du chemin n'est pas prise en compte dans ce cas. Cette fonction modifie l'actif réel du chemin, et affectera donc de façon permanente la façon dont le chemin est utilisé par toutes les instances du jeu, depuis le moment où la fonction est utilisée jusqu'à la fin du jeu. Si ce n'est pas pas ce dont vous avez besoin, utilisez une fonction comme path_duplicate() pour créer d'abord une copie du chemin, puis appelez cette fonction sur l'actif dupliqué (n'oubliez pas d'appeler la fonction path_delete() sur la ressource lorsqu'elle n'est plus nécessaire). Syntaxe : path_shift(index, xshift, yshift) ; Arguments Description indice L'index du chemin à modifier. xshift Le nombre de pixels horizontaux pour décaler le chemin. Négatif=gauche, positif=droite. yshift Le nombre de pixels verticaux pour décaler le chemin. Négatif=haut, positif=bas. Les retours : N/A Exemple : my_path = path_duplicate(pth_AI) ; path_shift(my_path, 50, 50) ; path_start(my_path, 4, path_action_reverse, true) ; Le code ci-dessus duplique un actif de chemin et stocke sa valeur ID dans une variable. Cette variable est ensuite utilisée pour déplacer la position du chemin dupliqué, puis l'instance démarre le chemin.
