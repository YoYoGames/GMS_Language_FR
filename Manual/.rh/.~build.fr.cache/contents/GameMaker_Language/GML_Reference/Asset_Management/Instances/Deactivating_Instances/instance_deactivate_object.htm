instance_deactivate_object Avec cette fonction, vous pouvez désactiver une seule instance ou toutes les instances d'un objet spécifique parmi toutes celles qui ont été activées précédemment. Notez que si vous avez désactivé une instance ou un objet qui a été marqué comme Persistant, vous devrez le réactiver à nouveau avec la fonction instance_activate_object() avant le vestiaire, sinon il ne sera pas reporté et sera jeté à la place. Notez également que la désactivation n'est pas instantanée et qu'une instance ainsi désactivée ne sera considérée comme inactive qu'à la fin de l'événement au cours duquel la fonction a été appelée. REMARQUE: Si vous désactivez une instance au démarrage de la salle (c'est-à-dire à partir du code de création de salle, ou à partir d'un événement de création d'instance d'une instance dans la salle), toutes les instances placées dans la salle à partir de l'éditeur de salle continueront d'exécuter leur événement de création. avant d'être désactivé. AVERTISSEMENT: la désactivation des instances sur lesquelles la physique est activée n'empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur variable phy_active sur true ou false lorsque vous activez / désactivez les instances. Syntaxe:instance_deactivate_object(obj); Argument Description obj L'objet ou l'instance à désactiver (le mot-clé all peut également être utilisé). Retour:N/A Exemple:instance_deactivate_object(obj_Enemy); var _vx = camera_get_view_x(view_camera[0]); var _vy = camera_get_view_y(view_camera[0]); var _vw = camera_get_view_width(view_camera[0]); var _vh = camera_get_view_height(view_camera[0]); instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false); Le code ci-dessus désactive toutes les instances de l'objet "obj_Enemy" puis active une région dans la pièce.
