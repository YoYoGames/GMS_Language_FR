instance_destroy Vous appelez cette fonction chaque fois que vous souhaitez "détruire" une instance, en déclenchant normalement un événement de destruction et également un événement de nettoyage. Cela le supprimera de la salle jusqu'à ce que la salle soit redémarrée (sauf si la salle est persistante). L'appel de la fonction sans argument détruira simplement l'instance actuellement dans la portée et exécutant le code, mais vous pouvez fournir un argument "id" facultatif et cibler une instance spécifique en utilisant l'instance id value, ou vous pouvez cibler toutes les instances d'un objet particulier en utilisant un object_index. Par exemple:instance_destroy(other);      // destroy the "other" instance in a Collision Event instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet" Le deuxième indicateur facultatif vous permet de "désactiver" l'événement Destroy pour l'instance en cours de destruction. Par défaut, l'événement Destroy sera toujours exécuté, mais si vous définissez cet indicateur sur false alors vous pouvez faire en sorte que l'instance se détruise d'elle-même et ne pas exécuter cet événement. IMPORTANT! Cela ignorera l'événement Destroy, mais l'événement Clean Up sera toujours appelé. Il est à noter que lorsque vous détruisez une instance, son événement destroy est appelé immédiatement après le code ou l'action qui appelle la fonction destroy. De plus, bien que l'événement de destruction soit effectué, l'instance n'est pas immédiatement supprimée du jeu et elle continuera à exécuter le code contenu dans l'événement en cours. Ce n'est qu'à la fin de l'événement en cours qu'il sera retiré du jeu. Donc, si vous avez, par exemple, ce code:f hp <= 0 instance_destroy(); score += 10; La variable "score" sera incrémentée même si le instance_destroy a été appelée, et l'instance sera finalement supprimée de votre jeu à la fin de l'événement. Faites attention à cela, comme si vous aviez (par exemple) créé une ressource dynamique pour l'instance, comme une structure de données, puis détruisiez cette ressource dans l'événement de destruction, s'il y avait des références à celle-ci après la fonction ou l'action de destruction a été effectuée, vous obtiendrez une erreur "ressource inconnue", car l'événement de destruction l'a supprimée du jeu. Syntaxe:instance_destroy([id, execute_event_flag]); Argument Description id L'ID de l'instance ou object_index à détruire (facultatif, l'instance appelante par défaut) execute_event_flag Mis à true ou false pour exécuter l'événement destroy ou non (facultatif, la valeur par défaut est true ) Retour:N/A Exemple:if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height     {     instance_destroy(id, false);     } Le code ci-dessus détruira l'instance exécutant le code sans appeler l'événement Destroy si l'instance est en dehors des limites de la salle.
