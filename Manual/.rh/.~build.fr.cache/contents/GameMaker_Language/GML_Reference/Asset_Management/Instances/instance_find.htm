instance_find Toutes les instances ont un identifiant unique (id ) qui peut être utilisé pour les modifier et les manipuler pendant qu'un jeu est en cours d'exécution, mais vous ne savez peut-être pas toujours quel est l'identifiant d'une instance spécifique et cette fonction peut donc vous aider car vous pouvez l'utiliser pour les parcourir tous pour trouver ce que vous avez besoin. Vous spécifiez l'objet dont vous souhaitez trouver l'instance et un nombre, et s'il y a une instance à cette position dans la liste d'instances, la fonction renvoie l'id de cette instance, et sinon, elle renvoie le mot-clé spécial noone. Vous pouvez également utiliser le mot-clé all pour parcourir toutes les instances d'une salle, ainsi qu'un objet parent pour parcourir toutes les instances qui font partie de cette hiérarchie parent / enfant, et vous pouvez même spécifier une instance (si vous avez son id ) pour vérifier s'il existe réellement dans la pièce actuelle. Veuillez noter que comme les instances sont triées de manière arbitraire, il n'y a pas d'ordre spécifique dans la manière dont les instances sont vérifiées par cette fonction, et toute instance peut être dans n'importe quelle position. La valeur maximale de "n" dans cette fonction serait Pour le mot-clé all:instance_count - 1 Pour un index d'objet:instance_number(OBJ) - 1 Syntaxe:instance_find(obj, n); Argument Description obj L'objet pour trouver la nième instance de n Le numéro de l'instance à rechercher. Retour:Real (either an instance ID value or the keyword noone) Exemple:for (var i = 0; i < instance_number(obj_Enemy); ++i;)     {     enemy[i] = instance_find(obj_Enemy,i);     } Le code ci-dessus utilisera un for boucle pour parcourir toutes les instances de "obj_Enemy" et stocker leur identifiant dans le tableau "ennemi []".
