instance_désactivation_région Cette fonction vous permet de définir une région dans la salle pour désactiver les instances qui ont été précédemment activées. Cette région peut être marquée comme "intérieure" ou "extérieure", comme le montre l'image suivante : Vous pouvez voir dans l'image ci-dessus que l'instance "apple" est toujours inactive car, même si le sprite lui-même ne chevauche pas la région, sa boîte englobante le fait. Ainsi, les instances sont considérées comme étant dans la région spécifiée lorsque leur boîte englobante la recouvre, et l'état du masque de collision (précis ou non) n'est pas pris en compte. (précis ou non) n'est pas pris en considération. Notez que la désactivation n'est pas instantanée, et une instance qui a été désactivée de cette manière ne sera pas considérée comme inactive avant la fin de l'événement dans lequel la fonction a été appelée. appelée. NOTE: Si vous désactivez une instance au démarrage de la room (c'est-à-dire à partir du code de création de la room, ou à partir d'un événement de création d'une instance dans la room), toutes les instances qui sont placées dans la room à partir de l'éditeur de room exécuteront toujours leur événement de création avant d'être désactivées. AVERTISSEMENT: Désactiver des instances dont la physique est activée n' empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur phy_active variable à true ou false lorsque vous activez/désactivez les instances. Syntaxe : instance_deactivate_region(left, top, width, height, inside, notme) ; Arguments Description gauche La coordonnée x de la partie gauche de la région rectangulaire à désactiver. top La coordonnée y du haut de la région rectangulaire à désactiver. largeur La largeur de la région à désactiver. hauteur La hauteur de la région à désactiver. à l'intérieur de Si vous souhaitez désactiver les instances à l'intérieur de la région (true) ou à l'extérieur (false). notme Exclure l'instance appelante de la désactivation (true) ou non (false).
