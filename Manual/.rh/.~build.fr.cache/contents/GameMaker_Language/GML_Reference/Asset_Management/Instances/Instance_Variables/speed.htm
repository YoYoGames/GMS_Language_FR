speed Toutes les instances de GameMaker Studio 2 ont certaines propriétés «intégrées» que vous pouvez utiliser et définir pour régir leur apparence et leur comportement.speed est l'une de ces propriétés et définit le nombre de pixels que l'instance déplacera à chaque étape. contrairement à hspeed et vspeed, speed n'a aucune direction qui lui est associée car elle est régie par le direction valeur de l'instance, mais elle peut avoir une valeur négative, auquel cas l'instance se déplacera dans la direction opposée à celle définie par le direction fonction (c'est-à-dire:direction - 180 °). Notez que le réglage du speed et / ou le direction, modifiera également les valeurs de hspeed et vspeed variables, et quegravity,gravity_direction etfriction peuvent tous modifier la valeur de cette variable lorsqu'ils sont utilisés dans vos jeux. Syntaxe:speed; Retour:Real (single precision floating point value) Exemple:if keyboard_check(vk_up) speed = 2; if keyboard_check(vk_left) direction += 5; if keyboard_check(vk_right) direction -= 5; Le code ci-dessus utilisera les touches fléchées pour définir la vitesse et la direction de l'instance.
