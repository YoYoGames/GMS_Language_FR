vitesse Toutes les instances de GameMaker Studio 2 possèdent certaines propriétés "intégrées" que vous pouvez utiliser et définir pour déterminer leur apparence et leur comportement. La vitesse est l'une de ces propriétés et définit le nombre de pixels que l'instance déplacera à chaque pas. Contrairement à hspeed et vspeedla vitesse n'est pas associée à une direction car elle est régie par la valeur de l'instance. direction de l'instance, mais elle peut avoir une valeur négative, dans ce cas l'instance se déplacera dans la direction opposée à celle définie par la fonction direction (ex : direction - 180°). Notez que le paramétrage de speed et/ou de direction, modifiera également les valeurs des variables hspeed et vspeed, et que la fonction gravity, gravity_direction et friction peuvent toutes modifier la valeur de cette variable lorsqu'elles sont utilisées dans vos jeux. Syntaxe : vitesse ; Les retours : Réel (valeur à virgule flottante de simple précision) Exemple : si keyboard_check(vk_up) speed = 2 ; si keyboard_check(vk_left) direction += 5 ; si keyboard_check(vk_right) direction -= 5 ; Le code ci-dessus utilisera les touches fléchées pour définir la vitesse et la direction de l'instance.
