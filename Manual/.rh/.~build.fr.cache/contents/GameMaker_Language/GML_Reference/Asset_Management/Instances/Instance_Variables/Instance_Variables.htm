Variables d'instance Lorsque vous créez un nouvel objet, il viendra avec certaines variables déjà initialisées avec des valeurs par défaut. Nous appelons ces variables des variables d' instance, car elles seront créées pour chaque instance que vous placez de l'objet dans vos salles de jeux, et une fois qu'une instance est créée, les valeurs de ces variables sont uniques à cette instance et uniquement à cette instance. Certaines de ces variables que vous utiliserez beaucoup, comme x et y, tandis que d'autres sont moins nécessaires en fonction de ce que vous voulez faire, mais en général, ils sont très utiles et, dans la mesure du possible, vous devriez chercher à les utiliser plutôt que de créer vos propres variables personnalisées. La plupart des variables d'instance peuvent être définies aussi bien que lues, ce qui vous permet de modifier les propriétés et le comportement d'une instance simplement en modifiant la valeur d'une certaine variable - vous pouvez empêcher une instance de dessiner, par exemple, en définissant simplement le visible variable intégrée à false. Vous trouverez ci-dessous les différentes variables initialisées pour toutes les instances de tous les objets de votre jeu. IMPORTANT! Si une instance est dans unsequence puis certaines de ces variables - commeimage_xscale /image_yscale / image_angle /x /y - sera écrasé lorsque la séquence sera mise à jour à chaque étape après le début de la lecture.
