La structure de la voie La structure de piste peut être utilisée soit pour le niveau supérieur Asset track s ou pour sous-track paramètres des pistes, et le comportement de la piste est défini par deux choses, son nom et son type. Les structures de suivi sont créées à l'aide de la fonction sequence_track_new() et peuvent être récupérés à partir des actifs de séquence à l'aide de activeTracks propriété de la structure d'instance de séquence ou tracks propriété de la structure de l'objet de séquence. Les propriétés disponibles pour la structure de piste sont:type: Les constantes suivantes peuvent toutes être utilisées pour définir le type de piste en cours de création dans la séquence (ces constantes sont également utilisées lors de la génération d' keyframes et de keyframe, expliquées dans les sections ci-dessous): Constant Description Évaluer seqtracktype_graphic Il s'agit d'une piste d'actifs graphiques (sprite). 1 seqtracktype_audio Ceci est une piste d'actif audio. 2 seqtracktype_instance Ceci est une piste d'actif d'instance. 14 seqtracktype_sequence Il s'agit d'une piste d'actif de séquence. 7 seqtracktype_clipmask Il s'agit d'une piste d'actif de groupe de masques de clip. 8 seqtracktype_clipmask_mask Il s'agit d'une piste d'actif de sprite de masque de clip utilisée pour générer le masque de clip. 9 seqtracktype_clipmask_subject Il s'agit d'une piste d'actif de sprite de masque de clip qui est masquée. 10 seqtracktype_group Cette piste d'actif de dossier de groupe. 11 seqtracktype_colour Il s'agit d'une piste de paramètres de données de couleur. 4 seqtracktype_real Il s'agit d'une piste de paramètre de valeur de nombre réel. 3 seqtracktype_message Ceci est une piste de message de diffusion. 15 seqtracktype_moment Ceci est une piste d'événement / moment. 16 seqtracktype_bool Non utilisé actuellement. 5 seqtracktype_string Non utilisé actuellement. 6 seqtracktype_spriteframes Non utilisé actuellement. 13 seqtracktype_empty Non utilisé actuellement. 12 name: Lors de la création d'une piste d'actif, le nom que vous donnez ici peut être n'importe quelle chaîne dont vous avez besoin pour identifier la piste, mais pour les pistes de paramètres, vous devez spécifier des chaînes spécifiques pour indiquer à GameMaker Studio 2 type de piste de paramètres que vous créez (ces les chaînes ne s'appliquent qu'aux seqtracktype_real et seqtracktype_colour types de piste de paramètres): " position " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme données de position. " scale " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme données de mise à l'échelle. " rotation " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme données de rotation. " blend_multiply " - lorsqu'il est défini sur une piste de type seqtracktype_colour cela indique que le contenu de cette piste doit être utilisé comme données de multiplication des couleurs. Les keyframe pour une telle piste doivent être un tableau de quatre valeurs au format [A, R, V, B]. Notez que les valeurs de chaque composant sont exprimées entre 0 et 1, où 0 correspond à la valeur HEX # 00 et 1 correspond à la valeur HEX #FF (0 à 255 comme indiqué dans le sélecteur de couleur pour les pistes de fusion d'image dans les séquences Éditeur). " image_speed " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme valeur de vitesse d'image. " image_index " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme valeur d'index de l'image. " image_angle " - lorsqu'il est défini sur une piste de type seqtracktype_real cela indique que le contenu de cette piste doit être utilisé comme valeur d'angle de l'image. " image_blend " - lorsqu'il est défini sur une piste de type seqtracktype_colour cela indique que le contenu de cette piste doit être utilisé comme valeur de fusion d'image. Les keyframe pour une telle piste doivent être un tableau de quatre valeurs au format [A, R, V, B]. Notez que les valeurs de chaque composant sont exprimées entre 0 et 1, où 0 correspond à la valeur HEX # 00 et 1 correspond à la valeur HEX #FF (0 à 255 comme indiqué dans le sélecteur de couleur pour les pistes de multiplication de couleurs dans les séquences Éditeur).tracks: Cette propriété permet d'accéder à la liste des pistes qui sont enfants de cette piste. Lors de l'obtention de cette propriété, un tableau de structures de suivi est renvoyé et lors de la définition de cette propriété, un tableau de structures de suivi doit être spécifié.visible: Ceci indique si cette piste est visible (la valeur est true ) ou non (la valeur est false ). Vous pouvez obtenir ou définir cette valeur et si une piste n'est pas visible, aucune de ses pistes enfants ne sera dessinée non plus.keyframes: Cette propriété permet d'accéder à la liste des structures d' keyframe de la piste. Lors de l'obtention de cette propriété, un tableau de keyframe est renvoyé, et lors de la définition de cette propriété, un tableau de keyframe doit être spécifié.
