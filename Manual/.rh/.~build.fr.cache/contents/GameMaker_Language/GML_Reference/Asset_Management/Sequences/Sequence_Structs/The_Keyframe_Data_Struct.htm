La Keyframe données d'images clés Lorsque vous définissez une structure d' keyframe pour une piste, vous devez également définir les données qui lui seront associées. Celui-ci est composé de différents canaux et chaque canal peut recevoir un type de données spécifique en fonction du type de piste pour lequel vous définissez les données d' keyframe Ces propriétés sont:channel: Il s'agit du canal auquel les keyframe doivent être appliquées. Il s'agit d'une valeur entière positive commençant à 0, et il convient de noter que lors de la création de pistes de paramètres pour les keyframes «position» ou «échelle», vous devez utiliser des valeurs de canal très spécifiques. Ce sont: le canal 0 est la position X ou l'échelle X, le canal 1 est la position Y ou l'échelle Y.spriteIndex: L'index de la ressource de sprite à utiliser pour la piste. Cette propriété n'est disponible que pour les pistes du type seqtracktype_graphic et vous pouvez l'obtenir ou le définir.soundIndex: L'index de l'élément audio à utiliser pour la piste. Cette propriété n'est disponible que pour les pistes du type seqtracktype_audio et vous pouvez l'obtenir ou le définir.playbackMode: Le mode de lecture du son. Cette propriété n'est disponible que pour les pistes du type seqtracktype_audio et vous pouvez l'obtenir ou le définir. La propriété doit être l'une des constantes suivantes: Constant Description Évaluer seqaudiokey_loop Le son bouclera lors de la lecture. 0 seqaudiokey_oneshot Le son ne jouera qu'une seule fois puis s'arrêtera. 1 curve: Cette propriété nécessite une structure de courbe d'animation (voir ici pour plus d'informations) et n'est disponible que pour les pistes de type seqtracktype_real. Si aucune structure de courbe n'est utilisée, la valeur de cette propriété sera -1.value: Cette propriété est simplement une valeur associée au keyframe, et n'est disponible que pour les pistes du type seqtracktype_real quand non curve struct est fourni.colour: Cette propriété renvoie (ou requiert, si elle est définie) un tableau pour la valeur de couleur de l' keyframe au format [A, R, V, B]. Ceci n'est disponible que pour les pistes du type seqtracktype_colour. Notez que les valeurs de chaque composant doivent être exprimées entre 0 et 1, où 0 correspond à la valeur HEX # 00 et 1 correspond à la valeur HEX #FF (0 à 255 comme indiqué dans le sélecteur de couleur pour les pistes de couleur dans les séquences Éditeur).sequence: Cette propriété renverra (ou nécessite, si elle est définie) une structure d'objet de séquence et n'est disponible que pour les pistes de type seqtracktype_sequence.objectIndex: Cette propriété renverra (ou nécessite, si elle est définie) un index d'objet et n'est disponible que pour les pistes de type seqtracktype_instance.events: Cette propriété permet d'accéder aux événements et aux messages de diffusion associés à la structure de données d' keyframe Vous pouvez obtenir ou définir cette propriété, et lors de son obtention, un tableau de chaînes est renvoyé, et lors de sa définition, un tableau de chaînes doit être spécifié. Pour plus d'informations sur les événements, veuillez consulter la section Événements et moments de séquence. Cette propriété n'est disponible que pour les pistes du type seqtracktype_message.event: Cette propriété renverra (ou peut être définie sur) lemethod associé à la structure de données d' keyframe Si aucune méthode n'a été spécifiée ou si vous souhaitez supprimer la méthode, la propriété doit être -1.
