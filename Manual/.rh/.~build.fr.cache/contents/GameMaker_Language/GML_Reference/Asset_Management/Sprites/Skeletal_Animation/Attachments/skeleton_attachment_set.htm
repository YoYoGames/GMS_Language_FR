skeleton_attachment_set Un sprite d'animation squelettique peut avoir d'autres images ajoutées en tant que pièces jointes, ces images étant ajoutées à un emplacement nommé (le nom est donné lorsque vous créez l'emplacement de pièce jointe dans votre programme d'animation) et elles seront dessinées avec l'animation du sprite actuel. Avec cette fonction, vous pouvez définir une pièce jointe à un emplacement donné, où vous devez donner les noms (sous forme de chaînes) de l'emplacement et de la pièce jointe. Ces noms sont définis par le programme d'animation utilisé, ou (dans le cas de la pièce jointe) lorsque vous appelez skeleton_attachment_create(). Notez que vous pouvez également passer un sprite_index en tant que pièce jointe, et ce sprite sera utilisé, ou vous pouvez utiliser -1 pour supprimer la pièce jointe de l'emplacement. Lorsque vous passez un index de sprite en tant qu'argument, il créera un emplacement de pièce jointe en utilisant le nom du sprite comme chaîne pour nommer l'emplacement (donc en utilisant spr_sword, par exemple, créera un emplacement de pièce jointe "spr_sword"), et l'emplacement utilisera le premier index d'image (0) du sprite, ainsi que son décalage d'origine x et y, avec une échelle de (1,1) et une rotation de 0. Syntaxe:skeleton_attachment_set(slot, attachment); Argument Description slot Le nom de l'emplacement (une chaîne) pour obtenir la pièce jointe. attachment Le nom (sous forme de chaîne ou de sprite_index) de l'image de la pièce jointe. Retour:String Exemple:if skeleton_attachment_get("slot_leftHand") == ""     {     skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));     } Le code ci-dessus vérifierait le nom de pièce jointe actuellement attribué pour l'emplacement nommé "slot_leftHand" et si une chaîne vide est renvoyée, un nouveau sprite est attaché.
