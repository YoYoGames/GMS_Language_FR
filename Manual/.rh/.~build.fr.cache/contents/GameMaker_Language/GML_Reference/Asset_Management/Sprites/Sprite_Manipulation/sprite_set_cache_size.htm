sprite_set_cache_size Lors de la fusion d'images, HTML5 ne peut pas le faire de manière dynamique de la même manière qu'un exécutable pourrait être exécuté. Par conséquent, GameMaker Studio 2 doit temporairement enregistrer une copie mélangée de l'image et la charger. Cette fonction définit le nombre de copies mélangées du sprite donné pouvant être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. Ceci est appliqué à toutes les sous-images du sprite. Syntaxe:sprite_set_cache_size(ind, max); Argument Description ind L'index de l'image-objet pour modifier la taille du cache. max Le nombre maximal de copies mises en cache de l'image-objet pouvant être stockées. RetourN/A Exemple:sprite_set_cache_size(sprite0, 2); Cela définira le cache de sprite de toutes les sous-images de sprite0 à 2 copies.
