image_index Un sprite est composé d'une ou plusieurs sous-images qui peuvent faire apparaître le sprite animé en passant de l'une à l'autre, ou qui peuvent être basculées entre elles dans le code pour donner différents "états", un peu comme un bouton dans Windows. Si le sprite est animé, vous pouvez obtenir l'image actuelle de l'animation en vérifiant la variable image_index, ou si vous voulez changer l'état d'un sprite statique, vous pouvez sélectionner une nouvelle sous-image en réglant cette variable sur la sous-image souhaitée du sprite. sous-image souhaitée du sprite. Veuillez noter que pour que les modifications de cette variable soient visibles, l'instance ne doit pas avoir d'événement de dessin ( GameMaker Studio 2 dessinera donc le sprite par défaut) ou être dessinée à l'aide d'une des fonctions de dessin telles que draw_self() ou draw_sprite_ext() (en fournissant l'index de l'image dans l'argument approprié). Notez que le fait de définir cette variable directement sur une image ne déclenchera pas les messages de diffusion qui peuvent être présents sur cette image. Notez également que lorsque vous utilisez des sprites d'animation squelettique, vous pouvez toujours obtenir et définir les valeurs image_index - voir la fonction skeleton_animation_get_duration() pour pour des exemples sur la façon de le faire. Syntaxe : image_index ; Les retours : Réel (valeur à virgule flottante de simple précision) Exemple : si (image_speed > 0) { if (image_index >= image_number - 1) instance_destroy() ; } Le code ci-dessus vérifie si le sprite est en cours d'animation, et si c'est le cas, il vérifie si l'image_index actuelle est à la dernière image et dans ce cas, il détruit l'instance.
