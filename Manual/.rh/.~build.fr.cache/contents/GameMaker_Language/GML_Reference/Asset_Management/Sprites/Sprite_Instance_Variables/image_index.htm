image_index Un sprite est composé d'une ou de plusieurs sous-images qui peuvent donner l'impression que le sprite est animé lors de son passage de l'une à l'autre, ou peuvent-ils être basculés entre eux dans le code pour donner différents "états", un peu comme un bouton dans les fenêtres. Si le sprite est animé, vous pouvez obtenir l'image actuelle de l'animation en vérifiant la image_index, ou si vous souhaitez changer l'état d'un sprite statique, vous pouvez sélectionner une nouvelle sous-image en définissant cette variable sur la valeur souhaitée sous-image du sprite. Veuillez noter que pour que les modifications de cette variable soient visibles, l'instance doit avoir soit aucun événement draw (et donc GameMaker Studio 2 dessine par défaut le sprite) soit être dessinée à l'aide de l'une des fonctions de dessin comme draw_self() ou draw_sprite_ext() ( en fournissant l'image_index dans l'argument approprié). Notez que la définition de cette variable directement sur une trame ne déclenchera aucun message de diffusion qui peut être présent sur cette trame. Notez également que lorsque vous utilisez des sprites d'animation squelettiques, vous pouvez toujours obtenir et définir les image_index - voir la fonction skeleton_animation_get_duration() pour des exemples de la manière de procéder. Syntaxe:image_index; Retour:Real (single precision floating point value) Exemple:if (image_speed > 0)     {     if (image_index >= image_number - 1) instance_destroy();     } Le code ci-dessus vérifie si le sprite est animé, et si c'est le cas, il vérifie si l'image_index actuelle est à la dernière image et dans ce cas détruit l'instance.
