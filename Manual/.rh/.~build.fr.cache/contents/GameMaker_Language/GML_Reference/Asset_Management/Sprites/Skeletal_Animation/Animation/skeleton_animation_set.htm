jeu_animation_squelette Une fois que vous avez assigné un sprite d'animation squelettique à votre instance, vous pouvez définir le jeu d'animation qu'il doit utiliser à tout moment avec cette fonction. Lorsque vous avez créé votre sprite (dans Spine), vous avez défini plusieurs jeux d'animations pour différents états et les avez nommés de manière appropriée. pour différents états et nommé chacun d'entre eux de manière appropriée. Ces noms vous permettront d'indiquer à GameMaker Studio 2 quel jeu d'animations utiliser à tout moment dans votre jeu. Par exemple, si votre sprite est destiné à un jeu de plateforme, vous pouvez avoir un jeu de "saut", un jeu de "course" et un jeu de "ralenti", tous contenus dans le même sprite, et vous pouvez passer de l'un à l'autre dans votre jeu à l'aide de cette fonction. Notez que le fait de paramétrer une nouvelle animation de cette manière réinitialisera le paramètre image_index à 0, faisant démarrer la nouvelle animation depuis le début de sa boucle d'animation. Si ce n'est pas le comportement Si ce n'est pas le comportement souhaité, vous pouvez définir une valeur de "mélange" à l'aide de la commande mélange_animation_squelette() et lorsque vous définissez une nouvelle animation, elle sera interpolée en douceur avec l'ensemble précédent. Vous devez également vous d'avoir une image-clé au début de chaque animation afin d'éviter que des données ne soient conservées à partir des animations précédentes, à moins que vous ne prévoyiez d'utiliser des pistes d'animation séparées. Syntaxe : skeleton_animation_set(animname) ; Arguments Description nom de l'animateur Le nom (une chaîne) du jeu d'animation à utiliser. Les retours : N/A Exemple : if keyboard_check_pressed(vk_space) { if skeleton_animation_get() != "jump" { skeleton_animation_set("jump") ; } } Le code ci-dessus changera le jeu d'animation utilisé en jeu de "saut" lorsque la touche espace sera pressée, mais seulement si le jeu actuel utilisé n'est pas déjà "saut".
