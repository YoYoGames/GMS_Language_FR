skeleton_animation_set Une fois que vous avez attribué un sprite d'animation squelettique à votre instance, vous pouvez définir le jeu d'animations à utiliser à tout moment avec cette fonction. Lorsque vous avez créé votre sprite (dans Spine), vous aurez défini plusieurs ensembles d'animations pour différents états et nommé chacun d'eux de manière appropriée. Ces noms sont ce que vous utiliserez pour indiquer à GameMaker Studio 2 jeu d'animations à utiliser à tout moment dans votre jeu. Par exemple, si votre sprite est pour un jeu de plateforme, alors vous pouvez avoir un ensemble "jump", un set "run" et un set "idle" tous contenus dans un sprite et vous pouvez basculer entre eux dans votre jeu en utilisant ce fonction. Notez que définir une nouvelle animation de cette manière réinitialisera le image_index à 0, en commençant le nouvel ensemble d'animations depuis le début de sa boucle d'animation. Si ce n'est pas le comportement souhaité, vous pouvez définir une valeur "mix" en utilisant skeleton_animation_mix() et lorsque vous définissez une nouvelle animation, elle sera interpolée en douceur avec l'ensemble précédent. Vous devez également vous assurer d'avoir une keyframe au début de chaque animation pour éviter que les données ne soient conservées des animations précédentes, sauf si vous prévoyez d'utiliser des pistes d'animation distinctes. Syntaxe:skeleton_animation_set(animname); Argument Description animname Le nom (une chaîne) du jeu d'animations à utiliser. Retour:N/A Exemple:if keyboard_check_pressed(vk_space)     {     if skeleton_animation_get() != "jump"         {         skeleton_animation_set("jump");         }     } Le code ci-dessus changera le jeu d'animation utilisé en jeu de "saut" lorsque la touche d'espace est pressée, mais seulement si le jeu actuel utilisé n'est pas déjà "saut".
