dessiner_squelette Cette fonction ne doit être utilisée qu' avec des sprites créés à l'aide d'un programme d'animation de squelette comme Spine. Bien que vous puissiez dessiner ces sprites en utilisant les fonctions de dessin normales (comme draw_self()), il arrive que vous souhaitiez dessiner une seule image ou montrer un changement de texture de peau sans modifier les propriétés du sprite. Dans ces cas, vous pouvez dessiner le sprite avec cette fonction, où vous donnez l'index du sprite et ensuite le nom de l'animation dont il faut extraire l'image (une chaîne, telle que définie dans le programme utilisé pour créer le sprite). Ensuite, vous donnez le nom du skin à utiliser (encore une fois, tel que défini lors de la création du sprite), et une valeur de frame. La valeur d'image sert à indiquer à GameMaker Studio 2 quelle partie de l'animation doit être affichée. Il s'agit d'une valeur comprise entre 0 et la valeur . nombre_d'images. Les autres arguments sont les mêmes que ceux utilisés pour le dessin normal d'un sprite. l'échelle, l'angle, la couleur de fusion et l'alpha de l'image d'animation finale dessinée. ATTENTION ! En raison de la façon dont les animations squelettiques sont interpolées entre les différentes images "clés", il y a une surcharge supplémentaire associée à cet appel et il est donc recommandé d'éviter de dessiner des sprites basés sur des squelettes en utilisant cette méthode, sauf en cas de nécessité absolue. cette méthode, sauf en cas de nécessité absolue. Syntaxe : draw_skeleton(sprite, animname, skinname, frame, x, y, xscale, yscale, rot, colour, alpha) ; Arguments Description sprite L'index du sprite à dessiner. nom de l'animateur Le nom de l'animation dont il faut récupérer l'image (une chaîne). nom de peau Le nom du skin à utiliser (une chaîne). cadre Le cadre d'animation à dessiner (de 0 à nombre_d'images - 1). x La coordonnée x de l'endroit où dessiner le sprite. y La coordonnée y de l'endroit où dessiner le sprite. xscale La mise à l'échelle horizontale du sprite, sous forme de multiplicateur : 1 = mise à l'échelle normale, 0.5 est la moitié etc... yscale La mise à l'échelle verticale du sprite, sous forme de multiplicateur : 1 = mise à l'échelle normale, 0.5 est la moitié etc... pourriture La rotation du sprite. 0=normal, 90=tourné de 90 degrés dans le sens inverse des aiguilles d'une montre, etc. couleur La couleur avec laquelle le sprite doit être mélangé. alpha L'alpha du sprite (de 0 à 1 où 0 est transparent et 1 opaque). Les retours : N/A Exemple : draw_skeleton(spr_Jump, "jump", "skin1", 0, x, y, image_xscale, image_yscale, image_angle, c_white, 0.5) ; Le code ci-dessus dessinera la première image du sprite d'animation squelettique donné, en utilisant l'ensemble d'animation "jump", le skin défini comme "skin1" et à la même échelle et position que les propriétés de l'instance définissent, mais avec un alpha de 0,5.
