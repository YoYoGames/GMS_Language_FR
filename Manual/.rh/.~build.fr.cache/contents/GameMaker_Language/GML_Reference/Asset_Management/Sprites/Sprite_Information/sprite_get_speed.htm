sprite_get_speed Cette fonction peut être utilisée pour récupérer la vitesse du sprite telle que définie pour la ressource sprite dans l'éditeur de sprites. La valeur renvoyée peut alors être utilisée, par exemple, pour calculer le nombre d'images pouvant être dessinées pour différents types de sprites. combien d'images peuvent être dessinées pour différentes image_speed. Notez que la valeur de retour sera très différente en fonction du type de vitesse qui a été appliquée dans l'éditeur de Sprite, soit Frames Per Second, soit Frames Per Game Frame. Les deux exemples suivants illustrent cela : Si vous avez un sprite qui dessine 1 image par seconde et que vous réglez la vitesse de l'image sur 0,5, il sera dessiné à 0,5 image par seconde. Si la fréquence d'images de votre jeu est de 60 images par seconde, le sprite dessinera une image toutes les 120 images du jeu. images du jeu. Si vous avez un sprite qui dessine 1 image par image du jeu et que vous réglez la vitesse de l'image sur 0,5, il dessinera 0,5 image par image du jeu. Si la fréquence d'images de votre jeu est de 60 images par seconde, le sprite affichera 30 images pour 60 images de jeu. toutes les 60 images du jeu. Vous pouvez connaître le type de vitesse d'animation utilisé lors de la définition du sprite à l'aide de la fonction sprite_get_speed_type()et vous pouvez définir la vitesse et le type d'animation à l'aide de la fonction sprite_set_speed(). Syntaxe : sprite_get_speed(index) Arguments Description indice L'index du sprite dont on veut obtenir la vitesse Les retours : Real Exemple : s_speed = sprite_get_speed(sprite_index) ; s_type = sprite_get_speed_type(sprite_index) ; Le code ci-dessus récupère la vitesse du sprite et le type d'animation du sprite et les stocke dans des variables d'instance pour une utilisation ultérieure.
