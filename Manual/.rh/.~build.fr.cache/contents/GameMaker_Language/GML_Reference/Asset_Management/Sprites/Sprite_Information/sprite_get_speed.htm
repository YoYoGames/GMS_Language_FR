sprite_get_speed Cette fonction peut être utilisée pour récupérer la vitesse de sprite telle que définie pour la ressource de sprite dans l' éditeur de sprite. La valeur renvoyée peut ensuite être utilisée, par exemple, pour calculer le nombre d'images pouvant être dessinées pour différents image_speed. Notez que la valeur de retour sera très différente selon le type de vitesse qui a été appliqué dans l'éditeur de sprite, soit images par seconde, soit images par image de jeu. Les deux exemples suivants illustrent ceci: Si vous avez un sprite qui dessine 1 image par seconde et définissez la vitesse de l'image à 0,5, il dessinera à 0,5 image par seconde. Si la fréquence d'images de votre jeu est de 60 images par seconde, le sprite dessinera 1 image pour 120 images de jeu. Si vous avez un sprite qui dessine 1 image par image de jeu et définissez la vitesse d'image sur 0,5, il dessinera 0,5 image par image de jeu. Si la fréquence d'images de votre jeu est de 60 images par seconde, alors le sprite dessinera 30 images pour 60 images de jeu. Vous pouvez savoir quel type de vitesse d'animation a été utilisé lors de la définition du sprite à l'aide de la fonction sprite_get_speed_type(), et vous pouvez définir la vitesse et le type d'animation à l'aide de la fonction sprite_set_speed(). Syntaxe:sprite_get_speed(index) Argument Description index L'index du sprite pour obtenir la vitesse de Retour:Real Exemple:s_speed = sprite_get_speed(sprite_index); s_type = sprite_get_speed_type(sprite_index); Le code ci-dessus obtient la vitesse du sprite et le type d'animation du sprite et les stocke dans des variables d'instance pour une utilisation future.
