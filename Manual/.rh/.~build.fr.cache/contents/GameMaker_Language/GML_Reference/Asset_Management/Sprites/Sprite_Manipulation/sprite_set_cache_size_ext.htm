sprite_set_cache_size_ext Lorsqu'il effectue le mélange d'images, HTML5 ne peut pas le faire de manière dynamique de la même manière qu'un exécutable. GameMaker Studio 2 doit donc enregistrer temporairement une copie mélangée de l'image et la charger. Cette fonction définit le nombre de copies mélangées du sprite donné peuvent être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. Ceci est appliqué à une seule sous-image donnée du sprite. Syntaxe : sprite_set_cache_size_ext(ind, index, max) ; Arguments Description ind L'index du sprite dont la taille du cache doit être modifiée. indice La sous-image de ind pour changer la taille du cache. max Le nombre maximum de copies en cache du sprite qui peuvent être stockées. Renvoie à N/A Exemple : sprite_set_cache_size_ext(sprite0, 0, 2) ; Le cache du sprite de la première sous-image du sprite0 sera ainsi réglé sur 2 copies.
