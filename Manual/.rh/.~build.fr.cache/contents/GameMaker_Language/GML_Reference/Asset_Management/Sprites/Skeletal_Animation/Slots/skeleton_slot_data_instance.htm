skeleton_slot_data_instance Avec cette fonction, vous pouvez remplir une liste DS (pré-créée) avec des données pour chacun des emplacements de pièce jointe disponibles pour le sprite Spine associé à l'instance actuelle (y compris toutes les modifications de pièce jointe). Ces données sont renvoyées dans le forum d'une carte DS qui contient les paires clé / valeur suivantes: "nom": le nom de l'emplacement. "bone": Le nom de l'os. "pièce jointe": le nom de la pièce jointe si elle est utilisée ou "(aucune)" si aucune pièce n'est attribuée. Les valeurs de chaque clé seront des chaînes et peuvent ensuite être utilisées dans les autres fonctions de rattachement squelette pour ces types de sprite. Notez que le DS Map créés ne sont pas détruits, vous devrez donc parcourir la liste DS et détruire vous-même chacune des cartes créées. Syntaxe:skeleton_slot_data_instance(list); Argument Description list ID de la liste DS à remplir avec les cartes DS. Retour:N/A Exemple:var list = ds_list_create(); var open = true; slot_name = ""; skeleton_slot_data_instance(list); for (var i = 0; i < ds_list_size(list); i++;)     {     var map = list[| i];     if open         {         if map[? "attachment"] == "(none)"             {             open = false;             slot_name = map[? "name"];             }         }     ds_map_destroy(map);     } ds_list_destroy(list); Le code ci-dessus crée une liste DS, puis la remplit avec les données d'emplacement pour l'image-objet d'instance. Ces données sont ensuite analysées pour extraire les cartes DS individuelles avec les données de slot. Ceci est ensuite vérifié pour voir s'il y a un emplacement vide, et si tel est le cas, la variable "nom_emplacement" reçoit le nom de l'emplacement vide avant que la carte DS ne soit détruite.
