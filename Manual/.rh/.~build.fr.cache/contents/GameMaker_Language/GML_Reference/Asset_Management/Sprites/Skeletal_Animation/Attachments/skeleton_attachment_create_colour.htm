attachement_squelette_créer_couleur Lorsque vous créez votre sprite d'animation squelettique, vous pouvez lui attribuer des emplacements d'attachement et des attachements à y placer. Il s'agit simplement d'images (sprites) qui sont séparées de l'animation mais qui, lorsqu'elles sont attachées, se déplacent avec l'os auquel elles sont attachées. attachés. Normalement, vous attribuez ces pièces jointes dans votre programme d'animation (Spine), mais avec cette fonction, vous pouvez créer les vôtres au moment de l'exécution en utilisant un sprite de votre jeu. une chaîne de caractères), puis de définir les paramètres sprite_index et image_index à utiliser, ainsi que l'origine x et y (qui peut être différente de celle définie par le sprite). être différente de celle définie par le sprite dans les propriétés du sprite), et vous pouvez également définir toutes les transformations que vous souhaitez appliquer à l'image lorsqu'elle est jointe, ainsi que la couleur à mélanger avec l'image et sa valeur alpha (transparence). de l'image. Si la pièce jointe a été créée avec succès, la fonction renverra 1, sinon (vous avez fourni un index de sprite invalide, ou le sprite de base n'est pas un sprite Spine), elle renverra -1. Il est intéressant de noter que le sprite d'instance peut avoir une couleur de mélange et un paramètre alpha (image_blend et image_alpha), tout comme le slot d'attachement (voir la fonction skeleton_slot_colour_set()), et donc la couleur et l'alpha finaux que le sprite d'attachement assigné pour le slot aura seront une combinaison de toutes ces valeurs. de toutes ces valeurs. Syntaxe : skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha) ; Arguments Description nom Le nom (sous forme de chaîne) de la pièce jointe à créer. sprite Le sprite_index pour obtenir l'image de la pièce jointe. ind L'index d'image pour obtenir l'image de la pièce jointe. xorigine L'origine x de l'image utilisée. yorigine L'origine y de l'image utilisée. xscale La mise à l'échelle horizontale de l'image, sous forme de multiplicateur : 1 = mise à l'échelle normale, 0.5 est la moitié etc... yscale La mise à l'échelle verticale de l'image, sous forme de multiplicateur : 1 = mise à l'échelle normale, 0.5 est la moitié etc... pourriture La rotation de l'image : 0 = normale, 90 = tournée de 90° dans le sens inverse des aiguilles d'une montre, etc. Couleur La valeur de la couleur à utiliser (une constante, un nombre entier ou une valeur hexadécimale). Alpha La valeur alpha à utiliser (de 0 à 1). Les retours : 1 en cas de succès, -1 en cas d'échec Exemple : skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1) ; skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1) ; skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5) ; skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow")) ; Le code ci-dessus vérifie le nom d'attachement actuellement attribué au slot nommé "slot_leftHand" et si une chaîne vide est retournée, un nouveau sprite est attaché.
