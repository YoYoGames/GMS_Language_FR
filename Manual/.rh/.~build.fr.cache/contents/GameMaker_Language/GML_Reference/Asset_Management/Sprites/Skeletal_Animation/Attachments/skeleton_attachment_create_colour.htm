skeleton_attachment_create_colour Lorsque vous créez votre sprite d'animation squelettique, vous pouvez attribuer des emplacements de pièce jointe et des pièces jointes pour y accéder. Ce sont simplement des images (sprites) séparées de l'animation, mais lorsqu'elles sont attachées, elles se déplacent avec l'os auquel elles sont attachées. Normalement, vous attribuez ces pièces jointes dans votre programme d'animation (Spine), mais avec cette fonction, vous pouvez créer les vôtres au moment de l'exécution en utilisant un élément de sprite de votre jeu. la fonction nécessite que vous donniez un nom à la pièce jointe (sous forme de chaîne), puis que vous définissiez le sprite_index et image_index à utiliser, ainsi que l'origine x et y (qui peut être différente de celle définie par le sprite dans les propriétés du sprite), et vous pouvez également définir les transformations que vous souhaitez appliquer à l'image lorsqu'elle est attachée, ainsi que la couleur à mélanger avec l'image et sa valeur alpha (transparence). Si la pièce jointe a été créée avec succès, la fonction retournera 1 et si ce n'était pas le cas (vous avez fourni un index de sprite invalide, ou le sprite de base n'est pas un sprite Spine) alors il retournera -1. Il est à noter que l'image-objet d'instance peut avoir une couleur de fusion et un paramètre alpha (image_blend et image_alpha ), tout comme la fente de fixation (voir la fonction skeleton_slot_colour_set() ), et donc la couleur finale et l'alpha que le sprite d'attachement assigné pour l'emplacement aura sera un composite de toutes ces valeurs. Syntaxe:skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha); Argument Description name Le nom (sous forme de chaîne) de la pièce jointe à créer. sprite Le sprite_index à partir duquel obtenir l'image de la pièce jointe. ind Image_index à partir de laquelle obtenir l'image de la pièce jointe. xorigin L'origine x de l'image utilisée. yorigin L'origine y de l'image utilisée. xscale La mise à l'échelle horizontale de l'image, sous forme de multiplicateur: 1 = mise à l'échelle normale, 0,5 est la moitié etc... yscale La mise à l'échelle verticale de l'image, sous forme de multiplicateur: 1 = mise à l'échelle normale, 0,5 est la moitié etc... rot La rotation de l'image: 0 = normale, 90 = tournée de 90 ° dans le sens antihoraire, etc. Colour La valeur de couleur à utiliser (valeur constante, entière ou hexadécimale). Alpha La valeur alpha à utiliser (de 0 à 1). Retour:1 if successful, -1 if not Exemple:skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1); skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1); skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5); skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow")); Le code ci-dessus vérifierait le nom de pièce jointe actuellement attribué pour l'emplacement nommé "slot_leftHand" et si une chaîne vide est renvoyée, un nouveau sprite est attaché.
