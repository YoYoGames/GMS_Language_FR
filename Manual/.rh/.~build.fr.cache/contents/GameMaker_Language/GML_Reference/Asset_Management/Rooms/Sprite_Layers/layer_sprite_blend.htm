couche_sprite_blend Cette fonction contrôle le mélange (ou "teinte") du sprite. La valeur par défaut est -1 (qui représente la constante c_white, qui peut également être utilisée). Toute autre valeur (y compris les constantes de couleur internes comme c_red, ou c_aqua) mélangera la couleur spécifiée avec le sprite original. Vous donnez l'ID de l'élément sprite (que vous obtenez lorsque vous créez un élément sprite à l'aide de la commande layer_sprite_create() ou lorsque vous utilisez la fonction layer_sprite_get_id()), puis définissez la couleur de mélange à utiliser. Vous pouvez voir ci-dessous un exemple de sprite qui a été mélangé avec différentes couleurs : Veuillez noter que vous devez essayer de limiter les mélanges sur la plateforme HTML5 (sauf si vous utilisez WebGL), car chaque sprite mélangé doit être mis en cache séparément et le fait de disposer de nombreux sprites mélangés peut nuire aux performances (vous pouvez également définir la taille du cache à l'aide de la fonction sprite_set_cache_size()). Syntaxe : layer_sprite_blend(sprite_element_id, blend) ; Arguments Description sprite_element_id La valeur d'identification unique de l'élément du sprite à modifier. mélange La couleur à mélanger avec le sprite (par défaut, c_white) Les retours : N/A Exemple : var lay_id = layer_get_id("Asset_sky") ; var spr_id = layer_sprite_get_id(lay_id, "Clouds") ; layer_sprite_blend(spr_id, c_gray) ; Le code ci-dessus récupère la valeur ID du sprite appelé "Clouds" attribué au calque "Asset_sky", puis le teinte d'une couleur.
