script_execute Cette fonction a été conçue à l'origine pour appeler les ressources de script de manière dynamique, mais avec l'introduction de fonctions définies par l'utilisateur, elle peut maintenant être utilisée de deux manières: La première consiste à prendre en charge les scripts hérités, où chaque script était une fonction unique et le titre du script était également le nom de la fonction. Utilisé de cette manière, il fonctionnera comme avant et exécutera un actif de script créé par l'utilisateur et y passera les arguments donnés. Notez qu'il n'appelle pas réellement le script dans ce cas, mais la fonction de script du même nom dans le script. La deuxième méthode consiste à appeler une fonction à partir d'un actif de script qui contient plusieurs fonctions ou définitions de variables. Dans ce cas, vous devez fournir le nom de la fonction et les arguments requis plutôt que le nom de l'actif de script. Notez que cela ne fonctionnera que pour les fonctions de script et non pour les méthodes, car seules les fonctions de script ont un ID de script qui leur est attribué. Bien que cette fonction soit conçue principalement pour le support hérité, elle n'est pas obsolète car elle a toujours sa place dans les situations de mise en réseau et d'appel de procédure à distance. Il convient également de noter que cette fonction est lente et que son utilisation fréquente peut nuire aux performances. REMARQUE: Cette fonction ne peut pas être utilisée avec les fonctions intégrées. Syntaxe:script_execute(scr, arg0, arg1, arg2..., etc...); Argument Description scr Le nom de la fonction / du script que vous souhaitez appeler arg0, arg1, arg2..., etc... Les différents arguments que vous souhaitez passer à la fonction / script Retour:Will depend on the return value from the script/function being called Exemple:script_execute(choose(move_up, move_down, move_left, move_right), irandom(5)); L'exemple de code ci-dessus utilisera script_execute pour choisir l'une des quatre fonctions définies par l'utilisateur (dans ce cas lié au mouvement) et lui passer un entier aléatoire comme premier argument.
