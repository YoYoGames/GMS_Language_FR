audio_sync_group_debug sa fonction peut être utilisée pour afficher des informations de débogage sur un groupe de synchronisation donné (l'index du groupe est la valeur renvoyée lorsque le groupe a été créé à l'aide de la fonction audio_create_sync_group() ). Vous pouvez appeler la fonction avec une valeur de -1 pour désactiver la superposition. Lorsque vous déboguez les groupes de synchronisation audio, la superposition affichera: Quel groupe de synchronisation est en cours de débogage Combien de canaux y a-t-il Combien de fois le groupe de synchronisation a échoué (cela signifie qu'il a essayé de faire trop de travail) Une liste des chaînes Le rouge signifie que cette chaîne ne fait actuellement aucun travail Le jaune signifie que le canal est dans son état initial (pas encore de décodage ou de lecture) Le vert signifie que la piste est en train de rechercher / décoder Le violet signifie que le canal n'a plus de données Une barre de profilage qui montre le travail effectué par le thread de décodage bleu signifie qu'un fichier est en cours de lecture pour la première fois rouge signifie que le thread lit un fichier pour savoir où lire vert signifie que le thread décode les données violet signifie que le fil dort jusqu'à ce qu'il ait besoin de fonctionner Notez que pour le débogage général de l'audio, vous disposez également de la fonction audio_debug(). REMARQUE: cette fonctionnalité n'est pas disponible pour la plate-forme cible HTML5. Syntaxe:audio_sync_group_debug(group_index); Argument Description group_index L'index de groupe à déboguer ou -1 pour désactiver le débogage. Retour:N/A Exemple:if (debug_mode)    {    audio_sync_group_debug(sg);    } else    {    audio_sync_group_debug(-1);    } Le code ci-dessus activera ou désactivera la superposition de débogage du groupe de synchronisation audio montrant le groupe de synchronisation indexé dans la variable «sg» selon que le jeu est exécuté en mode débogage ou non.
