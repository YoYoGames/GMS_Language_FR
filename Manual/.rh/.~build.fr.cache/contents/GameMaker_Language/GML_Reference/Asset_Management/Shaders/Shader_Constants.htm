Constantes de l'ombrage Outre les fonctions et les constantes de shader définies dans les pages de référence du langage d'ombrage OpenGL ES (GLSL ES), vous disposez également d'un certain nombre de constantes de shader propres à GameMaker Studio 2. de constantes d'ombrage disponibles qui sont propres à GameMaker Studio 2. Les constantes de matrice d'affichage suivantes peuvent être utilisées comme indices de tableau lors de l'utilisation de la constante de tableau du shader gm_Matrices: Constant Description MATRIX_VIEW Cette constante d'index de tableau contient l'index de la matrice de vue actuelle. L'index renvoyé sera utilisé comme une valeur de tableau lors de l'appel de la constante gm_Matrices dans le code du shader. MATRICE_PROJECTION Cette constante d'index de tableau contient l'index de la matrice de projection actuelle. L'index renvoyé sera utilisé comme une valeur de tableau lors de l'appel de la constante gm_Matrices dans le code du shader. MATRIX_WORLD Cette constante d'index de tableau contient l'index de la matrice du monde actuel. Cela peut être utilisé pour des choses comme l'éclairage si vous avez des informations sur la lumière dans l'espace monde. L'index renvoyé sera utilisé comme une valeur de tableau lors de l'appel de la constante gm_Matrices dans le code du shader. MATRICE_VUE DU MONDE Cette constante d'index de tableau contient l'index du résultat des matrices de monde et de vue multipliées ensemble. Elle est souvent utilisée pour des choses comme le brouillard. L'index renvoyé sera utilisé comme une valeur de tableau lors de l'appel de la constante gm_Matrices dans le code du shader. PROJECTION_MATRICE_DE_VUE_DU_MONDE Cette constante d'index de tableau contient l'index du résultat des matrices de monde, de vue et de projection multipliées ensemble. Il s'agit de la matrice de transformation normale utilisée pour les positions des sommets. L'index renvoyé est utilisé comme une valeur de tableau lors de l'appel de la constante gm_Matrices dans le code du shader. l'appel de la constante gm_Matrices dans le code du shader. MATRIX_MAX Il ne s'agit pas d'un indice de tableau, mais plutôt de la taille du tableau de la matrice dans le vertex shader. MAX_VS_LIGHTS Il ne s'agit pas d'un index de tableau, mais plutôt du nombre de lumières dans le vertex shader. La constante suivante est également disponible pour l'éclairage : Constant Description MAX_VS_LIGHTS Le nombre maximum de lumières ponctuelles et directionnelles disponibles dans le shader. Les uniformes et constantes de matrice prédéfinis suivants peuvent être utilisés dans votre shader pour accéder aux valeurs spécifiques de GameMaker Studio 2: Constant Description gm_Matrices [matrice] Cette constante de tableau renvoie une matrice de transformation et est l'un des uniformes prédéfinis disponibles que GameMaker Studio 2 crée pour que vous puissiez l'utiliser dans l'éditeur de code de shaders. L'index du tableau est choisi parmi l'une des constantes énumérées ci-dessus. gm_BaseTexture Il s'agit d'une constante d'échantillonneur 2D qui renvoie la texture de l'objet actuel, telle que définie par GameMaker Studio 2. Il s'agit donc du sprite, de la surface ou de la texture actuelle qui serait normalement utilisée lors du dessin sans que le shader soit appelé. gm_LightingEnabled Cette option peut être utilisée pour obtenir ou définir l'éclairage de GameMaker Studio 2 lors de l'utilisation de la 3D. gm_FogStart Ceci peut être utilisé pour obtenir la distance à laquelle les polygones commencent à être mélangés avec la couleur du brouillard. gm_RcpFogRange Cela peut être utilisé pour obtenir la distance à laquelle le brouillard est maximal et où l'on ne peut plus rien voir. gm_PS_FogEnabled Ceci retournera vrai ou faux si le brouillard de pixels est activé ou non sur le GPU. gm_FogColour Ceci peut être utilisé pour obtenir la couleur du brouillard utilisé par GameMaker Studio 2. gm_VS_FogEnabled Ceci retournera vrai ou faux si le brouillard de vertex est activé ou non sur le GPU. gm_AlphaTestEnabled Ceci peut être utilisé pour obtenir le test alpha dans le shader. Voir gpu_set_alphatestenable() pour plus d'informations sur le test d'alpha. gm_AlphaRefValue Cette fonction peut être utilisée pour obtenir la valeur de référence actuelle du test alpha.
