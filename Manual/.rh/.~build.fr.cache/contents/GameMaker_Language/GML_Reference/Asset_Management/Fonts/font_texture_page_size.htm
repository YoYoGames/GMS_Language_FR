font_texture_page_size Cette variable intégrée peut être utilisée pour obtenir ou définir la taille de la page de texture lors de l'utilisation de la fonction font_add(). Lors de l'ajout d'une police à l'aide de cette fonction, GameMaker Studio 2 créera un cache de page de texture des glyphes requis jusqu'à la taille définie par cette variable. Le processus est le suivant: Chaque caractère que vous utilisez est mis en cache de la police dans une page de texture Lorsque la page de texture est pleine (car vous utilisez de gros caractères ou beaucoup de personnages différents), GameMaker Studio 2 supprimera les caractères du cache et les remplacera par les nouveaux dont vous avez besoin GameMaker Studio 2 essaie de ne pas rendre la texture trop grande car tous les appareils ne peuvent pas gérer de grandes textures, et il essaie de ne pas rendre la texture trop petite car la suppression et la réajout de caractères à la texture prennent du temps CPU et font que votre jeu s'exécute. tout doucement Si vous pensez avoir besoin d'une taille de page de texture plus grande ou plus petite que la taille par défaut de GameMaker Studio 2 par GameMaker Studio 2, vous pouvez définir la valeur à l'aide de cette variable intégrée. Si vous n'utilisez pas le font_add() la fonction, alors la définition de cela n'aura aucun effet sur les performances de vos jeux car les polices ajoutées dans l'EDI sont mises en cache sur les pages de texture régulières (que vous pouvez contrôler à partir de l' éditeur de polices et des options de jeu pour la plate-forme cible). Syntaxe:font_texture_page_size; Retour:Real (maximum width/height in pixels) Exemple:font_texture_page_size = 512; newfont = font_add("Arial", 24, true, true, 32, 128); Le code ci-dessus définit la page de texture du cache de polices sur une taille maximale de 512 pixels, puis ajoute une nouvelle police au jeu.
