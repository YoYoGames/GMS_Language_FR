font_set_cache_size Lors de la fusion et de la coloration d'images, HTML5 ne peut pas le faire de manière dynamique de la même manière qu'un exécutable pourrait être exécuté. Par conséquent, GameMaker Studio 2 doit temporairement enregistrer une copie mélangée des images et les charger si nécessaire. Cette fonction définit le nombre de copies mélangées de la police donnée pouvant être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. REMARQUE: pour les polices de sprite, vous devez utiliser la fonction équivalente pour les sprites, sprite_set_cache_size(). Syntaxe:font_set_cache_size(ind, max); Argument Description ind L'index de la police pour modifier la taille du cache. max Le nombre maximal de copies mises en cache de la police qui peuvent être stockées. Retour:N/A Exemple:font_set_cache_size(fnt_MainMenu, 2); Cela définira le cache de police de la police indexée dans la variable "fnt_MainMenu" à 2 copies.
