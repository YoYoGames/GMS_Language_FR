font_set_cache_size Lorsqu'il s'agit de mélanger et de colorer des images, HTML5 ne peut pas le faire dynamiquement de la même manière qu'un exécutable. C'est pourquoi GameMaker Studio 2 doit enregistrer temporairement une copie mélangée des images et la charger en cas de besoin. et les charger en cas de besoin. Cette fonction définit le nombre de copies mélangées de la police donnée qui peuvent être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. NOTE : pour les polices sprites, vous devez utiliser la fonction équivalente pour les sprites, sprite_set_cache_size(). Syntaxe : font_set_cache_size(ind, max) ; Arguments Description ind L'index de la police dont la taille du cache doit être modifiée. max Le nombre maximum de copies en cache de la police qui peuvent être stockées. Les retours : N/A Exemple : font_set_cache_size(fnt_MainMenu, 2) ; Cela va définir le cache de la police indexée dans la variable "fnt_MainMenu" à 2 copies.
