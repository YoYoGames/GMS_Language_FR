Génération d'événements d'objets GameMaker Studio 2 est entièrement piloté par les événements et toutes les actions se produisent à la suite de certains événements déclenchés dans un ordre spécifique ou par une occurrence spécifique dans le jeu. Il y a un certain nombre d'événements différents, chacun avec une tâche ou un timing spécifique et certains d'entre eux sont même divisés en événements "sous" séparés. Normalement, ces événements sont exécutés par GameMaker Studio 2 lorsqu'il détecte un déclencheur dans le jeu, ou chaque tick de jeu (comme c'est le cas avec les événements d'étape et de tirage), mais vous pouvez également appliquer un événement à une instance de la salle actuelle à partir de dans un morceau de code. Les fonctions suivantes permettent de générer des événements: event_inheritedevent_performevent_perform_objectevent_user Il existe également un certain nombre de variables en lecture seule qui peuvent être utilisées avec les fonctions répertoriées ci-dessus et qui sont particulièrement utiles pour le débogage: event_actionevent_numberevent_objectevent_type Pour plus d'informations sur les événements, veuillez consulter la section Evénements d'objets.
