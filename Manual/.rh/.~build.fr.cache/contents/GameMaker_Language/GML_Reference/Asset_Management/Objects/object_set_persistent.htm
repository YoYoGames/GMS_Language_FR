object_set_persistent Avec cette fonction, vous pouvez définir la persistance d'un objet spécifique. Un objet persistant est "transporté" d'une pièce à l'autre et conserve ses propriétés à tout moment (pour plus d'informations sur la persistance, voir la section sur l' éditeur d'objets ). La définition de la persistance de l'objet signifie que toutes les instances de cet objet créées après la modification de la persistance seront créées de manière persistante, tandis que les instances qui se trouvent déjà dans la salle peuvent ne pas être affectées. Lorsque vous créez une instance avec la persistance activée dans une pièce, un calque ou une profondeur lui sera attribué (en fonction de la fonction que vous avez utilisée pour créer l'instance de l'objet). Lorsque la pièce est modifiée et si la pièce suivante n'a pas de calque portant le même nom ou la même profondeur que celui attribué, un nouveau calque est créé pour l'instance qui persiste dans les pièces. Si vous avez donné un calque nommé lors de la création de l'instance, le nouveau calque sera également nommé de la même manière que le calque d'origine, tandis que si vous attribuez une profondeur à l'instance, le nouveau calque sera nommé "_layer_XXX ", où" XXX "est une valeur hexadécimale utilisée pour donner au calque un nom unique. Ces calques seront supprimés de la pièce à l'extrémité de la pièce. Une autre chose à noter concernant les calques et les instances persistantes est que si vous avez attribué à l'instance un nom lors de la création, et il existe un autre calque dans les pièces suivantes portant le même nom, l'instance persistante sera affectée au calque portant le même nom, quelle que soit la profondeur du calque. Enfin, si une occurrence persistante se déplace vers une pièce avec un calque à la même profondeur que celle sur laquelle l'instance a été créée, il ne sera pas affecté à ce calque, mais un nouveau calque sera créé à la même profondeur (selon la convention de dénomination expliquée précédemment). Veuillez noter qu'il ne s'agit pas d'une fonction d'instance! Vous pouvez définir l'indicateur persistant d'instances individuelles à l'aide du persistent variable et donc dix instances, toutes marquées comme persistantes même si l'objet en question a l'indicateur défini sur false dans l'EDI, et même si vous changez l'indicateur persistant de l'objet sur true à l'aide de cette fonction, toutes les instances qui se trouvent actuellement dans la salle resteront telles qu'elles étaient, et seules les instances créées après l'appel de la fonction commenceront par persistent marquées comme true. Syntaxe:object_set_persistent(index, pers); Argument Description index L'index de l'objet à modifier. pers Si l'objet doit être marqué comme persistant (true ) ou non (false ). Retour:N/A Exemple:if (room == rm_final)     {     object_set_persistent(obj_Player, false);     } instance_create_layer(32, 32, "Instances", obj_Player); Le code ci-dessus vérifie pour voir dans quelle pièce se trouve actuellement l'instance et si elle se trouve dans le "rm_final" spécifié, il définit la persistance de l'objet "obj_Player" sur false puis crée l'instance.
