Objets Dans GameMaker Studio 2 vous avez des objets et des instances. Les objets sont essentiellement le modèle de base d'une instance, et en tant que tels ne sont jamais présents directement dans une pièce... seules les instances de l'objet sont placées dans la pièce. Cela signifie que si vous souhaitez modifier quelque chose pour toutes les instances que vous allez créer, vous pouvez le faire en modifiant l' objet. IMPORTANT! Changer quoi que ce soit sur un objet ne changera aucune instance actuellement présente dans la salle, seulement celles qui sont créées après la modification. Vous pouvez également accéder aux informations sur un objet de base et les utiliser pour dicter des comportements ou des changements dans le jeu. GameMaker Studio 2 vous offre un certain nombre de fonctions qui vous permettent d'obtenir divers détails sur un objet. Notez qu'un objet n'est pas une instance (une entité dans le jeu), c'est purement une ressource à partir de laquelle toutes les instances sont créées. Cela signifie que les valeurs de retour de ces fonctions peuvent être différentes des valeurs réelles affichées par les instances de la salle, car une fois qu'une instance est créée, elle peut être modifiée via le code et les DnD™. object_indexobject_existsobject_get_nameobject_get_maskobject_get_parentobject_get_persistentobject_get_solidobject_get_spriteobject_get_visibleobject_get_physicsobject_is_ancestor Il existe également un certain nombre de fonctions qui vous permettent de définir les propriétés d'un objet. Il convient de noter que toutes les instances de cet objet qui existent déjà dans la salle peuvent ne pas être affectées par ces fonctions, mais que toutes les nouvelles instances de cet objet créées dans la salle le seront, il est donc recommandé de ne jamais modifier les propriétés d'un objet lorsque des instances de cet objet sont présentes dans la salle actuelle. object_set_maskobject_set_persistentobject_set_solidobject_set_spriteobject_set_visible Vous pouvez également utiliser certaines fonctions pour générer des événements à partir de code à partir d'une instance ou d'un objet spécifique.
