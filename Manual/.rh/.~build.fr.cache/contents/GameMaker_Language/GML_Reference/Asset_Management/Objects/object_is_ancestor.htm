object_is_ancestor Cette fonction peut être utilisée pour vérifier un objet et voir s'il est un ancêtre ou non d'un autre objet. Ainsi, les deux arguments doivent être des indices d'objets et non des ids d'instance, et le premier est toujours l'objet que vous voulez savoir s'il est l'enfant du second argument, qui est toujours l'objet que vous voulez vérifier comme l'ancêtre (parent). Syntaxe : object_is_ancestor(obj, par) ; Arguments Description obj. L'objet qui est contrôlé en tant qu'enfant. par L'objet qui est contrôlé en tant qu'ancêtre (parent). Les retours : Booléen Exemple : if object_is_ancestor(object_index, obj_Enemy) { instance_destroy() ; } Le code ci-dessus vérifie si l'instance qui exécute le code est un enfant de l'objet "obj_Enemy", et si c'est le cas, elle est détruite.
