exception_unhandled_handler Cette runtime peut être utilisée pour «reprendre» le message d'erreur GameMaker Studio 2 runtime se produit. Vous fournissez une méthode ou une fonction qui est appelée lorsque l'exception se produit, et son premier argument sera une structure contenant des informations sur l'exception. Cette structure a le format suivant:{ message : "",               // a string that is a short message for this exception longMessage : "",           // a string that is a longer message for this exception script : "",                // a string that describes where the exception came from line: 0,                    // a real that is the line number where the exception came from stacktrace : [ "", "" ],    // an array of strings that is the stack frame the exception was generated } Vous pouvez également fournir undefined comme argument, ce qui supprimera le gestionnaire d'exceptions personnalisé (s'il est défini) et reviendra au message d'erreur par défaut GameMaker Studio 2 Cette fonction renverra la méthode du gestionnaire d'exceptions précédemment définie, cependant si elle n'a jamais été modifiée ou si elle a été précédemment réinitialisée à l'aide de undefined, alors il retournera simplement undefined. Passer une méthode à cette fonction signifie qu'aucun message d'erreur ne sera affiché à l'utilisateur et vous serez autorisé à gérer l'exception vous-même. Cependant, vous ne pouvez pas l'utiliser pour "sortir" de l'exception pour continuer votre partie, car le jeu se fermera toujours après l'appel du gestionnaire d'exceptions. Notez que le contexte d'exécution de la méthode n'est pas dans un événement et qu'il est donc hautement improbable que quoi que ce soit puisse être rendu à ce moment à l'écran. En général, la chose la plus sûre qui puisse être faite à ce stade est de sauvegarder un fichier avec les informations de contexte d'exception afin qu'il puisse être - par exemple - analysé et traité lors de la prochaine exécution du jeu, puis traité. Notez que si vous utilisez l' return dans la méthode fournie, la valeur donnée sera convertie en entier et utilisée comme code de sortie du runner. Syntaxe:exception_unhandled_handler(user_handler); Argument La description user_handler La méthode (fonction définie par l'utilisateur) à utiliser comme gestionnaire d'exceptions, ou undefined pour le réinitialiser à la valeur par défaut. Retour:Method (or undefined) Exemple:exception_unhandled_handler(function(ex))     {     show_debug_message( "--------------------------------------------------------------");     show_debug_message( "Unhandled exception " + string(ex) );     show_debug_message( "--------------------------------------------------------------");     if file_exists("crash.txt") file_delete("crash.txt");     var _f = file_text_open_write("crash.txt");     file_text_write_string(_f, string(ex));     file_text_close(_f);     return 0;     } ) Le code ci-dessus créera une méthode de gestionnaire d'exceptions et l'utilisera pour enregistrer un fichier avec les données d'exception.
