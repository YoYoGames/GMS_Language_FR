Débogage Lors de la création d'un jeu, il est souvent très important de connaître certains détails de la mécanique du jeu à des moments précis, ou de pouvoir saisir certaines valeurs clés pour certaines fonctions afin de tester différentes méthodes et idées pour votre jeu. À cette fin, GameMaker GameMaker Studio 2 vous fournit un certain nombre de fonctions de débogage qui afficheront des messages et demanderont des entrées en mode de débogage (ou sur une cible Windows ), mais qui seront ignorées lors de la compilation pour d'autres modules cibles, étant donné que ces fonctions placent le jeu dans une boucle fermée jusqu'à ce que l'entrée soit résolue. NOTE : Bien qu'ils puissent fonctionner sur certaines plateformes, ils ne doivent pas être utilisés pour autre chose que le débogage. debug_mode débogage_événement debug_get_callstack Gestionnaire d'exceptions non gérées get_integer get_string montrer_erreur montrer_message show_question montrer_debug_message show_debug_overlay code_est_compilé fps fps_real Outre ces fonctions spécialisées, qui vous aident à créer votre jeu, GameMaker Studio 2 dispose également d'une longue liste de messages d'erreur qui peuvent être affichés lors de l'écriture du code du jeu, de la compilation du jeu ou de l'exécution du jeu lui-même. Ces messages indiquent l'erreur spécifique en question et peuvent être utilisés pour localiser un problème dans un objet ou une fonction de script spécifique, ce qui facilite grandement le processus de débogage de votre jeu. Cependant, la signification exacte du message d'erreur n'est pas toujours évidente.
