Débogage Lors de la création d'un jeu, il est souvent très important que vous connaissiez certains détails de la mécanique du jeu à des moments spécifiques, ou que vous puissiez saisir certaines valeurs clés pour certaines fonctions afin de tester différentes méthodes et idées pour votre jeu. À cette fin, GameMaker Studio 2 vous fournit un certain nombre de fonctions de débogage qui afficheront des messages et demanderont des entrées en mode débogage (ou sur une Windows ) mais qui seront ignorées lors de la compilation pour d'autres modules cibles en raison du fait que ces les fonctions mettent le jeu en boucle fermée jusqu'à ce que l'entrée soit résolue. REMARQUE: bien qu'ils puissent fonctionner sur certaines plates-formes, ils ne doivent pas être utilisés pour autre chose que le débogage. debug_modedebug_eventdebug_get_callstackexception_unhandled_handlerget_integerget_stringshow_errorshow_messageshow_questionshow_debug_messageshow_debug_overlaycode_is_compiledfpsfps_real Outre ces fonctions spécialisées, pour vous aider lors de la création de votre jeu, GameMaker Studio 2 dispose également d'une longue liste de messages d'erreur qui peuvent être affichés soit lors de l'écriture de votre code de jeu, de la compilation de votre jeu, soit lorsque le jeu lui-même est en cours d'exécution. Ces messages pointent vers l'erreur spécifique en question et peuvent être utilisés pour identifier un problème dans un objet ou une fonction de script spécifique, ce qui facilite grandement le processus de débogage de votre jeu. Cependant, il n'est pas toujours évident de savoir exactement ce que signifie le message d'erreur.Par conséquent, les sections suivantes répertorient tous les messages possibles et expliquent ce qu'ils signifient pour vous faciliter la vie: Erreurs de syntaxe Erreurs du compilateur Erreurs de coureur Vous pouvez également trouver plus d'informations sur la façon de déboguer votre jeu dans la section sur le débogueur.
