debug_get_callstack Cette fonction génère un tableau de chaînes de caractères en tant que "callstack" où le script actuel est listé en premier, puis tous les autres scripts qui ont été exécutés pour que le script actuel soit exécuté. Le format exact de la chaîne varie en fonction de la de la plate-forme cible choisie, mais il comprendra généralement le nom du script/événement, puis deux points : et le numéro de ligne, comme ceci : "gml_Script_script2:1" "gml_Script_script1:32" "gml_Script_script0:64" "gml_Object_object0_Create_0:1" La fonction permet de passer un argument facultatif, qui est la profondeur maximale de la pile d'appels retournée. Cette valeur est le nombre de scripts qui sont inclus dans le tableau retourné en commençant par le script actuel. Si cet argument n'est pas spécifié, alors la pile d'appels complète sera retournée. Notez que le tableau retourné contiendra toujours 0 dans sa dernière position, après avoir listé les scripts de la pile d'appels. Syntaxe : debug_get_callstack([maxdepth]) Arguments Description [maxdepth] La profondeur maximale de la pile d'appels Les retours : Array Exemple : if debug_mode { if keyboard_check(vk_escape) { var _a = debug_get_callstack(4) ; for (var i = 0 ; i < array_length_id(_a) ; ++i ;) { show_debug_message(_a[i]) ; } } } Le code ci-dessus vérifie si le mode débogage est activé et si c'est le cas, il vérifie si une touche est maintenue enfoncée. Dans ce cas, il affiche la pile d'appels actuelle sur la console, avec une profondeur maximale de 4.
