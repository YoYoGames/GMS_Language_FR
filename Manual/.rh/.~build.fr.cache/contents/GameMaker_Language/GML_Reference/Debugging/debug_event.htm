débogage_événement Cette fonction génère un événement de débogage personnalisé qui sera affiché dans la vue graphique du débogueur lorsqu'un jeu est exécuté en mode débogage. Si vous souhaitez que les messages s'affichent en dehors du mode de débogage vous devriez utiliser show_debug_message(). Il convient de noter que la fonction prend également deux chaînes réservées pour faciliter le débogage à l'aide d'applications externes comme Visual Studio. Ces chaînes chaînes sont : "OutputDebugOn" - Ceci active un appel à OutputDebugString pour la cible. Windows ce qui signifie que toute la sortie de débogage - tout ce que vous voyez dans la fenêtre Output - peut être visualisée par Visual Studio ou par des applications tierces. "BreakOnError" - Cette option est réservée aux versions YYC de Windows et signifie que les projets n'afficheront pas l'écran d'erreur de code habituel si le runtime détecte une erreur, mais qu'ils continueront et se planteront. Cela vous permet de voir la trace de la pile dans Visual Studio en cas de débogage. Syntaxe : debug_event(string) Arguments Description chaîne de caractères La chaîne d'événement de débogage personnalisée à utiliser. Les retours : N/A Exemple : if !surface_exists(global.EffectsSurface) { debug_event("Recreating Effects Surface") ; global.EffectsSurface = surface_create(room_width, room_height) ; } Le code ci-dessus vérifie si une surface existe et si ce n'est pas le cas, un événement de débogage est déclenché dans la vue graphique du débogueur (le jeu doit avoir été exécuté en mode débogage pour que cela soit visible) et la surface est recréée.
