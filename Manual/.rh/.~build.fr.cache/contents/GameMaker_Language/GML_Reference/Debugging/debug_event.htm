debug_event Cette fonction génère un événement de débogage personnalisé qui sera affiché dans la vue graphique du débogueur lorsqu'un jeu est exécuté en mode débogage. Si vous souhaitez que les messages s'affichent lorsque vous n'êtes pas en mode débogage, vous devez utiliser show_debug_message(). Il est à noter que la fonction prendra également deux chaînes réservées pour aider à effectuer le débogage à l'aide d'applications externes telles que Visual Studio. Ces chaînes sont: "OutputDebugOn "- Cela permet un appel à OutputDebugString pour la Windows, ce qui signifie que toute la sortie de débogage - tout ce que vous voyez dans la fenêtre Sortie - peut être affichée par Visual Studio ou par des applications tierces. "BreakOnError "- Cette option concerne uniquement les Windows YYC et signifie que les projets n’afficheront pas l’écran d’erreur de code habituel si le runtime détecte une erreur, mais qu’ils continuent et se bloquent. Cela vous permet de voir le stacktrace dans Visual Studio cas de débogage. Syntaxe:debug_event(string) Argument Description string La chaîne d'événement de débogage personnalisée à utiliser. Retour:N/A Exemple:if !surface_exists(global.EffectsSurface)    {    debug_event("Recreating Effects Surface");    global.EffectsSurface = surface_create(room_width, room_height);    } Le code ci-dessus vérifie si une surface existe et si ce n'est pas le cas, un événement de débogage est déclenché dans la vue graphique du débogueur (le jeu doit avoir été exécuté en mode débogage pour que cela soit visible) et la surface est recréée.
