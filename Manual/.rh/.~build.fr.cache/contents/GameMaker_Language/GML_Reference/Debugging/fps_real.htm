fps_real Dans GameMaker Studio 2, deux méthodes principales peuvent être utilisées pour déterminer la vitesse à laquelle votre jeu s'exécute. Le game speed (comme spécifié dans les options de jeu) et le fps (images par seconde). Ces valeurs sont souvent confuses, mais en gros, l'une est le nombre d'étapes de jeu que GameMaker Studio 2 est censé terminer en une seconde (vitesse de jeu), tandis que l'autre est le nombre d'étapes du processeur que GameMaker Studio 2 est en train de terminer dans un seconde (le vrai fps), et cette valeur est généralement beaucoup plus élevée que la vitesse du jeu, mais diminuera à mesure que votre jeu devient plus complexe et utilise plus de puissance de traitement pour maintenir la vitesse de la pièce définie. Cette variable en lecture seule renvoie le fps actuel sous forme de valeur entière. Veuillez noter que la fonction ne sera mise à jour qu'une seule fois à chaque étape de votre jeu et peut donc sembler "sauter" d'une valeur à une autre, mais c'est tout à fait normal. REMARQUE: sur la cible HTML5, cette variable renverra à la place un multiple de la fréquence de rafraîchissement du moniteur car GameMaker Studio 2 doit s'appuyer sur le navigateur pour la synchronisation et l'envoi. Syntaxe:fps_real Retour:Real Exemple:if debug_mode    {    draw_text(32, 32, "FPS = " + string(fps_real));    } Le code ci-dessus vérifiera si le jeu est en mode débogage et si tel est le cas, il affichera les fps réels actuels à l'écran.
