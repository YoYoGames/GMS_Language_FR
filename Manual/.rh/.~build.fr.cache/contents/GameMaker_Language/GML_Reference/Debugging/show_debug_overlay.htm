show_debug_overlay Cette fonction peut être utilisée pour activer et désactiver la superposition de débogage standard lors du test de votre jeu et est désactivée par défaut. La superposition de débogage montre une barre graphique d'utilisation CPU / GPU dans la fenêtre de jeu réelle elle-même avec la valeur fps réelle actuelle, le nombre de permutations de texture et le nombre de lots de sommets (notez que les échanges de texture et les lots de sommets ne seront jamais zéro et seront normalement afficher des valeurs de 2 ou 3, car même avec une salle vide et sans objet, GameMaker Studio 2 doit toujours dessiner et regrouper des objets). Cette barre est divisée en sections, chaque section étant au 1 / 60e de seconde. Comme vous pouvez le voir sur l'image ci-dessus, la barre est composée de différentes couleurs: Vert - Traitement d'entrée / sortie (c'est-à-dire: clavier, souris, manette de jeu, réseau, etc.) Rouge - La vitesse de mise à jour de l'événement d'étape Jaune - Le temps requis pour l'événement de tirage au sort Orange - Heure de mise à jour du débogage, qui n'est normalement visible que lorsque vous utilisez le module de débogage Blanc - GPU restant dans le temps, c'est-à-dire le temps passé à attendre que le GPU termine le rendu de l'image avant que la suivante ne puisse commencer Cyan - Le temps de rendu du texte Gris - Le temps nécessaire pour effacer l'écran à chaque étape de tirage Bleu - Le temps requis pour que le Garbage Collector s'exécute Rouge foncé - Le rinçage du GPU, qui correspond au temps nécessaire au GPU pour effacer les images de la mémoire En utilisant cette fonction, vous pouvez ajouter la superposition que ce soit en mode débogage ou non, et de cette façon, vous pouvez voir avec quelle efficacité votre jeu fonctionne et obtenir un repère visuel sur la façon dont il utilise les ressources disponibles, sans le surcroît d'avoir le débogueur en cours d'exécution à côté. REMARQUE: cette fonction ne fonctionnera pas sur la cible HTML5 en raison de la différence de son fonctionnement par rapport à toutes les autres cibles. Syntaxe:show_debug_overlay(enable); Argument Description enable allumer (true ) ou désactivé (false ) the debug overlay. Retour:N/A Exemple:if global.debug    {    show_debug_overlay(true);    } else    {    show_debug_overlay(false);    } Le code ci-dessus activera ou désactivera le débogage en fonction de la valeur d'une variable globale.
