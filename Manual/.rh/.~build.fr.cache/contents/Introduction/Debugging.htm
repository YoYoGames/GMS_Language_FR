Débogage Lors de la programmation de votre jeu, que ce soit en utilisant le GameMaker Language ( GML ) ou Drag and Drop™ ( DnD™), il peut être très facile de faire des erreurs - utiliser les mauvaises variables, passer les mauvais arguments ou utiliser les mauvaises fonctions ne sont que quelques exemples des erreurs les plus courantes que nous faisons tous - et ces erreurs ne sont pas toujours détectées par le vérificateur de syntaxe inclus avec l'éditeur de script / action. Cela signifie que ces erreurs ne deviendront apparentes que lorsque vous exécuterez ou compilerez votre jeu, et même dans ce cas, il peut parfois être difficile de savoir exactement ce qui ne va pas. Il peut donc être d'une importance vitale de savoir comment votre jeu fonctionne sur l'appareil choisi, ainsi que de voir quelles instances se trouvent dans la salle, quelles variables globales ont été créées et une foule d'autres détails. Vérifier ces éléments s'appelle « débogage de votre code », et toutes les erreurs ou problèmes que vous pouvez trouver sont appelés « bogues ». Maintenant, vous pouvez vérifier beaucoup de ces choses vous-même en ajoutant un soi-disant «code de débogage» à vos jeux pour afficher à l'écran les informations dont vous avez besoin, ou vous pouvez faire en sorte qu'il affiche des messages en cas d'erreur, mais c'est souvent beaucoup de travail et si vous ne savez pas où se trouve le problème en premier lieu, cela peut prendre du temps pour traquer votre bogue. Pour cela, GameMaker Studio 2 dispose d'un mode de test spécial appelé Mode de débogage qui lancera le module de débogage, et il existe également un système de rapport d'erreurs complet qui affiche des messages lors de la compilation pour vous aider à détecter tout problème avec votre jeu. Les sections ci-dessous couvrent les différentes manières de déboguer votre jeu: Rapport d'erreurs Lorsqu'une erreur se produit pendant la compilation et l'exécution d'un projet, elle est signalée par un message dans la fenêtre de sortie des erreurs de compilation, un peu comme celle illustrée ci-dessous: La plupart des messages d'erreur du compilateur suivent le même schéma, avec un message indiquant l'objet dans lequel l'erreur s'est produite, puis l'événement avec le numéro de ligne, et enfin une brève description de l'erreur elle-même. Ce type d'informations fournies par le rapport d'erreurs est très important car il détecte et signale les bogues liés à la syntaxe et au langage qui vont faire planter votre jeu fini, mais il ne rapportera pas de bogues plus subtils, ni ne donnera de commentaires. sur les performances. Pour ces types de choses, vous devez utiliser le mode de débogage (expliqué ci-dessous). En dehors de ces erreurs de compilation, vous pouvez également obtenir des erreurs de syntaxe. Celles-ci seront détectées lorsque vous ajoutez vos DnD™ et sont affichées dans la fenêtre de sortie des erreurs de syntaxe, similaire à l'image ci-dessous: Comme pour les erreurs du compilateur, la fenêtre des erreurs de syntaxe vous dirigera vers l'objet, l'événement et les lignes spécifiques qui posent le problème, et sera souvent affichée avec une brève explication de l'erreur. Mode débogage Si vous avez besoin de vérifier les choses plus attentivement, vous pouvez exécuter le jeu en mode débogage en cliquant sur le bouton d'exécution de débogage. Lorsque vous le faites, le module de débogage s'ouvrira avec votre jeu. Ce module vous permet de surveiller comment tout fonctionne dans votre jeu, des performances globales aux valeurs des variables individuelles dans des instances individuelles. Vous pouvez trouver plus d'informations sur le module de débogage et ses possibilités et comment l'utiliser à partir de la page suivante: Le débogueur Notez que lorsque vous utilisez le module de débogage, vous pouvez également ajouter des points d'arrêt à votre code ou DnD™. Un point d'arrêt est un point dans la boucle de jeu où vous voulez que le module de débogage se mette en pause et vous permette de parcourir le code de jeu réel ligne par ligne. Vous pouvez ajouter des points d'arrêt à l'aide de la touche "F9", et ils seront affichés dans la fenêtre de sortie où vous pouvez les activer / désactiver selon les besoins. Outils supplémentaires En tant qu'outil supplémentaire pour le débogage des jeux compilés, certains paramètres de ligne de commande peuvent être utilisés lorsque vous exécutez le package de jeu final sous Windows.
