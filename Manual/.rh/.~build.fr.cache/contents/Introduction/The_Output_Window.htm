Résultats de la recherche Vous pouvez ouvrir la fenêtre Rechercher et remplacer en utilisant le raccourci clavier / + + " F ", ou aller dans le menu Edition. Une fois que vous avez saisi vos termes de recherche, les résultats s'affichent dans cette fenêtre au format : objet] - [événement] - [numéro de ligne] : [chaîne de recherche]. Si le terme recherché se trouve dans un script, il sera tout simplement : [script] - [Numéro de ligne] : [chaîne de recherche] Vous pouvez ensuite double-cliquer sur sur l'une des entrées retournées pour ouvrir l'actif donné à la position correcte pour l'édition. Points d'arrêt Les points d'arrêt sont des endroits de votre code de jeu ou de DnD™où vous avez demandé à GameMaker Studio 2 de mettre en pause l'exécution du projet pendant qu'il est en mode débogage. Vous pouvez ajouter un point d'arrêt n'importe où dans la boucle de jeu en utilisant la touche " F9 ", et lorsque vous le faites, il apparaîtra dans cet onglet de sortie. Vous pouvez les activer et les désactiver (sans les supprimer) en cliquant sur la case à cocher à gauche et si vous utilisez le bouton droit de la souris sur l'un d'eux, vous obtenez un petit menu qui vous permet de supprimer le point d'arrêt ou d'ouvrir la fenêtre de code/DnD™ où se trouve le point d'arrêt. Erreurs de syntaxe Les erreurs de syntaxe sont les erreurs de votre code que le vérificateur de syntaxe de GameMaker Studio 2 a détectées. a détecté dans votre code. Il existe de nombreuses raisons pour lesquelles vous pouvez obtenir une erreur de syntaxe, comme la présence d'un " ;" dans votre code alors qu'un " :" est attendu, et vous devez les corriger avant d'essayer de tester ou de compiler votre jeu. Chaque entrée d'erreur de syntaxe dans la fenêtre de sortie peut être double-cliquée à l'aide du bouton gauche de la souris pour ouvrir une fenêtre à la position indiquée afin de la résoudre. Les erreurs de syntaxe seront mises à jour au fur et à mesure que vous écrirez votre code (il y aura une courte pause entre le moment où vous tapez quelque chose et celui où les erreurs apparaissent dans cette fenêtre, afin d'éviter que des erreurs soient signalées pour un code inachevé), et suivront le format suivant : [objet] - [événement] - [numéro de ligne] - [position dans la ligne] : [chaîne d'erreur]. Ou si l'erreur est dans un script, elle le sera : [script] - [numéro de ligne] - [position dans la ligne] : [chaîne d'erreur]. Notez que les erreurs de syntaxe empêchent généralement la compilation du jeu, mais il existe des erreurs qui seront signalées et qui n'empêcheront pas la compilation, mais qui doivent néanmoins être résolues. Les erreurs en question sont : pour les variables déclarées qui ne sont utilisées nulle part pour les variables utilisées quelque part qui n'ont pas encore été déclarées En général, l'un ou l'autre de ces cas suggère une faute de frappe dans le nom d'une variable, ou un autre type d'erreur facile à commettre - et ils sont donc signalés pour que vous les révisiez. Mais cela peut aussi être intentionnel, car vous avez peut-être déclaré la variable pour une utilisation future, mais vous ne l'avez pas encore utilisée. C'est pourquoi ces erreurs permettront quand même à votre jeu de compiler. Pour une liste complète des messages d'erreur de syntaxe possibles, veuillez consulter cette page. Dans ce cas, le jeu affiche une erreur de compilation et cesse de fonctionner. Toute erreur de compilation sera également affichée dans la fenêtre General Output, mais elle sera également répertoriée ici séparément. (car elles peuvent se "perdre" dans le reste de la sortie générale). Les messages d'erreur du compilateur suivront tous le même format : objet] - [événement] - [numéro de ligne] : [chaîne d'erreur]. Si l'erreur se trouve dans un script, ce sera simplement : script] - [numéro de ligne] : [chaîne d'erreur]. Vous pouvez ensuite double-cliquer sur sur l'une des entrées d'erreur de compilation pour ouvrir le poste donné à la position signalée comme donnant lieu à l'erreur.
