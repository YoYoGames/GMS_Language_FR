Créer un Marketplace de marché Lorsque vous sélectionnez l' option Créer un package dans le menu Marketplace vous serez redirigé vers l'espace de travail suivant: Ici, vous devez d'abord sélectionner l'identité de l'éditeur sous laquelle vous souhaitez publier le package d'actifs (que vous auriez dû configurer à partir de votre tableau de bord de l'éditeur ), puis donner un nom d'affichage et un numéro de version. Un identifiant de package sera ensuite créé pour vous en fonction du nom d'affichage et du compte d'éditeur utilisé, mais vous pouvez le modifier pour qu'il soit autre chose si vous le souhaitez, à condition qu'il conserve le format d'URL inversé. Vous devez ensuite choisir le type de package d'actifs à réaliser: Actif: il s'agit d'un ensemble d'actifs (sprites, scripts, objets, etc...) que vous souhaitez vendre sur la Marketplace Démo: Ceci est une démo pour démontrer un concept ou une idée de jeu Tutoriel: l'élément en cours de téléchargement est un tutoriel, conçu pour être exécuté à partir de l' GameMaker Studio 2 (voir cette page pour plus de détails) Une fois que vous avez fait cela, vous pouvez ensuite choisir les actifs du projet que vous souhaitez ajouter au package d'actifs en utilisant la section de droite. Vous pouvez ajouter tous vos éléments, ou seulement certains d'entre eux, ainsi que supprimer ceux que vous avez peut-être ajoutés par erreur. On suppose que vous avez créé l'actif à télécharger précédemment dans GameMaker Studio 2 et que le projet actuel contient tout ce dont vous avez besoin pour le télécharger. RAPPELLES TOI! Pour pouvoir créer ou utiliser des packages d'actifs, vous devez avoir créé au préalable un certificat de profil utilisateur. Vous pouvez le faire à partir des Marketplace, qui sont expliquées ici. Lorsque vous êtes satisfait de la sélection d'actifs et que vous souhaitez télécharger votre package sur la place de Marketplace, cliquez simplement sur le bouton OK en bas, et le package d'actifs sera créé et téléchargé sur la place de Marketplace. Une fois le téléchargement terminé, un nouvel espace de travail s'ouvrira avec une vue du navigateur de la page d'actif que vous pourrez modifier. Pour plus d'informations sur la façon de configurer cela, veuillez consulter la page suivante: Liste d'un actif Si vous avez besoin de mettre à jour un Marketplace, vous faites plus ou moins la même chose que lorsque vous avez créé le package d'origine, seulement maintenant vous devez spécifier le package à mettre à jour à partir d'une liste déroulante des packages disponibles: Lors de la mise à jour, vous ne pourrez pas modifier l'ID du package, et vous devez être conscient que le numéro de version affiché sera le numéro de version actuelle et vous devez l' incrémenter si nécessaire, sinon la mise à jour échouera (car vous ne pouvez pas avoir deux actifs paquets avec le même numéro de version). Il existe également un champ dans lequel vous pouvez ajouter du texte pour le journal des modifications de l'actif, qui sera téléchargé avec l'actif et affiché sur la page Marketplace IMPORTANT! Les GameMaker Studio avec GameMaker Studio version 2.3+ ne seront disponibles que pour les utilisateurs qui utilisent la version 2.3 et supérieure. Si vous mettez à jour un actif à partir d'une version précédente, cet actif sera toujours disponible pour les utilisateurs qui n'ont pas mis à jour et coexistera avec la version 2.3+ mise à jour. Téléchargement via la page Web Vous pouvez télécharger des actifs via la Marketplace plutôt que via l'EDI, ce qui peut être nécessaire si le téléchargement via l'EDI a échoué pour une raison quelconque. Pour commencer, vous créez le package et tentez de télécharger via l'EDI comme indiqué ci-dessus, et si cela a échoué, vous allez alors dans la fenêtre de sortie et vérifiez le journal pour une ligne qui montre où le package que vous créez a été enregistré à avant de tenter le téléchargement. Par exemple: Sous Windows:Created package file: C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMakerStudio2\GMS2USERNAME\Marketplace\Packages\com.publishername.assetname.yymp Sur macOS:Created package file: /Users/MACUSERNAME/.config/GameMakerStudio2/GMS2USERNAME/Marketplace/Packages/com.publishername.assetname.yymp Vous devez accéder à cet emplacement de fichier et copier le *.YYMP fichier dans un emplacement sûr, car ce sera le fichier que vous téléchargerez sur la place de Marketplace. À ce stade, vous devez ouvrir la place de Marketplace dans votre navigateur Web et accéder à la liste des actifs de l'actif, puis accéder à la section Versions, puis cliquer sur le bouton Télécharger la version, comme indiqué dans l'image ci-dessous: Cela vous mènera à la page de téléchargement de la nouvelle version où vous pouvez ajouter des informations sur la version en cours de téléchargement si nécessaire et également fournir le fichier en cliquant sur le bouton Choisir un fichier et en sélectionnant le fichier précédemment créé. *.YYMP déposer: L'ajout du fichier va télécharger les versions et afficher une barre de "progression", et lorsque le téléchargement est terminé et que vous êtes satisfait des détails, vous pouvez alors cliquer sur le bouton Enregistrer pour enregistrer l'élément dans la liste. Notez que l'actif sera en phase de traitement pendant quelques minutes avant de pouvoir être publié.
