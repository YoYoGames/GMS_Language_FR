Erreurs de compilation Le deuxième type d'erreur qui peut être signalé pour votre jeu est l'erreur de compilation. Une erreur de compilation se produit lorsque votre jeu rencontre un type d'erreur que le vérificateur de syntaxe n'a pas pu détecter. oublier de déclarer une variable ou de taper par erreur une chaîne de caractères comme un réel, par exemple - ou lorsqu'une erreur est liée à la façon dont vous avez configuré les outils de compilation dans les préférences de la plate-forme pour la plate-forme cible. Ces erreurs seront détectées par GameMaker Studio 2 lors de la compilation du jeu et ces informations seront également affichées dans la sortie des erreurs de compilation, comme indiqué ci-dessous : REMARQUE: Ces erreurs n'entraînent généralement pas l'ouverture d'une fenêtre et affichent simplement "Build Failed" dans la sortie du compilateur. Si une fenêtre s'affiche, il s'agit généralement d'une erreur Runner, qui est expliquée ici. Les messages d'erreur du compilateur suivront tous le même format : objet] - [événement] - [numéro de ligne] : [chaîne d'erreur]. Si l'erreur se trouve dans un script, ce sera simplement : script] - [numéro de ligne] : [chaîne d'erreur]. Vous pouvez ensuite double-cliquer sur sur l'une des entrées d'erreur de compilation pour ouvrir l'actif donné à la position signalée comme donnant lieu à l'erreur. pouvez utiliser les informations contenues dans le message pour déterminer exactement où l'erreur s'est produite dans l'objet ou le script.
