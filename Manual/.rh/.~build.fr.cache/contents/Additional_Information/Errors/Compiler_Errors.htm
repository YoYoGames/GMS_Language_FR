Erreurs de compilation Le deuxième type d'erreur qui peut être signalé pour votre jeu est l' erreur du compilateur. Une erreur de compilation se produit lorsque votre jeu rencontre un type d'erreur que le vérificateur de syntaxe n'a peut-être pas été en mesure de détecter - il est très facile d'oublier de déclarer une variable ou de taper par erreur une chaîne en tant que réel, par exemple - ou lorsqu'une erreur est lié à la façon dont vous avez configuré les outils de compilation à partir des préférences de la plate-forme pour la plate-forme cible. Ces erreurs seront GameMaker Studio 2 lors de la compilation du jeu et ces informations seront également affichées dans la sortie d'erreur du compilateur, comme indiqué ci-dessous: REMARQUE: ces erreurs ne font généralement pas apparaître de fenêtre et afficheront simplement « Échec de la construction » dans la sortie du compilateur. Si une fenêtre apparaît, c'est généralement une erreur Runner, qui est expliquée ici. Les messages d'erreur du compilateur suivront tous le même format:[object] - [event] - [Line Number]: [error string] Si l'erreur est trouvée dans un script, ce sera simplement:[script] - [Line Number]: [error string] Vous pouvez ensuite double-cliquer sur l'une des entrées d'erreur du compilateur pour ouvrir l'actif donné à la position marquée comme donnant l'erreur, puis vous pouvez utiliser les informations contenues dans le message pour localiser exactement où dans l'objet ou le script l'erreur s'est produite.
