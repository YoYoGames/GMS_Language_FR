Erreurs de coureur Même après la vérification de la syntaxe dans l'éditeur de code et après que le compilateur vérifie votre code pour les erreurs du compilateur, il y a encore des occasions où quelque chose peut mal tourner. Dans la plupart des cas, cela générera une runtime machine virtuelle (VM) (également appelée exception d' exécution) qui ressemble à ceci: Les erreurs de runner sont généralement plus graves que les erreurs de compilation car cela signifie qu'il y a quelque chose de grave dans votre code que ni l'éditeur de code ni le compilateur n'ont pu détecter, et en tant que tel, vous devez faire attention à toutes ces erreurs. Lorsque cela se produit, vous pouvez utiliser le bouton Copier de la fenêtre contextuelle pour copier l'erreur dans le presse-papiers que vous pouvez ensuite coller dans un fichier texte (ou ailleurs) pour référence future. La structure de cette erreur est la suivante: Cela montre l'événement dans lequel l'erreur s'est produite ainsi que l'objet (le cas échéant). Cela montre l'erreur réelle. Ainsi, il montre la portée de l'erreur et le type d'erreur qu'il s'agit. La première valeur entre parenthèses sera l'ID d'instance de l'instance qui exécutait le code - ou il pourrait s'agir de l'une des valeurs spéciales répertoriées dans le tableau ci-dessous - et la seconde est une valeur interne que YoYo Games utilise pour identifier les bogues et peut être ignoré. Dans le cas de l'image ci-dessus, il est dit que la variable _image n'a pas été défini (c'est-à-dire: n'existe pas avant d'être accédé) sur l'objet obj_EA_DEMO_Control. Cela montre le numéro de ligne sur lequel l'erreur se produit ainsi qu'un extrait de code indiquant la position. Notez que ce n'est pas toujours précis à 100% en raison de la façon dont le jeu est compilé, mais cela devrait suffire pour vous aider à trouver exactement où se situe le problème. Comme mentionné ci-dessus, certains messages d'erreur identifieront la portée non pas par une valeur d'ID d'instance, mais plutôt par une valeur négative. Ces valeurs peuvent être utilisées pour identifier la nature exacte de l'erreur et à quoi elle fait référence avec les valeurs suivantes possibles: Préfixe Portée -1 Soi -2 Autre -3 Tout -4 Personne -5 Global -6 Non spécifié -7 Local Les erreurs possibles du VM runner sont les suivantes: Erreur Un message Opération Description DoSet Type de comparaison non valide Types de données Cela indique que le coureur a essayé de comparer deux types de données incompatibles, comme un nombre réel avec une chaîne. DoConv Erreur d'exécution Types de données Cela dénote une erreur lors de la conversion d'un type de données en un autre. DoAdd Erreur d'exécution Mathématiques Une erreur s'est produite lors de l'utilisation de l'addition (+ ) expression. DoMul Erreur d'exécution Mathématiques Une erreur s'est produite lors de l'utilisation de la multiplication (* ) expression. DoSub Erreur d'exécution Mathématiques Une erreur s'est produite lors de l'utilisation de la soustraction (- ) expression. DoSub Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Mathématiques Vous essayez de soustraire le mauvais type de variables (par exemple soustraire une chaîne d'un réel). DoDiv Erreur d'exécution Mathématiques Un problème est survenu lors de l'utilisation de la division (/ ou div ) expression. DoDiv Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Mathématiques Vous essayez de diviser le mauvais type de variables (par exemple, divisez une chaîne par un réel). DoDiv Diviser par zéro Mathématiques Vous essayez de diviser par 0 (notez que cela ne se produira que lors de l'utilisation de la division entière, la division d'un réel (non nul) par 0 donnera l'infini comme réponse, la division de zéro par zéro donnera NaN comme réponse). Vous pouvez vérifier ces valeurs avec (is_infinity ) et (is_nan ) DoMod Erreur d'exécution Mathématiques Une erreur s'est produite lors de l'utilisation du modulo (mod ) expression. DoMod Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Mathématiques Vous essayez d'utiliser modulo (mod ) sur le mauvais type de variables (par exemple mod une chaîne par un réel). DoAnd Erreur d'exécution Bitwise Une erreur s'est produite lors de l'utilisation du bit "et" (& ) expression. DoAnd Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Bitwise Vous essayez d'utiliser au niveau du bit "et" (& ) sur le mauvais type de variables (par exemple en essayant de "et" une chaîne avec un réel). DoOr Erreur d'exécution Bitwise Une erreur s'est produite lors de l'utilisation du bit "ou" (| ) expression. DoOr Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Bitwise Vous essayez d'utiliser "ou" (| ) sur le mauvais type de variables (par exemple en essayant de "ou" une chaîne avec un réel). DoXor Erreur d'exécution Bitwise Une erreur s'est produite lors de l'utilisation du "xor" au niveau du bit (^ ) expression. DoXor Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Bitwise Vous essayez d'utiliser "xor" (^ ) sur le mauvais type de variables (par exemple en essayant de "xor" une chaîne avec un réel). DoShl Erreur d'exécution Bitwise Un problème est survenu lors du changement de bits à gauche (<< ) une valeur. DoShl Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Bitwise Vous essayez de quitter le bithift (<< ) le mauvais type de variables (par exemple en essayant de décaler une chaîne de bits). DoShr Erreur d'exécution Bitwise Quelque chose s'est mal passé lors du changement de bits à droite (>> ) une valeur. DoShr Moteur d'exécution - Ne peut pas fonctionner sur le type de chaîne Bitwise Vous essayez de changer de bits à droite (>> ) le mauvais type de variables (par exemple en essayant de décaler une chaîne de bits). DoNeg Erreur d'exécution Nier Vous essayez de transformer un type de variable en négatif lorsque ce type ne permet pas une telle opération. DoNot Erreur d'exécution Nier Vous essayez de "ne pas" un type de variable lorsque ce type ne permet pas une telle opération. Push Erreur d'exécution - Index variable hors limites (var) Empiler La variable en cours d'accès est sortie avec la plage établie pour le coureur. Push Variable Get (var) Empiler La variable donnée n'a pas été définie ou est inconnue. Pop Index variable hors plage (var) Empiler La variable en cours d'accès est sortie avec la plage établie pour le coureur. Pop Variable Get (var) Empiler La variable donnée n'a pas été définie ou est inconnue. With Impossible d'utiliser global dans l'instruction with Avec Vous avez essayé d'utiliser "global "en tant que variable dans un"with "déclaration, c'est-à-dire: with (global)    {    //do something;    }With Impossible d'utiliser local dans l'instruction with Avec Vous avez essayé d'utiliser "local "en tant que variable dans un"with "déclaration, c'est-à-dire: with (local)    {    //do something;    }DoCall Erreur de type de moteur d'exécution Moteur Il s'agit d'une erreur non définie dans la machine virtuelle. Vous devez déposer un rapport de bogue si cela se produit (voir: Le menu Aide pour plus de détails sur la façon de procéder. Stack Overflow - Moteur Un débordement de pile se produit lorsque trop de mémoire est utilisée sur la pile d'appels et lorsque votre jeu tente d'utiliser plus d'espace que celui disponible sur la pile d'appels (c'est-à-dire lorsqu'il tente d'accéder à la mémoire au-delà des limites de la pile d'appels, ce qui est essentiellement un buffer overflow), on dit que la pile déborde, ce qui entraîne un plantage du programme. Redémarrez votre ordinateur et GameMaker: Studio et si l'erreur persiste, veuillez contacter l'assistance et / ou signaler un bogue (comme expliqué ci-dessus).
