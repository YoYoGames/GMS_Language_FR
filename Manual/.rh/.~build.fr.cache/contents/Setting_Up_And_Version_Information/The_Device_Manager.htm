Le gestionnaire de périphériques Le gestionnaire de périphériques est accessible en cliquant sur sur le bouton crayon dans la liste cible: Cliquez sur ce bouton pour ouvrir le Gestionnaire de périphériques où vous pouvez configurer différents périphériques pour les plates-formes cibles sur lesquelles vous souhaitez compiler et tester (notez que les plates-formes disponibles dépendront de la licence dont vous disposez et que toutes ne seront pas visibles). Vous trouverez ci-dessous des détails sur la configuration des appareils pour chacune des plates-formes cibles: Android Ici, vous pouvez cliquer sur le bouton Ajouter un nouveau périphérique pour ajouter un nouveau Android appareil à la liste manuellement ou vous pouvez cliquer sur le bouton de l' appareil Détecter pour avoir GameMaker Studio 2 détecte automatiquement tous les périphériques connectés via une connexion filaire. Vous pouvez également configurer un Android virtuel Android en cliquant sur le bouton Exécuter AVD. Cela ouvrira le Android où vous pouvez créer un nouvel AVD (voir ici pour plus d'informations). Une fois que vous en avez créé un, vous devez le lancer à partir du gestionnaire AVD, puis dans GameMaker Studio 2 cliquez sur le bouton Détecter l'appareil et il devrait être ajouté automatiquement à la liste des appareils: Une fois que vous avez ajouté un appareil (physique ou virtuel), vous pouvez alors vérifier la connexion à l'aide du bouton Tester la connexion ou le supprimer à nouveau de la liste en cliquant sur le bouton Supprimer l'appareil. HTML5 Le gestionnaire d'appareils HTML5 permet d'ajouter différents navigateurs à la liste d'appareils pour le ciblage lors de l'exécution. Pour commencer, vous devez cliquer sur le bouton Détecter les navigateurs, qui devrait créer des périphériques pour chacun des navigateurs installés sur votre machine. Si votre navigateur n'est pas répertorié, vous pouvez également cliquer sur le bouton Ajouter un nouveau navigateur, puis attribuer un nom à l'appareil et pointer GameMaker Studio 2 vers le chemin de l'exécutable du navigateur. Les champs supplémentaires disponibles peuvent être configurés pour aider à contrôler la façon dont le navigateur exécute vos projets, avec les options suivantes disponibles: Arguments de ligne de commande du navigateur: Ici, vous pouvez ajouter des arguments de ligne de commande à ajouter au chemin du navigateur. Vous pouvez en trouver une liste à partir des liens suivants vers les principaux navigateurs populaires: Chrome (Chrome) Firefox Internet Explorer Adresse du serveur Web: adresse du serveur Web à cibler (la valeur par défaut est 127.0.0.1). Vous devrez peut-être changer cela en fonction des spécifications de différentes API, par exemple, Facebook exigerait que ce soit défini sur http://localhost (puis mettez cela en liste blanche sur leur console d'application). Port du serveur Web: définissez le numéro de port du serveur Web (la valeur par défaut est 51264). IP de serveur Web / plages d'adresses IP CIDR autorisées: définissez le serveur Web, les noms d'hôte et les plages d'adresses IP CIDR autorisés selon vos besoins. iOS Avant de pouvoir configurer vos iOS, vous devez avoir configuré votre appareil Mac et testé sa connexion (voir ci-dessous pour plus de détails). Une fois que cela est fait, vous pouvez sélectionner le Mac que vous souhaitez cibler dans le menu déroulant en haut de la fenêtre, puis cliquer sur le bouton Détecter l'appareil pour que GameMaker Studio 2 trouve automatiquement tous iOS appareils iOS connectés, y compris les simulateurs. Une fois connecté, les détails de l'appareil seront affichés dans la liste de cette fenêtre, et vous pouvez modifier le nom de l'appareil pour le rendre plus convivial (ce changement n'affecte que la façon dont l'appareil est présenté dans Target Manager et rien d'autre). tvOS Avant de pouvoir configurer vos tvOS, vous devez avoir configuré votre appareil Mac et testé sa connexion (voir ci-dessous pour plus de détails). Une fois que cela est fait, vous pouvez sélectionner le Mac que vous souhaitez cibler dans le menu déroulant en haut de la fenêtre, puis cliquer sur le bouton Détecter l'appareil pour que GameMaker Studio 2 trouve automatiquement tous les tvOS connectés, y compris les simulateurs. Une fois connecté, les détails de l'appareil seront affichés dans la liste de cette fenêtre, et vous pouvez modifier le nom de l'appareil pour le rendre plus convivial (ce changement n'affecte que la façon dont l'appareil est présenté dans Target Manager et rien d'autre). macOS Pour ajouter un appareil Mac au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Ajouter un nouvel appareil qui ajoutera l'appareil et créera des champs vides que vous devrez remplir pour activer la connexion. Vous devez d'abord nommer le périphérique afin de pouvoir l'identifier dans la liste cible, puis donner le nom d'hôte (ou IP) du Mac. Vous devez ensuite donner le nom d'utilisateur de l'utilisateur auquel vous souhaitez vous connecter et le mot de passe qu'il a utilisé. Si vous le souhaitez, vous pouvez également modifier le dossier d'installation sur le Mac à l'endroit où vous préférez que GameMaker Studio 2 stocke les fichiers de construction pour vos projets. Lorsque tout cela est fait, vous pouvez cliquer sur le bouton Tester la connexion pour vous assurer que GameMaker Studio 2 peut communiquer avec le Mac. REMARQUE: Si vous développez pour iOS vous devez d'abord l'avoir fait et avoir établi une connexion fonctionnelle avec le Mac que vous souhaitez utiliser pour le développement. REMARQUE: vous devez avoir activé la connexion à distance sur le mac cible. Cela peut être fait en allant dans Préférences -> Partage et Activer la connexion à distance pour l'utilisateur que vous souhaitez pouvoir vous connecter. Ubuntu ( Linux ) Pour ajouter un Ubuntu ( Linux ) au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Ajouter un nouveau périphérique qui ajoutera le périphérique et créera des champs vides que vous devrez remplir pour activer la connexion. Vous devez d'abord nommer le périphérique afin de pouvoir l'identifier dans la liste cible, puis donner le nom d'hôte (ou IP) de la Ubuntu (ou VM). Vous devez ensuite donner le nom d'utilisateur de l'utilisateur auquel vous souhaitez vous connecter et le mot de passe qu'il a utilisé. Vous devez également fournir un dossier d'installation sur la Ubuntu que vous souhaitez utiliser pour installer vos projets. Lorsque tout cela est fait, vous pouvez cliquer sur le bouton Tester la connexion pour vous assurer que GameMaker Studio 2 peut communiquer avec la machine Ubuntu Notez que l'exécutable final créé par GameMaker dépendra de l'architecture de l'appareil ciblé, qui peut être x64 ou ARM. Par exemple, si vous connectez un Raspberry Pi 3 ou 4, l'exécutable créé sera un exécutable ARM 32 bits, et si vous connectez un Linux ce sera un exécutable 64 bits x64. Windows UWP Avant de pouvoir ajouter un appareil pour la Windows 10 UWP, vous devez avoir activé le mode développeur et configuré le portail d'appareils sur l'appareil. Si vous ne le faites pas, GameMaker Studio 2 ne pourra pas communiquer avec l'appareil ni créer correctement vos applications UWP. Le moyen le plus simple d'activer le mode développeur consiste simplement à taper « Mode développeur » dans Windows Search, puis à cliquer sur l' entrée «Paramètres » qui s'affichera. Cela ouvrira la fenêtre suivante: Vous devez sélectionner l' option Mode développeur (qui peut nécessiter que Windows télécharge et installe des packages supplémentaires), puis sélectionnez l'option en dessous, Activer le portail de l'appareil. Le portail de l'appareil permet à GameMaker Studio 2 de communiquer avec le Windows 10, et lorsqu'il est activé, vous devrez fournir un nom d' utilisateur et un mot de passe: Notez que le nom d'utilisateur et le mot de passe doivent être uniques au portail et ne doivent pas être ceux utilisés par votre compte Microsoft ou d'autres informations d'identification Windows Une fois que vous avez donné ces informations, cliquez sur le bouton intitulé Appliquer, puis fermez le gestionnaire de paramètres et revenez à GameMaker Studio 2. Pour plus d'informations sur le portail des appareils, veuillez consulter cet article Microsoft: Portail des appareils pour ordinateur de bureau. Vous pouvez maintenant ouvrir le Gestionnaire de périphériques dans GameMaker Studio 2 et configurer un ou plusieurs périphériques à utiliser: Pour ajouter un Windows 10 UWP au gestionnaire, vous devez d'abord cliquer sur le bouton intitulé Ajouter un nouvel appareil qui ajoutera l'appareil et créera des champs vides que vous devrez remplir pour activer la connexion. Vous devez d'abord nommer l'appareil afin de pouvoir l'identifier dans la liste cible, puis indiquer l' hôte - il s'agit de l'adresse IP de l' Windows 10 sur le réseau qui doit héberger le jeu et doit être suffixée par " :50443 "pour définir le numéro de port. Ensuite, ajoutez le nom d'utilisateur et le mot de passe que vous avez définis dans le portail de l'appareil (comme mentionné précédemment). Vous pouvez laisser le code PIN vide car il n'est pas requis pour UWP Windows 10 Desktop. Pour des détails de configuration plus spécifiques qui couvrent les différentes sorties UWP (bureau, XboxOne), veuillez consulter le helpdesk YYG: Configurer UWP. Toutes les différentes fenêtres de périphériques ont une option en bas pour Effacer les périphériques qui, lorsque vous cliquez dessus, supprimera tous les périphériques connectés de la liste.
