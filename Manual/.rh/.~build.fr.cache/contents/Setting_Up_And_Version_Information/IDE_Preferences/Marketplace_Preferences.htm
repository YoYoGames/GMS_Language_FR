Préférences dumarché Les préférences de la place de marché sont utilisées pour configurer le certificat de la place de marché pour le téléchargement ou l'achat de biens. Le formulaire de certificat se compose des sections suivantes, lorsque vous avez déjà créé un certificat (cf. plus bas sur cette page pour plus de détails sur la façon de créer un certificat) : Fichier de certificat: Cette section vous permet de spécifier le fichier *.pem que le Marketplace utilise pour la certification des actifs. Si vous avez un certificat précédemment créé, vous pouvez cliquer sur le bouton du navigateur de fichiers ici et l'ajouter dans le forum, mais si vous n'avez pas de certificat et souhaitez en créer un, vous devez omettre toute information ici et cliquer sur le bouton Créer un nouveau certificat (expliqué ci-dessous). Certificate Password: Ici, vous devez saisir le mot de passe du certificat pour le fichier sélectionné ci-dessus. Le mot de passe est créé lorsque vous créez pour la première fois le fichier *.pem (voir Créer un nouveau certificat, ci-dessous) et sera lié de manière irréversible au fichier de certificat créé. Vérifiez: Une fois que vous avez rempli les champs ci-dessus, vous devez cliquer sur ce bouton pour vous assurer que le mot de passe et la sélection du fichier sont corrects et fonctionnent ensemble. Dialogues de confirmation Vous pouvez sélectionner ici le comportement des dialogues de confirmation lorsque vous travaillez avec une ressource depuis le navigateur d'actifs. Les options disponibles sont les suivantes : Réponse automatique à la suppression de contenu hors ligne: Lorsque vous supprimez une ressource du marché de votre bibliothèque sans connexion Web, vous recevez normalement un avertissement vous demandant si vous êtes sûr de vouloir procéder. En réglant cette option sur "OK", le message le message sera supprimé comme si vous aviez cliqué sur le bouton "OK" pour continuer. La valeur par défaut est "Afficher le message". Créer un nouveau certificat Si vous n'avez pas encore créé de fichier de certificat pour l'utilisation de la Place de marché, vous devez cliquer sur le bouton intitulé Créer un nouveau certificat au bas de ce formulaire. Une nouvelle fenêtre s'ouvrira alors pour vous demander de de saisir les informations suivantes : Nom de l'éditeur: Il s'agit du nom qui sera utilisé comme nom d'éditeur pour la ressource. Dans le tableau de bord Marketplace, vous pouvez configurer différents éditeurs si vous le souhaitez et chacun d'entre eux doit avoir son propre fichier de certificat *.pem correspondant. Certificate Password: Ici vous devez donner un mot de passe unique pour le fichier de certificat à créer. Faites attention à ce que vous choisissez car vous ne pouvez pas le changer plus tard et si vous n'avez pas le mot de passe correct pour le fichier de certificat dans les sections ci-dessus, vous ne pourrez pas mettre à jour les actifs existants du Marché (ni en télécharger de nouveaux). les sections ci-dessus, vous ne serez pas en mesure de mettre à jour les actifs Marketplace existants (ni d'en télécharger de nouveaux). Avec ces détails donnés, vous pouvez alors cliquer sur le bouton Créer un certificat pour générer le fichier *.pem. Vous serez invité à indiquer un emplacement de sauvegarde et un nom pour le fichier, et vous devriez en prendre note pour référence future. Une fois que vous avez fait cela, le nouveau fichier et le mot de passe seront automatiquement automatiquement dans les champs du formulaire énumérés ci-dessus et vous pouvez cliquer sur le bouton Vérifier pour vérifier que le certificat a été généré correctement. REMARQUE: Nous vous recommandons de faire une capture d'écran des paramètres actuels ainsi qu'une copie du fichier *.pem . Les deux fichiers doivent être enregistrés ensemble dans un endroit sûr pour référence ultérieure, ou au cas où vous auriez à réinstaller GameMaker Studio 2 à un moment donné. réinstaller GameMaker Studio 2 à un moment donné. Vous ne pouvez pas mettre à jour les ressources qui ont déjà été téléchargées, à moins que le paquet mis à jour ne soit signé à l'aide du même certificat que celui utilisé pour la création de l'original.
