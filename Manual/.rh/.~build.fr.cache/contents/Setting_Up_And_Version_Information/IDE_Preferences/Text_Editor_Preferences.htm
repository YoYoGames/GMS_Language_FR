Préférences de l'éditeur de l'éditeur de texte Les préférences des éditeurs de texte sont utilisées pour définir l'apparence et la convivialité des différents éditeurs de code / script. La page principale propose les options suivantes: Délai de vérification de la syntaxe (ms): délai (en millisecondes) avant l'exécution du vérificateur de syntaxe. Cette valeur par défaut est de 2000 ms. Délai d'achèvement du code (ms): le temps de retard (en millisecondes) Ouvrir les scripts dans l'éditeur plein écran: cocher cette option ouvrira automatiquement les ressources de script dans un nouvel espace de travail. Si cette case n'est pas cochée, ils s'ouvriront dans une fenêtre de l'espace de travail actuel. Cette option est décochée par défaut. Ouvrir les shaders dans l'éditeur plein écran: cocher cette option shader dans un nouvel espace de travail. Si cette case n'est pas cochée, ils s'ouvriront dans une fenêtre de l'espace de travail actuel. Cette option est décochée par défaut. Ouvrir les événements d'objet dans l'éditeur plein écran: cocher cette option ouvrira automatiquement les éditeurs de code d'événement d'objet dans un nouvel espace de travail. Si cette case n'est pas cochée, ils s'ouvriront dans une fenêtre enchaînée à l'éditeur d'événement d'objet. Cette option est décochée par défaut. Ouvrir les scripts d'événement sous forme d'onglets dans la même fenêtre: lors de l'ajout de code ou du Drag and Drop à un événement, la fenêtre de code est enchaînée à la fenêtre d'événement, et lorsque cette option est cochée, tout autre événement ouvert pour modification sera ajouté à la même fenêtre sous forme d'onglets en haut. Si vous décochez cette option, chaque événement sera ouvert comme sa propre fenêtre chaînée unique. Cette option est cochée par défaut. Ouvrir les scripts liés sous forme d'onglets dans la même fenêtre: lorsque vous avez plusieurs scripts liés et que vous les ouvrez, ils seront par défaut regroupés dans la même fenêtre sous forme d'onglets. Décocher cette option les forcera à ouvrir chacun dans leur propre fenêtre / espace de travail. Cette option est cochée par défaut. Afficher l'éditeur de texte au focus: lorsque vous ouvrez un éditeur de texte à partir d'un événement d'objet, la sélection de cette option fait automatiquement un panoramique de l'espace de travail pour amener l'éditeur dans les limites. Cette option est désactivée par défaut. Largeur de la fenêtre de code par défaut: définissez la largeur par défaut (en pixels) de la fenêtre de code à l'ouverture. La valeur par défaut est 900. Hauteur de la fenêtre de code par défaut: définissez la hauteur par défaut (en pixels) de la fenêtre de code à l'ouverture. La valeur par défaut est 600. Largeur minimale de la fenêtre de code: définissez la largeur minimale (en pixels) autorisée par la fenêtre de code. La valeur par défaut est 300. Hauteur minimale de la fenêtre de code: définissez la hauteur minimale (en pixels) autorisée par la fenêtre de code. La valeur par défaut est 200. Outre les options ci-dessus, il existe également une sous-catégorie pour les préférences de l'éditeur de code: Éditeur de code Les préférences de couleur sont utilisées pour modifier la façon dont le code et l'éditeur de script affichent différents textes. Chaque entrée a un certain nombre de sous-entrées pour vous permettre de définir exactement à quoi doit ressembler chaque morceau de texte séparé, comme le montre l'exemple d'image ci-dessus. En général, vous pouvez définir la couleur, la police, la taille et le style de chacune des différentes options. Plus bas, vous pouvez également définir un style plus général pour des éléments tels que la sélection de lignes de crochets de code, etc. Sous les préférences de couleur, vous avez les différentes options de style. Ces préférences sont utilisées pour définir le comportement de l'éditeur de code / script, avec les options suivantes disponibles: Conserver les onglets: si coché, cela conservera les touches de tabulation sous forme de caractères de tabulation - ' \t '- dans l'éditeur, si elle n'est pas cochée, cela signifie que les onglets sont convertis en espaces. La valeur par défaut est désactivée. Taille de la tabulation: le nombre d'espaces de caractères que le fait d'appuyer sur "Tab" doit indenter le code. La valeur par défaut est 4. Afficher les espaces: Afficher " . "pour afficher les espaces (voir l'image ci-dessous). La valeur par défaut est désactivée. Afficher les numéros de ligne: afficher ou non les numéros de ligne vers le bas à gauche. La valeur par défaut est activée. Afficher les lignes de tabulation: Afficher " .... "pour afficher les tabulations. La valeur par défaut est désactivée. Activer le codage couleur: activez ou désactivez le codage couleur pour l'éditeur. Si elle est désactivée, le code ne sera pas stylisé, tandis que lorsqu'il est activé, il sera stylisé en utilisant la mise en forme donnée ci-dessus dans la section Couleur. La valeur par défaut est activée. Limite de ligne de document: définissez le nombre maximum de lignes pour tout document éditeur donné. La valeur par défaut est 0 (0 ou moins est essentiellement infini) et toute autre valeur limitera les lignes autorisées. Faire correspondre le retrait sur les nouvelles lignes: activez ou désactivez l'indentation automatique pour votre code. La valeur par défaut est activée, ce qui conservera tous les onglets définis sur une ligne lorsqu'une nouvelle ligne est ajoutée. Indentation automatique après accolade: activez ou désactivez l'indentation automatique pour votre code lors de l'ajout d'accolades {}. La valeur par défaut est activée, ce qui tabulera la ligne de code suivante lors d'une nouvelle accolade { et une nouvelle ligne est ajoutée. Activer la complétion de code: activez ou désactivez la fenêtre de complétion de code. Lorsque vous écrivez du code et que cela est activé, vous obtiendrez une fenêtre contextuelle montrant les fonctions possibles que vous pouvez utiliser en fonction du texte d'entrée actuel. La valeur par défaut est activée. Ajout automatique de crochets de fonction: permet d'activer ou de désactiver l'ajout automatique de crochets () lorsque vous utilisez la saisie semi-automatique. Lors de l'écriture de code et ceci est activé, l'EDI ajoutera automatiquement les deux crochets () requise par une fonction, en y plaçant le curseur si la fonction prend des arguments, ou en plaçant le curseur après eux si ce n'est pas le cas. Cette fonctionnalité ne fonctionne que lorsque la fonction a été sélectionnée à partir de la saisie semi-automatique. La valeur par défaut est activée. Limite de pile Annuler / Rétablir: Ici, vous pouvez définir la limite de pile Annuler / Rétablir. Tout ce qui est sous 0 est essentiellement infini (limité par la mémoire disponible) tandis que les valeurs positives limiteront la pile à ce nombre, supprimant toute action à la queue de la pile si la limite est atteinte. La valeur par défaut est -1. Mettre en évidence les correspondances de sélection: cela mettra légèrement en évidence toute section de code dupliquée ailleurs. Ainsi, si vous sélectionnez une fonction, par exemple, toutes les instances de cette fonction dans la fenêtre de code actuelle seront également mises en surbrillance. La valeur par défaut est activée. Codage couleur / complétion de code pour les fonctions obsolètes: Cela mettra en évidence toute section de code obsolète. La valeur par défaut est activée.
