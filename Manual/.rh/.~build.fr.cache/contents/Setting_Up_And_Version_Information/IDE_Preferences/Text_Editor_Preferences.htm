Préférences de l'éditeur de texte Les préférences des éditeurs de texte sont utilisées pour définir l'aspect et la convivialité des différents éditeurs de code/script. La page principale comporte les options suivantes : Délai de vérification syntaxique (ms): Le délai (en millisecondes) avant l'exécution du vérificateur syntaxique. La valeur par défaut est de 2000ms. Délai d'achèvement du code (ms): Le temps de retard (en millisecondes) Ouvrir les scripts dans l'éditeur plein écran: En cochant cette option, les ressources de script s'ouvriront automatiquement dans un nouvel espace de travail. Si cette option n'est pas cochée, elles s'ouvriront dans une fenêtre de l'espace de travail actuel. Cette option n'est pas cochée par par défaut. Ouvrir les shaders dans l'éditeur plein écran: En cochant cette option, les ressources de shaders s'ouvriront automatiquement dans un nouvel espace de travail. Si cette option n'est pas cochée, elles s'ouvriront dans une fenêtre de l'espace de travail actuel. Cette option n'est pas cochée par par défaut. Ouvrir les événements d'objets dans un éditeur plein écran: En cochant cette option, les éditeurs de code d'événements d'objets s'ouvriront automatiquement dans un nouvel espace de travail. Si cette option n'est pas cochée, ils s'ouvriront dans une fenêtre chaînée à l'éditeur d'événements d'objets. Cette option n'est pas cochée par défaut. Ouvrir les scripts d'événements sous forme d'onglets dans la même fenêtre: Lorsque vous ajoutez du code ou un glisser-déposer à un événement, la fenêtre de code est enchaînée à la fenêtre de l'événement, et lorsque cette option est cochée, tous les autres événements ouverts à l'édition seront ajoutés à la même fenêtre sous forme d'onglets en haut. dans la même fenêtre sous forme d'onglets en haut. Si vous décochez cette option, chaque événement s'ouvrira comme sa propre fenêtre enchaînée. Cette option est cochée par défaut. Ouvrir les scripts liés sous forme d'onglets dans la même fenêtre: Lorsque vous avez plusieurs scripts liés et que vous les ouvrez, par défaut ils seront regroupés dans la même fenêtre sous forme d'onglets. Si vous décochez cette option, ils s'ouvriront chacun dans leur propre fenêtre/espace de travail. Cette option est cochée par défaut. Panoramique de l'éditeur de texte en fonction du focus: Lorsque vous ouvrez un éditeur de texte à partir d'un événement d'objet, la sélection de cette option permet d'orienter automatiquement l'espace de travail afin d'amener l'éditeur à l'intérieur des limites. Cette option est désactivée par défaut. Largeur par défaut de la fenêtre de code: définissez la largeur par défaut (en pixels) de la fenêtre de code lorsqu'elle est ouverte. La valeur par défaut est 900. Hauteur par défaut de la fenêtre de code: Définissez la hauteur par défaut (en pixels) de la fenêtre de code lorsqu'elle est ouverte. La valeur par défaut est 600. Largeur minimale de la fenêtre de code: Définit la largeur minimale (en pixels) autorisée par la fenêtre de code. La valeur par défaut est 300. Hauteur minimale de la fenêtre de code: Définissez la hauteur minimale (en pixels) autorisée par la fenêtre de code. La valeur par défaut est 200. Chaque entrée comporte un certain nombre de sous-entrées qui vous permettent de définir exactement l'aspect de chaque élément de texte, comme le montre l'exemple suivant ci-dessus. En général, vous pouvez définir la couleur, la police, la taille et le style pour chacune des différentes options. Plus bas, vous pouvez également définir un style plus général pour des éléments tels que la sélection de lignes, les parenthèses de code, etc... Sous les préférences de couleur, vous avez les différentes Options de style. Ces préférences sont utilisées pour définir le comportement de l'éditeur de code/script, avec les options suivantes disponibles : Conserver les tabulations: Si cette option est cochée, les pressions sur les touches de tabulation seront conservées comme caractères de tabulation - ' \t ' - dans l'éditeur, tandis que si cette option n'est pas cochée, les tabulations seront converties en espaces. Par défaut, cette option est désactivée. Taille de la tabulation: Le nombre d'espaces de caractères que l'appui sur "Tab" devrait indenter le code. La valeur par défaut est 4. Afficher les espaces blancs: Affichez " . " pour afficher les espaces blancs (voir l'image ci-dessous). La valeur par défaut est désactivée. Afficher les numéros de ligne: Afficher ou non les numéros de ligne en bas à gauche. Par défaut, cette option est activée. Afficher les lignes d'onglets: Affichez " .... " pour afficher les lignes de tabulation. La valeur par défaut est désactivée. Activer le codage couleur: Activez ou désactivez le codage couleur pour l'éditeur. S'il est désactivé, le code ne sera pas stylé, tandis que s'il est activé, il sera stylé en utilisant le formatage donné ci-dessus dans la section Couleur. Par défaut, cette option est activée. Limite de lignes du document: Définit le nombre maximum de lignes pour un document d'éditeur donné. La valeur par défaut est 0 (0 ou moins est essentiellement infini) et toute autre valeur limitera les lignes autorisées. Correspondre à l'indentation sur les nouvelles lignes: Activez ou désactivez l'indentation automatique pour votre code. La valeur par défaut est activée, ce qui maintient les tabulations définies sur une ligne lorsqu'une nouvelle ligne est ajoutée. Indentation automatique après une accolade: Activez ou désactivez l'indentation automatique de votre code lors de l'ajout d'accolades. {} . La valeur par défaut est activée, ce qui permet de tabuler la ligne de code suivante lorsqu'une nouvelle accolade { et une nouvelle ligne sont ajoutées. Activer la complétion de code: Activez ou désactivez la fenêtre de complétion de code. Lorsque vous écrivez du code et que cette option est activée, vous obtiendrez une fenêtre popup montrant les fonctions possibles que vous pouvez utiliser en fonction du texte d'entrée actuel. Par défaut, cette option est activée. Ajout automatique de parenthèses de fonction: Activez ou désactivez l'ajout automatique des crochets de fonction () lorsque vous utilisez l'autocomplétion. Lorsque vous écrivez du code et que cette option est activée, l'IDE ajoutera automatiquement les deux parenthèses () qu'une fonction requiert, en plaçant le curseur à l'intérieur si la fonction prend des arguments, ou après si elle n'en prend pas. Cette fonctionnalité ne fonctionne que lorsque la fonction a été sélectionnée dans l'autocomplétion. Par défaut, elle est activée. Limite de la pile Undo / Redo: Ici, vous pouvez définir la limite de la pile Undo/Redo. Tout ce qui est inférieur à 0 est essentiellement infini (limité par la mémoire disponible) tandis que les valeurs positives limiteront la pile à ce nombre, supprimant toute action à la fin de la pile si la limite est atteinte. la queue de la pile si la limite est atteinte. La valeur par défaut est -1. Mettre en évidence les correspondances de sélection: Cette option met légèrement en évidence toute section de code qui est dupliquée ailleurs. Ainsi, si vous sélectionnez une fonction, par exemple, toutes les instances de cette fonction dans la fenêtre de code actuelle seront également mises en évidence. également. Cette option est activée par défaut. Codage couleur/complétion de code pour les fonctions obsolètes: Ceci mettra en évidence toute section de code qui est obsolète. La valeur par défaut est activée.
