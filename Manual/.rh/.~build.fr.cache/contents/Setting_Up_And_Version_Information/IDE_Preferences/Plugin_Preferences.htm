Préférences des plugins Les préférences des plugins sont utilisées pour contrôler tous les plugins supplémentaires qui ont été ajoutés à l'IDE GameMaker Studio 2. Par défaut, une section est consacrée au plugin Source Control pour l'intégration des projets GameMaker Studio 2 avec Git. Contrôle des sources (Git) Cette section traite des préférences que vous devez définir pour obtenir l'application Git SCM fonctionne avec GameMaker Studio 2. Tout d'abord, vous avez le jeu de préférences G options générales : Ouvrir automatiquement la fenêtre de sortie du contrôle de la source: Lorsque cette option est cochée, la fenêtre de sortie du contrôle de la source s'affiche automatiquement lorsque quelque chose est modifié. Cette option est activée par défaut. Journalisation verbeuse: Active la journalisation verbeuse pour le journal de sortie de Git. Cette option est désactivée par défaut. Vient ensuite la section Identité de l'utilisateur avec les champs suivants qui doivent être remplis : Nom d'utilisateur pour commiter: Vous devez saisir ici le nom d'utilisateur que vous utiliserez pour effectuer des commits dans Git (ce doit être le même que le nom d'utilisateur spécifié lorsque vous avez configuré votre dépôt Git ). Email pour commiter en tant que: L'email qui est associé à cette identité Git. Après avoir configuré l'identité de l'utilisateur, vous pouvez ensuite ajouter une nouvelle combinaison nom d'utilisateur/mot de passe pour l'authentification en cliquant sur le bouton intitulé Ajouter un nouvel utilisateur/mot de passe d'authentification, ce qui ouvrira la fenêtre suivante avec différents champs à remplir : Vous pouvez également Ajouter une nouvelle authentification par paire de clés SSH en cliquant sur le bouton portant ce libellé. Cela ouvrira la fenêtre suivante que vous devrez remplir : Sous ces boutons, vous pouvez alors configurer l'outil de Outil de fusion et l'outil de Diff Tool pour SCM avec ces options : Chemin d'accès à l'outil de fusion: vous indiquez ici le chemin d'accès complet au fichier de l'outil de fusion Git. Options pour l'outil de fusion: Dans ce champ, vous pouvez ajouter toute commande supplémentaire à exécuter lorsque vous utilisez l'outil de fusion. Chemin d'accès à l'outil de comparaison: vous indiquez ici le chemin d'accès complet à l'outil de comparaison Git. Options pour l'outil Diff: Dans ce champ, vous pouvez ajouter des commandes supplémentaires à exécuter lorsque vous utilisez l'outil de diffusion. Enfin, après les avoir mis en place, vous avez une dernière option : Activer automatiquement le contrôle de source: Lorsque cette option est cochée, le contrôle de la source sera automatiquement activé pour tous les nouveaux projets.
