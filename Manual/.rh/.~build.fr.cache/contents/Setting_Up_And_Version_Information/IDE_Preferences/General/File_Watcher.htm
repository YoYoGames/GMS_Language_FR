Préférences de l'observateur de fichiers Le File Watcher est le nom donné à la partie de l'IDE qui surveille les fichiers liés à un projet pour détecter toute modification due à un logiciel externe (par exemple, l'édition d'une image dans un éditeur externe) ou à des modifications internes comme l'édition d'un script actif. En général, ces paramètres ne devraient pas avoir besoin d'être modifiés, mais sur certains systèmes - comme ceux qui utilisent un SSD comme lecteur de sauvegarde pour les fichiers du projet - l'observateur de fichiers peut vous informer des changements lorsque vous faites des choses comme simplement sauvegarder le projet ou lorsque vous modifiez une ressource du projet, auquel cas vous devrez peut-être modifier ces paramètres. Les paramètres disponibles sont les suivants : Activer l'observateur de fichiers: Ceci peut être utilisé pour activer ou désactiver le système File Watcher. Si cette option est désactivée, l'IDE ne détectera plus les fichiers modifiés par des sources externes, il faut donc faire attention lorsque l'on désactive cette fonction. cette fonction. Le File Watcher est activé par défaut. Définir la tolérance du File Watcher dans MS après la sauvegarde Dans l'IDE: Ceci ajuste le temps entre la sauvegarde d'un fichier et la vérification des changements par le File Watcher. Vous ajustez cette valeur si vous sauvegardez votre projet et que le File Watcher s'affiche par erreur en disant que quelque chose a changé. Vous ajusteriez cette valeur si vous sauvegardiez votre projet et que le File Watcher apparaissait par erreur en disant que quelque chose avait changé, dans ce cas vous augmenteriez le délai jusqu'à ce que le message n'apparaisse plus.
