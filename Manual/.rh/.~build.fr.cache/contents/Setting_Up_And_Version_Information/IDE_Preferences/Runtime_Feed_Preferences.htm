Préférences de flux d' Runtime Les Runtime Préférences Les flux sont utilisés pour afficher et modifier les actuellement installé Runtime ainsi que l' runtime en cours d' utilisation. GameMaker Studio 2 peut avoir plusieurs runtime différents installés à tout moment et vous pouvez définir celui à utiliser à partir de cette préférence. Idéalement, vous souhaitez utiliser le runtime le plus à jour, mais parfois cela peut avoir un problème, ou vous devrez peut-être utiliser un plus ancien pour la comparabilité, etc... et ainsi vous pouvez changer les outils d' runtime GameMaker Studio 2 utilise pour créer votre jeu ici. La section initiale de cette préférence est liée à l'endroit où vous extrayez les runtime et vous oblige à saisir une URL dans un flux RSS. Ce flux sera ensuite analysé pour les mises à jour runtime outils d'exécution qui peuvent ensuite être téléchargés et installés. Cette section affiche l' runtime principal par défaut, mais vous pouvez cliquer sur le bouton Ajouter pour créer un nouveau Runtime lequel vous donnez le nom et l'URL du flux RSS, puis cliquez sur Appliquer pour indiquer à GameMaker Studio 2 d'y accéder.. Vous devrez fermer et rouvrir la fenêtre Préférences pour que le nouveau flux s'affiche. Une fois qu'un Runtime a été créé, il sera répertorié dans cette section en tant que sous-catégorie, et vous devriez toujours avoir au moins un flux répertorié (par défaut, il s'agirait du flux principal ). Quel que soit l' runtime utilisé, ils auront tous les mêmes fonctionnalités, décrites ci-dessous en utilisant le flux principal comme exemple: Les Runtime Feed permettent de sélectionner, d'installer et de désinstaller les différents outils d' runtime En haut, vous avez les runtime actuellement utilisés, la date à laquelle il a été créé et l'emplacement dans lequel il est stocké sur le disque dur. Sous ces détails, vous pouvez trouver une liste de tous les runtimes disponibles, et vous pouvez choisir n'importe lequel de cette liste pour devenir le runtime "en cours d'exécution" utilisé pour compiler vos projets. Pour sélectionner un autre runtime à utiliser, recherchez-le simplement dans la liste, puis double-cliquez dessus. Si le runtime n'a pas été téléchargé et installé, vous serez invité à le faire, et une fois qu'il a été installé, il vous sera demandé de fermer et de redémarrer l'EDI afin que le nouveau runtime puisse être initialisé. Cette liste détaille tous les runtimes disponibles, leur numéro de version, la date à laquelle ils ont été publiés et, s'ils sont installés, le chemin d'accès à l'endroit où ils ont été installés. Le côté droit montre également les différentes icônes d'état qui sont les suivantes: Cela marque le runtime actuel qui a été sélectionné pour être utilisé lors de la création de vos projets. Cliquez dessus pour lancer le téléchargement et l'installation du runtime sélectionné (une barre d'état sera affichée en haut à droite de l' EDI ). Cliquez dessus pour supprimer le runtime sélectionné de votre disque dur. Vous pouvez mettre à jour la runtime à tout moment à partir du flux RSS donné en cliquant sur le bouton d'actualisation, et vous pouvez nettoyer votre disque dur d'anciens runtimes d'exécution en cliquant sur le bouton "Nettoyer les Runtimes maintenant".
