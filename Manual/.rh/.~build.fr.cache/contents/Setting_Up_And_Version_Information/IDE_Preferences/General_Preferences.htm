Préférences générales Les préférences générales sont utilisées pour définir le comportement général de l' GameMaker Studio 2 IDE. Les plus générales sont répertoriées sur la page principale tandis que beaucoup d'autres sont divisées en plusieurs sous-catégories, expliquées plus loin dans cette page. Langue IDE: Cette option vous permet de sélectionner la langue de l'EDI dans la liste des options fournies. La valeur par défaut est l'anglais. IDE Skin: Ici, vous pouvez sélectionner parmi les différents skins que l'EDI peut utiliser. Par défaut, GameMaker Studio 2 est livré avec deux skins, un clair et un foncé, et la configuration initiale utilise le skin foncé. Fréquence d'images maximale de l' GameMaker Studio 2: cette option vous permet de définir la fréquence d'images maximale à laquelle l' GameMaker Studio 2 mettra à jour, ce qui vous permet de définir la fréquence d'images en fonction de la vitesse de mise à jour de votre moniteur. La valeur par défaut est 60. Périphérique audio par défaut: Ici, vous pouvez choisir le périphérique audio par défaut que l' GameMaker Studio 2 doit utiliser. Vous pouvez actualiser la liste en cliquant sur le bouton Actualiser. Réponse automatique à la confirmation de sortie: cette option vous permet de définir si un message de confirmation doit être affiché lorsque vous quittez GameMaker Studio 2. Par défaut, il est défini sur "afficher le message", mais peut être configuré pour ignorer le message comme si vous aviez cliqué sur OK. Réponse automatique à la boîte de dialogue Changement de police système: Cette option vous permet de définir si un message de confirmation doit être affiché lorsqu'une police système a été ajoutée, supprimée ou modifiée. Par défaut, il est défini sur "afficher le message", mais peut être configuré pour ignorer le message comme si vous aviez cliqué sur OK. Activer les notifications: cocher cette option permettra à l'EDI d'afficher des notifications sur les nouveaux runtimes, etc... Ceci est activé par défaut. Délai d'expiration de la notification (ms): Ceci définit la durée (en millisecondes) pendant laquelle une notification est affichée. La valeur par défaut est 6000. Déconnexion à la sortie: Si cette option est cochée, l'EDI déconnectera automatiquement la session utilisateur lorsque GameMaker Studio 2 est fermé. Ceci est principalement destiné aux utilisateurs de l'éducation, car - une fois coché - cela signifie que chaque fois que le programme s'arrête, l'utilisateur est déconnecté et libère ainsi le siège de session pour que d'autres utilisateurs l'utilisent. Ceci est désactivé par défaut. Activer le remplacement PPP: cette option vous permet de remplacer le paramètre PPP par défaut par le vôtre. Vous devez d'abord activer l'option, puis définir soit la valeur Raw DPi Override (la valeur par défaut est 96), soit définir le pourcentage du DPi natif (valeur par défaut 100%), et une fois modifiée, vous serez invité à redémarrer l'EDI, ce qui peut vous obliger à enregistrer tout travail en cours sur le projet actuellement ouvert. Notez que le DPi a un pourcentage minimum de 50% et un maximum de 500%, et si vous passez la souris sur l'icône d'information, vous pouvez voir le paramètre maximum recommandé pour l'affichage utilisé: Notez que si vous dépassez le paramètre DPi recommandé pour le moniteur, vous verrez une notification vous permettant de réinitialiser le DPi. Afficher l'écran de bienvenue au démarrage: Ceci est activé par défaut et indique simplement à GameMaker Studio 2 d'afficher un bref écran de démarrage lorsque vous démarrez le programme. Si vous décochez cette option, l'écran de démarrage ne s'affichera pas. Autoriser la fermeture des onglets vides via un raccourci: vous pouvez fermer tous les onglets ouverts à l'aide du raccourci clavier / + W, mais si vous désactivez cette option, ce raccourci ne fonctionnera plus. C'est activé par défault. Ouvrir de nouvelles ressources à modifier en faisant glisser des fichiers dans l'EDI: certaines ressources (comme des sprites ou des sons) peuvent être glissées sur l'espace de travail principal dans l' GameMaker Studio 2 et seront détectées et ajoutées en tant que nouvelle ressource dans le navigateur d'actifs. Par défaut, l'éditeur de ressources correspondant sera également ouvert pour modification lorsque cela se produit, mais décocher cette option empêchera l'ouverture des éditeurs et vous devrez les ouvrir manuellement. Ouvrir de nouvelles ressources pour modification lors de la création: si cette case est cochée, chaque fois que vous créez une nouvelle ressource dans le navigateur d'actifs, l'éditeur de ressources correspondant s'ouvre automatiquement. Ceci est coché par défaut et décocher cela signifiera que vous devrez ouvrir manuellement toutes les ressources nouvellement créées. Recharger automatiquement les fichiers modifiés: si cette case est cochée, l'EDI analysera et vérifiera automatiquement les fichiers pour voir si l'un d'entre eux a été modifié en dehors de GameMaker Studio 2 et les rechargera le cas échéant. Désactiver les animations de transition IDE: l' GameMaker Studio 2 a un certain nombre de transitions animées pour des choses comme l'ouverture de fenêtres, etc., mais elles ne sont pas du goût de tout le monde et vous pouvez bénéficier d'une amélioration des performances si elles sont désactivées sur des machines plus anciennes ou sous-alimentées. s'ils sont désactivés. Vous pouvez les désactiver ici, et ils sont activés par défaut. Inscrivez-vous aux sondages de rétroaction: lorsque cette option est cochée, vous GameMaker Studio 2 que vous souhaitez participer à des sondages de rétroaction, ce qui signifie que de temps en temps, pendant que vous travaillez, un message sera affiché concernant le domaine sur lequel vous travaillez et demandez une évaluation et retour d'information. Cette option est activée par défaut.
