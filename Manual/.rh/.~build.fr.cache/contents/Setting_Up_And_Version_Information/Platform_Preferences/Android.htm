Préférences Android Les Android sont divisées en différentes sections répertoriées ci-dessous: En haut des préférences, vous pouvez définir la taille maximale du tas de mémoire JDK (la valeur par défaut est de 2 Go). Ensuite, vous devez configurer les différents chemins afin que GameMaker Studio 2 sache où chercher les différents outils nécessaires pour construire le package exécutable final et tester votre jeu. Ceci est divisé en trois sections pour le Android SDK, Android NDK et Java JDK. Ces chemins doivent avoir été remplis automatiquement pour vous, mais si vous rencontrez des problèmes, vous pouvez cliquer sur le bouton à droite pour ouvrir l'explorateur de fichiers et sélectionner manuellement les emplacements pour chaque élément. Vous pouvez trouver des détails sur les versions requises pour chacun des éléments du SDK ici. Les chemins doivent être validés au fur et à mesure que vous les appliquez et un message affichera « Trouvé » ou vous informera de toute erreur (dans ce cas, vous devez réviser toutes les informations pour vous assurer qu'elles sont correctes). Ci-dessous, vous avez les différentes options d'emballage qui sont: Effectuez toujours une installation complète de l'APK: si vous le GameMaker Studio 2 supprimera tous les précédents coureurs de votre jeu de votre appareil, ainsi que tous les fichiers associés (par exemple, *.ini fichiers) à partir de votre appareil de test avant d'installer une nouvelle version du jeu. Installer sur le package: lorsque cette option est marquée, compiler un exécutable final *.apk package l'installera également automatiquement sur le périphérique cible actuellement sélectionné. Enfin, nous arrivons au KeyStore. Il s'agit d'un fichier qui sera utilisé pour «signer» toutes vos Android. Veuillez donc remplir tous les détails correctement et gardez à l'esprit que ce fichier sera nécessaire pour créer et mettre à jour toutes vos Android à l'avenir, ce qui signifie qu'après la configuration il est conseillé de prendre une capture d'écran des paramètres utilisés et de faire une sauvegarde du fichier final. Si vous perdez ce fichier, vous ne pourrez pas mettre à jour les jeux existants qui ont été téléchargés dans la boutique. Pour créer un nouveau fichier de keystore, les détails suivants sont requis: Nom de fichier: il s'agit du nom du fichier de clés. REMARQUE: ce n'est pas votre nom ou le nom de l'entreprise, mais le nom du fichier à générer! Si vous avez déjà un fichier de keystore de projets précédents, vous pouvez cliquer ici sur le bouton du navigateur de fichiers et diriger GameMaker Studio 2 vers le fichier de keystore précédemment créé (ou cliquer sur le bouton Importer ). Dans ce cas, vous devez toujours remplir le reste des informations requises ( exactement comme lors de la première génération du fichier ) mais vous ne devez pas cliquer sur le bouton Générer le magasin de clés!. Cela générera un nouveau fichier keystore, écrasant le précédent. Nom commun: ce serait normalement votre nom. Mot de passe: votre mot de passe de sécurité pour le fichier de clés, qui doit comporter au moins six caractères. Alias: il s'agit d'un autre nom qui peut être utilisé pour le fichier de clés et doit être différent du nom saisi ci-dessus. Mot de passe d'alias: votre mot de passe de sécurité d'alias pour le fichier de clés, qui doit comporter au moins six caractères. Unité organisationnelle: le service de l'entreprise dont vous faites partie. Organisation: le nom de votre entreprise. Lieu: Le nom de la ville ou de la ville où vous êtes basé. Pays: code standard à deux lettres du pays dans lequel vous êtes basé. Une fois que ces informations sont remplies et que vous en êtes satisfait, appuyez sur le bouton Générer le magasin de GameMaker Studio 2 pour créer le fichier de magasin de clés nécessaire. Cela peut prendre un moment, mais une fois terminé, il ne sera normalement pas nécessaire de le modifier à nouveau. Notez que vous pouvez cliquer sur le bouton Importer pour importer un fichier de keystore précédemment créé (voir la section Nom de fichier, ci-dessus) et vous pouvez également récupérer le hachage du keystore SHA1 en cliquant sur le bouton Generate Key Hash. En appuyant dessus, vous générerez deux clés de hachage de sécurité spéciales directement liées à votre fichier de keystore dont certains sites Web ont besoin pour une fonctionnalité correcte (par exemple, Facebook ou Google Play pour le hachage Sha1). Si vous en avez besoin, appuyez simplement sur le bouton, puis copiez et collez les clés de hachage générées.
