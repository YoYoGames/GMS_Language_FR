Informations sur la configuration et la version de GMS2 Il existe différentes versions des licences que vous pouvez obtenir pour GameMaker Studio 2, dont le choix dépendra en grande partie du public que vous souhaitez cibler ainsi que du fait que vous souhaitiez simplement vous amuser à créer des jeux ou que vous souhaitiez vendre. et distribuez-les également. Vous trouverez ci-dessous les détails de chaque type de licence: Procès L'inscription à un compte YoYo vous accorde une licence d'essai gratuite de 30 jours pour démarrer GameMaker Studio 2 avec, puis vous pouvez acheter différentes mises à niveau en fonction de vos besoins. La licence d'essai vous offre une grande partie de l' GameMaker Studio 2 et est disponible pour une exécution sur un ordinateur macOS ou Windows, mais a une limite de 30 jours d' utilisation. Une fois les 30 premiers jours écoulés, vous devrez passer à l'une des licences payantes pour continuer à utiliser le produit. Notez que la licence d'évaluation a les limitations suivantes: Il ne permet d'accéder à GMS2 que pendant 30 jours à compter de la date à laquelle vous avez activé votre YoYo via votre e-mail de confirmation. Vous ne pouvez pas compiler un exécutable final pour une plate-forme cible. Vous n'avez pas accès pour tester les exportations cibles "YYC" plus performantes. Vous ne pouvez pas désactiver les notifications IDE concernant votre licence d'essai expirant bientôt (mais les autres notifications seront désactivées si vous désactivez les notifications dans les préférences). Vous ne pouvez pas créer de Marketplace (mais votre YoYo peut accéder à Marketplace et acheter des actifs payants / gratuits). Créateur Avec la version Creator GameMaker Studio 2 la limite de 30 jours de la licence d'essai (voir ci-dessus) est supprimée et vous pouvez exporter vos jeux pour la plate-forme choisie, qui peut être macOS ou Windows. La plate-forme cible disponible dépendra de la version de l'EDI que vous avez acheté ( Creator pour macOS ou Creator pour Windows ) et utilisera la VM (un runner + code interprété) pour exécuter et créer vos jeux. Pour plus d'informations sur la configuration, consultez la section du support technique sur les SDK requis ou la section sur les cibles de bureau. Notez qu'il existe deux limites mineures imposées à la licence Creator: Vous ne pouvez pas modifier l'écran de démarrage par défaut lorsque votre jeu est en cours de chargement. Vous n'avez pas accès pour créer vos projets à l'aide des YoYo Compiler). Bureau Avec la version Desktop GameMaker Studio 2 toutes les limitations de la licence d'évaluation (voir ci-dessus) sont supprimées et vous pouvez compiler des projets vers les cibles Desktop suivantes: Windows macOS Ubuntu ( Linux ) Toutes les cibles sont livrées avec deux options de compilation, l'une est la VM qui utilise un runner et du code interprété et l'autre qui est le YYC ( YoYo Compiler) qui crée du code natif et un exécutable compilé. Notez que l'option YYC nécessitera une configuration supplémentaire. Pour plus d'informations, consultez la section sur les SDK requis ou la section sur les cibles de bureau. la toile Avec la version Web GameMaker Studio 2 toutes les limitations de la licence d'essai (voir ci-dessus) sont supprimées et vous pouvez compiler des projets vers la cible HTML5. Les jeux sont compilés en utilisant JavaScript. Vous pouvez trouver plus d'informations sur le helpdesk de YoYo Games ici: Configuration pour le Web (HTML5). Mobile Avec la version mobile GameMaker Studio 2 toutes les limitations de la licence d'essai (voir ci-dessus) sont supprimées et vous pouvez compiler des projets vers les cibles suivantes: Android iOS tvOS Toutes les cibles sont livrées avec deux options de compilation, l'une est la VM qui utilise un runner et du code interprété et l'autre qui est le YYC ( YoYo Compiler) qui crée du code natif et un exécutable compilé. Notez que l'option YYC nécessitera une configuration supplémentaire. Pour plus d'informations sur la configuration, consultez la section sur les SDK requis ou la section d'assistance sur les cibles mobiles. UWP Avec la version UWP GameMaker Studio 2 toutes les limitations de la licence d'essai (voir ci-dessus) sont supprimées et vous pouvez compiler des projets en tant qu'exécutables compatibles UWP. Ces exécutables peuvent être configurés pour s'exécuter sur l'une des deux plates-formes UWP: Windows 10 XBox One La cible UWP est livrée avec deux options de compilation, l'une est la VM qui utilise un runner et du code interprété et l'autre qui est le YYC ( YoYo Compiler) qui crée du code natif et un exécutable compilé. Notez que l'option YYC nécessitera une configuration supplémentaire. Pour plus d'informations sur la configuration, consultez la section sur les SDK requis ou la section du support technique sur les cibles UWP. Consoles Avec la version Consoles GameMaker Studio 2 toutes les limitations de la licence d'essai (voir ci-dessus) sont supprimées et vous pouvez compiler des projets pour une ou plusieurs des plates-formes cibles suivantes: XBox One Playstation 4 Switch Nintendo Chacune des cibles de la console est livrée avec deux options de compilation, l'une est la VM qui utilise un runner et du code interprété et l'autre qui est le YYC ( YoYo Compiler) qui crée du code natif et un exécutable compilé. Notez que l'option YYC nécessitera une configuration supplémentaire. Pour plus d'informations sur la configuration, consultez la section sur les SDK requis. IMPORTANT! En raison des restrictions des divers NDA associés aux exportations de la console, il n'y a aucune information sur la configuration ou la programmation de ces cibles dans le manuel de GameMaker Studio 2 Une fois que vous êtes un développeur enregistré pour la plateforme choisie, veuillez contacter le Helpdesk via votre compte YoYo et demander l'accès à la documentation appropriée. Éducation GameMaker Studio 2 peut également être utilisé dans un environnement scolaire ou universitaire. Pour plus d'informations à ce sujet, veuillez contacter YoYo Games via le portail éducatif YoYo Games. Quelle que soit la version de GameMaker Studio 2 que vous utilisez, ils nécessiteront tous la même configuration de base avant utilisation. Les sections répertoriées ci-dessous traitent toutes de cela et incluent des informations sur la mise à jour de l'EDI et des Runtimes, les SDK requis pour des modules cibles spécifiques et les préférences de l'IDE pour adapter le fonctionnement de GameMaker Studio 2 à votre propre style de travail.
